# /save-session — 세션 인계 파일 업데이트

현재 세션의 최종 상태를 `docs/team/session.md` 에 **전면 덮어쓰기**한다.
기존 내용은 모두 삭제되고, 최신 스냅샷만 남는다. 다음 세션은 이 파일을 먼저 읽고 이어간다.

---

## 1단계: 현재 세션 맥락 수집

아래 항목을 모아 요약한다. 부족한 부분은 조용히 추정하지 말고 생략한다.

1. **이번 세션에서 실제로 한 일** — 코드/문서/DB 변경, 배포, 디버깅 결과 등 구체 사실
2. **현재 시스템/작업 상태** — 실행 중인 서비스, DB 설정, 환경 변수, 미커밋 변경 등
3. **미해결/진행 중 항목** — 이어서 할 일, 대기 중 결정, 차단 요인
4. **다음 세션 예상 첫 질문/요청** — 사용자가 가장 먼저 할 말 후보

참고 명령:
```bash
git log --oneline -10
git status --short
git diff --stat
```

---

## 2단계: `docs/team/session.md` 전면 교체

Write 도구로 파일 전체를 덮어쓴다(Edit이 아닌 Write). 아래 뼈대를 따른다.

```markdown
# 세션 인계 파일 (cho)

> 이 파일은 **세션 종료 시마다 최신 작업 내용으로 전면 교체**된다.
> 다음 세션은 이 파일을 먼저 읽고 이어간다. 오래된 내역은 보존하지 않음.

**최종 업데이트**: {YYYY-MM-DD}

---

## 최근 세션 요약

- **한 일**: {1~3줄 핵심 사실}
- **현재 운영 상태**: {서비스/환경/DB 등 한눈에}
- **다음 할 일**: {가장 먼저 이어갈 작업}

---

## 현재 상태 상세

{필요한 표/목록 — 호스트, 경로, 포트, 스택 버전, DB 설정 등}

---

## 다음 세션 시작 시 체크리스트

{실제 실행 가능한 단계별 명령/SQL/경로 — 추상적 설명 금지}

---

## 미해결 / 검토 대상

- [ ] {항목 1}
- [ ] {항목 2}

---

## 다음 세션에 전달할 컨텍스트

**사용자가 다음 세션에서 가장 먼저 말할 가능성이 높은 내용**
1. {후보 1} → {대응 방안/첫 명령}
2. {후보 2} → {대응 방안}
```

규칙:
- 추상어(예: "설정을 잘 했다") 금지. **파일 경로, 명령, SQL, 포트, 버전** 등 구체적 사실만
- 표·코드블록 적극 사용. 산문 최소화
- "이번 세션 이전"의 맥락은 기록하지 않는다. 오직 **현재 스냅샷**만
- 민감 정보(비밀번호 등)는 `docs/` 에 이미 적혀 있던 것만 유지. 새로 추가하지 않음

---

## 에러 케이스

- 변경된 것이 거의 없을 때: 그래도 최신 날짜·상태로 갱신 (파일을 덮어쓰되 큰 변동 없음)
- 파일 접근 실패: 원인 출력 후 중단
