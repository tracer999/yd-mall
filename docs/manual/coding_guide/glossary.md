# 용어집

이 페이지는 코딩가이드 전반에서 반복해서 등장하는 핵심 용어를 모아, 초보자도 빠르게 찾아볼 수 있게 정리한 참고 자료입니다.

- 범위: `tech_stack.md`, `nodejs.md`, `express_libs.md`, `mvc.md`, `mysql.md`, `mysql2.md`, `project_structure.md`, `vibe_coding.md`, `workflow.md`, `example_google_login.md`, `example_notice.md`
- 정렬: **가나다순**

---

## ㄱ

### 가상 DOM (Virtual DOM)
React 같은 프론트엔드 라이브러리에서 UI 변경을 최적화하기 위해 사용하는 메모리상의 가상 트리 구조입니다. 이 프로젝트는 EJS 서버 렌더링 기반이라 핵심 기술은 아니지만, 프론트 개념 비교 시 자주 등장합니다.

### 검증(Validation)
사용자가 입력한 값(예: 이메일, 비밀번호, 제목)이 규칙에 맞는지 확인하는 과정입니다. 서버 검증은 보안과 데이터 무결성을 위해 필수입니다.

### 관계형 데이터베이스(RDBMS)
데이터를 테이블 형태로 저장하고, 테이블 간 관계를 기반으로 조회/관리하는 DBMS 유형입니다. 이 프로젝트의 MySQL 8이 여기에 해당합니다.

### 기본키(PK, Primary Key)
테이블의 각 행을 유일하게 식별하는 컬럼입니다. 보통 `id` 컬럼을 자동 증가 정수로 둡니다.

### 깃(Git)
코드 변경 이력을 관리하는 버전 관리 시스템입니다. 기능 단위 브랜치, 커밋, 병합 등의 작업에 사용됩니다.

### 깃허브(GitHub)
Git 저장소를 원격으로 협업/관리하는 플랫폼입니다. 코드 리뷰, 이슈, PR(풀 리퀘스트) 흐름에 자주 사용됩니다.

### 꼬리질문(Refinement Prompt)
AI에게 1차 결과를 받은 뒤, 조건을 더 구체화해 결과 품질을 높이는 추가 프롬프트입니다. 바이브코딩에서 매우 중요합니다.

## ㄴ

### 난수 Salt
비밀번호 해시 시 원문 비밀번호 앞뒤에 더해지는 랜덤 데이터입니다. 같은 비밀번호라도 서로 다른 해시 결과를 만들도록 도와줍니다.

### 널(Null)
값이 비어 있음을 나타내는 데이터베이스 값입니다. `NULL` 허용 여부는 테이블 설계 시 의도적으로 결정해야 합니다.

### 노드 모듈(Node Modules)
`npm install`로 설치된 의존성 패키지들이 저장되는 `node_modules/` 폴더입니다.

### 노드몬(nodemon)
개발 중 파일 변경을 감지해 Node.js 서버를 자동 재시작해 주는 도구입니다.

### Node.js
브라우저 밖(서버)에서 JavaScript를 실행할 수 있게 해 주는 런타임입니다. 이 프로젝트 백엔드의 실행 기반입니다.

---

## ㄷ

### 데이터 무결성(Data Integrity)
데이터가 정확하고 일관된 상태로 유지되는 성질입니다. 키 제약조건, 검증, 트랜잭션으로 보장합니다.

### 데이터베이스(Database)
애플리케이션이 사용하는 데이터를 구조적으로 저장/조회/수정/삭제하는 저장소입니다.

### DBMS
데이터베이스를 생성·관리·질의할 수 있도록 해 주는 소프트웨어입니다. MySQL, PostgreSQL 등이 대표적입니다.

### dotenv
`.env` 파일의 환경 변수를 Node.js 런타임으로 로드해 주는 패키지입니다.

### 동시성(Concurrency)
여러 요청이 겹쳐 실행되는 상황을 다루는 개념입니다. 세션 처리, 트랜잭션, 락 전략과 연계됩니다.

---

## ㄹ

### 라우터(Router)
URL과 HTTP 메서드를 컨트롤러 함수에 연결하는 계층입니다. 프로젝트에서는 `routes/` 폴더가 해당 역할을 담당합니다.

### 라우팅(Routing)
들어온 요청 경로(`GET /products` 등)에 따라 실행할 로직을 분기하는 과정입니다.

### 런타임(Runtime)
코드가 실제로 실행되는 환경입니다. 이 프로젝트 서버 런타임은 Node.js입니다.

### 레이아웃(Layout)
여러 화면에서 공통으로 사용하는 템플릿 골격(헤더/푸터/사이드바 등)입니다. EJS + `express-ejs-layouts`로 관리합니다.

### 로깅(Logging)
애플리케이션 실행 중 발생한 상태/이벤트/오류를 기록하는 행위입니다. 장애 분석과 운영 모니터링에 필요합니다.

### 리다이렉트(Redirect)
서버가 브라우저를 다른 URL로 이동시키는 응답입니다. 로그인 후 이동, 권한 체크 실패 처리 등에 사용합니다.

### 리팩터링(Refactoring)
동작은 유지하면서 코드 구조를 개선하는 작업입니다. 중복 제거, 함수 분리, 가독성 향상 등이 포함됩니다.

---

## ㅁ

### 마이그레이션(Migration)
DB 스키마 변경 이력을 관리하며 안전하게 반영하는 작업입니다. 테이블/컬럼 추가, 제약조건 변경 등을 포함합니다.

### 마크다운(Markdown)
문서 작성을 쉽게 하는 경량 마크업 문법입니다. 이 코딩가이드 문서들이 Markdown으로 작성되었습니다.

### 멱등성(Idempotency)
같은 요청을 여러 번 보내도 최종 결과가 같게 유지되는 성질입니다. API 설계에서 중요합니다.

### 모델(Model)
MVC에서 데이터 구조와 데이터 처리 책임을 갖는 영역입니다. 이 프로젝트에서는 MySQL 테이블과 컨트롤러 SQL이 Model 역할을 함께 담당합니다.

### 뮤터블/이뮤터블(Mutable/Immutable)
값이 직접 변경 가능한지 여부를 나타내는 개념입니다. JavaScript 객체/배열 상태 관리 시 자주 등장합니다.

### 뷰(View)
MVC에서 사용자에게 보여 줄 화면을 담당하는 영역입니다. 프로젝트에서는 `views/`의 EJS 파일들이 해당됩니다.

### 미들웨어(Middleware)
요청과 응답 사이에서 공통 처리를 수행하는 함수입니다. 인증 체크, 바디 파싱, 세션 처리, 공통 데이터 주입 등에 사용됩니다.

---

## ㅂ

### 바디 파싱(Body Parsing)
HTTP 요청 본문(JSON, form-data 등)을 서버가 읽을 수 있는 객체로 변환하는 과정입니다.

### 바이브코딩(Vibe Coding)
요구사항을 자연어로 설명하고 AI와 대화하며 설계/코드를 점진적으로 완성하는 개발 방식입니다.

### 배포(Deploy)
개발한 애플리케이션을 실제 운영 환경에서 실행 가능한 상태로 올리는 과정입니다.

### 백엔드(Backend)
브라우저 뒤에서 실행되며 데이터 처리, 인증, 비즈니스 로직, DB 연동을 담당하는 서버 영역입니다.

### bcrypt
비밀번호를 안전하게 해시하는 라이브러리입니다. 평문 저장을 피하고 인증 보안을 강화합니다.

### 브랜치(Branch)
기존 코드와 분리해 독립적으로 작업하는 Git의 가지입니다. 기능 개발 단위를 안전하게 관리할 수 있습니다.

### 브레이크포인트(Breakpoint)
디버깅 시 코드 실행을 특정 줄에서 멈추는 지점입니다. 변수 상태와 흐름 분석에 유용합니다.

### 브라우저 렌더링
브라우저가 HTML/CSS/JS를 해석해 실제 화면으로 그리는 과정입니다.

---

## ㅅ

### 세션(Session)
서버가 사용자 상태(로그인 여부 등)를 요청 간에 유지하기 위해 사용하는 저장 방식입니다.

### 서버 사이드 렌더링(SSR)
서버에서 HTML을 완성해 브라우저로 보내는 방식입니다. 이 프로젝트의 EJS 렌더링 방식이 SSR에 해당합니다.

### 소스맵(Source Map)
압축/변환된 코드와 원본 코드의 위치를 매핑해 디버깅을 돕는 파일입니다.

### 스키마(Schema)
DB 구조 정의입니다. 어떤 테이블/컬럼/타입/제약이 있는지 규정합니다.

### 스택 트레이스(Stack Trace)
오류 발생 시 함수 호출 경로를 출력한 정보입니다. 장애 원인 파악에 핵심 단서가 됩니다.

### 스트레칭 목표(Stretch Goal)
필수 범위를 넘는 확장 목표입니다. 기능 우선순위 정리에 사용됩니다.

### 스타일시트(Stylesheet)
화면 스타일을 정의한 CSS 파일입니다. 이 프로젝트는 Tailwind 기반 CSS를 빌드해 사용합니다.

---

## ㅇ

### 앱 진입점(Entry Point)
애플리케이션 실행이 시작되는 파일입니다. 이 프로젝트에서는 `app.js`가 진입점입니다.

### 에러 핸들링(Error Handling)
예외 상황을 감지/기록하고 사용자에게 적절한 응답을 반환하는 처리입니다.

### 엔드포인트(Endpoint)
클라이언트가 요청을 보내는 API/URL 지점입니다. 예: `GET /notices`.

### 요청(Request)
클라이언트가 서버에 보내는 HTTP 메시지입니다. URL, 메서드, 헤더, 본문을 포함합니다.

### 응답(Response)
서버가 클라이언트로 돌려주는 HTTP 결과입니다. HTML, JSON, 상태코드 등이 포함됩니다.

### 외래키(FK, Foreign Key)
다른 테이블의 기본키를 참조해 관계를 연결하는 컬럼입니다.

### ORM(Object Relational Mapping)
객체와 관계형 DB를 매핑해 SQL 작성을 줄이는 접근입니다. 이 프로젝트는 ORM 대신 `mysql2` 직접 쿼리 방식을 사용합니다.

### OAuth 2.0
외부 인증 제공자(예: Google)로 로그인 권한을 위임받는 표준 프로토콜입니다.

### 유효성 검사(Validation)
입력값 형식/범위/필수 여부를 확인하는 절차입니다. 클라이언트와 서버 양쪽에서 수행할 수 있습니다.

### 의존성(Dependency)
프로젝트가 동작하기 위해 필요한 외부 라이브러리/패키지입니다. `package.json`에서 관리합니다.

### 인덱스(Index)
DB 조회 성능을 높이기 위해 컬럼에 부여하는 검색 구조입니다. 과도한 인덱스는 쓰기 성능 저하를 유발할 수 있습니다.

### 인증(Authentication)
사용자가 누구인지 확인하는 절차입니다. 예: 아이디/비밀번호 로그인, 구글 OAuth 로그인.

### 인가(Authorization)
인증된 사용자에게 특정 기능 접근 권한이 있는지 판단하는 절차입니다. 예: 관리자 페이지 접근 제한.

## ㅈ

### 자바스크립트(JavaScript)
이 프로젝트의 프론트/백엔드 모두에서 사용하는 프로그래밍 언어입니다.

### 조인(JOIN)
여러 테이블을 관계 기준으로 결합해 조회하는 SQL 연산입니다.

### 정적 파일(Static Assets)
서버 로직과 무관하게 그대로 제공되는 리소스(CSS, JS, 이미지)입니다. 보통 `public/`에서 제공합니다.

### 정규화(Normalization)
중복 데이터를 줄이고 갱신 이상을 방지하도록 테이블 구조를 설계하는 방법론입니다.

### 쿼리(Query)
DB에 데이터 조회/변경을 요청하는 SQL 문장입니다.

### 트랜잭션(Transaction)
여러 DB 작업을 하나의 원자적 단위로 실행하는 메커니즘입니다. 중간 실패 시 롤백으로 일관성을 유지합니다.

---

## ㅊ

### 캐시(Cache)
자주 조회되는 데이터를 빠르게 제공하기 위한 임시 저장소입니다. Redis가 대표적인 캐시 도구입니다.

### 컨트롤러(Controller)
MVC에서 요청을 받아 비즈니스 로직을 실행하고 응답(뷰/JSON/리다이렉트)을 결정하는 영역입니다.

### 커넥션 풀(Connection Pool)
DB 연결을 미리 여러 개 만들어 재사용하는 방식입니다. `mysql2.createPool()`로 설정합니다.

### 커밋(Commit)
Git에서 변경 사항 스냅샷을 이력으로 남기는 작업입니다.

### 콜백 URL (OAuth Callback)
외부 인증 후 사용자를 다시 돌려보내는 서버 URL입니다. 예: `/auth/google/callback`.

---

## ㅌ

### 테이블(Table)
관계형 DB에서 동일한 성격의 데이터를 행/열 형태로 저장하는 구조입니다.

### 템플릿 엔진(Template Engine)
데이터를 HTML에 바인딩해 동적 페이지를 만드는 도구입니다. 이 프로젝트는 EJS를 사용합니다.

### 트러블슈팅(Troubleshooting)
오류 원인을 찾고 해결책을 적용하는 문제 해결 과정입니다.

---

## ㅍ

### 패스포트(Passport)
Node.js(Express)에서 인증을 구조적으로 구현할 수 있게 도와주는 미들웨어입니다.

### 패키지 매니저(Package Manager)
라이브러리 설치/업데이트/버전 관리를 담당하는 도구입니다. Node.js 생태계에서는 npm을 사용합니다.

### 풀 리퀘스트(Pull Request)
브랜치 변경사항을 리뷰 후 메인 브랜치에 병합하도록 요청하는 협업 단위입니다.

### 프레임워크(Framework)
웹 서버/라우팅/미들웨어 같은 공통 구조를 제공해 개발 생산성을 높이는 기반 도구입니다. Express가 대표적입니다.

### 프롬프트(Prompt)
AI에게 원하는 결과를 요청할 때 입력하는 지시문입니다. 바이브코딩 품질은 프롬프트 품질에 크게 좌우됩니다.

---

## ㅎ

### 해시(Hash)
원문 데이터를 일정 길이 문자열로 변환한 값입니다. 비밀번호 저장 시 해시를 사용해 보안을 높입니다.

### 환경 변수(Environment Variables)
실행 환경별 설정(DB 계정, API 키 등)을 코드 밖에서 주입하는 값입니다. `.env`에 저장하고 `dotenv`로 로드합니다.

### HTTP 메서드
요청 의도를 나타내는 동사입니다. 주로 `GET`(조회), `POST`(생성), `PUT/PATCH`(수정), `DELETE`(삭제)를 사용합니다.

### REST API
리소스를 URL로 표현하고 HTTP 메서드 중심으로 조작하는 API 설계 방식입니다.
