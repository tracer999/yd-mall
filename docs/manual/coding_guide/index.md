# 코딩 가이드

이 코딩 가이드는 **비개발자** 또는 **Node.js, Express, MySQL, Tailwind CSS, MVC 패턴 등이 처음인 초보 개발자**가, **바이브코딩(Vibe Coding)**으로 이 쇼핑몰 프로젝트를 이해하고 비슷한 웹 서비스를 만들 수 있도록 돕는 교육 자료입니다.

코드를 한 줄도 몰라도 괜찮습니다. 이 가이드는 **"코드를 직접 다 치는 능력"**이 아니라, **"AI에게 무엇을 어떻게 요청해야 하는지 이해하는 능력"**을 키우는 데 초점을 맞추고 있습니다.

---

## 대상 독자

- 프로그래밍을 처음 접하는 분
- Node.js, Express, MySQL, MVC 등 용어가 낯선 분
- AI와 대화하며 코드를 만드는 **바이브코딩**을 배우고 싶은 분
- 이 프로젝트를 예시로 삼아 소형 쇼핑몰을 만들어 보고 싶은 분
- 이미 다른 언어(Python, Java 등)를 알지만 Node.js/Express는 처음인 분

## 이 프로젝트의 핵심 구조 한눈에 보기

이 쇼핑몰이 어떻게 동작하는지를 **한 장의 그림**으로 표현하면 아래와 같습니다.

```text
[쇼핑몰 아키텍처 – 요청이 흐르는 경로]

  ┌─────────────┐
  │   브라우저    │  사용자가 URL을 입력하거나 버튼을 클릭
  │  (Chrome 등) │
  └──────┬──────┘
         │  HTTP 요청 (GET /products, POST /cart/add 등)
         ▼
  ┌──────────────────────────────────────────────────────────────────┐
  │                        Express 서버 (app.js)                     │
  │                                                                  │
  │  ┌────────────────────────────────────────────────────────────┐  │
  │  │                    미들웨어 체인                             │  │
  │  │  session → passport → bodyParser → 메뉴로드 → 장바구니수량  │  │
  │  └───────────────────────────┬────────────────────────────────┘  │
  │                              │                                   │
  │                              ▼                                   │
  │  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐       │
  │  │   라우터(R)   │───▶│ 컨트롤러(C)  │───▶│   뷰(V/EJS)  │       │
  │  │  routes/     │    │ controllers/ │    │   views/     │       │
  │  │              │    │              │    │              │       │
  │  │ /products    │    │ 데이터 가공   │    │ HTML 생성     │       │
  │  │ /cart        │    │ DB 조회/저장  │    │ 화면 렌더링   │       │
  │  │ /admin       │    │ 응답 결정     │    │              │       │
  │  │ /auth        │    │              │    │              │       │
  │  └──────────────┘    └──────┬───────┘    └──────┬───────┘       │
  │                             │                    │               │
  └─────────────────────────────┼────────────────────┼───────────────┘
                                │                    │
                   ┌────────────▼──────┐             │  HTTP 응답 (HTML)
                   │    MySQL 8 (DB)   │             │
                   │                   │             ▼
                   │ users, products,  │      ┌─────────────┐
                   │ orders, carts ... │      │   브라우저    │
                   │                   │      │  화면 표시    │
                   │  config/db.js     │      └─────────────┘
                   │  (pool 연결)       │
                   └───────────────────┘

  ┌───────────────────────────────────────────────────┐
  │              정적 파일 (public/)                    │
  │  CSS (Tailwind), JavaScript, 이미지, 업로드 파일    │
  │  → 브라우저가 직접 요청해서 받아감                    │
  └───────────────────────────────────────────────────┘
```

**흐름 요약 (7단계)**

```text
1. 사용자가 브라우저에서 URL을 요청한다        (예: GET /products)
2. Express(app.js)가 요청을 받는다
3. 미들웨어가 세션/로그인/공통 데이터를 처리한다
4. 라우터가 URL에 맞는 컨트롤러를 찾아 호출한다  (routes/ → controllers/)
5. 컨트롤러가 MySQL에서 데이터를 조회/저장한다    (pool.query)
6. 컨트롤러가 EJS 뷰에 데이터를 넘겨 HTML을 만든다 (res.render)
7. 완성된 HTML이 브라우저로 돌아가고, 화면에 표시된다
```

> 이 흐름이 이해되지 않아도 괜찮습니다. `mvc.md`에서 각 단계를 비유와 실제 코드로 자세히 설명합니다.

지금 당장 바이브코딩을 시작하고 싶다면, 아래 5단계를 따라가 보세요.

### Step 1: 바이브코딩의 감을 잡는다

`vibe_coding.md`를 읽습니다. 전부 읽을 필요 없이, **"1. 바이브코딩이란?"** 섹션과 **"2. 기본 원칙 4가지"** 섹션만 읽어도 됩니다.

> 핵심 한 줄: "내가 원하는 것을 명확하게 설명하면, AI가 코드를 만들어 준다."

### Step 2: 프로젝트 구조를 파악한다

`project_structure.md`를 열어서, **폴더 트리**를 한 번 훑어 봅니다.

> 핵심 한 줄: "routes/에 URL, controllers/에 로직, views/에 화면이 들어 있다."

### Step 3: 만들고 싶은 기능을 하나 고른다

예시:
- "공지사항 게시판을 추가하고 싶다"
- "상품 리뷰 기능을 만들고 싶다"
- "관리자 대시보드에 통계를 넣고 싶다"

너무 크게 잡지 말고, **한 가지 기능**만 고릅니다.

### Step 4: 워크플로우를 따라간다

`workflow.md`를 열고, **"기능 하나를 추가할 때의 큰 흐름"** 섹션의 순서를 따릅니다.

```text
기획 → DB 설계 → 컨트롤러 → 라우터 → 뷰 → 메뉴/버튼 연결 → 테스트
```

각 단계에서 AI에게 이렇게 말하면 됩니다:

```text
"이 프로젝트는 Node.js + Express + MySQL + EJS 기반 쇼핑몰이야.
 MVC 패턴으로 routes/, controllers/, views/ 폴더를 쓰고 있어.
 여기에 [원하는 기능]을 추가하고 싶어.
 먼저 [현재 단계]부터 시작해 줘."
```

### Step 5: 예제를 참고한다

막히면 `example_notice.md`(공지사항 예제)나 `example_google_login.md`(구글 로그인 예제)를 펼쳐서, 비슷한 단계의 코드를 참고합니다.

> 이 5단계를 한 번 돌고 나면, 그다음부터는 스스로 **"기획 -> 구현 -> 테스트"** 사이클을 반복할 수 있게 됩니다.

## 용어 사전 (간단)

이 가이드에서 자주 등장하는 용어를 한 줄씩 정리했습니다. 읽다가 모르는 단어가 나오면 여기로 돌아와 확인하세요.

### 구조/패턴 관련

| 용어 | 설명 |
|------|------|
| **MVC** | Model-View-Controller의 줄임말. 데이터/화면/처리 로직을 분리하는 설계 패턴입니다. |
| **Router (라우터)** | URL 경로(예: `/products`)와 실행할 함수(컨트롤러)를 연결해 주는 역할입니다. |
| **Controller (컨트롤러)** | 요청을 받아서, DB를 조회/저장하고, 어떤 화면을 보여줄지 결정하는 함수입니다. |
| **View (뷰)** | 사용자에게 실제로 보이는 화면(HTML)입니다. 이 프로젝트에서는 EJS 파일이 뷰입니다. |
| **Model (모델)** | 데이터의 구조와 데이터를 읽고 쓰는 로직입니다. 이 프로젝트에서는 MySQL 테이블 + SQL 쿼리가 모델 역할을 합니다. |
| **Middleware (미들웨어)** | 요청과 응답 사이에서 공통적으로 처리하는 함수입니다. (예: 로그인 확인, 세션 처리) |

### 웹 개발 관련

| 용어 | 설명 |
|------|------|
| **Session (세션)** | 서버가 사용자를 기억하는 방법입니다. 로그인 상태 유지 등에 사용됩니다. |
| **Cookie (쿠키)** | 브라우저에 저장되는 작은 데이터 조각입니다. 세션 ID를 쿠키에 담아 서버와 주고받습니다. |
| **CRUD** | Create(생성), Read(조회), Update(수정), Delete(삭제)의 줄임말. 데이터 처리의 기본 4가지 동작입니다. |
| **REST** | URL 설계 규칙 중 하나입니다. "자원(resource)"을 URL로 표현하고, HTTP 메서드(GET/POST/PUT/DELETE)로 동작을 구분합니다. |
| **API** | Application Programming Interface. 프로그램끼리 데이터를 주고받는 약속된 방식입니다. |

### Node.js / npm 관련

| 용어 | 설명 |
|------|------|
| **npm** | Node Package Manager. Node.js용 라이브러리를 설치하고 관리하는 도구입니다. |
| **package.json** | 이 프로젝트가 어떤 라이브러리를 쓰는지, 어떤 명령어로 실행하는지 적어 둔 설정 파일입니다. |
| **node_modules/** | `npm install`을 하면 라이브러리들이 설치되는 폴더입니다. 직접 수정하지 않습니다. |

### 바이브코딩 관련

| 용어 | 설명 |
|------|------|
| **Vibe Coding (바이브코딩)** | AI와 대화하면서, 원하는 기능을 설명하고, AI가 만든 코드를 검토/수정하며 개발하는 방식입니다. |
| **프롬프트 (Prompt)** | AI에게 보내는 요청 메시지입니다. 좋은 프롬프트를 쓰면 좋은 코드를 받을 수 있습니다. |
| **컨텍스트 (Context)** | AI에게 알려 주는 배경 정보입니다. 프로젝트 구조, 사용 기술, 기존 코드 등을 포함합니다. |

## 자주 묻는 질문

### Q. 코딩을 전혀 몰라도 읽을 수 있나요?

**네, 읽을 수 있습니다.** 이 가이드는 비개발자를 주 대상으로 만들었습니다. ★☆☆ 난이도 문서(`vibe_coding.md`, `tech_stack.md`)부터 시작하면, 코드를 몰라도 개념을 잡을 수 있습니다. 코드가 등장하는 문서(★★☆, ★★★)에서도 한 줄씩 뜻을 풀어 설명합니다.

### Q. 어떤 순서로 읽어야 하나요?

위의 **학습 로드맵**을 참고하세요. 요약하면:

```text
사용 기술 → Node.js → Express → 예제(Express 로그인&REST API) → MVC → 프로젝트구조 → MySQL → 바이브코딩이란 → 워크플로우 → 예제
```

하지만 반드시 순서대로 읽을 필요는 없습니다. **"목적별 빠른 찾기"** 표를 보고, 지금 필요한 문서만 골라 읽어도 됩니다.

### Q. 바이브코딩만으로 쇼핑몰을 만들 수 있나요?

**네, 가능합니다.** 이 프로젝트 자체가 바이브코딩을 활용하여 만들어진 쇼핑몰입니다. 다만 몇 가지 알아두면 좋은 점이 있습니다:

- AI가 만든 코드를 **그대로 붙여넣기만 하면 안 되고**, 결과를 확인하고 수정하는 과정이 필요합니다.
- 프로젝트 구조(MVC, 폴더 구성)를 이해할수록 AI에게 **더 정확한 요청**을 할 수 있습니다.
- `workflow.md`의 순서를 따르면, 한 번에 큰 기능을 요청하는 대신 **작은 단계로 나눠서** 안전하게 만들 수 있습니다.

### Q. 프로젝트를 실행하려면 뭘 설치해야 하나요?

최소한 아래 세 가지가 필요합니다:

1. **Node.js** (18 이상) -- [nodejs.org](https://nodejs.org)에서 다운로드
2. **MySQL 8** -- 로컬에 설치하거나 클라우드 DB 사용
3. **텍스트 에디터** -- VS Code를 추천합니다

설치 후 `npm install`로 라이브러리를 내려받고, `.env` 파일을 설정하면 `npm run dev`로 실행할 수 있습니다. 자세한 내용은 `workflow.md`의 **"프로젝트 세팅"** 섹션에 있습니다.

### Q. 문서에서 모르는 용어가 나오면 어떻게 하나요?

이 페이지의 **"용어 사전"** 섹션을 참고하세요. 더 자세한 설명이 필요하면 해당 용어가 다루어지는 개별 문서(예: MVC가 궁금하면 `mvc.md`)를 읽어 보세요.

### Q. 이 가이드에 없는 내용은 어디서 찾나요?

- **공식 문서**: [Node.js Docs](https://nodejs.org/docs), [Express Guide](https://expressjs.com), [MySQL Reference](https://dev.mysql.com/doc/)
- **바이브코딩**: AI에게 직접 물어보세요! 예를 들어 "Express에서 라우터를 쓰는 방법을 초보자에게 설명하듯 알려 줘"라고 프롬프트하면 됩니다.

### Q. 가이드 내용 중 틀린 부분이나 개선할 점이 있으면 어떻게 하나요?

이 프로젝트의 GitHub 저장소에 이슈(Issue)를 남기거나, 직접 수정하여 풀 리퀘스트(PR)를 보내 주셔도 됩니다.
