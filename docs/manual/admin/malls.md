# 몰 리스트 관리

이 시스템은 **쇼핑몰 하나를 운영하는 도구가 아니라, 쇼핑몰을 찍어내는 빌더**입니다. 하나의 관리자 안에서 여러 개의 몰을 만들어 두고, 필요할 때 만들고 · 확인하고 · 지우고 · 다시 만들 수 있습니다. 이 화면이 그 몰들을 관리하는 곳입니다.

여기서는 **몰 만들기 · 편집할 몰 고르기(선택) · 몰 정보 수정 · 몰 삭제 · 테마(프리셋) 재적용**을 다룹니다.

## 1. 가장 중요한 개념 — "편집 중인 몰"

### 1-1. 편집 중인 몰이란

관리자 화면 **오른쪽 위**에는 몰 이름이 적힌 버튼이 하나 있습니다. 이것이 **지금 편집 중인 몰**입니다.

상품을 등록하든, 배너를 걸든, 테마를 바꾸든 — **여기 적힌 몰에 저장**됩니다. 화면 어디에도 "어느 몰에 저장할까요?"라고 묻는 칸이 없는 이유가 이것입니다. **묻지 않고 이 버튼에 적힌 몰에 저장**합니다.

> **"저장했는데 화면에 안 나와요"라는 문의의 절반은 편집 몰이 달랐던 경우입니다.** 작업을 시작하기 전에 오른쪽 위를 한 번 보는 습관을 들이세요.

### 1-2. 편집 몰을 바꾸는 두 가지 방법

| 방법 | 위치 | 설명 |
|---|---|---|
| **몰 선택기** | 관리자 화면 오른쪽 위 | 버튼을 누르면 몰 목록이 펼쳐집니다. 고르면 즉시 바뀝니다. |
| **몰 목록의 [선택] 버튼** | 이 화면(`/admin/malls`)의 각 줄 | 위 선택기와 **완전히 같은 동작**입니다. |

바꾸면 화면 위쪽에 "이제 **○○** 을(를) 편집합니다. 카테고리·상품·메뉴·테마·페이지 빌더가 모두 이 몰 기준으로 동작합니다." 라는 안내가 뜹니다.

**한 번 고르면 계속 유지됩니다.** 다른 메뉴로 이동해도, 브라우저를 닫았다 열어도 그대로입니다. 다만 **로그아웃하면 초기화**되어 기본몰로 돌아갑니다.

### 1-3. 무엇이 몰마다 나뉘고, 무엇이 공용인가

| 몰마다 따로 (편집 몰 기준) | 모든 몰이 함께 쓰는 것 |
|---|---|
| 상품 | 카테고리(일반) · 브랜드 |
| 홈 화면 섹션 · 페이지 | 회원 |
| 배너 · 메인 슬라이더 | 주문 |
| 메뉴 구성 · 헤더 설정 | 관리자 계정 |
| 테마 · 사이트 설정(로고·상호) | 시스템 설정(결제 키·메일·소셜 로그인) |
| 기획전 · 쿠폰 · 이벤트 · 특가 · 공동구매 · 쇼핑라이브 | 약관·정책 |
| 배송비 정책 · FAQ · 아울렛 | |

> **회원·주문은 몰로 나뉘지 않습니다.** 그래서 **대시보드와 매출 통계는 전체 기준**으로 나옵니다. "몰별 매출"을 보여 주는 화면이 아닙니다.

### 1-4. 비활성 몰은 선택할 수 없습니다

몰을 **비활성(OFF)** 으로 꺼 두면 목록의 **[선택]** 버튼이 회색으로 잠깁니다. 편집하려면 먼저 그 몰을 **활성**으로 바꿔야 합니다.

## 2. 몰 목록 화면 읽는 법

목록의 각 줄은 다음을 보여 줍니다.

| 열 | 내용 |
|---|---|
| **ID** | 몰의 내부 번호입니다. |
| **코드** | 몰을 구분하는 영문 식별자. 고객이 `?mall=코드` 로 접속할 때 쓰입니다. |
| **이름** | 몰 이름과, 그 옆에 **헤더 스킨 배지**(기본형 / 드로어형)가 붙습니다. 지금 편집 중인 몰에는 **편집중** 배지가 추가로 붙습니다. 도메인을 넣어 두었으면 이름 아래에 작게 보입니다. |
| **데이터** | 이 몰이 가진 `카테고리 ○ · 상품 ○` 개수입니다. **삭제할 때 중요한 숫자**입니다. |
| **기본/활성** | `기본` = 기본몰, `ON`/`OFF` = 활성 여부. |
| **스토어 링크** | `?mall=코드 ↗` 를 누르면 그 몰의 고객 화면이 새 창으로 열립니다. |
| **관리** | **선택 · 수정 · 삭제** 버튼. |

## 3. 새 몰 만들기

  <a class="manual-goto" href="/admin/malls/new" target="_blank">몰 등록 화면 열기</a>

### 3-1. 기본 정보

| 입력란 | 필수 | 설명 | 예시 |
|---|---|---|---|
| **코드** | 필수(조건부) | 몰의 고정 식별자입니다. **소문자·숫자·`-`·`_` 만 2~50자.** `?mall=코드` 로 접속에 쓰입니다. | `general` |
| **몰 이름** | 필수 | 관리자와 화면에 보이는 이름. 최대 100자. | `와이디몰 종합관` |
| **도메인** | 선택 | 나중에 도메인별 접속을 붙이기 위한 칸입니다. **지금은 저장만 되고 동작하지 않습니다.** | `general.ydata.co.kr` |
| **활성** | 체크박스 | 끄면 그 몰로 접속할 수 없고 편집 대상으로도 고를 수 없습니다. |  |
| **기본몰** | 체크박스 | `?mall=` 없이 `/` 로 접속했을 때 보여 줄 몰입니다. **항상 정확히 하나만** 기본몰입니다. |  |

> **기본몰로 체크하면 코드 입력칸이 사라집니다.** 이건 정상입니다. 몰을 하나만 쓰는 환경에서는 코드를 알 필요가 없으므로 **서버가 `main` 같은 코드를 자동으로 붙여 줍니다.** 기본몰이 아니면 코드는 필수입니다.

> **코드는 나중에 바꾸지 마세요.** 고객에게 `?mall=코드` 링크를 뿌린 뒤 코드를 바꾸면 **기존 링크가 전부 깨집니다.**

### 3-2. 테마 선택 (이지 모드)

**등록 화면에서만** 나오는 영역입니다. 테마를 고르면 그에 맞는 **디자인 · 헤더 · 메인 화면 골격**이 자동으로 구성됩니다.

| 테마 | 배지 | 무엇이 달라지나 |
|---|---|---|
| **테마 1 — 좌 히어로 + 우 상품 카드** | 2단형 | 홈 상단 히어로를 두 칸으로 나눠 왼쪽에 큰 슬라이드, 오른쪽에 추천 상품 카드를 세웁니다. |
| **테마 2 — 전체폭 히어로** | 전체폭 | 히어로를 가로 전체폭 한 칸으로 크게 씁니다. 가장 보편적인 형태이며 **기본 선택**입니다. |
| **테마 3 — 풀블리드(에디토리얼)** | 풀블리드 | 투명 오버레이 헤더 위로 화면을 꽉 채우는 히어로를 깝니다. 명조 계열 디스플레이 폰트의 라이프스타일 무드. |

세 테마 모두 히어로 아래에 **베스트 · 쇼핑특가 · 베스트 카테고리 · 베스트 브랜드** 캐러셀이 기본으로 깔립니다.

> **테마는 "배치 뼈대"만 정합니다.** 히어로 안에 무엇이 들어가는지(상품 쇼케이스 / 이미지 배너)는 테마가 아니라 **배너 관리 > 메인 슬라이더**에서 고릅니다. 그래서 테마를 바꿔도 등록해 둔 배너·상품은 그대로 남습니다.

### 3-3. 메뉴 구성 방식

테마와 **완전히 별개인** 선택입니다. 테마가 "룩"이라면 이건 "메뉴를 어떻게 조립할지"입니다. 어느 테마와도 자유롭게 조합할 수 있습니다.

| 방식 | 고객 화면 |
|---|---|
| **카테고리 분리형** | 상단에 **[☰ 카테고리]** 버튼을 두고 그 옆에 일반 메뉴를 한 줄로 놓습니다. 카테고리는 버튼을 누르면 3단 드롭다운 패널로 열립니다. |
| **통합형 (드로어)** | 헤더에는 **[☰]·로고·장바구니**만 두고 메뉴 전체를 좌측 슬라이드 드로어에 담습니다. 카테고리 1단계가 일반 메뉴와 같은 목록에 놓이고 하위는 **[+]** 로 펼칩니다. |

만든 뒤에 바꾸려면 **Header 설정**으로 갑니다.

### 3-4. 샘플 데이터 포함

**기본으로 켜져 있습니다.** 켜 두면 몰을 만들 때 **상품 6개 · 특가 1건 · 배너 3개(영상 2개 포함)** 를 함께 심어, **생성 직후 메인 화면이 바로 뜨게** 합니다.

- 카테고리·브랜드는 **새로 만들지 않고 공용 카테고리에 연결**합니다. 어느 카테고리에 붙일지는 서비스 관리의 샘플 데이터 관리에서 지정합니다.
- 최소한만 넣으므로 상품 관리에서 쉽게 지우거나 진짜 상품으로 바꿀 수 있습니다.

> **샘플 데이터를 끄면 갓 만든 몰의 메인 화면이 거의 비어 보입니다.** 상품이 없으면 베스트·특가 같은 섹션이 표시할 게 없어 **조용히 건너뛰기** 때문입니다. 고장이 아닙니다. 상품을 등록하면 채워집니다.

### 3-5. 저장

**저장하고 구성 채우기** 버튼을 누르면 몰이 만들어지고, 이어서 **GNB 메뉴 · 헤더 정책 · 테마 · 메인 화면 골격**이 자동으로 채워집니다. 성공하면 목록으로 돌아오며 초록색 안내가 뜹니다.

새 몰의 GNB에는 **카테고리 · 쇼핑특가 · 베스트 · 신상품 · 이벤트 · 브랜드** 메뉴가 켜진 상태로 들어갑니다. 기획전·전문관은 **일부러 꺼진 채로** 시작합니다 — 갓 만든 몰에는 내용이 없어 눌러도 빈 화면이기 때문입니다. 내용을 넣은 뒤 메뉴 관리에서 직접 켜세요.

> 드물게 `몰은 만들어졌지만 초기 구성에 실패했습니다` 라는 빨간 안내가 뜰 수 있습니다. 몰 자체는 살아 있으니 걱정하지 말고, **페이지 빌더의 테마 설정 탭에서 테마를 다시 적용**하면 됩니다(아래 5장).

## 4. 몰 정보 수정

목록에서 **수정**을 누르면 편집 화면이 열립니다. 여기서는 **코드 · 이름 · 도메인 · 활성 · 기본몰**만 바꿉니다. **테마와 메뉴 구성은 이 화면에 없습니다** — 페이지 빌더로 옮겨졌습니다.

화면 아래쪽 **화면 구성 상태** 카드에서 이 몰의 준비 상태를 확인할 수 있습니다.

| 표시 | 뜻 |
|---|---|
| **내비 설정** | 헤더·GNB 정책이 만들어져 있는지 |
| **테마** | 테마가 지정되어 있는지 |
| **사이트 설정** | 이 몰의 로고·상호 정보가 있는지 |
| **홈 섹션** | 메인 화면에 배치된 섹션 개수 |

넷 다 초록색이면 정상입니다. 회색 항목이 있으면 아래 5장의 **테마 재적용**으로 채울 수 있습니다.

**기본몰 관련 규칙**

- 기본몰은 **비활성화할 수 없습니다.** 시도하면 `기본몰은 비활성화할 수 없습니다. 다른 몰을 먼저 기본몰로 지정하세요.` 가 뜹니다.
- 다른 몰을 기본몰로 지정하면 **기존 기본몰은 자동으로 내려갑니다.** 기본몰은 항상 하나입니다.
- 기본몰로 지정되는 몰은 **자동으로 활성**이 됩니다.
- 코드가 다른 몰과 겹치면 `코드 '○○' 는 다른 몰이 쓰고 있습니다.` 로 거부됩니다.

## 5. 테마(프리셋) 재적용

이미 만든 몰의 **히어로 배치와 메뉴 구성 방식을 다시 고르는** 기능입니다. 몰 수정 화면이 아니라 **페이지 빌더 > 테마 설정 탭**에 있습니다.

<div class="manual-goto-bar">
  <span class="manual-goto-label">테마 재적용 화면</span>
  <a class="manual-goto" href="/admin/page-builder" target="_blank">페이지 빌더 열기 → [테마 설정] 탭</a>
</div>

**적용하면 무엇이 바뀌나**

| 항목 | 기본 동작 (체크 안 함) |
|---|---|
| 히어로 배치(테마 1·2·3) | **바뀝니다** |
| 메뉴 구성 방식(분리형/통합형)·헤더 정책 | **바뀝니다** |
| 테마 색·폰트·모서리 값 | **테마 기본값으로 되돌아갑니다** |
| 홈 섹션(히어로 아래 캐러셀들) | **그대로 유지됩니다** |
| 상품 · 카테고리 | 건드리지 않습니다 |
| 사이트 설정(로고·상호·연락처) | 건드리지 않습니다 |

**홈 섹션까지 기본 구성으로 교체** 체크박스를 켜면, 히어로 아래 캐러셀이 **테마 기본 구성(베스트·쇼핑특가·베스트 카테고리·베스트 브랜드)으로 통째로 교체**됩니다.

> **이 체크박스는 파괴적입니다.** 지금까지 꾸며 둔 홈 섹션이 전부 삭제되며 **되돌릴 수 없습니다.** 기본 구성으로 완전히 리셋하려는 게 아니라면 체크하지 마세요.

> **디자인 스타일 화면에서 손본 모서리·폰트·간격 값은 재적용 시 테마 기본값으로 초기화됩니다.** 세부 다듬기는 테마 재적용을 끝낸 뒤에 하세요.

## 6. 몰 삭제

몰 빌더에서는 **만들고 → 확인하고 → 지우고 → 다시 만드는** 것이 정상적인 작업 흐름입니다. 그래서 데이터가 있어도 삭제할 수 있습니다.

### 6-1. 기본몰은 삭제할 수 없습니다

기본몰의 **삭제** 버튼은 회색으로 잠겨 있습니다. 지우려면 **다른 몰을 먼저 기본몰로 지정**한 뒤 삭제하세요.

### 6-2. 데이터가 없는 몰

카테고리·상품이 0개이면 **삭제** 버튼을 누르는 순간 간단한 확인창만 뜨고 바로 지워집니다.

### 6-3. 데이터가 있는 몰 — 강제 삭제

카테고리나 상품이 하나라도 있으면 **강제 삭제 확인창**이 뜹니다. 여기에는 지워질 내용이 나열되고, **몰 코드를 그대로 입력**해야만 삭제 버튼이 활성화됩니다.

**함께 지워지는 것**

- 이 몰의 **상품**과 거기 딸린 리뷰·장바구니·이미지
- 이 몰의 **테마 카테고리·아울렛 카테고리**
- **페이지 · 홈 섹션 · 배너 · 메인 슬라이더**
- **쿠폰 · 이벤트 · 기획전 · 공동구매 · 쇼핑라이브 · 특가 · 베스트 설정**
- **메뉴 구성 · 헤더 정책 · 테마 · 사이트 설정 · 배송비 정책 · FAQ**
- 외부몰 연동으로 가져온 상품·실행 이력

**지워지지 않는 것**

- **주문 이력** — 남습니다. 다만 지워진 상품과의 연결만 끊깁니다.
- **회원**
- **공용 카테고리와 브랜드** — 전 몰이 함께 쓰는 자산이라 보존됩니다.
- **외부몰 연동 계정 정보** — 같은 몰을 다시 만들 때 재입력하지 않도록 일부러 남깁니다.

> **삭제는 되돌릴 수 없습니다.** 코드 재입력을 요구하는 이유가 이것입니다. 지우기 전에 **정말 그 몰이 맞는지, 편집 중인 몰과 헷갈린 건 아닌지** 확인하세요.

## 7. 자주 막히는 상황

**Q. 상품을 등록했는데 고객 화면에 안 보입니다.**
등록할 때의 **편집 중인 몰**이 지금 보고 있는 몰과 다를 수 있습니다. 몰 목록에서 각 몰의 **데이터(카테고리·상품 개수)** 를 보면 어디에 들어갔는지 금방 알 수 있습니다.

**Q. [선택] 버튼이 회색이라 눌리지 않습니다.**
그 몰이 **비활성(OFF)** 상태입니다. **수정** 화면에서 **활성**을 켠 뒤 다시 선택하세요.

**Q. 몰을 만들었는데 메인 화면이 텅 비어 있습니다.**
샘플 데이터를 끄고 만들었고 상품도 아직 없는 경우입니다. 상품이 없는 섹션은 표시할 게 없어 자동으로 건너뜁니다. 상품을 등록하면 채워집니다.

**Q. 기본몰을 지우고 싶습니다.**
바로는 안 됩니다. 다른 몰을 **기본몰로 지정** → 원래 기본몰이 일반 몰이 됨 → 그때 삭제하세요.

**Q. 코드를 바꿨더니 고객이 링크가 안 열린다고 합니다.**
`?mall=코드` 링크는 코드에 묶여 있습니다. 코드를 되돌리거나, 새 코드로 링크를 다시 안내해야 합니다.

**Q. 몰별 매출을 보고 싶습니다.**
지원하지 않습니다. 주문·회원은 몰로 나뉘어 있지 않아 통계는 전체 기준입니다.

**Q. 테마를 바꿨더니 히어로에 있던 배너가 사라졌나요?**
아닙니다. 테마는 배치만 바꾸고 내용은 그대로 둡니다. 배너가 안 보인다면 **배너 관리 > 메인 슬라이더** 쪽을 확인하세요.

**Q. 도메인을 넣었는데 그 주소로 접속이 안 됩니다.**
도메인 칸은 아직 **저장만 되고 동작하지 않습니다.** 지금은 `?mall=코드` 로 접속합니다.

## 8. 아직 지원하지 않는 것

- **도메인별 자동 접속** — 도메인 칸은 저장만 됩니다
- **몰 복제(기존 몰을 그대로 베껴 새 몰 만들기)**
- **몰별 매출·회원 통계**
- **삭제한 몰 복구**
- 몰마다 다른 결제·소셜 로그인 키 — 시스템 설정은 전 몰 공용입니다
