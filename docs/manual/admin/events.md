# 이벤트 관리

고객 화면의 **이벤트 & 혜택**(주소 `/event`)에 보이는 이벤트를 등록·수정하는 메뉴입니다. (`/admin/events`)

> **매뉴얼 목차에는 "(준비 중)"으로 적혀 있지만, 이 화면은 실제로 동작합니다.** 등록 · 수정 · 삭제, 고객 화면 노출, 고객 응모 접수까지 모두 됩니다. 다만 **응모자 명단 조회와 당첨자 선정은 아직 없습니다.** (맨 아래 [아직 안 되는 것](#아직-안-되는-것) 참고)

---

## 이벤트 목록 화면

목록에서는 등록된 이벤트를 **운영 상태**로 거르거나, **이벤트명·슬러그**로 검색할 수 있습니다.

표에는 다음이 보입니다.

| 칸 | 뜻 |
|---|---|
| 썸네일 | 목록에 쓰이는 작은 이미지 |
| 이벤트명 | 아래에 고객 주소(`/event/슬러그`)가 함께 표시됩니다. |
| 유형 | 공지형 / 응모 |
| 운영상태 | 운영자가 직접 고른 값 (임시저장 · 발행 · 숨김) |
| 노출상태 | **기간에서 자동 계산**된 값 (예정 · 진행중 · 종료) |
| 기간 | 시작일 ~ 종료일. 종료일이 없으면 **상시** |
| 참여 | 참여한 인원 수 (선착순 인원을 정했다면 "참여수 / 정원") |

오른쪽 **수정 · 삭제** 버튼으로 각 이벤트를 관리합니다.

---

## 운영 상태와 노출 상태는 다릅니다

이 화면에서 가장 헷갈리는 부분입니다. **두 가지는 별개**입니다.

- **운영 상태**는 운영자가 고릅니다. 고객에게 보일지 말지를 정합니다.
- **노출 상태**는 운영자가 못 고릅니다. **시작·종료 일시를 보고 시스템이 자동으로 계산**합니다.

| 운영 상태 | 고객에게 보이나요? |
|---|---|
| **임시저장** | 안 보입니다. 작성 중일 때 씁니다. |
| **발행** | 보입니다. |
| **숨김** | 안 보입니다. 내렸다가 나중에 다시 올릴 때 씁니다. |

| 노출 상태 | 언제 붙나요? |
|---|---|
| **예정** | 시작 일시가 아직 오지 않음 |
| **진행중** | 시작했고 아직 안 끝남 |
| **종료** | 종료 일시가 지남 |

> 즉 **"종료"를 만들려고 운영 상태를 숨김으로 바꿀 필요가 없습니다.** 종료 일시가 지나면 알아서 종료로 바뀌고, 고객 화면의 "종료" 목록에 남습니다. 숨김으로 바꾸면 **아예 사라집니다.**

## 이벤트 등록하기

**이벤트 등록** 버튼을 누르면 입력 화면이 나옵니다. 항목은 네 덩어리입니다.

### 기본 정보

- **이벤트명** *(필수)*
- **슬러그 (URL)** — 고객 주소가 `/event/슬러그` 로 만들어집니다. **비워두면 이벤트명에서 자동으로 만들어 줍니다.**
- **한 줄 요약** — 목록 카드에 보이는 짧은 설명
- **상세 본문** — 이벤트 상세 페이지 내용 (편집기로 작성)
- **유의사항** — 상세 페이지 아래쪽 안내 문구

### 이미지

**목록 썸네일**, **상세 PC 대표 이미지**, **상세 모바일 대표 이미지** 세 가지를 넣습니다.

> ⚠️ **이 화면에는 이미지 업로드 버튼이 없습니다.** 이미지 **주소(URL)를 직접 입력**해야 합니다. **배너 관리**에서 이미지를 먼저 올린 뒤, 그 주소를 복사해 붙여넣으세요.

### 발행

- **운영 상태** — 임시저장(고객 미노출) / 발행 / 숨김
- **이벤트 목록에 노출** — 체크를 풀면 목록에서 빠집니다.

### 기간 · 참여 설정

- **시작 일시** *(필수)*
- **종료 일시** — **비우면 상시 진행**입니다.
- **당첨자 발표** — 상세 페이지에 발표 예정일을 안내하는 **날짜 표시일 뿐**입니다. (실제 당첨자를 뽑아주지 않습니다)
- **참여 방식** — **공지형** 또는 **응모**
- **참여 시 로그인 필요**
- **선착순 인원** — **비우면 무제한**

---

## 참여 방식은 두 가지뿐입니다

| 참여 방식 | 하는 일 |
|---|---|
| **공지형** | 알리기만 합니다. **참여 버튼이 없습니다.** |
| **응모** | 고객이 **참여하기** 버튼을 눌러 응모합니다. 1인 1회. |

목록에서 **쿠폰팩(준비중)**, **출석체크(준비중)**, **구매인증(준비중)** 이라고 적힌 이벤트를 볼 수도 있습니다. 이 세 가지는 **아직 동작하지 않아서 등록 화면에서 고를 수 없습니다.** 골라도 아무 일이 일어나지 않기 때문에 일부러 막아둔 것입니다.

---

## 고객에게는 이렇게 보입니다

**이벤트 & 혜택(`/event`)** 은 이벤트만 모아둔 곳이 아니라 **혜택 허브**입니다. 이벤트 목록과 **멤버십 등급 안내**가 함께 보입니다. 그래서 **발행된 이벤트가 하나도 없어도 이 페이지는 정상적으로 열립니다.** (목록 자리에만 "없음" 문구가 뜹니다)

- 고객은 **전체 · 진행중 · 예정 · 종료** 로 걸러 볼 수 있습니다.
- 목록은 **진행중 → 예정 → 종료** 순서로 정렬됩니다.
- 이벤트를 누르면 상세 페이지(`/event/슬러그`)가 열립니다.

**응모형**에서 고객이 참여하기를 누르면 결과에 따라 이런 메시지가 뜹니다.

| 상황 | 고객에게 보이는 문구 |
|---|---|
| 성공 | 참여가 완료되었습니다. |
| 이미 참여함 | 이미 참여하셨습니다. |
| 정원 마감 | 선착순이 마감되었습니다. |
| 기간이 아님 | 지금은 참여할 수 없는 이벤트입니다. |
| 로그인 안 함 | 로그인 후 참여할 수 있습니다. |

선착순 인원을 정했다면 상세 페이지에 **"선착순 N명 중 M명 참여"** 가 함께 보입니다.

---

## ⚠️ 삭제하면 참여자 기록도 함께 사라집니다

**이벤트를 삭제하면 그 이벤트에 응모한 참여자 기록도 전부 함께 지워집니다. 되돌릴 수 없습니다.**

> 응모를 받은 이벤트는 **삭제하지 말고 "숨김"으로 내리세요.** 삭제하면 누가 응모했는지 확인할 방법이 완전히 없어집니다. (참여자 명단을 따로 볼 수 있는 화면도 없으므로, 지우는 순간 끝입니다)

---

## ⚠️ 그 밖에 주의할 점

- **"목록에 노출" 체크를 풀어도 완전히 숨겨지지 않습니다.** 목록에서만 빠질 뿐, **주소를 아는 사람은 그대로 들어갈 수 있습니다.** 진짜로 감추려면 운영 상태를 **숨김**으로 바꾸세요. (반대로, 링크를 아는 사람만 참여하는 비공개 이벤트를 만들 때 이 성질을 이용할 수 있습니다)
- **슬러그를 바꾸면 기존 링크가 깨집니다.** 이미 배포한 주소가 있다면 슬러그는 건드리지 마세요.
- **같은 슬러그는 만들 수 없습니다.** 중복이면 "같은 슬러그의 이벤트가 이미 있습니다." 라는 안내가 뜹니다.
- **이벤트는 현재 편집 중인 몰에만 등록됩니다.** 다른 몰에는 나타나지 않습니다. 이벤트가 안 보인다면 상단에서 **편집 중인 몰**이 맞는지 먼저 확인하세요.
- 수정 화면의 참여 인원 숫자가 실제 참여 기록과 어긋나면 **빨간 글씨로 불일치 안내**가 뜹니다. 이 경우 숫자를 그대로 믿지 마세요.

---

## 아직 안 되는 것

- **응모자 명단을 볼 수 없습니다.** 화면에는 **참여 인원 숫자만** 나옵니다. 누가 응모했는지 조회하거나 내려받는 기능이 없습니다.
- **당첨자를 뽑을 수 없습니다.** "당첨자 발표"는 상세 페이지에 날짜를 적어주는 **안내 문구**일 뿐이고, 당첨자를 선정하거나 표시하는 기능이 없습니다.
- **쿠폰팩 · 출석체크 · 구매인증** 참여 방식은 준비 중이라 고를 수 없습니다. 특히 **이벤트로 쿠폰을 자동 지급할 수 없습니다.** 쿠폰을 주려면 **쿠폰 관리**에서 따로 지급해야 합니다.
- **이벤트로 포인트를 자동 지급할 수 없습니다.** **포인트 관리**에서 수동으로 지급해야 합니다.
- **이미지 업로드가 없습니다.** 주소를 직접 입력해야 합니다.
- **고객의 참여를 취소하거나 되돌릴 수 없습니다.**
- **이벤트 복사(복제) 기능이 없습니다.** 비슷한 이벤트도 매번 새로 입력해야 합니다.
