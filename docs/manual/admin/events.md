# 이벤트 관리

고객 화면의 **이벤트 페이지(`/event`)에 올라갈 이벤트**를 만드는 메뉴입니다. 단순히 알리기만 하는 **공지형** 이벤트와, 고객이 참여 버튼을 눌러 접수하는 **응모형** 이벤트를 등록할 수 있습니다.

화면 위치: <span class="manual-path">관리자 &gt; 프로모션 관리 &gt; 이벤트 관리</span> (`/admin/events`)

> **이벤트는 "응모를 받는 곳"까지입니다.** 참여 접수는 시스템이 자동으로 처리하지만, **당첨자 선정과 경품 지급은 자동으로 되지 않습니다.** 당첨자에게 쿠폰이나 포인트를 주려면 쿠폰 관리·포인트 관리에서 **운영자가 직접 지급**해야 합니다.

## 1. 시작하기 전에 — 참여 방식은 2가지뿐입니다

이벤트 등록 화면의 **참여 방식**에서 고를 수 있는 것은 딱 두 가지입니다.

| 참여 방식 | 고객 화면 | 언제 쓰나 |
|---|---|---|
| **공지형** | **참여 버튼이 없습니다.** 내용만 보여 줍니다. | 행사 안내, 쿠폰 사용법 안내, 당첨자 발표 게시 |
| **응모** | 참여 버튼이 있고, 누르면 응모가 접수됩니다. | 경품 추첨, 사전 신청 접수 |

> **출석체크·쿠폰팩·구매인증 같은 방식은 아직 선택할 수 없습니다.** 목록 화면에서 예전 데이터가 `(준비중)` 표시와 함께 보일 수는 있지만, 등록 화면의 참여 방식 목록에는 나오지 않습니다. **선택할 수 있는데 아무 일도 안 일어나는 것보다, 아예 안 보이게 막아 둔 것입니다.**

### 준비물

| 준비물 | 왜 필요한가 | 어디서 |
|---|---|---|
| **이미지 3종의 주소(URL)** | 목록 썸네일 / PC 대표 / 모바일 대표. **이 화면에는 업로드 기능이 없습니다.** | 배너 관리에서 올린 뒤 경로를 복사 |
| **경품·혜택 계획** | 응모형이라면 당첨자에게 무엇을 줄지 미리 정해야 합니다. | 쿠폰 관리 / 포인트 관리 |

## 2. 전체 흐름 한눈에 보기

```
① 이미지 준비          (배너 관리에서 올리고 주소 복사)
        ↓
② 이벤트 등록          (제목·본문·기간·참여 방식)
        ↓
③ 운영 상태를 '발행' + '이벤트 목록에 노출' 켜기
        ↓
④ 고객이 응모          (응모형인 경우)
        ↓
⑤ 참여 수 확인 → 당첨자 선정(수동) → 쿠폰·포인트 지급(수동)
```

## 3. 1단계 — 이벤트 등록하기

<div class="manual-goto-bar">
  <span class="manual-goto-label">이 단계 화면</span>
  <a class="manual-goto" href="/admin/events/add" target="_blank">이벤트 등록 화면 열기</a>
</div>

화면은 왼쪽(기본 정보·이미지)과 오른쪽(발행·기간·참여 설정)으로 나뉩니다.

### 3-1. 기본 정보

| 입력란 | 필수 | 설명 | 예시 |
|---|---|---|---|
| **이벤트명** | 필수 | 고객에게 보이는 제목. 최대 200자. | `가을맞이 홍삼 응모 이벤트` |
| **슬러그 (URL)** | 선택 | 주소 `/event/` 뒤에 붙습니다. **비우면 이벤트명에서 자동 생성**됩니다. | `autumn-hongsam` |
| **한 줄 요약** | 선택 | 목록에서 제목 아래 한 줄로 보입니다. 최대 500자. | `추첨을 통해 100분께 홍삼정 증정` |
| **상세 본문** | 선택 | 이벤트 페이지의 본문. 에디터로 이미지·표를 넣을 수 있습니다. | — |
| **유의사항** | 선택 | 참여 조건·당첨자 발표 방법 등 반드시 알릴 내용. | — |

> **슬러그를 나중에 바꾸지 마세요.** 고객에게 링크를 뿌린 뒤 이 값을 바꾸면 **기존 링크가 전부 깨집니다.** 문자·카카오톡으로 발송한 링크도 마찬가지입니다.

> **같은 슬러그의 이벤트는 두 개 만들 수 없습니다.** 중복이면 *"같은 슬러그의 이벤트가 이미 있습니다."* 라는 오류로 저장이 막힙니다. 지난해 이벤트를 같은 이름으로 다시 열 때 자주 걸리니, `2026-autumn-hongsam` 처럼 연도를 붙이세요.

### 3-2. 이미지 3종

**이 화면에는 파일 업로드 기능이 없습니다. 이미지 주소(URL)를 직접 입력합니다.** 배너 관리에서 이미지를 올린 뒤 그 경로를 복사해 붙여넣으세요.

| 입력란 | 어디에 보이나 | 없으면 |
|---|---|---|
| **목록 썸네일** | 이벤트 목록의 카드 이미지 | 회색 빈 칸으로 보입니다 |
| **상세 PC 대표 이미지** | 이벤트 상세 상단의 큰 이미지 | 상단이 비어 허전해집니다 |
| **상세 모바일 대표 이미지** | 모바일 전용 상단 이미지 | 모바일에서 가로 이미지가 잘려 보일 수 있습니다 |

### 3-3. 발행 (오른쪽 위)

| 항목 | 설명 |
|---|---|
| **운영 상태** | 아래 표 참고 |
| **이벤트 목록에 노출** | 체크박스. 끄면 목록에서 빠집니다. 기본은 켜짐. |

**운영 상태 3가지**

| 상태 | 의미 |
|---|---|
| **임시저장 (고객 미노출)** | 준비 중. 고객에게 보이지 않습니다. 새 이벤트의 기본값입니다. |
| **발행** | 고객에게 공개됩니다. |
| **숨김** | 공개를 잠시 내립니다. 내용은 그대로 남습니다. |

> **고객에게 보이려면 `운영 상태 = 발행` + `이벤트 목록에 노출 = 켜짐` 두 가지가 모두** 필요합니다. 하나만 해 두고 "안 보인다"고 하는 경우가 가장 많습니다.

### 3-4. 기간

| 입력란 | 필수 | 설명 |
|---|---|---|
| **시작 일시** | **필수** | 날짜와 시각을 함께 지정합니다. |
| **종료 일시** | 선택 | **비우면 상시 진행**됩니다. |
| **당첨자 발표** | 선택 | 발표 예정 일시. |

> **`예정 / 진행중 / 종료`는 직접 고르는 값이 아닙니다.** 시작·종료 일시에서 **자동으로 계산**됩니다. 화면에도 *"예정·진행중·종료는 아래 기간에서 자동 계산됩니다."* 라고 안내됩니다. 즉 **상태를 `발행`으로 해 두어도 시작 일시가 미래면 아직 `예정`** 이고, 응모 버튼이 동작하지 않습니다.

### 3-5. 참여 설정

| 입력란 | 설명 |
|---|---|
| **참여 방식** | `공지형` 또는 `응모`. 공지형은 참여 버튼이 없습니다. |
| **참여 시 로그인 필요** | 체크박스. 기본은 켜짐. |
| **선착순 인원** | **비우면 무제한.** 이 인원을 채우면 더 이상 응모되지 않습니다. |

수정 화면에서는 아래에 **현재 참여 N명** 이 표시됩니다.

> **선착순 인원은 되도록 지정하세요.** 경품 수량이 정해져 있는데 응모를 무제한으로 받으면, 나중에 "응모했는데 왜 못 받느냐"는 문의를 전부 수동으로 처리해야 합니다.

> **참여 시 로그인 필요를 끄면 같은 사람이 여러 번 응모할 수 있습니다.** 중복 참여는 **회원 계정 기준**으로만 막힙니다. 경품이 걸린 이벤트라면 켜 두세요.

**저장** 버튼을 누르면 등록되고 수정 화면으로 이동합니다.

## 4. 목록 화면 읽는 법

<div class="manual-goto-bar">
  <span class="manual-goto-label">이 단계 화면</span>
  <a class="manual-goto" href="/admin/events" target="_blank">이벤트 목록 열기</a>

**검색**: 운영 상태 / 이벤트명·슬러그로 좁힐 수 있습니다.

| 열 | 의미 |
|---|---|
| **썸네일** | 목록 썸네일 이미지. 없으면 회색 칸 |
| **이벤트명** | 아래 회색 글씨는 고객 주소(`/event/슬러그`) |
| **유형** | `공지형` / `응모`. 예전 데이터면 `(준비중)` 이 붙을 수 있습니다 |
| **운영상태** | `DRAFT`(임시저장) / `PUBLISHED`(발행) / `HIDDEN`(숨김) |
| **노출상태** | `예정` / `진행중` / `종료` — **기간에서 자동 계산된 값** |
| **기간** | 시작일 ~ 종료일. 종료일이 없으면 `상시` |
| **참여** | `참여 수` 또는 `참여 수 / 선착순 인원` |

> **운영상태와 노출상태는 다른 축입니다.** 운영상태가 `PUBLISHED`여도 노출상태가 `예정`이면 아직 시작 전이라 응모가 되지 않고, `종료`면 이미 끝난 것입니다. "발행했는데 참여가 안 된다"면 **노출상태 열을 먼저 보세요.**

**삭제** 버튼을 누르면 확인창이 뜹니다.

> ⚠️ **이벤트를 삭제하면 참여자 기록도 함께 지워집니다.** 응모자 명단이 영구히 사라지므로 되돌릴 수 없습니다. 끝난 이벤트는 삭제하지 말고 **운영 상태를 `숨김`으로** 두세요. 그래야 나중에 참여 이력을 확인할 수 있습니다.

## 5. 응모형 이벤트 운영하기

### 5-1. 고객이 참여하면 무슨 일이 일어나나

응모 버튼을 누른 순간 시스템이 아래를 한 번에 검사합니다.

| 검사 항목 | 통과하지 못하면 |
|---|---|
| 운영 상태가 `발행`인가 | 참여 불가 |
| 참여 방식이 `응모`인가 | 참여 버튼 자체가 없음 |
| 시작 일시가 지났고 종료 전인가 | 참여 불가 |
| 선착순 인원이 남았는가 | **마감** 처리 |
| 이미 참여하지 않았는가 | **중복 참여** 처리 |

전부 통과하면 참여자로 기록되고 참여 수가 1 올라갑니다.

> **선착순은 동시에 몰려도 초과되지 않습니다.** 100명 정원에 101명이 동시에 눌러도 100명까지만 접수됩니다. 안심하고 선착순 인원을 지정하세요.

### 5-2. 당첨자 선정과 경품 지급은 수동입니다

**여기가 이 기능의 가장 중요한 한계입니다.**

- 관리자 화면에서 **응모자 명단을 조회할 수 없습니다.** 볼 수 있는 것은 **참여 인원 수**뿐입니다.
- **당첨자를 자동으로 뽑아 주지 않습니다.**
- **경품(쿠폰·포인트)을 자동으로 지급하지 않습니다.**

따라서 실제 운영은 이렇게 됩니다.

1. 참여 수를 확인합니다.
2. 당첨자를 별도의 방법으로 정합니다.
3. **쿠폰 관리**의 쿠폰 지급, 또는 **포인트 관리**의 포인트 지급으로 **직접 지급**합니다.
4. 당첨자 발표는 **공지형 이벤트**나 공지사항으로 별도 게시합니다.

  <a class="manual-goto is-sub" href="/admin/notices" target="_blank">공지사항으로 당첨자 발표</a>

> ⚠️ **쿠폰·포인트 지급은 실제 돈이 나가는 작업입니다.** 당첨자 명단을 정확히 확인하고 지급하세요. **포인트 지급에는 취소 버튼이 없고, 쿠폰은 개별 회수가 불가능합니다.**

> **참여 수와 참여자 테이블 건수가 다르면** 수정 화면에 빨간 글씨로 *"(참여자 테이블 N건 — 불일치)"* 가 표시됩니다. 정상 운영에서는 나타나지 않는 경고이니, 보이면 기록해 두세요.

## 6. 자주 막히는 상황

**Q. 이벤트를 만들었는데 고객 화면에 안 보입니다.**
아래를 순서대로 확인하세요.
1. **운영 상태**가 `발행`인가? (`임시저장`이 기본값입니다)
2. **이벤트 목록에 노출** 체크가 켜져 있는가?
3. **시작 일시**가 이미 지났는가? (미래면 목록에 `예정`으로 나옵니다)
4. **편집 중인 몰**이 맞는가?

**Q. `발행`으로 했는데 참여 버튼이 눌리지 않습니다.**
① 참여 방식이 `공지형`이면 애초에 참여 버튼이 없습니다. ② 노출상태가 `예정`(시작 전)이거나 `종료`면 참여할 수 없습니다. ③ 선착순 인원이 다 찼을 수 있습니다.

**Q. 저장하려니 "같은 슬러그의 이벤트가 이미 있습니다"가 뜹니다.**
같은 주소를 쓰는 이벤트가 이미 있습니다. 슬러그를 다르게 지정하세요. (연도를 붙이는 것을 권장합니다.)

**Q. "이벤트명은 필수입니다" / "시작일시는 필수입니다"가 뜹니다.**
이 둘은 반드시 채워야 저장됩니다. 종료 일시는 비워도 됩니다.

**Q. 이미지 업로드 버튼이 없습니다.**
이 화면은 이미지 **주소(URL)만 입력**받습니다. 배너 관리에서 이미지를 올린 뒤 경로를 복사해 붙여넣으세요.

**Q. 응모자 명단을 보고 싶습니다.**
현재 관리자 화면에서는 **참여 인원 수만** 확인할 수 있습니다. 명단 조회 기능이 없습니다.

**Q. 당첨자에게 쿠폰이 자동으로 나가나요?**
나가지 않습니다. 쿠폰 관리 또는 포인트 관리에서 **직접 지급**해야 합니다.

**Q. 출석체크 이벤트를 만들고 싶습니다.**
참여 방식 목록에 없습니다. 현재는 `공지형`과 `응모` 두 가지만 지원합니다.

**Q. 끝난 이벤트를 정리하고 싶습니다.**
**삭제하지 마세요.** 참여자 기록이 함께 사라집니다. 운영 상태를 `숨김`으로 바꾸거나, `이벤트 목록에 노출`만 끄세요.

**Q. 종료일을 비워 두면 어떻게 되나요?**
`상시` 진행이 되어 계속 `진행중` 상태로 남습니다. 응모형이라면 선착순 인원을 채울 때까지 응모가 계속 들어옵니다.

---

## 7. 아직 지원하지 않는 것

- **응모자 명단 조회·내려받기** (참여 인원 수만 볼 수 있습니다)
- **당첨자 자동 추첨**·당첨 처리
- **경품 자동 지급** (쿠폰·포인트를 직접 지급해야 합니다)
- 출석체크 / 쿠폰팩 / 구매인증 참여 방식
- 이벤트 화면 내 이미지 직접 업로드
