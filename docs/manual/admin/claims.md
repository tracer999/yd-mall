# 클레임 관리 (취소 · 반품 · 환불)

고객이 신청한 **주문 취소**와 **반품**을 확인하고, **승인 또는 거절**하는 화면입니다. 승인하면 재고·쿠폰·포인트가 복원되고 **결제 취소(환불)까지 한 번에 실행**됩니다.

> ⚠️ **승인 버튼은 되돌릴 수 없습니다.** 누르는 순간 **고객 카드로 실제 환불이 나가고**, 재고·쿠폰·포인트가 복원되며, 주문이 `취소`로 확정됩니다. 취소 버튼도, 되돌리기 기능도 없습니다. **금액과 귀책을 확인한 다음에 누르세요.**

## 1. 시작하기 전에 — 알아야 할 3가지

### ① 클레임은 관리자가 만들 수 없습니다

목록에 뜨는 건은 전부 **고객이 마이페이지에서 신청한 것**입니다. 관리자는 신청된 건을 **처리만** 합니다.

### ② 고객이 언제 무엇을 신청할 수 있는지

| 주문 상태 | 신청 가능한 것 |
|---|---|
| **대기 · 결제완료 · 배송준비** | **취소** |
| **배송중 · 배송완료** | **반품** — 단, 배송완료 처리 후 **7일 이내**만 |
| **취소 · 환불** | 신청 불가 (이미 취소된 주문) |

- 이미 처리 중인 클레임이 있으면 같은 주문에 또 신청할 수 없습니다.
- 반품 가능 기간 7일은 **배송 관리에서 `배송완료 처리`를 누른 시각**부터 셉니다.
- **교환은 아직 지원하지 않습니다.** 교환을 원하는 고객에게는 **반품 후 재주문**을 안내하세요.

### ③ 출고 전 취소는 자동으로 끝납니다

주문이 **`대기` 또는 `결제완료`** 상태일 때 고객이 취소를 신청하면, **관리자 승인 없이 그 자리에서 자동 승인·환불까지 완료**됩니다. 목록에는 이미 `완료` 상태로 나타납니다.

즉 **이 화면에서 실제로 손댈 일은 두 가지**입니다.

```
① 배송준비 이후의 취소 신청   → 승인 여부 판단
② 모든 반품 신청              → 물건을 회수해야 하므로 항상 승인 대상
```

## 2. 클레임 목록 보기

### 검색 조건

| 조건 | 고를 수 있는 값 |
|---|---|
| **상태** | 전체 / 접수 / 승인 / 완료 / 거절 / 철회 |
| **유형** | 전체 / 취소 / 반품 |

값을 고르고 **`검색`** 을 누릅니다.

### 목록 열

| 열 | 내용 |
|---|---|
| **신청일** | 고객이 신청한 날짜와 시각. |
| **주문번호** | 어떤 주문에 대한 클레임인지. |
| **고객** | 신청한 회원 이름(비회원이면 `비회원`). |
| **유형** | `취소` 또는 `반품` 배지. |
| **사유** | 단순 변심 / 불량 / 오배송 / 기타. |
| **금액** | 그 주문의 결제금액. |
| **상태** | 아래 표 참고. |
| **처리** | `접수` 상태면 `처리`, 그 외엔 `상세` 버튼. |

### 상태 5가지

| 상태 | 뜻 | 할 일 |
|---|---|---|
| **접수** | 고객이 신청했고 **아직 처리되지 않았습니다.** | **이 건들을 처리해야 합니다.** |
| **승인** | 승인 처리가 진행 중인 상태입니다. | — |
| **완료** | 승인·복원·환불까지 끝났습니다. | 없음 |
| **거절** | 관리자가 거절했습니다. 주문은 원래 상태 그대로입니다. | 없음 |
| **철회** | **고객이 스스로 신청을 취소**했습니다. | 없음 |

**`접수` 건은 목록 맨 위에 모이고 줄 배경이 옅은 노란색으로 표시됩니다.** 매일 이 건들만 확인하면 됩니다.

> **"목록에서 건이 사라졌어요"** — 고객이 승인 전에 **철회**했을 가능성이 큽니다. 상태 필터를 `철회`로 놓고 검색해 보세요.

## 3. 클레임 상세 화면

목록에서 `처리` 또는 `상세`를 누르면 열립니다.

**왼쪽에 확인할 정보, 오른쪽에 처리 패널**이 있습니다.

### 3-1. 왼쪽 — 확인할 정보

- **신청 정보** — 신청자(관리자 신청이면 `관리자`), 신청일, 사유, 주문 상태. 고객이 적은 **상세 사유**가 있으면 회색 상자로, 관리자 메모가 있으면 파란 상자로 표시됩니다.
- **주문 상품** — 상품명 × 수량과 금액. 아래에 상품금액 / 배송비 / **결제금액**이 정리됩니다.
- **환불 내역** — 환불이 발생한 뒤에만 나타납니다. 금액, 방식(`PG 취소` / `수동 환불` / `결제 없음`), 반품비 차감액, 상태(요청·완료·실패)를 보여 줍니다.

### 3-2. 오른쪽 — 처리 패널

상태가 **`접수`일 때만** 처리 폼이 나옵니다. 이미 처리된 건은 "처리 완료"와 처리일만 표시됩니다.

## 4. 승인하기 (되돌릴 수 없음)

### 4-1. 입력할 값

| 입력란 | 언제 나오나 | 설명 |
|---|---|---|
| **귀책** | 항상 | `고객 (단순 변심 — 반품비 청구)` 또는 `판매자 (불량·오배송 — 전액 환불)` 중 선택. |
| **반품 배송비 (원)** | **반품일 때만** | 환불액에서 차감할 금액. 자동으로 채워집니다. |
| **메모** | 항상 | 처리 사유를 적어 둡니다. 나중에 상세에서 확인할 수 있습니다. |

그 아래 초록 상자에 **예상 환불액**이 계산되어 나옵니다.

```
예상 환불액 = 결제금액 − 반품 배송비
```

### 4-2. 귀책을 어떻게 정하나

**귀책은 곧 "반품 배송비를 누가 내는가"입니다.**

| 귀책 | 이런 경우 | 반품 배송비 |
|---|---|---|
| **고객** | 단순 변심, 색상·사이즈가 마음에 안 듦, 잘못 주문 | **고객에게 청구** (환불액에서 차감) |
| **판매자** | 상품 불량·파손, 다른 상품이 왔음 | **청구하지 않음** (0원, 전액 환불) |

고객이 신청할 때 고른 사유에 따라 **처음부터 자동으로 잡혀 있습니다.**

- 사유가 `불량` 또는 `오배송` → **판매자** 귀책
- 사유가 `단순 변심` 또는 `기타` → **고객** 귀책

관리자가 판단해서 **바꿀 수 있습니다.** 셀렉트를 `판매자`로 바꾸면 반품 배송비 칸이 자동으로 `0`이 되고, `고객`으로 되돌리면 추천 금액이 다시 채워집니다.

> **귀책은 승인 전에 확정하세요.** 승인 후에는 귀책도 반품 배송비도 고칠 수 없습니다. 애매하면 고객에게 사진을 요청한 뒤 판단하세요.

### 4-3. 반품 배송비 기본값

**반품이면서 고객 귀책**일 때만 금액이 붙습니다. 기본값은 **그 몰의 기본 배송비 × 2**(왕복)입니다.

기본 배송비가 3,000원이면 반품 배송비 기본값은 **6,000원**입니다. 화면에서 직접 다른 금액으로 고칠 수 있습니다.

### 4-4. `승인 (복원 + 환불)` 을 누르면 일어나는 일

> ⚠️ **확인 창 없이 즉시 실행됩니다. 되돌릴 수 없습니다.**

한 번에 다음이 진행됩니다.

1. **재고 복원** — 팔린 수량이 상품 재고로 되돌아갑니다.
2. **쿠폰 복원** — 이 주문에 쓴 쿠폰이 다시 사용 가능해집니다.
3. **포인트 복원** — 고객이 쓴 적립금은 돌려주고, 이 주문으로 지급했던 구매적립은 회수합니다.
4. **특가 수량 복원** — 선착순 특가 소진분이 되돌아갑니다.
5. **멤버십 실적 차감** — 이 주문으로 쌓인 등급 실적이 빠집니다. (등급 자체는 이 자리에서 내려가지 않고 다음 정기 평가에서 반영됩니다.)
6. **결제 취소** — 결제사(토스)로 `결제금액 − 반품 배송비` 만큼 취소 요청이 나갑니다. **실제로 고객에게 돈이 돌아갑니다.**
7. **상태 확정** — 클레임이 `완료`, 주문이 `취소`로 바뀌고 이력에 남습니다.

**결제 내역이 없는 주문**(테스트 결제, 전액 포인트·쿠폰으로 0원 결제 등)은 6번이 생략됩니다. 이 경우 승인 화면 초록 상자에 "PG 결제 내역이 없어 실제 환불은 발생하지 않습니다(재고·쿠폰·적립금만 복원)" 라는 안내가 미리 나옵니다.

> **반품은 "물건을 돌려받는 것"이 먼저입니다.** 승인은 곧 환불입니다. **상품이 실제로 회수되어 상태를 확인한 뒤에 승인**하세요. 먼저 승인하면 돈은 나갔는데 물건은 안 오는 상황이 생깁니다.

## 5. 거절하기

승인 폼 아래에 **거절 폼**이 있습니다. **거절 사유**를 적고 `거절`을 누르면 확인 창이 뜹니다.

> "이 클레임을 거절할까요?"

거절하면 **클레임만 닫히고 주문 상태는 그대로 유지**됩니다. 환불도, 재고 복원도 일어나지 않습니다.

거절이 맞는 경우는 예를 들면 이렇습니다.

- 반품 가능 기간(수령 후 7일)이 지난 신청
- 사용 흔적이 뚜렷해 재판매가 불가능한 상품
- 이미 다른 방법으로 처리된 건

> **거절 사유를 반드시 적으세요.** 비워도 저장은 되지만, 나중에 왜 거절했는지 확인할 방법이 없어집니다. 고객 응대에서 그대로 문제가 됩니다.

**이미 처리된 클레임(완료·거절·철회)은 다시 승인하거나 거절할 수 없습니다.** "이미 처리된 클레임입니다" 라는 오류가 나옵니다.

## 6. 환불이 실패했을 때 (반드시 확인)

결제사 쪽 사정으로 환불 요청이 실패할 수 있습니다. 이때 **클레임은 이미 승인·완료로 확정되고, 환불만 `실패`로 남습니다.**

> ⚠️ **이 상태는 "주문은 취소됐는데 고객 돈은 그대로 있는" 상태입니다.** 그냥 두면 고객은 영영 환불을 못 받습니다.

### 어떻게 알아채나

- 승인 직후 상세 화면 위에 주황색 안내가 뜹니다 — *"승인은 처리됐으나 PG 환불에 실패했습니다. 아래 환불 내역에서 수동 환불로 마감하세요."*
- 클레임 상세의 **환불 내역**에 빨간 `실패` 배지와 실패 사유가 표시됩니다.
- 주문 상세(판매 관리)의 **시스템 정보 → 환불** 항목이 `실패`로 나옵니다.

### 처리 순서

1. **먼저 실제로 돈을 보냅니다.** 고객에게 환불 계좌를 받아 송금하거나, 결제사 관리자 화면에서 직접 취소합니다.
2. 클레임 상세의 환불 내역에서 실패한 줄 옆 **`처리 메모`** 칸에 처리 내용을 적습니다. (예: `2026-07-20 계좌환불 완료`)
3. **`수동 완료`** 버튼을 누릅니다.

> **`수동 완료`는 돈을 보내는 기능이 아닙니다.** 이미 밖에서 처리한 환불을 **기록으로만 남기는 버튼**입니다. 송금하지 않은 채 누르면 시스템상으로는 환불 완료인데 고객은 돈을 못 받은 상태가 되고, 아무도 눈치채지 못합니다. **반드시 송금을 먼저 하세요.**

## 7. 자주 막히는 상황

**Q. 고객이 취소를 신청했다는데 목록에 없습니다.**
`대기`·`결제완료` 주문의 취소는 **자동 승인**되어 곧바로 `완료`로 넘어갑니다. 상태 필터를 `전체`나 `완료`로 놓고 검색하세요.

**Q. 고객이 취소 신청을 못 한다고 합니다.**
주문 상태를 확인하세요. `배송중`·`배송완료`에서는 취소가 아니라 **반품**만 신청할 수 있습니다. 반품도 배송완료 후 7일이 지나면 신청할 수 없습니다.

**Q. 배송 전인데 고객이 반품밖에 못 신청합니다.**
송장을 등록해 주문이 이미 `배송중`이 된 상태입니다. 정상 동작입니다.

**Q. 승인했는데 재고가 안 늘었습니다.**
`대기` 상태였던 주문은 애초에 재고가 차감되지 않았으므로 복원할 것이 없습니다. 상품 관리에서 실제 재고를 확인해 보세요.

**Q. 환불액이 결제금액보다 적습니다.**
반품 배송비가 차감된 것입니다. 클레임 상세 환불 내역에 `(반품비 -6,000)` 처럼 표시됩니다.

**Q. 판매자 잘못인데 고객에게 반품비가 청구됐습니다.**
승인 전에 귀책을 `판매자`로 바꿨어야 합니다. 승인 후에는 고칠 수 없으므로, 차액을 별도로 환불하거나 쿠폰·포인트로 보상해야 합니다.

**Q. 같은 주문을 판매 관리에서도 취소했습니다. 재고가 두 번 늘어나나요?**
아닙니다. 복원은 주문당 한 번만 실행되도록 잠금이 걸려 있습니다. 다만 사유가 남지 않으니 **클레임으로 처리하는 습관**을 들이세요.

**Q. 부분 취소(일부 상품만 취소)를 하고 싶습니다.**
현재 지원하지 않습니다. 클레임은 **주문 단위**로만 처리됩니다.

---

## 8. 아직 지원하지 않는 것

- **교환** — 반품 후 재주문으로 안내해야 합니다
- **부분 취소·부분 반품** — 주문 단위로만 처리
- 관리자가 직접 클레임 생성
- 승인 취소(되돌리기)
- 반품 회수 송장 등록·회수 추적
- 클레임 처리 결과 자동 알림 발송
