# 관리자 로그인 · 로그아웃

관리자 화면에 들어가는 문입니다. 아이디와 비밀번호로 들어가며, 계정에 **이중인증(2단계 인증)** 이 켜져 있으면 이메일로 받은 6자리 숫자를 한 번 더 입력해야 합니다.

화면 위치: <span class="manual-path">관리자 로그인</span> (`/admin/login`)

> **이 화면은 관리자 전용입니다.** 고객이 쓰는 로그인과 완전히 다른 문이며, 계정도 서로 다릅니다. 고객 계정으로는 관리자 화면에 들어올 수 없습니다.

---

## 1. 로그인 화면의 생김새

로그인 화면 위쪽에는 **사이트 설정에 등록한 로고**가 뜹니다. 로고가 없으면 대신 **회사명(상호)** 이 글자로 나옵니다. 카드 아래쪽에는 **헤더 슬로건**과 회사명이 표시됩니다.

즉 이 화면이 허전하거나 엉뚱한 회사 이름이 떠 있다면 로그인 기능의 문제가 아니라 **사이트 설정이 비어 있는 것**입니다.

## 2. 전체 흐름 한눈에 보기

```
① 아이디 · 비밀번호 입력 → [로그인]
        ↓
   이중인증이 꺼진 계정 ──────────→ 바로 관리자 화면 진입
        ↓
   이중인증이 켜진 계정
        ↓
② 등록된 이메일로 6자리 코드 발송
        ↓
③ 코드 입력 → [인증 후 로그인]
        ↓
④ 관리자 화면 진입 (권한이 있는 첫 메뉴로 이동)
```

---

## 3. 1단계 — 아이디와 비밀번호

| 입력란 | 필수 | 설명 | 예시 |
|---|---|---|---|
| **아이디** | 필수 | 운영자 관리에서 만든 관리자 아이디입니다. 이메일이 아니라 아이디입니다. | `tracer999` |
| **비밀번호** | 필수 | 대소문자를 구분합니다. | — |

**로그인** 버튼을 누르면 확인이 시작됩니다. 확인하는 동안 버튼이 **로그인 중...** 으로 바뀌고 잠깁니다. 여러 번 누르지 마세요.

**여기서 나오는 오류 문구와 뜻**

| 화면에 뜨는 문구 | 뜻 | 할 일 |
|---|---|---|
| `존재하지 않는 계정입니다.` | 그 아이디가 아예 없습니다. | 오타를 확인하세요. 그래도 안 되면 다른 관리자에게 계정 생성을 요청하세요. |
| `비밀번호가 일치하지 않습니다.` | 아이디는 맞고 비밀번호만 틀렸습니다. | 대소문자·한영 전환을 확인하세요. |
| `이 계정은 이중인증이 설정되어 있으나 등록된 이메일이 없습니다. 관리자에게 이메일 등록을 요청하세요.` | 2단계 인증은 켜져 있는데 코드를 보낼 주소가 없습니다. | 다른 관리자가 운영자 관리에서 **이메일을 등록**해 주어야 합니다. |
| `이메일 발송에 실패했습니다. SMTP 설정을 확인하세요.` | 비밀번호는 맞았지만 인증 메일을 못 보냈습니다. | 시스템 설정의 **SMTP 메일 설정**을 점검하세요. |

> **비밀번호를 스스로 재발급하는 기능은 없습니다.** 이 화면에는 "비밀번호 찾기"가 없습니다. 다른 최고 관리자 계정으로 들어가 운영자 관리에서 비밀번호를 새로 지정해야 합니다. **관리자 계정을 하나만 두면 그 계정을 잃는 순간 들어갈 방법이 없습니다.** 반드시 2개 이상 만들어 두세요.

## 4. 2단계 — 이중인증(이메일 인증코드)

계정에 이중인증이 켜져 있으면, 비밀번호가 맞은 뒤 화면 제목이 **이중인증**으로 바뀌고 인증코드 입력칸이 나타납니다. 화면에는 `abc***@example.com 으로 6자리 인증코드를 발송했습니다` 처럼 **일부만 가린 이메일 주소**가 표시됩니다.

| 항목 | 내용 |
|---|---|
| **코드 형식** | 숫자 6자리 |
| **유효 시간** | **5분.** 지나면 처음(아이디 입력)부터 다시 해야 합니다. |
| **재사용** | 한 번 쓴 코드는 다시 쓸 수 없습니다. |
| **메일 제목** | `[회사명] 관리자 로그인 인증코드` |
| **버튼 문구** | **인증 후 로그인** |

**여기서 나오는 오류 문구**

| 문구 | 뜻 |
|---|---|
| `6자리 인증코드를 입력해 주세요.` | 자릿수가 모자라거나 숫자가 아닙니다. |
| `인증코드가 일치하지 않거나 만료되었습니다. 처음부터 다시 시도하세요.` | 코드가 틀렸거나 5분이 지났습니다. |

**아이디부터 다시 하려면** 코드 입력칸 아래의 **아이디/비밀번호 다시 입력** 링크를 누르세요. 진행 중이던 인증이 취소되고 1단계 화면으로 돌아갑니다.

> **메일을 여러 번 받았다면 반드시 가장 마지막 메일의 코드를 쓰세요.** 로그인을 다시 시도할 때마다 새 코드가 발송되며, 이전 메일의 숫자를 넣으면 "일치하지 않거나 만료되었습니다"가 뜹니다.

## 5. 이중인증 켜기 · 끄기

이중인증은 **계정마다 따로** 설정합니다. 로그인 화면에서는 바꿀 수 없고, **운영자 관리**에서 해당 계정을 열어 **이중인증 사용** 체크박스로 조절합니다.

- 새 운영자를 만들 때는 **기본으로 켜져 있습니다.**
- 켜 두려면 그 계정에 **이메일이 반드시 등록**되어 있어야 합니다.
- 끄면 아이디·비밀번호만으로 바로 들어갑니다.

<div class="manual-goto-bar">
  <span class="manual-goto-label">이중인증 설정</span>

> **모든 관리자 계정의 이중인증을 켜기 전에, 메일 발송이 실제로 되는지 먼저 확인하세요.** 메일이 안 나가는 상태에서 전부 켜면 **아무도 로그인할 수 없게 됩니다.** 확인은 시스템 설정의 **테스트 발송** 기능으로 합니다.

## 6. 로그인한 다음 — 어디로 가나

로그인에 성공하면 **접근 권한이 있는 첫 번째 메뉴**로 이동합니다. 보통은 **몰 관리** 화면입니다. 대시보드가 첫 화면이 아니라는 점에 주의하세요.

<div class="manual-goto-bar">
  <span class="manual-goto-label">로그인 후 자주 가는 곳</span>
  <a class="manual-goto is-sub" href="/admin/malls" target="_blank">몰 관리</a>
  <a class="manual-goto is-sub" href="/admin/dashboard" target="_blank">대시보드</a>
</div>

또 하나 중요한 것이 있습니다. 관리자 화면 **오른쪽 위에는 "지금 편집 중인 몰"** 이 표시됩니다. 상품·카테고리·메뉴·테마·페이지 빌더 작업은 전부 **여기 적힌 몰 기준으로 저장**됩니다. 로그인한 뒤 작업을 시작하기 전에 이 표시를 확인하는 습관을 들이세요.

---

## 7. 로그아웃

관리자 화면 오른쪽 위의 **로그아웃 아이콘**을 누르면 로그아웃되어 로그인 화면으로 돌아갑니다.

- 로그아웃하면 세션이 완전히 지워집니다. **편집 중인 몰 선택도 초기화**되어, 다음에 로그인하면 기본몰부터 시작합니다.
- **저장하지 않은 편집 내용은 사라집니다.** 작성 중인 상품·배너·페이지가 있으면 먼저 저장하세요.

> **공용 PC에서는 반드시 로그아웃하세요.** 브라우저 창만 닫으면 세션이 남아 있어 다음 사람이 그대로 들어갈 수 있습니다.

## 8. 자주 막히는 상황

**Q. 아이디·비밀번호가 맞는데 계속 로그인 화면으로 돌아옵니다.**
브라우저의 쿠키가 차단되어 세션이 유지되지 않는 경우입니다. 시크릿(프라이빗) 모드이거나 쿠키 차단 확장 프로그램이 켜져 있는지 확인하세요.

**Q. 인증코드 메일이 안 옵니다.**
① 스팸함 확인 → ② 운영자 관리에서 그 계정의 **이메일 주소가 맞는지** 확인 → ③ 시스템 설정의 SMTP 항목 확인 순서로 점검하세요.

**Q. 인증코드를 넣었는데 "만료되었습니다"가 뜹니다.**
5분이 지났거나, 메일을 여러 번 받은 뒤 **오래된 메일의 코드**를 입력한 경우입니다.

**Q. 로그인은 되는데 메뉴가 몇 개 안 보입니다.**
계정 권한(역할)에 따라 보이는 메뉴가 다릅니다. 운영자 관리에서 그 계정의 역할을 확인하세요.

**Q. 로그인했더니 지난번과 다른 몰이 편집 대상으로 잡혀 있습니다.**
로그아웃하면 편집 몰 선택이 초기화되어 **기본몰**로 돌아갑니다. 상단의 몰 선택기에서 원하는 몰로 바꾸세요.

**Q. 작업 도중에 갑자기 로그인 화면으로 튕깁니다.**
세션이 만료된 것입니다. 다시 로그인하면 되지만 **저장하지 않은 입력은 복구되지 않습니다.** 긴 글을 쓸 때는 중간중간 저장하세요.

**Q. 비밀번호를 아무도 모릅니다. 관리자 계정이 하나뿐입니다.**
화면에서 해결할 방법이 없습니다. 시스템 담당자가 직접 복구해야 합니다.

---

## 9. 아직 지원하지 않는 것

- 관리자 비밀번호 찾기·재발급(본인 요청)
- 로그인 실패 횟수에 따른 자동 계정 잠금
- 문자(SMS)·앱(OTP) 방식의 이중인증 — **이메일 방식만** 지원합니다
- 로그인 이력 조회 화면
