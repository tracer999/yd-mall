# 쿠폰 관리

고객이 결제할 때 **금액을 깎아 주는 쿠폰**을 만들고, 회원에게 지급하고, 얼마나 쓰였는지 확인하는 메뉴입니다. 신규 가입 축하 쿠폰, 이벤트 쿠폰, 쿠폰존에서 직접 받아 가는 다운로드 쿠폰, 배송비 무료 쿠폰까지 모두 여기서 만듭니다.

> ⚠️ **쿠폰은 고객이 실제로 내는 금액을 바꿉니다.** 잘못 설정하면 그대로 손실이 됩니다. 특히 **정률(%) 쿠폰의 최대 할인액**과 **최소 주문 금액**을 반드시 확인하세요. 뿌린 뒤에는 이미 받아 간 고객이 있어 되돌리기 어렵습니다.

> **쿠폰에는 삭제 버튼이 없습니다. 대신 종료합니다.** 삭제하면 회원이 보유한 쿠폰과 과거 사용 이력까지 함께 사라져 정산이 불가능해지기 때문입니다.

## 1. 시작하기 전에 — 쿠폰을 이루는 3개의 축

쿠폰 등록 화면이 복잡해 보이는 이유는 **성격이 다른 3가지를 한 번에 정하기 때문**입니다. 셋을 분리해 이해하면 화면이 쉬워집니다.

| 축 | 정하는 것 | 동작에 영향 |
|---|---|---|
| **목적 (분류)** | 이 쿠폰이 왜 있는가 | **없음.** 순수한 분류 라벨입니다 |
| **발급 방식** | 고객이 이 쿠폰을 **어떻게 받는가** | **큼.** 입력란과 동작이 바뀝니다 |
| **혜택 유형** | **무엇을 얼마나** 깎아 주는가 | **큼.** 입력란이 바뀝니다 |

> **목적(분류)은 아무 동작도 바꾸지 않습니다.** `신규가입`으로 골랐다고 가입 시 자동으로 나가지 않습니다. 자동 지급은 **발급 방식**에서 `가입 시 자동지급`을 골라야 합니다. 이 둘을 혼동해 "가입 쿠폰이 안 나간다"고 하는 경우가 가장 많습니다.

### 준비물

| 준비물 | 언제 필요한가 |
|---|---|
| **쿠폰 썸네일 이미지** | 쿠폰존 카드에 쓰입니다. 없으면 기본 아이콘이 나옵니다 |
| **적용할 카테고리·브랜드 이름** | 전 상품이 아니라 일부에만 적용할 때 |
| **지급할 회원 목록** | 발급 방식이 `관리자 수동지급`일 때 |

## 2. 전체 흐름 한눈에 보기

```
① 쿠폰 등록            (혜택·발급 방식·기간 설정)
        ↓
② 상태를 '진행중'으로   ('작성중'이면 아무 일도 일어나지 않습니다)
        ↓
③ 고객에게 전달
   · 가입 자동지급   → 가입하면 자동으로 나감
   · 관리자 수동지급 → '쿠폰 지급' 화면에서 직접 지급
   · 쿠폰 코드       → 고객에게 코드를 알림
   · 쿠폰존 다운로드 → 고객이 직접 받아 감
        ↓
④ 상세 화면에서 수령·사용·총 할인액 확인
        ↓
⑤ 행사가 끝나면 '종료'  (삭제 아님)
```

### 3-1. 기본 정보

| 입력란 | 필수 | 설명 | 예시 |
|---|---|---|---|
| **쿠폰명** | 필수 | 고객에게 그대로 보입니다. | `신규 가입 3,000원 할인` |
| **한 줄 소개 (요약)** | 선택 | 쿠폰존 카드에서 혜택 아래에 표시됩니다. 최대 255자. | `여름맞이 전상품 특가 쿠폰!` |
| **썸네일 이미지** | 선택 | JPG·PNG 파일을 고르면 자동 업로드됩니다. **이미지 제거**로 지웁니다. | — |
| **목적 (분류)** | 선택 | `신규가입` / `이벤트` / `시즌` / `특별`. **분류 라벨일 뿐 동작은 바뀌지 않습니다.** | `이벤트` |
| **몰** | 선택 | `전 몰 공용` 또는 특정 몰. | `전 몰 공용` |
| **상태** | 선택 | 아래 표 참고. 새 쿠폰의 기본값은 `진행중`입니다. | `작성중` |

**상태 4가지**

| 상태 | 동작 |
|---|---|
| **작성중 (노출 안 함)** | 준비 중. 고객에게 노출되지 않고 지급도 되지 않습니다. |
| **진행중** | **실제로 살아 있는 유일한 상태입니다.** 지급·수령·사용이 이 상태에서만 됩니다. |
| **일시중지** | 잠시 멈춥니다. 새 지급이 막힙니다. |
| **종료** | 끝난 쿠폰. 이미 발급된 쿠폰도 사용할 수 없게 됩니다. |

> **`진행중`이 아니면 쿠폰 지급 화면 목록에 아예 뜨지 않습니다.** "만든 쿠폰이 지급 목록에 없다"면 상태부터 확인하세요. 반대로 **새 쿠폰의 기본값이 `진행중`**이므로, 준비만 해 두려면 **직접 `작성중`으로 바꿔야** 합니다. 이걸 놓치면 미완성 쿠폰이 곧바로 살아납니다.

### 3-2. 상세 소개

쿠폰존 상세 페이지에 노출되는 내용입니다.

| 입력란 | 설명 |
|---|---|
| **상세 본문** | 쿠폰 소개 본문. 에디터로 이미지·표를 넣을 수 있습니다. |
| **유의사항** | 사용 조건·제외 대상 등 고객에게 반드시 알릴 내용. |

> **혜택 조건은 유의사항에 글로도 적으세요.** "5만원 이상 구매 시", "일부 상품 제외" 같은 조건을 시스템이 자동으로 안내 문구로 만들어 주지 않습니다. 적어 두지 않으면 "결제 화면에서 쿠폰이 안 먹힌다"는 문의가 몰립니다.

### 3-3. 혜택 — 여기가 금액에 직결됩니다

**혜택 유형 4가지**

| 유형 | 무엇을 깎나 | 추가 입력 |
|---|---|---|
| **정액 할인 (원)** | 주문 금액에서 정해진 금액 | 할인 금액 |
| **정률 할인 (%)** | 주문 금액의 비율 | 할인율 + **최대 할인액(필수)** |
| **무료배송** | 배송비 **전액** | 없음 (금액 입력란이 사라집니다) |
| **배송비 정액 할인 (원)** | 배송비에서 정해진 금액 | 할인 금액 |

> ⚠️ **정률(%) 쿠폰에는 최대 할인액이 반드시 필요합니다.** 비워 두면 *"정률 쿠폰에는 최대 할인액이 필요합니다"* 라는 오류로 저장이 막힙니다. **이 검증이 없다면 100만원짜리 주문에 10% 쿠폰이 붙어 10만원이 그대로 나가는 사고가 납니다.** 화면이 막아 주더라도, 최대 할인액은 **감당 가능한 금액**으로 직접 계산해 넣으세요.

> **할인율은 1~100 사이여야 합니다.** 벗어나면 저장되지 않습니다.

**최소 주문 금액 (원)** — 이 금액 이상일 때만 쿠폰이 적용됩니다. `0` 이면 제한 없음.

> ⚠️ **최소 주문 금액이 0인 정액 쿠폰은 위험합니다.** 3,000원 할인 쿠폰에 최소 주문 금액이 없으면 3,000원짜리 상품을 **거의 0원에** 사 갈 수 있습니다. 배송비 정책까지 겹치면 주문 한 건마다 손실이 납니다. **정액 할인액의 3배 이상**을 최소 주문 금액으로 두는 것을 권장합니다.

**배송비 쿠폰은 주문 쿠폰과 함께 쓰입니다.** 한 주문에 주문 쿠폰 1장 + 배송비 쿠폰 1장이 동시에 붙습니다. 다만 **배송비보다 많이 할인되지는 않습니다.**

### 3-4. 적용 대상

**비우면 전 상품에 적용됩니다.** 일부에만 적용하려면 아래를 지정합니다.

| 항목 | 설명 |
|---|---|
| **적용 카테고리** | 이름을 입력하면 후보가 뜨고, 고르면 파란 칩으로 추가됩니다. 칩의 `×` 로 뺍니다. |
| **적용 브랜드** | 같은 방식으로 브랜드를 지정합니다. |
| **제외할 상품 뱃지** | 체크한 뱃지가 붙은 상품은 이 쿠폰에서 **제외**됩니다. |

> **적용 대상을 지정하면 그 범위 밖의 상품에는 쿠폰이 전혀 먹히지 않습니다.** 고객은 "쿠폰이 있는데 적용이 안 된다"고 느낍니다. 범위를 좁혔다면 **쿠폰명이나 유의사항에 반드시 명시**하세요. (예: `홍삼 카테고리 전용`)

> ⚠️ **이미 할인 중인 상품에 쿠폰까지 주면 이중 할인이 됩니다.** 특가·마감세일 상품을 빼려면 `제외할 상품 뱃지`에서 해당 뱃지를 체크하세요. 범위가 걸린 쿠폰은 목록에서 노란 글씨 **적용범위 제한** 으로 표시됩니다.

### 3-5. 발급 — 고객이 어떻게 받는가

| 발급 방식 | 고객이 받는 방법 | 추가로 나타나는 입력란 |
|---|---|---|
| **가입 시 자동지급** | 회원가입하면 자동으로 들어갑니다. | 없음 |
| **관리자 수동지급** | 운영자가 **쿠폰 지급** 화면에서 직접 지급합니다. | 없음 |
| **쿠폰 코드 입력** | 고객이 쿠폰존에서 코드를 입력해 받습니다. | **쿠폰 코드** |
| **쿠폰존 다운로드** | 고객이 쿠폰존에서 직접 눌러 받습니다. | **수령 시작 / 수령 종료** |

**쿠폰 코드** — `WELCOME2026` 처럼 고객이 입력할 문자열입니다. 발급 방식이 `쿠폰 코드 입력`일 때만 저장됩니다. 다른 방식으로 바꿔 저장하면 코드가 지워집니다.

**선착순 수령 수량** — 비우면 무제한. **이 한도를 넘으면 더 이상 발급되지 않습니다.** 수정 화면에는 아래에 `현재 발급 수`가 함께 표시됩니다.

> ⚠️ **선착순 수량은 되도록 지정하세요.** 무제한 쿠폰을 SNS·문자로 뿌리면 몇 명이 받을지 통제할 수 없습니다. 예상 인원 + 여유분 정도로 못 박는 것이 안전합니다.

### 3-6. 사용 조건

| 입력란 | 설명 |
|---|---|
| **사용 시작일 / 사용 종료일** | 이 기간에만 결제에 쓸 수 있습니다. **비우면 오늘부터 1년 뒤까지**가 자동으로 들어갑니다. |
| **발급 후 N일 (상대 유효기간)** | 값이 있으면 **"받은 날부터 N일"** 이 그 사람의 만료일이 됩니다. 비우면 위의 사용 종료일을 씁니다. |
| **전체 사용 한도** | 결제에 **사용**될 수 있는 총 횟수. 비우면 무제한. |

> **사용 기간을 비워 두면 1년짜리 쿠폰이 됩니다.** 기본값이 그렇습니다. 짧게 쓰고 끝낼 프로모션이라면 **종료일을 반드시 직접 지정**하세요. 안 그러면 1년 뒤에도 결제 화면에서 할인이 나갑니다.

> **`수령 기간`과 `사용 기간`은 다릅니다.** "7월에 받아서 8월까지 쓴다"가 가능합니다. 수령 기간은 다운로드 쿠폰에만 적용됩니다.

> **`선착순 수령 수량`과 `전체 사용 한도`도 다른 축입니다.** 앞은 **몇 장을 나눠 줄지**, 뒤는 **몇 번이나 결제에 쓰일지**입니다. 1,000장을 뿌려도 사용 한도를 100으로 두면 101번째 결제부터는 적용되지 않습니다. **비용을 통제하려면 사용 한도**를, **참여 인원을 통제하려면 수령 수량**을 쓰세요.

마지막으로 **등록**(수정 화면에서는 **수정**) 버튼을 누릅니다.

## 4. 2단계 — 회원에게 지급하기

<div class="manual-goto-bar">
  <span class="manual-goto-label">이 단계 화면</span>
  <a class="manual-goto" href="/admin/coupons/issue" target="_blank">쿠폰 지급 화면 열기</a>
</div>

발급 방식이 `관리자 수동지급`인 쿠폰을 직접 나눠 주는 화면입니다. (1인 1장, 이미 보유 중이면 건너뜁니다.)

1. **지급할 쿠폰**을 고릅니다. 목록에는 **상태가 `진행중`이고 사용 종료일이 지나지 않은 쿠폰만** 나옵니다.
2. **지급 방식**을 고릅니다.
   - **전체 회원 지급** — 가입된 모든 회원에게 한 장씩
   - **특정 회원 지급** — **회원 검색** 버튼으로 이름·이메일·연락처·생년월일로 찾아 여러 명 선택
3. **지급하기** 를 누릅니다.

지급이 끝나면 화면 위에 **쿠폰 지급 결과**가 표시됩니다.

| 구분 | 내용 |
|---|---|
| **✓ 발급 완료 (N명)** | 실제로 쿠폰이 들어간 회원 목록 |
| **✗ 미발급 (N명)** | 못 받은 회원과 그 이유 |

**미발급 사유**

| 사유 | 뜻 |
|---|---|
| **이미 보유** | 이 쿠폰을 아직 안 쓴 채로 갖고 있습니다. 1인 1장 원칙이라 건너뜁니다. |
| **발급 한도 소진** | 선착순 수령 수량을 다 썼습니다. |
| **활성 상태가 아님** | 쿠폰 상태가 `진행중`이 아닙니다. |
| **만료된 쿠폰** | 사용 종료일이 지났습니다. |

> ⚠️ **`전체 회원 지급`은 되돌릴 수 없습니다.** 개별 회수 기능이 없습니다. 회원 수가 많다면 그 수만큼 할인 재원이 즉시 확정된다고 생각하세요. 누르기 전에 **혜택 금액 × 회원 수**를 계산하고, 최소 주문 금액과 사용 기간이 맞는지 다시 확인하세요.

> **먼저 특정 회원 1명에게 시험 지급해 보세요.** 결제 화면에서 의도한 대로 할인되는지 확인한 뒤 전체 지급하는 것이 안전합니다.

**검색**: 상태 / 발급 방식 / 쿠폰명·코드로 좁힐 수 있습니다.

| 열 | 의미 |
|---|---|
| **쿠폰명** | 아래 회색 글씨는 쿠폰 코드입니다. |
| **몰** | `전 몰 공용` 또는 특정 몰 이름 |
| **혜택** | `3,000원 할인` · `10% (최대 20,000원)` · `무료배송` 등. 아래에 최소 주문 금액과 **적용범위 제한** 표시가 붙습니다. |

| **수령** | `발급된 수 / 선착순 수량`. 한도가 없으면 `(무제한)` |
| **사용** | `사용된 수 / 발급된 수`, 그 아래 사용률(%)과 사용 한도 |
| **기간** | 다운로드 쿠폰이면 수령 기간이 함께 표시됩니다 |
| **상태** | `작성중` / `진행중` / `일시중지` / `종료` |

**관리 버튼 3개** — **상세** / **수정** / **종료**

> ⚠️ **`종료`를 누르면 확인창이 뜨고, 이미 발급된 쿠폰도 사용할 수 없게 됩니다.** 고객이 갖고 있던 쿠폰이 결제 화면에서 갑자기 사라진다는 뜻입니다. 사용 기간이 남은 쿠폰을 종료할 때는 공지를 먼저 하세요. 잠시 멈추기만 할 거라면 수정 화면에서 상태를 **`일시중지`** 로 바꾸는 편이 낫습니다.

## 6. 쿠폰 상세 — 성과 확인하기

목록에서 **상세**를 누르면 위쪽에 카드 3개가 뜹니다.

| 카드 | 의미 |
|---|---|
| **수령** | `발급된 수 / 선착순 수량`. 아래는 수령률, 한도가 없으면 `수령 한도 없음` |
| **사용** | `사용된 수 / 발급된 수`. 아래는 사용률(%) |
| **총 할인액** | **이 쿠폰이 실제로 깎아준 금액 합계** |

그 아래에는 등록 내용과 **받은 회원 목록**(발급일시·사용 여부·연결된 주문번호)이 표시됩니다.

> **총 할인액이 곧 이 쿠폰의 비용입니다.** 행사를 이어 갈지 끊을지 판단할 때 이 숫자를 먼저 보세요. 수령률은 높은데 사용률이 낮다면 **최소 주문 금액이 너무 높거나 적용 범위가 지나치게 좁은** 경우가 많습니다.

## 8. 자주 막히는 상황

**Q. 쿠폰을 만들었는데 지급 화면 목록에 없습니다.**
상태가 `진행중`이 아니거나 사용 종료일이 지났습니다. 수정 화면에서 상태와 사용 기간을 확인하세요.

**Q. 목적을 `신규가입`으로 했는데 가입해도 쿠폰이 안 옵니다.**
목적(분류)은 동작을 바꾸지 않습니다. **발급 방식**을 `가입 시 자동지급`으로 바꿔야 합니다.

**Q. 저장하려니 "정률 쿠폰에는 최대 할인액이 필요합니다"가 뜹니다.**
혜택 유형이 `정률 할인 (%)`이면 최대 할인액이 필수입니다. 상한이 없으면 고액 주문에서 할인이 무한정 커지기 때문에 시스템이 막습니다.

**Q. 고객이 쿠폰을 갖고 있는데 결제 화면에서 적용되지 않습니다.**
아래를 순서대로 확인하세요.
1. **최소 주문 금액**에 미달하지 않는가?
2. **적용 대상**(카테고리·브랜드)이 지정되어 있고 장바구니 상품이 그 범위 밖인가?
3. **제외할 상품 뱃지**에 걸리는 상품인가?
4. 쿠폰 **사용 기간**이 지났거나, `발급 후 N일`로 그 사람의 만료일이 지났는가?
5. 쿠폰 **상태**가 `종료`·`일시중지`로 바뀌지 않았는가?
6. **전체 사용 한도**를 다 썼는가?

**Q. 배송비 쿠폰과 할인 쿠폰을 같이 쓸 수 있나요?**
됩니다. 한 주문에 주문 쿠폰 1장 + 배송비 쿠폰 1장이 함께 적용됩니다. 다만 같은 종류를 2장 겹쳐 쓸 수는 없습니다.

**Q. 쿠폰을 삭제하고 싶습니다.**
삭제 기능은 없습니다. **종료**를 쓰세요. 삭제하면 회원 보유 쿠폰과 사용 이력이 함께 사라져 정산이 불가능해집니다.

**Q. 잘못 지급한 쿠폰을 회수할 수 있나요?**
개별 회수 기능은 없습니다. 쿠폰 전체를 **종료**하면 아직 안 쓴 쿠폰도 사용할 수 없게 되지만, 정상적으로 받은 다른 고객에게도 똑같이 적용되니 신중히 판단하세요.

**Q. `이미 보유`로 미발급됐습니다. 한 명에게 2장 주고 싶습니다.**
지급은 1인 1장이 원칙이라, 아직 쓰지 않은 쿠폰을 갖고 있으면 건너뜁니다. 그 회원이 쿠폰을 사용한 뒤 다시 지급하거나, 별도의 쿠폰을 하나 더 만들어 지급하세요.

**Q. 수령 수량과 사용 한도 중 무엇을 써야 하나요?**
**비용을 막고 싶으면 전체 사용 한도**, **참여 인원을 막고 싶으면 선착순 수령 수량**입니다. 둘 다 지정해도 됩니다.

**Q. 쿠폰 코드를 넣었는데 저장 후 사라졌습니다.**
발급 방식이 `쿠폰 코드 입력`이 아니면 코드가 저장되지 않습니다. 발급 방식을 먼저 바꾸세요.

## 9. 아직 지원하지 않는 것

- 개별 회원의 쿠폰 **회수**·회수 취소
- 회원 등급·구매 이력 조건으로 자동 지급 (등급 쿠폰은 멤버십 관리에서 다룹니다)
- 1인 1코드 형태의 쿠폰 코드 대량 생성
- 사용 내역 엑셀 내려받기
