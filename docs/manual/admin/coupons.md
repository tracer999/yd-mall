# 쿠폰 관리

고객에게 나눠 줄 **할인 쿠폰·무료배송 쿠폰**을 만들고, 누가 얼마나 받아서 얼마나 썼는지 확인하는 메뉴입니다.

화면 위치: **관리자 > 프로모션 관리 > 쿠폰 관리** (`/admin/coupons`)

---

## 쿠폰 관리에서 할 수 있는 일

- 쿠폰을 **새로 만들고 수정**할 수 있습니다.
- 만든 쿠폰을 **특정 회원에게 직접 지급**할 수 있습니다.
- 쿠폰별로 **몇 명이 받았는지(수령률), 몇 장이 쓰였는지(사용률), 총 할인 금액이 얼마인지** 볼 수 있습니다.
- 더 이상 쓰지 않을 쿠폰을 **종료** 처리할 수 있습니다.

---

## 쿠폰은 세 가지를 각각 정합니다

쿠폰을 만들 때 헷갈리기 쉬운 부분입니다. **"어떤 혜택인가", "어떻게 나눠 주나", "어디에 쓸 수 있나"** 를 따로따로 고른다고 생각하면 됩니다.

### 1) 혜택 종류 — 무엇을 깎아 주나

| 혜택 종류 | 설명 |
|---|---|
| **정액 할인** | 정해진 금액을 깎아 줍니다. (예: 3,000원 할인) |
| **정률(%) 할인** | 주문 금액의 몇 %를 깎아 줍니다. (예: 10% 할인) |
| **무료배송** | 배송비를 전액 면제합니다. |
| **배송비 정액할인** | 배송비 중 일부 금액만 깎아 줍니다. (예: 배송비 1,500원 할인) |

### 2) 발급 방식 — 고객이 어떻게 받나

| 발급 방식 | 설명 |
|---|---|
| **가입 시 자동 지급** | 신규 회원이 가입하면 자동으로 들어갑니다. |
| **관리자가 직접 지급** | 쿠폰 상세 화면에서 운영자가 회원을 골라 지급합니다. |
| **쿠폰 코드 입력** | 고객이 코드를 직접 입력해야 받습니다. (오프라인 행사·제휴 등에 사용) |
| **고객이 쿠폰존에서 받기(다운로드)** | 쿠폰존에 걸어 두고 고객이 "받기" 버튼을 눌러 가져갑니다. |

### 3) 적용 대상 — 어디에 쓸 수 있나

| 적용 대상 | 설명 |
|---|---|
| **전 상품** | 모든 상품에 사용 가능 |
| **특정 카테고리** | 고른 카테고리의 상품에만 사용 가능 |
| **특정 브랜드** | 고른 브랜드의 상품에만 사용 가능 |
| **배송비** | 상품값이 아니라 배송비에 적용 |

---

## 쿠폰 등록하기

1. **쿠폰 관리** 화면에서 **쿠폰 등록** 버튼을 클릭합니다.
2. **쿠폰 이름**을 입력합니다. 고객의 쿠폰함과 주문서에 그대로 보이므로, "10월 신규가입 3,000원 할인"처럼 알아보기 쉽게 적습니다.
3. **혜택 종류**를 고르고 값을 입력합니다.
   - 정액 할인이면 **할인 금액**
   - 정률(%) 할인이면 **할인율** 과 **최대 할인액**
4. **발급 방식**을 고릅니다. (가입 시 자동 지급 / 관리자 직접 지급 / 쿠폰 코드 입력 / 쿠폰존에서 받기)
5. **적용 대상**을 고릅니다. 카테고리·브랜드를 골랐다면 대상 카테고리·브랜드를 지정합니다.
6. **사용 조건**을 입력합니다.
   - **최소 주문 금액:** 이 금액 이상 주문해야 쿠폰을 쓸 수 있습니다.
7. **사용 기간**을 지정합니다. 이 기간이 지나면 고객이 가지고 있어도 쓸 수 없습니다.
8. 쿠폰존에서 받는 쿠폰(다운로드)이라면 **수령 기간**과 **선착순 수량**을 함께 지정합니다.
9. **저장** 버튼을 눌러 등록합니다.
10. 상태를 **진행중** 으로 바꾸면 고객에게 노출·지급되기 시작합니다.

---

## 주요 입력 항목

| 항목 | 설명 |
|---|---|
| **쿠폰 이름** | 고객 쿠폰함·주문서에 그대로 보입니다. |
| **할인 금액 / 할인율** | 혜택 종류에 따라 둘 중 하나를 입력합니다. |
| **최대 할인액** | 정률(%) 쿠폰에서 "아무리 많이 사도 이 금액까지만 깎는다"는 상한선입니다. |
| **최소 주문 금액** | 이 금액 미만 주문에는 쿠폰이 뜨지 않습니다. |
| **사용 기간** | 쿠폰을 **쓸 수 있는** 기간입니다. |
| **수령 기간** | 쿠폰존에서 **받을 수 있는** 기간입니다. 사용 기간과 다릅니다. |
| **수령 한도(선착순 수량)** | 몇 명까지 받아 갈 수 있는지 정합니다. 다 차면 쿠폰존에서 마감됩니다. |
| **총 사용 한도** | 실제로 몇 장까지 쓰일 수 있는지 정합니다. **받은 수량과는 다른 개념**입니다. |
| **상태** | 작성중 / 진행중 / 일시중지 / 종료 |

---

## 쿠폰 상태

| 상태 | 의미 |
|---|---|
| **작성중** | 아직 고객에게 보이지 않습니다. 편하게 수정할 수 있습니다. |
| **진행중** | 고객에게 노출·지급됩니다. |
| **일시중지** | 잠시 발급·노출을 멈춥니다. 다시 진행중으로 되돌릴 수 있습니다. |
| **종료** | 더 이상 발급되지 않습니다. |

**쿠폰은 삭제할 수 없습니다.** 이미 받아 간 고객이 있을 수 있기 때문에, 쓰지 않을 쿠폰은 **종료** 로 처리합니다.

---

## 고객은 어디서 쿠폰을 보나요

- **쿠폰존(`/coupon`)**: 아직 받지 않은 쿠폰을 **받는 곳**입니다.
- **마이페이지 쿠폰함**: 이미 **받아 둔 쿠폰**을 보는 곳입니다.

이 둘은 다른 화면입니다. "쿠폰존에 안 보인다"는 문의와 "쿠폰함에 없다"는 문의는 원인이 다르므로 구분해서 확인하세요.

---

## 주문에 쿠폰이 적용되는 방식

- 한 주문에 **주문 쿠폰 1장 + 배송비 쿠폰 1장**까지 함께 쓸 수 있습니다. 즉 할인 쿠폰과 무료배송 쿠폰을 동시에 쓰는 것은 가능하지만, 할인 쿠폰 두 장을 겹쳐 쓸 수는 없습니다.
- 고객이 주문서에서 쿠폰을 선택하면 **결제가 끝날 때까지 30분간 그 쿠폰이 점유**됩니다. 그 동안에는 다른 주문에 그 쿠폰을 쓸 수 없습니다.
- 주문을 취소하면 사용했던 쿠폰은 **다시 쓸 수 있는 상태로 돌아옵니다.**

> 고객이 "쿠폰이 사라졌다"고 문의하면, 주문서를 열어 둔 채 결제를 마치지 않은 상태일 수 있습니다. 30분이 지나면 자동으로 풀립니다.

---

## 쿠폰 성과 확인하기

쿠폰 목록에서 쿠폰 이름을 클릭하면 상세 화면이 열립니다. 여기서 다음을 볼 수 있습니다.

- **수령률**: 준비한 수량 대비 몇 명이 받아 갔는지
- **사용률**: 받아 간 쿠폰 중 실제로 몇 장이 쓰였는지
- **총 할인액**: 이 쿠폰으로 나간 할인 금액 합계

---

## 주의

- **정률(%) 쿠폰은 최대 할인액을 반드시 입력해야 합니다.** 비워 두면 저장이 거부됩니다. 상한선이 없으면 고액 주문에서 할인이 무한정 커지기 때문입니다.
- **쿠폰은 삭제되지 않습니다.** 잘못 만든 쿠폰은 종료 처리하고 새로 만드세요.
- **수령 한도와 총 사용 한도를 혼동하지 마세요.** "선착순 100명"이면 수령 한도가 100이고, "총 100장까지만 할인"이면 총 사용 한도가 100입니다.
- **회원을 삭제하면 그 회원이 받은 쿠폰은 사라지지만, 쿠폰의 "발급 수" 숫자는 줄어들지 않습니다.** 선착순 쿠폰에서 회원 삭제가 반복되면, 실제로 쿠폰을 가진 사람은 적은데 숫자상으로는 먼저 마감될 수 있습니다. 선착순 쿠폰 운영 중에는 회원 삭제를 되도록 피하고, 마감이 이상하게 빠르면 수령 한도를 늘려 조정하세요.

---

## 자주 묻는 상황

**Q. 고객이 쿠폰존에서 쿠폰을 못 본다고 합니다.**
A. ① 쿠폰 상태가 **진행중** 인지, ② 발급 방식이 **쿠폰존에서 받기(다운로드)** 인지, ③ **수령 기간** 안인지, ④ 선착순 수량이 다 차지 않았는지 순서대로 확인하세요.

**Q. 쿠폰함에는 있는데 주문서에서 선택이 안 됩니다.**
A. **최소 주문 금액**을 못 채웠거나, **적용 대상**(카테고리·브랜드)이 장바구니 상품과 맞지 않거나, **사용 기간**이 지난 경우입니다.

**Q. 할인 쿠폰과 무료배송 쿠폰을 같이 쓸 수 있나요?**
A. 가능합니다. 주문 쿠폰 1장 + 배송비 쿠폰 1장까지 동시 사용됩니다.

**Q. 진행 중인 쿠폰의 할인율을 바꿔도 되나요?**
A. 이미 받아 간 고객에게도 영향이 가므로 권장하지 않습니다. 조건을 바꾸려면 기존 쿠폰을 **종료**하고 새 쿠폰을 만드는 편이 안전합니다.
