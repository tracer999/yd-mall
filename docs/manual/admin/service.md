# 서비스 관리

몰을 만들어 고객에게 **납품(포팅·배포)** 하는 쪽, 즉 **서비스 제공자**가 쓰는 메뉴입니다. **몰 하나를 운영하기 위한 메뉴가 아닙니다.** 상품을 등록하거나 주문을 처리하는 것과는 완전히 다른 층위의 화면입니다.

> **세 화면 모두 최고 관리자 전용입니다.** 다른 역할에게는 사이드바에 `서비스 관리` 대메뉴 자체가 나타나지 않고, 주소를 직접 입력해도 접근이 거부됩니다. 각 화면에도 "서비스 관리 · 제공자(super_admin) 전용" 이라는 표시가 붙어 있습니다.

> **여기서 관리하는 것은 "우리가 누구에게 몰을 팔았는가"입니다.** 고객사가 납품받은 뒤 스스로 만든 개별 몰까지는 추적하지 않습니다.

## 1. 왜 이 메뉴가 따로 있나

이 시스템은 쇼핑몰이 필요한 고객에게 **몰을 만들어 넘겨주는 몰 빌더**입니다. 그래서 관리자 화면 안에 성격이 다른 두 층이 섞여 있습니다.

| 층 | 무엇을 하나 | 해당 메뉴 |
|---|---|---|
| **몰 운영** | 상품 올리고 주문 처리하고 화면 꾸미기 | 상품 관리, 판매 관리, 배너 관리 등 대부분의 메뉴 |
| **서비스 운영** | 누구에게 납품했고, 그 고객이 어떤 기능을 쓸 수 있고, 새 몰에 무엇을 복제할지 | **서비스 관리** (이 문서) |

납품받은 고객사는 아래층(몰 운영)만 씁니다. **서비스 관리는 납품하는 우리 쪽만 씁니다.**

## 2. 세 화면 한눈에 보기

```
① 등급별 기능 설정   → 판매할 등급(플랜)을 정의한다
        ↓
② 배포·포팅 관리     → 납품한 고객을 등록하고 ①의 등급을 배정한다
        ↓
③ 샘플 데이터 관리   → 앞으로 만들 몰에 기본으로 들어갈 내용물을 다듬는다
```

**순서가 중요합니다.** 등급을 먼저 만들어 두어야 고객 등록 화면에서 배정할 등급이 선택지로 나옵니다.

| 화면 | 주소 | 무엇을 하나 |
|---|---|---|
| **배포·포팅 관리** | `/admin/service/porting` | 배포 절차 확인 + **납품 고객 명부** 관리 |
| **등급별 기능 설정** | `/admin/service/features` | **판매 등급(플랜)** 정의와 기능 자격 편집 |
| **샘플 데이터 관리** | `/admin/service/samples` | 새 몰 생성 시 복제될 **샘플 카테고리·상품** 편집 |

## 3. 1단계 — 등급별 기능 설정

<div class="manual-goto-bar">
  <span class="manual-goto-label">이 단계 화면</span>
  <a class="manual-goto" href="/admin/service/features" target="_blank">등급별 기능 설정 열기</a>
</div>

**등급(플랜)** 은 "얼마짜리 상품을 파는가"에 해당합니다. 기본형·표준형·엔터프라이즈처럼 등급을 만들어 두고, 각 등급이 어떤 기능을 쓸 수 있는지 정의합니다. 납품 고객에게는 이 중 하나를 배정합니다.

### 3-1. 등급 표의 각 칸

기존 등급들은 표에 한 줄씩 나오고, 표 안에서 바로 고칠 수 있습니다.

| 칸 | 설명 |
|---|---|
| **등급** | 등급 이름을 직접 고칠 수 있습니다(최대 100자). 이름 아래 작은 글씨로 **등급 코드**가 표시되는데, **코드는 만든 뒤 바꿀 수 없습니다.** |
| **서브몰 생성 가능 개수** | 이 등급의 고객이 만들 수 있는 몰 개수. 0~9999. |
| **네이버 스토어** | 켜면 네이버 스토어 연동 기능을 쓸 수 있는 등급이 됩니다. |
| **도매 연동 (도매꾹·온채널)** | 켜면 외부 도매몰에서 상품을 가져오는 기능을 쓸 수 있는 등급이 됩니다. |
| **AI 자동생성** | 켜면 상품 설명 등을 AI로 만드는 기능을 쓸 수 있는 등급이 됩니다. |
| **사용** | 끄면 그 등급은 **고객 등록 화면의 선택지에서 사라집니다.** 이미 배정된 고객의 등급 표시는 그대로 남습니다. |
| **배정 고객** | 지금 이 등급을 쓰는 납품 고객이 몇 곳인지. 읽기 전용입니다. |
| (맨 오른쪽 휴지통) | 등급 삭제 |

> **표를 고친 뒤 반드시 아래쪽 저장 버튼을 누르세요.** 표 안의 값을 바꾸기만 하고 화면을 벗어나면 변경이 전부 사라집니다.

### 3-2. ⚠️ 이 설정이 실제로 하는 일 (가장 중요)

화면 상단에 노란 안내가 떠 있습니다. 그 내용이 이 화면의 핵심입니다.

> **여기서 정하는 것은 "기능 자격(entitlement)"의 정의일 뿐, 실제로 기능을 막지는 않습니다.**
>
> 화면에도 "현재는 등급 정의·배정까지입니다. 실제 기능 차단(강제)은 각 기능 모듈 연동 시 적용됩니다." 라고 적혀 있습니다.

구체적으로 이런 뜻입니다.

- 어떤 등급의 **AI 자동생성** 을 꺼 두어도, 그 고객이 실제로 AI 기능을 쓰는 것을 **시스템이 막지 않습니다.**
- **서브몰 생성 가능 개수**를 3으로 정해 두어도, 그 고객이 몰을 4개 만드는 것을 **시스템이 막지 않습니다.**

따라서 지금 이 화면은 **계약 내용을 기록해 두는 장부**로 쓰는 것이 맞습니다. 실제 제한은 계약서와 사람이 지킵니다. 자동 차단은 배포·포팅 화면의 **로드맵**에 예정 항목으로 적혀 있습니다.

> **이 설정은 스토어프론트 메뉴 노출과 아무 관계가 없습니다.** 고객 화면에 어떤 메뉴가 보이는지는 메뉴 관리에서 정합니다. 여기서 체크를 껐다고 고객 화면에서 뭔가 사라지지 않습니다.

### 3-3. 새 등급 추가

표 아래 점선 상자의 **새 등급 추가** 영역을 사용합니다.

| 입력란 | 필수 | 설명 | 예시 |
|---|---|---|---|
| **등급 코드** | 필수 | **영문 대문자와 숫자, 밑줄만** 사용됩니다. 소문자를 넣어도 자동으로 대문자로 바뀌고 다른 문자는 제거됩니다. 최대 50자. | `ENTERPRISE` |
| **등급명** | 필수 | 화면에 보이는 이름. 최대 100자. | `엔터프라이즈` |

**추가** 버튼을 누르면 등급이 만들어지고, 세부 기능은 위 표에서 켜고 끄면 됩니다.

> **둘 중 하나라도 비면 "코드와 등급명을 입력하세요." 라는 빨간 안내가 뜹니다.** 이미 있는 코드를 넣으면 "이미 있는 등급 코드입니다." 가 뜹니다.

> **등급 코드는 나중에 바꿀 수 없습니다.** 화면에 회색 작은 글씨로만 표시되며 수정할 방법이 없습니다. `BASIC`, `STANDARD`, `PREMIUM` 처럼 의미가 오래 유지되는 이름을 쓰세요.

### 3-4. 등급 삭제

각 줄 오른쪽 끝의 휴지통을 누르면 확인 창이 뜹니다.

> **등급을 삭제하면 그 등급을 쓰던 납품 고객은 자동으로 `미배정` 상태가 됩니다.** 고객 정보 자체는 지워지지 않지만, 어떤 등급이었는지에 대한 기록은 사라집니다. 삭제 전에 **배정 고객** 칸이 `0곳` 인지 확인하세요.

**등급을 더 이상 팔지 않을 뿐이라면 삭제하지 말고 `사용` 체크만 끄세요.** 신규 고객에게는 선택지로 나오지 않으면서, 기존 고객의 등급 표시는 그대로 유지됩니다.

## 4. 2단계 — 배포·포팅 관리

<div class="manual-goto-bar">
  <span class="manual-goto-label">이 단계 화면</span>
  <a class="manual-goto" href="/admin/service/porting" target="_blank">배포·포팅 관리 열기</a>
  <a class="manual-goto" href="/admin/service/customers/new" target="_blank">납품 고객 등록 화면 열기</a>
</div>

이 화면은 네 부분으로 되어 있습니다.

### 4-1. 배포 파이프라인 (읽기 전용 안내)

현재 배포가 어떤 절차로 이루어지는지 보여 주는 **안내 카드**입니다. 세 단계로 표시됩니다.

| 단계 | 내용 |
|---|---|
| **STEP 1** | `git push origin main` |
| **STEP 2** | GitHub Actions 가 실행됨 |
| **STEP 3** | 서버 동기화 → build → start (PM2) |

> **이 카드에는 누를 수 있는 버튼이 없습니다.** 화면에서 배포를 실행하는 기능은 아직 없고, 배포는 개발 담당자가 코드를 푸시하는 방식으로 이루어집니다. 카드 아래에도 "현재 배포는 Git 기반입니다. 푸시 = 개발 서버 즉시 반영." 이라고 안내되어 있습니다.

### 4-2. 납품 고객 명부

우리가 몰을 납품한 고객사 목록입니다. **활성 고객이 먼저, 그다음 최근 등록순**으로 정렬됩니다.

| 칸 | 내용 |
|---|---|
| **고객** | 업체명. 메모를 적어 두었으면 그 아래 작은 글씨로 한 줄 미리보기가 나옵니다. |
| **담당자 / 연락처** | 담당자 이름과, 그 아래 이메일(없으면 연락처) |
| **배정 등급** | 등급 이름 배지. 지정하지 않았으면 `미배정` |
| **서브몰 한도** | 배정된 등급의 서브몰 생성 가능 개수 |
| **납품일** | `YYYY-MM-DD` 형태. 비어 있으면 `-` |
| **상태** | `활성`(초록) 또는 `비활성`(회색) |
| **관리** | 연필(수정) · 휴지통(삭제) |

**비활성 고객의 줄은 회색으로 흐리게** 표시됩니다.

### 4-3. 납품 고객 등록·수정

오른쪽 위 **납품 고객 등록** 버튼, 또는 목록의 연필 아이콘으로 들어갑니다.

| 입력란 | 필수 | 설명 | 예시 |
|---|---|---|---|
| **고객(업체)명** | **필수** | 최대 150자. 비우면 저장되지 않고 목록으로 되돌아갑니다. | `(주)한빛유통` |
| **담당자** | 선택 | 최대 100자 | `김담당` |
| **이메일** | 선택 | 최대 255자 | `kim@hanbit.co.kr` |
| **연락처** | 선택 | 최대 50자 | `02-1234-5678` |
| **배정 등급** | 선택 | 3단계에서 만든 등급 중 하나. 선택지에는 등급명과 서브몰 개수가 함께 표시됩니다. 기본값은 `미배정`. | `표준형 (서브몰 3개)` |
| **납품일** | 선택 | 날짜 선택기로 고릅니다. | `2026-03-15` |
| **메모** | 선택 | 최대 500자. 계약 조건, 특이사항 등을 적어 둡니다. | `2차 확장 협의 중` |
| **활성 (납품 유지 중)** | 체크박스 | 켜면 `활성`, 끄면 `비활성`으로 표시됩니다. | |

마지막에 **등록**(신규) 또는 **수정** 버튼을 누릅니다.

> **`사용` 을 끈 등급은 이 화면의 배정 등급 선택지에 나오지 않습니다.** 배정하려는 등급이 목록에 안 보이면 등급별 기능 설정에서 `사용` 이 켜져 있는지 먼저 확인하세요.

<div class="manual-goto-bar">
  <span class="manual-goto-label">등급이 없다면</span>
  <a class="manual-goto is-sub" href="/admin/service/features" target="_blank">등급별 기능 설정에서 등급 만들기</a>
</div>

**메모를 잘 쓰세요.** 이 명부에는 계약 금액·기간·갱신일 같은 전용 칸이 없습니다. 필요한 내용은 500자 안에서 메모에 규칙을 정해 적어 두는 것이 현실적입니다. (예: `계약 2026-03~2027-02 / 연 갱신 / 담당 영업 박OO`)

### 4-4. 납품 고객 삭제

목록의 휴지통을 누르면 확인 창이 뜨고, 확인하면 **완전히 지워집니다.** 되돌릴 수 없습니다.

> **거래가 끝난 고객은 삭제하지 말고 `활성` 체크를 끄세요.** 언제 누구에게 무엇을 납품했는지는 나중에 반드시 찾게 되는 기록입니다. 비활성으로 두면 목록 아래쪽에 회색으로 남습니다.

### 4-5. 포팅 & 프로비저닝 · 로드맵

화면 아래에 참고용 카드 두 개가 있습니다.

- **포팅 & 프로비저닝** — 새 몰이 자동으로 내비게이션·메뉴·테마·홈을 갖추도록 구성된다는 안내와, 관련 화면으로 가는 링크(몰 리스트 관리 / 몰 빌더 가이드 / 등급별 기능 설정)가 있습니다.
- **로드맵** — 아직 만들어지지 않은 기능 목록입니다. 서브몰 개수 한도 강제, 등급 기능 실제 차단, 앱 내 배포/롤백 버튼, 몰 내보내기·가져오기, 서비스 관리 전용 인스턴스 분리가 적혀 있습니다.

## 5. 3단계 — 샘플 데이터 관리

<div class="manual-goto-bar">
  <span class="manual-goto-label">이 단계 화면</span>
  <a class="manual-goto" href="/admin/service/samples" target="_blank">샘플 데이터 관리 열기</a>
</div>

새 몰을 만들 때 **자동으로 복제되어 들어가는 샘플 카테고리·브랜드·상품**을 편집하는 곳입니다. 빈 몰을 납품하면 고객이 무엇부터 해야 할지 몰라 막히므로, 처음부터 그럴듯한 내용물이 들어가 있도록 미리 다듬어 두는 것입니다.

**이 화면에서 꼭 지켜야 할 세 가지**

> **① 이미 만들어진 몰에는 아무 영향이 없습니다.** 여기서 바꾼 값은 **앞으로 생성될 몰**에만 복제됩니다. 이미 납품한 몰의 상품을 고치려면 그 몰의 상품 관리 화면에서 직접 해야 합니다.

> **② 이미지 경로는 반드시 `/images/...` 로 시작해야 합니다.** `/uploads/...` 로 시작하는 경로를 넣으면 **납품본에서 이미지가 깨집니다.** 업로드 폴더는 배포 대상에서 제외되기 때문입니다. 이미지 파일은 `public/images/sample/` 위치에 두세요.

> **③ 샘플 카테고리는 새로 만들어지지 않습니다.** 화면에서 지정한 **공용 카테고리**에 샘플 상품이 연결됩니다. 지정하지 않으면 같은 이름의 공용 카테고리를 찾고, 그것도 없으면 `미분류`로 들어갑니다.

각 줄에서 이름·연결할 공용 카테고리·이미지 경로·순서·사용 여부를 고칠 수 있고, 줄마다 삭제 버튼이 있습니다.

> **삭제 버튼을 누르면 저장하지 않은 다른 변경까지 사라집니다.** 화면 안내에도 "저장하지 않은 변경은 사라집니다."라고 적혀 있습니다. **먼저 저장한 뒤 삭제하세요.**

## 7. 자주 막히는 상황

**Q. 서비스 관리 메뉴가 사이드바에 없습니다.**
지금 로그인한 계정이 **최고 관리자가 아닙니다.** 세 화면 모두 최고 관리자 전용이며, 다른 역할에게는 대메뉴 자체가 나타나지 않습니다.

**Q. 등급 표를 고쳤는데 반영이 안 됩니다.**
표 아래 **저장** 버튼을 눌러야 합니다. 표 안의 입력값만 바꾸고 화면을 벗어나면 전부 사라집니다.

**Q. 등급의 AI 자동생성을 껐는데 고객이 여전히 AI 기능을 씁니다.**
정상입니다(현 단계 기준). 이 화면은 **자격을 기록**할 뿐 실제로 기능을 막지 않습니다. 자동 차단은 로드맵 항목입니다. 3-2장을 참고하세요.

**Q. 고객 등록 화면에 배정할 등급이 안 보입니다.**
그 등급의 **`사용` 체크가 꺼져** 있습니다. 등급별 기능 설정에서 켜세요. 등급 자체가 없다면 먼저 만들어야 합니다.

**Q. 등급을 지웠더니 고객이 `미배정`이 됐습니다.**
정상입니다. 등급을 지우면 배정이 함께 풀립니다. 되돌리려면 등급을 다시 만들고 고객마다 재배정해야 합니다.

**Q. 고객(업체)명만 넣고 저장했는데 목록에 안 나옵니다.**
업체명 칸이 실제로 비어 있었을 수 있습니다. 업체명이 비면 저장되지 않고 조용히 목록으로 돌아갑니다.

**Q. 납품일을 비워 두어도 되나요?**
됩니다. 목록에 `-` 로 표시됩니다. 다만 나중에 계약 시점을 추적하기 어려우니 되도록 채우세요.

**Q. 샘플 데이터를 고쳤는데 이미 납품한 몰이 그대로입니다.**
정상입니다. 샘플 변경은 **앞으로 생성될 몰에만** 적용됩니다.

**Q. 샘플 상품 이미지가 납품본에서 깨집니다.**
이미지 경로가 `/uploads/...` 로 되어 있을 가능성이 큽니다. 파일을 `public/images/sample/` 로 옮기고 경로를 `/images/sample/...` 로 고치세요.

**Q. 화면에서 배포 버튼을 누르고 싶습니다.**
아직 없습니다. 배포 파이프라인 카드는 절차 안내일 뿐이고, 실제 배포는 개발 담당자가 수행합니다.

**Q. 고객사가 실제로 몰을 몇 개 만들었는지 보고 싶습니다.**
이 명부는 고객이 만든 개별 몰을 추적하지 않습니다. 화면 안내에도 "고객이 만든 개별 몰은 추적하지 않습니다."라고 적혀 있습니다. 현재 만들어진 몰 전체는 **몰 리스트 관리**에서 확인하세요.

## 8. 아직 지원하지 않는 것

- 등급의 **서브몰 개수 한도 강제** (몰 생성 차단)
- 등급 기능 자격의 **실제 차단** (네이버·도매·AI 연동 게이팅)
- 앱 안에서 **배포 / 롤백 실행**
- 몰 **내보내기 · 가져오기**
- 서비스 관리 전용 인스턴스 분리
- 계약 금액·기간·갱신 관리 (메모로 대체)
- 고객사가 만든 개별 몰 현황 추적
