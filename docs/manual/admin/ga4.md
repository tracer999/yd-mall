# GA4 설정 · 추적

구글이 무료로 제공하는 방문 분석 도구인 **GA4(Google Analytics 4)** 를 쇼핑몰에 연결하는 방법을 설명합니다. 관리자 화면에 **측정 ID 한 줄만 입력하면** 연결이 끝나고, 그 뒤부터는 고객이 상품을 보고 장바구니에 담고 결제하는 과정이 자동으로 구글 쪽에 기록됩니다.

> **측정 ID는 지금 편집 중인 몰에만 저장됩니다.** 몰을 여러 개 운영한다면 **몰마다 각각 입력**해야 합니다. A몰에 넣은 ID는 B몰에 전혀 적용되지 않습니다.

---

## 1. GA4가 무엇이고 왜 필요한가

관리자 화면에도 **대시보드**와 **방문자 통계**가 있습니다. 그런데도 GA4를 붙이는 이유는 다음과 같습니다.

| | 이 몰의 내장 통계 | GA4 |
|---|---|---|
| 방문자 수 | 있음 | 있음 (더 정교함) |
| 유입 매체(어디서 왔나) | 있음 | 있음 (광고 캠페인 단위까지) |
| **상품별 조회 → 장바구니 → 결제 전환율** | 없음 | **있음** |
| **어떤 광고가 실제 매출을 만들었는가** | 없음 | **있음** |
| 기간 자유 지정·비교 | 30일까지, 비교 없음 | 자유 |
| 보관 기간 | 계속 쌓임 | 구글 정책에 따름 |

**핵심 차이는 "돈이 어디서 나왔는지"를 추적할 수 있다는 점입니다.** 광고를 집행할 계획이 있다면 GA4 연결은 사실상 필수입니다. 광고비를 쓰기 전에 먼저 붙여 두세요.

**서로 대체 관계가 아닙니다.** 일상 점검은 이 몰의 대시보드가 빠르고, 깊은 분석은 GA4가 강합니다. 둘 다 켜 두는 것이 맞습니다.

## 2. 시작하기 전에 — 준비물

| 준비물 | 어디서 구하나 |
|---|---|
| **구글 계정** | 회사 대표 계정을 권장합니다. 개인 계정으로 만들면 담당자가 바뀔 때 접근권이 함께 사라집니다. |
| **GA4 속성** | 구글 애널리틱스(analytics.google.com)에서 계정 → 속성 → 데이터 스트림 순으로 만듭니다. |
| **측정 ID** | 데이터 스트림을 만들면 발급됩니다. **`G-` 로 시작하는 문자열**입니다. |

> **`G-` 로 시작하지 않는 값은 GA4 측정 ID가 아닙니다.** `UA-` 로 시작하는 값은 지금은 쓰지 않는 예전 방식이고, `GTM-` 으로 시작하는 값은 태그 매니저 컨테이너 ID로 여기에 넣으면 동작하지 않습니다. **반드시 `G-` 로 시작하는 값**을 넣으세요.

## 3. 1단계 — 측정 ID 입력하기

<div class="manual-goto-bar">
  <span class="manual-goto-label">이 단계 화면</span>
  <a class="manual-goto" href="/admin/site-settings" target="_blank">사이트 설정 열기</a>

사이트 설정 화면을 아래로 내리면 **분석 & 추적 설정** 이라는 상자가 있습니다.

| 입력란 | 필수 | 설명 | 예시 |
|---|---|---|---|
| **Google Analytics 4 측정 ID** | 선택 | GA4에서 발급된 측정 ID입니다. 입력하면 고객 사이트에 자동으로 적용됩니다. | `G-XXXXXXXXXX` |

화면에도 "GA4에서 발급된 측정 ID를 입력하면 사용자 사이트에 자동으로 적용됩니다. 예: G-XXXXXXXXXX" 라고 안내되어 있습니다.

**입력 시 알아 둘 점**

- **소문자로 입력해도 자동으로 대문자로 저장**됩니다. 대소문자를 신경 쓰지 않아도 됩니다.
- 앞뒤 공백은 자동으로 정리됩니다.
- 다만 **값이 올바른 형식인지 검사하지는 않습니다.** 오타가 있어도 그대로 저장되고, 데이터만 조용히 쌓이지 않습니다. **복사·붙여넣기를 권장합니다.**

> **입력 후 반드시 페이지 아래쪽 저장 버튼을 누르세요.** 입력만 하고 화면을 벗어나면 저장되지 않습니다.

> **측정 ID를 비우고 저장하면 연결이 즉시 끊깁니다.** 그 시점부터 GA4로 데이터가 전혀 가지 않습니다. 이미 수집된 과거 데이터는 구글 쪽에 남아 있지만, 새 데이터는 쌓이지 않습니다. 잠시 끄는 용도로 쓸 수는 있으나, 그 기간은 통계에 공백으로 남습니다.

## 4. 2단계 — 자동으로 수집되는 것들

측정 ID를 저장하고 나면 **고객 화면을 열 때마다** 구글 추적 코드가 함께 실행됩니다. 별도 설정 없이 다음 네 가지 쇼핑 행동이 자동으로 기록됩니다.

| 이벤트 이름 | 언제 전송되나 | 무엇을 알 수 있나 |
|---|---|---|
| **view_item** | 고객이 **상품 상세 화면을 열 때** | 어떤 상품이 실제로 관심을 받는지. 목록 노출은 많은데 상세 조회가 적으면 **썸네일이나 가격에 문제**가 있는 것입니다. |
| **add_to_cart** | 상품 상세에서 **장바구니 담기를 누를 때** (수량 반영) | 조회 대비 장바구니 비율. 낮으면 **상세 설명·배송비·재고 표시**를 점검하세요. |
| **begin_checkout** | 상품 상세에서 **바로 구매하기를 누를 때** (수량·금액 반영) | 결제 화면으로 진입한 지점 |
| **purchase** | **주문 완료 화면이 뜰 때** (주문번호·총액·품목 목록 반영) | 실제 매출. 어떤 광고·검색어가 매출로 이어졌는지 여기서 연결됩니다. |

### 4-1. 각 이벤트에 함께 담기는 정보

| 항목 | 내용 |
|---|---|
| **상품 식별값** | 상품의 주소용 이름(slug). 없으면 상품 번호 |
| **상품명** | 등록된 상품명 |
| **브랜드** | 공급사 정보 |
| **카테고리** | 상품이 속한 카테고리명 |
| **가격** | 결제 단가 (회원가가 적용되면 회원가) |
| **수량** | 담거나 구매한 개수 |
| **통화** | `KRW` (원화 고정) |
| **금액** | 총 금액 |

**이 정보가 제대로 담기려면 상품 등록이 잘 되어 있어야 합니다.** 카테고리를 지정하지 않은 상품은 GA4에서도 카테고리가 비어 나와 분석이 어려워집니다.

### 4-2. 자동으로 수집되지 **않는** 것

다음 행동은 현재 GA4로 전송되지 않습니다. 필요하다면 별도 개발이 필요합니다.

- 상품 목록·기획전에서 **상품이 노출된 것** (`view_item_list`)
- 장바구니에서 **상품을 뺀 것** (`remove_from_cart`)
- 장바구니 화면에서 **결제로 넘어간 것** (결제 시작은 상품 상세의 바로 구매하기에서만 잡힙니다)
- **회원가입 완료** (`sign_up`)
- **검색어** (`search`) — 검색은 이 몰의 [검색 로그](/manual/admin/search_logs) 화면에서 확인하세요

## 5. 3단계 — 제대로 붙었는지 확인하기

측정 ID를 저장했다면 **반드시 확인까지 하세요.** 오타 하나로 몇 주치 데이터를 통째로 날리는 일이 흔합니다.

### 5-1. 가장 쉬운 방법 — 실시간 보고서

1. **다른 기기(휴대폰 등)** 로 쇼핑몰 첫 화면에 들어갑니다.
2. 구글 애널리틱스에서 **보고서 → 실시간** 을 엽니다.
3. **현재 사용자 수가 1 이상**으로 잡히면 연결 성공입니다.

**5분 정도 기다려도 0이라면** 아래 5-3의 원인을 확인하세요.

### 5-2. 구매 이벤트까지 확인하려면

가장 확실한 방법은 **저가 상품을 하나 만들어 실제로 끝까지 주문해 보는 것**입니다.

1. 테스트용 상품을 등록합니다.
2. 상품 상세를 열고(→ `view_item`), 장바구니에 담고(→ `add_to_cart`), 결제까지 완료합니다(→ `purchase`).
3. GA4 실시간 보고서의 이벤트 목록에 네 가지가 순서대로 나타나는지 봅니다.

> **테스트 주문은 반드시 처리를 마무리하세요.** 취소·환불하고, 테스트 상품은 판매중지로 바꾸거나 삭제합니다. 그대로 두면 매출 집계와 재고가 어긋납니다.

### 5-3. 데이터가 안 잡힐 때 확인 순서

| 순서 | 확인할 것 |
|---|---|
| 1 | **저장 버튼을 눌렀는가?** 사이트 설정을 다시 열어 값이 남아 있는지 봅니다. |
| 2 | **`G-` 로 시작하는가?** `UA-`·`GTM-` 은 동작하지 않습니다. |
| 3 | **오타는 없는가?** 화면의 값과 GA4의 값을 나란히 놓고 한 글자씩 대조하세요. |
| 4 | **편집 몰이 맞는가?** 다른 몰에 저장했을 수 있습니다. |
| 5 | **광고 차단 프로그램을 쓰고 있지 않은가?** 대부분의 광고 차단기가 GA4 수집을 막습니다. 확인은 차단기를 끄거나 다른 기기에서 하세요. |
| 6 | **GA4 속성과 데이터 스트림이 올바른가?** 여러 속성을 만들어 두고 엉뚱한 곳을 보고 있는 경우가 많습니다. |

## 6. 알아 두어야 할 제약

### 6-1. 이 사이트는 검색엔진 색인이 차단되어 있습니다

현재 이 몰은 **검색엔진이 페이지를 수집하지 않도록 설정**되어 있습니다. 이는 GA4 수집 자체와는 무관하지만(방문만 있으면 GA4는 기록합니다), **네이버·구글 검색을 통한 자연 유입이 거의 발생하지 않는다**는 뜻입니다.

따라서 GA4를 붙여도 **검색 유입 수치는 계속 0에 가까울 수 있습니다.** 이는 GA4 설정 문제가 아닙니다. 실제 서비스를 시작할 때 색인 차단을 해제해야 검색 유입이 생깁니다.

### 6-2. 몰마다 따로 설정해야 합니다

측정 ID는 **몰별로 저장**됩니다. 몰이 세 개면 세 번 입력해야 합니다.

| 방식 | 장점 | 단점 |
|---|---|---|
| **몰마다 다른 측정 ID** | 몰별 성과가 완전히 분리되어 명확합니다. | GA4에서 화면을 여러 번 오가야 합니다. |
| **모든 몰에 같은 측정 ID** | 한곳에서 전체를 봅니다. | 어느 몰의 매출인지 구분이 어렵습니다. |

**몰이 서로 다른 고객층·다른 브랜드라면 몰마다 다른 측정 ID**를 쓰는 편이 분석이 쉽습니다.

### 6-3. 관리자 화면은 추적되지 않습니다

GA4 코드는 **고객 화면에만** 들어갑니다. 운영자가 관리자 화면에서 아무리 작업해도 GA4 수치에 잡히지 않습니다. 이는 의도된 동작이며, 덕분에 운영자 활동이 통계를 오염시키지 않습니다.

다만 **운영자가 고객 화면을 확인하는 것은 방문으로 잡힙니다.** 방문이 적은 초기에는 이 영향이 꽤 클 수 있으니, 수치를 볼 때 감안하세요.

## 7. 자주 막히는 상황

**Q. 측정 ID를 넣었는데 GA4에 아무 데이터도 안 옵니다.**
5-3의 확인 순서를 위에서부터 하나씩 짚어 보세요. 실제로는 ① 저장 안 함, ③ 오타, ⑤ 광고 차단기 세 가지가 대부분입니다.

**Q. `UA-` 로 시작하는 값을 가지고 있습니다.**
예전 방식(유니버설 애널리틱스)의 ID로 지금은 동작하지 않습니다. 구글 애널리틱스에서 **GA4 속성**을 새로 만들어 `G-` 로 시작하는 측정 ID를 받으세요.

**Q. 구글 태그 매니저를 쓰고 싶습니다.**
이 화면에는 태그 매니저 컨테이너 ID(`GTM-`)를 넣는 칸이 없습니다. `GTM-` 값을 측정 ID 칸에 넣어도 동작하지 않습니다.

**Q. 방문자 수가 이 몰의 방문자 통계와 다릅니다.**
정상입니다. 세는 방식이 다릅니다. 이 몰의 통계는 **접속 위치 기준으로 하루 1회**만 세고, GA4는 **브라우저 단위 세션**으로 셉니다. 광고 차단기 사용자는 GA4에만 빠집니다. **두 숫자가 정확히 맞는 일은 없으니, 각각을 추세로만 보세요.**

**Q. 구매 이벤트만 안 잡힙니다.**
주문 완료 화면이 실제로 열려야 전송됩니다. 결제 도중 창을 닫거나 결제가 실패하면 전송되지 않습니다. 실제로 끝까지 완료된 테스트 주문으로 확인하세요.

**Q. 장바구니 화면에서 결제로 넘어간 것이 안 잡힙니다.**
현재 결제 시작 이벤트는 **상품 상세의 바로 구매하기**에서만 전송됩니다. 장바구니를 거친 경로는 잡히지 않습니다.

**Q. 상품 카테고리가 GA4에서 비어 있습니다.**
그 상품에 카테고리가 지정되어 있지 않습니다. 상품 관리에서 카테고리를 지정하세요.

**Q. 몰을 새로 만들었는데 GA4가 자동으로 붙나요?**
붙지 않습니다. 새 몰마다 사이트 설정에서 측정 ID를 직접 입력해야 합니다.

**Q. 개인정보 문제는 없나요?**
GA4에는 이름·이메일 같은 개인 식별 정보를 보내지 않도록 되어 있습니다. 다만 GA4를 사용한다는 사실은 **개인정보처리방침**에 명시하는 것이 원칙입니다.

**Q. 광고를 시작하려는데 무엇을 먼저 해야 하나요?**
① 측정 ID 입력 → ② 5-1·5-2로 수집 확인 → ③ 광고 집행 순서입니다. **확인 없이 광고를 시작하면 어떤 광고가 효과가 있었는지 영영 알 수 없습니다.**

## 8. 아직 지원하지 않는 것

- 구글 태그 매니저(GTM) 연결
- 목록 노출·장바구니 제거·회원가입·검색 이벤트 전송
- 장바구니 화면에서의 결제 시작 이벤트
- 관리자 화면 안에서 GA4 수치 보기 (구글 애널리틱스에서 확인해야 합니다)
- 측정 ID 형식 검증 (오타가 있어도 그대로 저장됩니다)
- 네이버·메타 등 다른 분석 도구 연결
- 여러 몰의 측정 ID 일괄 설정
