# 세트 · 기획상품 관리

이미 등록된 상품들을 **묶어서 하나의 상품으로 파는 화면**입니다. "헤어케어 3종 세트", "설 선물세트", "2개 묶음 할인"처럼 **여러 개를 한 번에 사는 상품**을 여기서 만듭니다.

낱개 상품과 옵션은 [상품 관리](/manual/admin/products)에서 만들고, 그것들을 **묶는 일**만 이 화면에서 합니다. 여기서 새 상품을 처음부터 만들 수는 없습니다.

화면 위치: <span class="manual-path">관리자 &gt; 상품 관리 &gt; 세트·기획상품</span> (`/admin/derived-products`)

> **세트의 재고는 직접 입력하지 않습니다.** 구성으로 넣은 상품들의 재고에서 **자동으로 계산**됩니다. 그래서 세트 화면에는 재고 입력칸이 아예 없습니다. (4장 참고)

> **"기획상품"은 "기획전"과 다릅니다.** 여기서 만드는 기획상품은 **여러 상품을 묶어 파는 하나의 상품**이고, [기획전](/manual/admin/exhibitions)은 상품을 모아 보여 주는 **전시 화면**입니다.

## 1. 시작하기 전에 — 준비물

| 준비물 | 왜 필요한가 | 어디서 만드나 |
|---|---|---|
| **묶을 상품들** | 세트는 이미 등록된 상품(정확히는 그 상품의 옵션 단위)을 골라 담는 화면입니다. | 상품 관리 |
| **각 상품의 재고** | 구성 상품에 재고가 없으면 세트의 판매 가능 수량이 0 이 됩니다. | 상품 관리 (옵션상품은 옵션·SKU 관리) |
| **세트 판매가** | 세트를 얼마에 팔지 정합니다. | — |

## 2. 유형 4가지 — 무엇을 고를까

세트를 만들 때 고르는 **유형**입니다. 상품 성격에 맞게 고르면 됩니다.

| 유형 | 어떤 경우에 쓰나 |
|---|---|
| **묶음** | 같은 계열 상품을 여러 개 묶어 파는 형태. (예: 같은 샴푸 2개 묶음) |
| **세트** | 서로 다른 상품을 한 세트로 파는 형태. 새로 만들 때의 기본값입니다. |
| **선물세트** | 선물용으로 구성한 세트. |
| **선택형세트** | 구성 중 일부를 고르는 형태를 염두에 둔 유형. |

> **네 유형 모두 만들고 관리하는 방법은 똑같습니다.** 재고 계산 방식도 같습니다. 유형은 **성격을 구분하는 표시**로 쓰이며, 목록에서 유형별로 알아보기 쉽게 해 줍니다.

## 3. 전체 흐름 한눈에 보기

```
① [+ 새로 만들기]
        ↓
② 상품명 · 유형 · 판매가 입력
        ↓
③ [만들고 구성하기 →]  ← 이 순간 상품이 만들어집니다
        ↓
④ 구성 SKU 검색해서 담기 (+ 구성수량 입력)
        ↓
⑤ [저장]
        ↓
⑥ (필요하면) [기본정보] 에서 이미지·설명·노출 설정 채우기
        ↓
⑦ 고객 화면에서 확인
```

**③번을 누르는 순간 상품이 실제로 만들어집니다.** 이때 판매 상태는 `판매중`, 노출은 `노출` 로 시작합니다. 즉 **구성을 채우기도 전에 이미 상품이 존재합니다.** 준비가 덜 됐다면 기본정보 화면에서 노출을 잠시 꺼 두세요.

---

## 4. 재고는 어떻게 계산되나 (가장 중요)

세트는 **자기 재고를 갖지 않습니다.** 구성으로 담은 상품들의 재고에서 **만들 수 있는 최대 세트 개수**를 계산합니다.

계산 규칙은 이렇습니다.

```
구성 A : 재고 100개, 구성수량 2  →  100 ÷ 2 = 50세트
구성 B : 재고  30개, 구성수량 1  →   30 ÷ 1 = 30세트
구성 C : 재고  80개, 구성수량 1  →   80 ÷ 1 = 80세트
                                       ↓
        가장 적은 것이 세트 재고  →  30세트
```

- 목록의 **가용수량** 칸이 이 계산 결과입니다.
- **구성이 하나도 없으면 가용수량은 0** 입니다. 즉 **구성을 채우기 전에는 팔리지 않습니다.**
- 고객이 세트를 1개 사면 **구성 상품이 각각 `구성수량` 만큼 차감**됩니다. 위 예에서 세트 1개가 팔리면 A 는 2개, B 는 1개, C 는 1개가 빠집니다.

> **구성 상품의 재고가 줄면 세트 재고도 같이 줄어듭니다.** 구성 중 하나라도 품절이면 **세트 전체가 팔리지 않습니다.** 세트가 갑자기 안 팔린다면 구성 상품의 재고부터 확인하세요.

## 5. 1단계 — 세트 만들기

<div class="manual-goto-bar">
  <span class="manual-goto-label">이 단계 화면</span>
  <a class="manual-goto" href="/admin/derived-products/new" target="_blank">새 세트·기획상품 화면 열기</a>
</div>

| 입력란 | 필수 | 설명 | 예시 |
|---|---|---|---|
| **상품명** | 필수 | 고객에게 보이는 세트 이름입니다. 최대 100자. | `헤어케어 3종 세트` |
| **유형** | 필수 | 위 2장의 4가지 중 하나. 기본값은 `세트`. | `세트` |
| **판매가 (원)** | 선택 | 세트를 얼마에 팔지. 0 으로 두고 나중에 고칠 수도 있습니다. | `39000` |

상품명을 비우고 저장하면 `상품명을 입력하세요.` 안내가 뜹니다.

**[만들고 구성하기 →]** 를 누르면 상품이 만들어지고 곧바로 구성 화면으로 넘어갑니다.

## 6. 2단계 — 구성 채우기

구성 화면 위쪽에 현재 **가용수량**이 표시됩니다. 구성을 바꾸고 저장할 때마다 다시 계산됩니다.

### 6-1. 유형 바꾸기

화면 위의 **유형** 선택 상자에서 언제든 바꿀 수 있습니다. 여기서는 조금 더 자세한 설명이 붙습니다.

| 화면 표기 | 의미 |
|---|---|
| **묶음(동일계열)** | 같은 계열 상품을 여러 개 묶음 |
| **세트(서로 다른 상품)** | 서로 다른 상품을 한 세트로 |
| **선물세트** | 선물용 구성 |
| **선택형 세트** | 구성 중 일부를 고르는 형태 |

### 6-2. 구성 SKU 추가하기

1. **구성 SKU 추가** 칸에 **상품명 또는 SKU코드**를 넣고 **검색**을 누릅니다. (Enter 도 됩니다)
2. 검색 결과에서 담을 항목을 클릭하면 아래 **구성 목록**에 추가됩니다.
3. 각 줄의 **구성수량**을 넣습니다. (세트 1개에 몇 개가 들어가는지. 최소 1)
4. 잘못 담았으면 오른쪽 **삭제**를 누릅니다.

검색 결과에는 상품명 옆에 **옵션 이름**과 **SKU코드**, 그리고 **현재 재고**가 함께 표시됩니다. 그래서 "블랙 / 라지"처럼 **정확히 어떤 옵션을 담는지 골라 넣을 수 있습니다.**

> **검색에 안 잡히는 것들이 있습니다.** 아래는 의도적으로 후보에서 빠집니다.
> - **이 세트 자신** (자기 자신을 구성에 넣는 것 방지)
> - **다른 세트·묶음 상품** (세트 안에 세트를 넣는 것 방지)
> 세트를 세트로 묶고 싶다면 그 세트의 구성 상품들을 직접 담으세요.

### 6-3. 저장

**[저장]** 버튼을 눌러야 반영됩니다. 성공하면 옆에 `저장됨 (유형 · 구성 N)` 이 표시됩니다.

> **저장하지 않고 화면을 벗어나면 담은 구성이 전부 사라집니다.** 구성수량만 고쳐도 마찬가지입니다.

> **구성을 모두 비우고 저장하면 이 상품은 일반 단일상품으로 되돌아갑니다.** 세트·기획상품 목록에서 사라지고, 그때부터는 자기 재고를 갖는 보통 상품이 됩니다. 세트를 잠시 내리려는 것이라면 구성을 비우지 말고 기본정보에서 **판매 상태**나 **노출 설정**을 바꾸세요.

## 7. 3단계 — 기본정보 채우기 (이미지 · 설명)

새로 만든 세트에는 **이미지도 설명도 없습니다.** 목록에서 **[기본정보]** 를 누르면 일반 상품과 똑같은 편집 화면이 열립니다. 거기서 아래를 채우세요.

- 대표 이미지 · 썸네일 이미지
- 카테고리 · 브랜드
- 상품 기본설명 · 상품 상세 설명
- 노출 설정 · 판매 상태
- 판매가 (여기서 고쳐도 됩니다)

> **기본정보 화면의 `재고 수량` 칸은 세트에 의미가 없습니다.** 세트 재고는 언제나 구성에서 계산되므로, 이 칸을 고쳐도 판매 가능 수량은 바뀌지 않습니다.

자세한 항목 설명은 상품 관리 매뉴얼을 보세요.

## 8. 목록 화면에서 할 수 있는 일

| 칸 | 설명 |
|---|---|
| **유형** | 묶음 / 세트 / 선물세트 / 선택형세트 |
| **상품명** | 세트 이름 |
| **구성** | 담긴 구성 SKU 가 몇 종인지 |
| **가용수량** | 지금 만들 수 있는 세트 수. **0 이면 빨간색**으로 표시됩니다. |
| **판매가** | 세트 판매가 |
| **상태** | 판매 상태 |

각 줄 오른쪽의 버튼은 이렇습니다.

| 버튼 | 하는 일 |
|---|---|
| **구성 관리** | 구성 SKU 를 담고 빼는 화면으로 갑니다. |
| **기본정보** | 이미지·설명·가격 등 일반 상품 편집 화면으로 갑니다. |
| **삭제** | 세트를 지웁니다. |

> **[삭제]는 세트만 지웁니다.** 구성으로 담았던 **원래 상품들은 그대로 남습니다.** 다만 구성 정보는 함께 사라지므로 되돌릴 수 없습니다.

## 9. 자주 막히는 상황

**Q. 세트를 만들었는데 가용수량이 0 입니다.**
① 구성을 하나도 담지 않았거나, ② 구성 상품 중 하나의 재고가 0 입니다. 구성 목록의 `재고` 칸을 보고 어떤 것이 비었는지 확인하세요.

**Q. 담고 싶은 상품이 검색에 안 나옵니다.**
아래를 확인하세요.
1. **편집 중인 몰**의 상품이 맞는가? (다른 몰 상품은 검색되지 않습니다)
2. 담으려는 것이 **다른 세트·묶음 상품**은 아닌가? (세트 안에 세트는 담을 수 없습니다)
3. 이 세트 자기 자신을 담으려는 것은 아닌가?

**Q. 구성 상품을 담았는데 저장 후에 사라졌습니다.**
**[저장]** 버튼을 누르지 않고 화면을 벗어났을 가능성이 큽니다. 담기만 해서는 반영되지 않습니다.

**Q. 세트 구성에 쓴 옵션을 상품 관리에서 지우려니 오류가 납니다.**
정상입니다. 세트가 그 옵션(SKU)을 참조하고 있어서 막습니다. 세트 구성에서 먼저 빼고 지우세요.

**Q. 상품 관리 목록에 이 세트가 안 보입니다.**
정상입니다. 세트·묶음·선물세트·선택형세트는 상품 목록에서 빠지고 이 메뉴에서만 보입니다.

**Q. 세트 이미지가 안 나옵니다.**
새로 만든 세트에는 이미지가 없습니다. **[기본정보]** 에서 대표 이미지와 썸네일을 넣으세요.

**Q. 세트를 잠깐만 내리고 싶습니다.**
구성을 비우면 **단일상품으로 되돌아가 버립니다.** 그러지 말고 [기본정보]에서 `판매 상태 = 중지(판매중단)` 또는 `노출 설정 = 노출중지` 를 쓰세요.

**Q. 세트 가격이 고객 화면에서 다르게 보입니다.**
그 세트에 **쇼핑특가**가 걸려 있을 수 있습니다.

---

## 10. 아직 지원하지 않는 것

- 구성 상품 가격을 합산해 세트가를 자동 계산하는 기능 (판매가는 직접 입력합니다)
- 구성 중 일부를 고객이 고르게 하는 화면 (`선택형세트` 유형은 있으나 구성 방식은 다른 유형과 같습니다)
- 세트 복사·복제
