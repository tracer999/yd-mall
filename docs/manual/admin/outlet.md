# 아울렛 관리

**아울렛은 "몰 안의 몰"입니다.** 지난 시즌 이월품, 유통기한이 임박한 상품, 전시·리퍼브·포장이 상한 상품처럼 **왜 싼지가 상품 자체에 있는 상품**을 모아 **상시로 파는 코너**입니다. 이 화면에서 아울렛 상품·아울렛 전용 카테고리·아울렛 운영 설정을 모두 관리합니다.

> **아울렛은 가격을 만들지 않습니다. 진열만 합니다.** 고객이 보는 가격·할인율은 전부 **상품 관리에 설정된 판매가**입니다. 아울렛 화면에는 가격 입력란이 아예 없습니다. 더 싸게 팔고 싶으면 **상품 관리에서 가격을 먼저 내린 뒤** 아울렛에 담으세요.

## 1. 시작하기 전에 — 아울렛과 쇼핑특가의 차이

이 둘을 헷갈리면 운영이 계속 꼬입니다. 먼저 구분하세요.

| | **아울렛** | **쇼핑특가** |
|---|---|---|
| 성격 | **상시** 할인 채널 | **기간 한정** 행사 |
| 왜 싼가 | **상품 자체에 사유가 있음** (이월·리퍼브·임박 등) | 프로모션 때문 |
| 언제 정상가로 돌아가나 | **돌아가지 않음.** 재고가 소진될 때까지 계속 쌈 | **기간이 끝나면 정상가로 복귀** |
| 가격을 어디서 정하나 | 상품 관리(판매가 그대로) | 특가 금액을 따로 지정 |

> **단순한 기간 할인 상품은 아울렛에 넣지 마세요.** "이번 주만 30% 할인"은 쇼핑특가입니다. 아울렛에 넣으면 행사가 끝나도 계속 아울렛에 남아 "왜 싼지 설명할 수 없는 상품"이 됩니다.

<div class="manual-goto-bar">
  <span class="manual-goto-label">기간 할인이라면</span>
  <a class="manual-goto is-sub" href="/admin/deals" target="_blank">쇼핑특가 관리로 이동</a>
</div>

### 준비물

| 준비물 | 왜 필요한가 | 어디서 |
|---|---|---|
| **최소 할인율을 넘긴 상품** | 할인율이 기준(기본 20%)에 못 미치면 **등록 자체가 막힙니다.** | 상품 관리에서 가격 조정 |
| **아울렛 카테고리** | 없어도 등록은 되지만 전부 `미분류`가 되어 고객이 탐색하기 어렵습니다. | 아울렛 카테고리 화면 |
| **하자 고지 문구** | 리퍼브·전시·포장훼손 상품은 **필수**입니다. | 미리 작성해 두세요 |

> **지금 편집 중인 몰을 반드시 확인하세요.** 아울렛 상품·카테고리·운영 설정은 모두 **몰별로 따로 저장**됩니다. 다른 몰을 편집 중이면 저장해도 원하는 몰에는 반영되지 않습니다.

## 2. 전체 흐름 한눈에 보기

```
① 상품 관리에서 가격/할인율 조정   (최소 할인율 이상으로)
        ↓
② 아울렛 카테고리 만들기            (의류 / 신발 / 건강식품 …)
        ↓
③ 아울렛 상품 등록                  (할인 사유 + 상태 고지 필수)
        ↓
④ 아울렛 운영 설정 확인             (특히 GNB 노출 최소 상품 수)
        ↓
⑤ 고객 화면 상단 메뉴에 아울렛이 뜨는지 확인
```

**④를 건너뛰면 "상품은 다 넣었는데 메뉴가 안 보인다"는 상황이 그대로 발생합니다.** 아래 6장을 반드시 읽으세요.

## 3. 1단계 — 아울렛 카테고리 만들기

<div class="manual-goto-bar">
  <span class="manual-goto-label">이 단계 화면</span>
  <a class="manual-goto" href="/admin/outlet/categories" target="_blank">아울렛 카테고리 화면 열기</a>
</div>

아울렛 목록 오른쪽 위의 **카테고리** 버튼으로 들어갑니다.

> **아울렛 카테고리는 일반 상품 카테고리와 완전히 별개입니다.** 여기서 만든 분류는 아울렛 안에서만 쓰이고, 일반 카테고리 목록에는 절대 섞이지 않습니다. 반대로 일반 카테고리도 여기에 나타나지 않습니다.

### 카테고리 등록

| 입력란 | 필수 | 설명 | 예시 |
|---|---|---|---|
| **카테고리명** | 필수 | 최대 50자. | `시즌오프 의류` |
| **상위 카테고리** | 선택 | 비우면 `최상위`. 계층을 만들 때 지정합니다. | `의류` |
| **순서** | 선택 | 작을수록 앞. | `0` |

**추가** 버튼을 누르면 등록됩니다. 만들 수 있는 계층 깊이는 화면 아래 *"최대 N뎁스까지 만들 수 있습니다"* 안내에 표시됩니다.

### 목록에서 바로 수정하기

등록된 카테고리는 표에서 **이름·상위 카테고리·순서·활성** 을 직접 고칠 수 있고, 줄 오른쪽 **저장** 버튼을 눌러야 반영됩니다.

| 항목 | 설명 |
|---|---|
| **상품** | 이 카테고리에 속한 **판매중** 아울렛 상품 수 |
| **활성 / 비활성** | 비활성으로 두면 고객 화면 분류에서 빠집니다 |
| **삭제** | 확인창 후 삭제. 연결된 아울렛 상품은 **미분류로 남습니다**(상품이 지워지진 않습니다) |

> **한 줄 안의 값을 바꿨으면 그 줄의 저장 버튼을 눌러야 합니다.** 여러 줄을 고치고 한 번에 저장되는 화면이 아닙니다. 저장하지 않고 다른 화면으로 넘어가면 변경이 사라집니다.

> **하위 카테고리가 있는 카테고리는 삭제되지 않습니다.** *"하위 카테고리가 있어 삭제할 수 없습니다."* 라는 오류가 뜹니다. 하위를 먼저 정리한 뒤 삭제하세요. (하위가 조용히 최상위로 올라가 계층이 망가지는 것을 막는 안전장치입니다.)

### 카테고리와 할인 사유는 다릅니다

| | 카테고리 | 할인 사유 |
|---|---|---|
| 답하는 질문 | **무슨 상품인가** | **왜 싼가** |
| 예 | 의류 / 신발 / 건강식품 | 시즌오프 / 리퍼브 / 임박상품 |
| 어디서 정하나 | 아울렛 카테고리 화면 | 상품 등록 화면 |

고객은 이 **두 축을 함께** 써서 아울렛을 탐색합니다. 둘 다 채워야 제 기능을 합니다.

### 4-1. 상품 고르기

검색창에 **상품명 또는 상품코드**를 **2글자 이상** 입력하면 후보가 뜹니다. 각 줄에는 판매가·할인율·재고가 함께 표시되므로 담아도 되는지 바로 판단할 수 있습니다.

**담을 수 없는 상품도 목록에 나오되, 빨간 글씨로 이유가 붙고 선택이 막힙니다.**

| 빨간 안내 | 뜻 | 해결 |
|---|---|---|
| **이미 아울렛에 있음** | 같은 상품을 두 번 담을 수 없습니다. | 기존 등록을 수정하세요 |
| **할인율 N% — 최소 M% 필요** | 최소 할인율에 미달합니다. | 상품 관리에서 가격을 내리거나, 운영 설정에서 최소 할인율을 낮추세요 |

<div class="manual-goto-bar">
  <span class="manual-goto-label">할인율이 모자라다면</span>
  <a class="manual-goto is-sub" href="/admin/products" target="_blank">상품 관리에서 판매가 조정</a>
</div>

> **수정 화면에서는 상품을 바꿀 수 없습니다.** *"상품은 변경할 수 없습니다. 다른 상품이면 새로 등록하세요."* 라고 안내됩니다. 상품을 잘못 골랐다면 이 등록을 **내리기**로 지우고 새로 등록하세요.

### 4-2. 할인 사유 고르기 (필수)

**아울렛의 존재 이유이자 필수 항목입니다.** 고객에게 "왜 이 상품이 싼지" 알려 주는 값입니다.

| 사유 | 어떤 상품에 | 추가로 요구하는 것 |
|---|---|---|
| **시즌오프** | 지난 시즌 이월상품 | — |
| **단종·구형** | 단종 예정·구형 모델 | — |
| **재고정리** | 재고 과다 소진 | — |
| **전시상품** | 매장·행사 전시품 | **상태 등급 필수** |
| **리퍼브** | 점검·수리 후 재판매 | **상태 등급 필수** |
| **포장훼손** | 외부 포장 손상, 제품은 정상 | **상태 등급 필수** |
| **임박상품** | 유통기한 임박 | **유통기한 날짜 필수** |

사유를 고르면 필요한 입력란이 **화면에 자동으로 나타납니다.** 안 보이는 칸은 그 사유에 필요 없는 칸입니다.

**고객 노출 문구** (선택, 최대 255자) — 상품 목록·상세의 사유 배지 옆에 표시됩니다. 비우면 사유 이름만 나옵니다.
예: `25FW 시즌 이월 — 제품은 새 상품입니다`

**유통기한** — `임박상품`을 골랐을 때만 나타나며 **필수**입니다. 고객 화면에 그대로 표시됩니다.

### 4-3. 상품 상태 고지 (전시·리퍼브·포장훼손 전용)

> ⚠️ **여기가 아울렛에서 가장 위험한 구간입니다.** 리퍼브·전시·포장훼손 상품을 일반 이월상품처럼 보여 주면 **교환·반품 분쟁이 발생합니다.** 화면에도 같은 경고가 붉은 글씨로 표시됩니다.

**상태 등급** (필수)

| 등급 | 의미 | 하자 고지 |
|---|---|---|
| **A급** | 미개봉·새제품 수준 | 선택 |
| **B급** | 경미한 사용·외관 하자 | **필수** |
| **C급** | 눈에 띄는 하자 있음 | **필수** |

**하자 고지** — B·C 등급이면 **비워 두면 저장되지 않습니다.** 저장 시 *"B·C 등급은 하자 고지 내용이 필수입니다. 미고지 시 교환·반품 분쟁이 발생합니다."* 라는 오류가 뜹니다.

작성 예: `우측 하단 3cm 스크래치. 기능 이상 없음. 박스 모서리 눌림.`

이 내용은 **상품 상세의 구매 버튼 위에 눈에 띄게 표시**됩니다. 상세 설명에 따로 적을 필요가 없습니다.

> **A급이어도 알릴 하자가 있으면 적으세요.** 하자 고지는 등급과 별개로, **내용이 있으면 고객에게 고지**됩니다. 분쟁을 줄이는 가장 값싼 방법입니다.

### 4-4. 진열 설정

| 입력란 | 필수 | 설명 |
|---|---|---|
| **아울렛 카테고리** | 선택 | 비우면 `미분류`. 아울렛 카테고리가 하나도 없으면 만들라는 안내가 뜹니다. |
| **정렬 순서** | 선택 | 작을수록 앞에 나옵니다. |
| **판매 시작** | 선택 | **비우면 즉시 시작.** |
| **판매 종료** | 선택 | **비우면 재고 소진까지 무기한.** |
| **고객 화면에 노출** | 체크박스 | 끄면 아울렛에서 숨겨집니다. 기본은 켜짐. |

마지막으로 **아울렛에 등록**(수정 화면에서는 **수정**) 버튼을 누릅니다.

> **고객에게 실제로 팔리려면 4가지가 모두 맞아야 합니다.** ① **고객 화면에 노출** 켜짐 ② **판매 기간**이 오늘을 포함 ③ 상품이 **판매중 또는 품절** 상태 ④ 상품 노출이 **`노출(공개)`**. 하나라도 어긋나면 아울렛에 뜨지 않습니다.

## 6. GNB 노출 — "메뉴가 안 보여요"의 정체

목록 화면 맨 위에는 초록 또는 노란 안내 박스가 항상 떠 있습니다. **아울렛 운영에서 가장 중요한 지표**입니다.

| 색 | 문구 | 뜻 |
|---|---|---|
| **초록** | 아울렛이 고객 메뉴(GNB)에 노출되고 있습니다 — 판매중 N개 | 정상. 고객 상단 메뉴에 아울렛이 보입니다. |
| **노랑** | 아울렛이 고객 메뉴(GNB)에서 숨겨져 있습니다 — 판매중 N개 / 최소 M개 필요 | **상품이 부족해 메뉴가 자동으로 숨겨졌습니다.** |

> **메뉴 관리에서 아울렛을 켜 두어도, 판매중 상품이 기준(기본 30개)에 못 미치면 고객 화면에 나타나지 않습니다. 고장이 아니라 빈 메뉴를 막는 안전장치입니다.**

해결 방법은 둘 중 하나입니다.

1. **아울렛 상품을 더 담는다** — 노란 박스에 *"N개를 더 등록하면 노출됩니다"* 라고 부족분이 표시됩니다.
2. **운영 설정에서 `GNB 노출 최소 상품 수`를 낮춘다** — `0` 으로 두면 상품 수와 관계없이 항상 노출됩니다.

여기서 세는 **판매중**은 등록 건수가 아니라 **실제로 고객에게 보이는 상품 수**입니다. 노출 꺼짐·기간 종료·판매중지 상품은 세지 않습니다.

## 7. 아울렛 운영 설정

목록 화면 **맨 아래**에 있는 영역입니다. 이 몰에서 아울렛을 어떻게 운영할지 정합니다.

| 항목 | 설명 |
|---|---|
| **사용할 할인 사유** | 체크한 사유만 상품 등록 폼에 나타납니다. 이 몰에서 발생하지 않는 사유는 꺼 두세요. (예: 건강식품몰은 시즌 이월이 없고 유통기한 임박만 있습니다) |
| **최소 할인율 (%)** | 이 할인율에 못 미치는 상품은 **아울렛에 등록할 수 없습니다.** 허위 할인을 막는 장치입니다. |
| **GNB 노출 최소 상품 수** | 판매중 아울렛 상품이 이 수에 못 미치면 고객 메뉴에서 **자동으로 숨습니다.** `0` 이면 항상 노출. |
| **아울렛 상품을 일반 상품 목록에도 함께 노출** | 끄면 아울렛 상품은 **아울렛 안에서만** 보입니다. 신상품·브랜드 이미지를 보호하려면 끄세요. |
| **아울렛 공통 고지** | 아울렛 목록·상세 상단에 표시되는 문구. 교환·반품 조건이 일반 상품과 다르면 **반드시 적으세요.** 예: `아울렛 상품은 단순 변심 반품 시 왕복 배송비가 부과됩니다.` |

입력 후 **설정 저장** 버튼을 눌러야 반영됩니다.

> **`사용할 할인 사유`를 껐다고 이미 등록된 상품이 사라지지는 않습니다.** 다만 그 상품을 **수정**하려고 하면 *"이 몰에서 사용하지 않는 할인 사유입니다"* 라는 오류로 저장이 막힙니다. 사유를 끌 때는 그 사유로 등록된 상품이 있는지 먼저 확인하세요.

> **`최소 할인율`을 올려도 이미 등록된 상품은 그대로 남습니다.** 하지만 수정 시 검증에 걸려 저장할 수 없게 됩니다. 기준을 올릴 때는 기존 상품의 할인율을 함께 점검하세요.

## 8. 자주 막히는 상황

**Q. 고객 화면 상단에 아울렛 메뉴가 없습니다.**
가장 흔한 문제입니다. 목록 맨 위 안내 박스가 **노란색**이라면 판매중 상품이 기준(기본 30개)에 못 미쳐 메뉴가 자동으로 숨겨진 것입니다. 상품을 더 담거나 운영 설정에서 최소 상품 수를 낮추세요. (위 6장 참고)

**Q. 상품을 검색했는데 회색으로 흐려져 선택이 안 됩니다.**
빨간 글씨로 이유가 적혀 있습니다. ① **이미 아울렛에 있음** — 중복 등록 불가 ② **할인율 부족** — 상품 관리에서 가격을 먼저 내리세요.

**Q. 아울렛에서 가격을 더 낮추고 싶습니다.**
아울렛에는 가격 입력란이 없습니다. **상품 관리에서 판매가를 수정**하면 아울렛 가격도 함께 바뀝니다.

**Q. 저장 버튼을 눌렀는데 "하자 고지 내용이 필수입니다"라고 나옵니다.**
상태 등급을 **B급 또는 C급**으로 골랐기 때문입니다. 하자 고지 칸을 채워야 저장됩니다.

**Q. "임박상품은 유통기한을 입력해야 합니다"가 뜹니다.**
할인 사유가 `임박상품`이면 유통기한 날짜가 필수입니다. 사유를 고르면 유통기한 칸이 화면에 나타납니다.

**Q. 등록은 됐는데 고객 아울렛 화면에 상품이 안 보입니다.**
① 목록에서 **노출**이 `숨김`인지 ② **판매 시작**이 미래이거나 **판매 종료**가 지났는지 ③ 상품 관리에서 그 상품이 **판매중지**이거나 **숨김/회원전용**인지 확인하세요.

**Q. 상품을 "내리기" 하면 상품이 삭제되나요?**
아닙니다. 아울렛에서만 내려갑니다. 상품 자체와 재고·주문 이력은 그대로입니다.

**Q. 아울렛 카테고리를 지우면 상품도 지워지나요?**
아닙니다. 연결됐던 상품은 **미분류**로 남습니다.

**Q. 일반 카테고리를 아울렛 카테고리 목록에서 고르고 싶습니다.**
불가능합니다. 두 분류는 완전히 분리되어 있습니다. 아울렛용 분류는 아울렛 카테고리 화면에서 새로 만드세요.

**Q. 할인율이 높은 상품을 자동으로 아울렛에 모을 수 있나요?**
없습니다. 아울렛 상품은 **사람이 사유를 붙여 하나씩** 담습니다. 할인율만으로 긁어오면 "왜 싼지 설명할 수 없는 상품"이 섞이기 때문입니다.

## 9. 아직 지원하지 않는 것

- 아울렛 전용 가격 설정 (가격은 언제나 상품 관리 값)
- 할인율 조건으로 상품 자동 수집
- 엑셀/CSV로 아울렛 상품 일괄 등록
- 아울렛 채널별 매출 집계
