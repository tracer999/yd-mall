# 공동구매 관리

정해진 **기간 동안만 특별한 가격으로 파는 판매 캠페인**을 만드는 메뉴입니다. "3일간 홍삼정 40% 공동구매"처럼 시작·종료 일시를 정하고, 그 기간에만 적용되는 **공동구매가**로 상품을 팝니다. 목표 수량을 걸어 "얼마나 모였는지"를 고객에게 보여 줄 수도 있습니다.

> **공동구매는 "실제 결제 금액을 바꾸는" 기능입니다.** 기획전이 전시용 화면인 것과 달리, 여기서 정한 **공동구매가가 고객이 실제로 내는 금액**이 됩니다. 가격을 잘못 넣으면 그 금액 그대로 팔립니다. 저장 전에 숫자를 두 번 확인하세요.

---

## 1. 시작하기 전에 — 준비물 3가지

| 준비물 | 왜 필요한가 | 어디서 만드나 |
|---|---|---|
| **팔 상품** | 공동구매는 이미 등록된 상품을 담아 가격만 바꿔 파는 방식입니다. 여기서 상품을 새로 만들 수는 없습니다. | 상품 관리 |
| **판매할 가격 결정** | 정상가와 공동구매가 두 개를 미리 정해 두어야 합니다. 할인율은 자동 계산됩니다. | — |
| **이미지 3종** | 목록 카드 썸네일 / 상세 PC 대표 / 상세 모바일 대표. | 이미지 편집 도구 |

## 2. 전체 흐름 한눈에 보기

```
① 기본 정보 입력 → 저장    (여기까지 해야 공동구매가 "생깁니다")
        ↓
② 상품 담기                 (검색 모달에서 상품 선택)
        ↓
③ 공동구매가 · 수량 제한 입력 → 상품 설정 저장
        ↓
④ 운영 상태를 "발행"으로 + 목록 노출 켜기
        ↓
⑤ 고객 화면 확인 → 기간이 시작되면 자동으로 '진행중'
```

**①번 저장을 먼저 해야 ②③이 가능합니다.** 등록 화면에는 상품 영역이 아직 나타나지 않습니다. 저장 후 이어지는 수정 화면에서 계속 작업하세요.

### 3-1. 기본 정보

| 입력란 | 필수 | 설명 | 예시 |
|---|---|---|---|
| **공동구매명** | 필수 | 고객에게 보이는 제목입니다. 최대 200자. | `3일간 홍삼정 공동구매` |
| **슬러그 (URL)** | 선택 | 주소 `/group-buy/` 뒤에 붙는 이름입니다. **비워 두면 공동구매명에서 자동 생성**됩니다. | `hongsam-3days` |
| **한 줄 요약** | 선택 | 목록 카드에 제목 아래로 보입니다. 최대 500자. | `단 3일, 최저가 도전` |
| **상세 본문** | 선택 | 상세 화면의 소개글입니다. HTML 을 쓸 수 있습니다. | 여러 줄 입력 가능 |
| **공동구매 유의사항** | 선택 | 주문 취소·환불 기준, 배송 예정일 등 고객이 반드시 알아야 할 내용. | `마감 후 일괄 발송` |

> **슬러그는 나중에 바꾸지 마세요.** 문자·카카오톡으로 링크를 뿌린 뒤 이 값을 바꾸면 **기존 링크가 전부 깨집니다.**

> **상세 본문·유의사항에 넣은 HTML 은 저장할 때 위험한 태그가 자동으로 제거됩니다.** 스크립트를 넣어도 저장 후 사라지니, 이미지·문단·표 정도로 구성하세요.

### 3-2. 이미지 3종

| 이미지 | 어디에 보이나 |
|---|---|
| **목록 카드 썸네일** | 공동구매 목록의 카드 이미지 |
| **상세 PC 대표 이미지** | 상세 화면 상단의 큰 가로 이미지 |
| **상세 모바일 대표 이미지** | 모바일 전용 상단 이미지 |

수정 화면에서는 이미 올린 이미지 옆에 **삭제** 체크박스가 나타납니다. 체크하고 저장하면 그 이미지가 지워집니다.

### 3-3. 발행 (우측)

| 항목 | 설명 |
|---|---|
| **현재 노출 상태** | 기간에서 자동 계산된 값을 배지로 보여 줍니다(수정 화면에만 표시). 직접 바꿀 수 없습니다. |
| **운영 상태** | 아래 표 참고. |
| **공동구매 목록에 노출** | 체크하면 고객의 공동구매 목록 화면에 나타납니다. |
| **검색엔진 색인 허용** | 체크하면 검색 결과에 잡히도록 허용합니다. |

**운영 상태 3가지** — 운영자가 직접 정하는 값입니다.

| 값 | 동작 |
|---|---|
| **임시저장** | 준비 중. 고객에게 보이지 않습니다. 새로 만들면 기본값입니다. |
| **발행** | 고객에게 공개됩니다. |
| **숨김** | 공개를 잠시 내립니다. 내용은 그대로 남습니다. |

> **고객에게 보이려면 `운영 상태 = 발행` + `공동구매 목록에 노출 = 켜짐` 두 가지가 모두** 필요합니다. 하나만 해 두고 "안 보인다"고 하는 경우가 가장 많습니다.

### 3-4. 기간

| 입력란 | 필수 | 설명 |
|---|---|---|
| **시작 일시** | 필수 | 공동구매가 시작되는 날짜·시각. |
| **종료 일시** | 필수 | 끝나는 날짜·시각. **공동구매는 기간이 본질이라 비울 수 없습니다.** |
| **마감임박 기준 (시간)** | 선택 | 종료 N시간 전부터 `마감임박` 배지가 붙습니다. 비우면 **24시간**이 적용됩니다. |
| **종료 후 구매** | — | 아래 표 참고. |
| **배송 예정 안내** | 선택 | 상세 화면에 표시되는 발송 안내 문구. 최대 200자. 예: `마감 후 3영업일 내 순차 발송` |

> **종료 일시는 시작 일시보다 뒤여야 합니다.** 같거나 앞서면 저장되지 않고 `종료일시가 시작일시보다 빠르거나 같습니다.` 오류가 뜹니다.

**종료 후 구매**

| 값 | 동작 |
|---|---|
| **구매 차단** | 기간이 끝나면 더 이상 살 수 없습니다. 기본값이며 대부분 이 값을 씁니다. |
| **구매 허용(일반가 아님, 공동구매가 유지)** | 기간이 끝나도 **공동구매가 그대로** 계속 팔립니다. |

> **"구매 허용" 은 할인가가 무기한 유지된다는 뜻입니다.** 기간 한정 특가를 열었는데 이 값을 켜 두면 종료 후에도 그 가격으로 계속 팔립니다. 특별한 이유가 없으면 `구매 차단`을 두세요.

**노출 상태(자동 계산)** — 기간에서 파생되며 저장되는 값이 아닙니다.

| 배지 | 언제 |
|---|---|
| **예정** | 시작 일시가 아직 오지 않음 |
| **진행중** | 기간 안에 있음 |
| **마감임박** | 기간 안이면서, 종료까지 `마감임박 기준` 시간 이하로 남음 |
| **종료** | 종료 일시가 지남 |

### 3-5. 목표 / 참여 표시

| 항목 | 설명 |
|---|---|
| **목표 수량 사용** | 체크하면 "100개 목표 중 42개" 같은 목표 표시가 켜집니다. |
| **목표 수량** | 목표로 잡을 개수. `목표 수량 사용`을 켰다면 **반드시 입력**해야 저장됩니다. |
| **달성률 막대 표시** | 진행률 막대를 고객 화면에 보여 줍니다. |
| **참여 수량 표시** | 지금까지 팔린 수량을 보여 줍니다. |
| **참여자 수 표시** | 참여한 주문 건수를 보여 줍니다. |

> **목표 수량은 "표시"일 뿐입니다.** 목표에 못 미쳐도 결제는 그대로 진행되고 자동 취소·환불은 일어나지 않습니다. 고객은 공동구매가로 **즉시 결제**합니다. 목표 미달 시 처리 방침은 **공동구매 유의사항**에 직접 적어 주세요.

> **`목표 수량 사용`을 끄고 저장하면 입력해 둔 목표 수량 값도 함께 지워집니다.** 나중에 다시 켜면 값이 비어 있으니 새로 입력하세요.

화면의 **현재 참여 N개 / N건** 은 결제 완료된 주문에서 자동으로 누적된 숫자입니다. 관리자가 직접 고칠 수 없습니다.

여기까지 입력했으면 우측의 **저장** 버튼을 누릅니다. 저장하면 수정 화면으로 넘어가고, 이제 상품을 담을 수 있습니다.

## 4. 2단계 — 상품 담기

수정 화면 중간의 **공동구매 상품 · 가격** 영역을 사용합니다.

1. 우측의 **상품 담기** 버튼을 누릅니다.
2. 상품 검색 창이 열립니다. **상품명**을 입력하면 결과가 나타납니다(최대 20건).
3. 결과에서 상품을 클릭하면 바로 담깁니다.

담긴 상품에는 **정상가**로 상품의 원래 정가가, **공동구매가**로 상품의 현재 판매가가 자동으로 채워집니다. 이 값을 원하는 가격으로 고쳐 주세요.

- **첫 번째로 담은 상품이 자동으로 `대표`** 가 됩니다. 이후에 담는 상품은 `추가` 입니다.
- 같은 상품을 두 번 담으면 `이미 담긴 상품입니다.` 오류가 납니다.
- **다른 몰의 상품은 담을 수 없습니다.**

> **현재는 대표 상품 하나로 구매합니다.** 여러 개를 담아도 고객 화면의 구매 카드는 맨 위 `대표` 상품 기준으로 만들어집니다.

## 5. 3단계 — 공동구매가 · 수량 제한 정하기

담긴 상품은 표로 나열되며, 각 줄마다 다음을 지정합니다.

| 항목 | 필수 | 설명 |
|---|---|---|
| **역할** | — | `대표` 또는 `추가`. 대표가 고객 화면의 주 구매 상품입니다. |
| **정상가** | 선택 | 할인 전 비교 가격입니다. 비우면 상품의 판매가가 대신 쓰입니다. |
| **공동구매가** | **필수** | **고객이 실제로 결제하는 단가입니다.** 비어 있거나 0이면 그 줄은 저장되지 않습니다. |
| **할인율** | — | 정상가 대비로 **자동 계산**됩니다. 직접 입력할 수 없습니다. |
| **최소** | — | 한 번에 살 수 있는 최소 수량. 비우면 1. |
| **최대** | 선택 | 한 번에 살 수 있는 최대 수량. 비우면 재고까지 살 수 있습니다. |
| **1인제한** | 선택 | 한 사람이 살 수 있는 총 수량. 비우면 무제한. |
| **구매** | — | 끄면 보여만 주고 구매 버튼이 막힙니다. |
| **노출** | — | 끄면 이 공동구매에서만 상품이 숨겨집니다. |
| **삭제** | — | 이 공동구매에서 상품을 빼냅니다. (상품 자체는 삭제되지 않습니다.) |

**표를 수정한 뒤에는 반드시 `상품 설정 저장` 버튼을 누르세요.** 누르지 않고 화면을 벗어나면 변경이 사라집니다.

> **결제 금액은 서버가 다시 계산합니다.** 화면에 보이는 값이 아니라 여기 저장된 **공동구매가**가 최종 결제 단가입니다. 그래서 가격 오타가 그대로 매출 손실이 됩니다.

> **`최대`를 `최소`보다 작게 넣으면 그 값은 무시되고 최대 제한이 없어집니다.** (그대로 두면 아무 수량도 살 수 없기 때문입니다.)

> **`1인제한`은 아직 주문 단계에서 적용되지 않습니다.** 값은 저장되지만 실제 구매를 막지는 않습니다.

**상품 원본이 우선입니다.** 표에서 `노출`·`구매`를 켜 두어도, 상품 관리에서 그 상품이 **판매중지·품절**이면 고객은 살 수 없습니다. 표의 상품명 아래에 그 상태가 빨간 글씨로 표시되니 확인하세요.

목록 상단에서 **운영 상태**로 거르거나 **공동구매명/슬러그**로 검색할 수 있습니다.

| 열 | 의미 |
|---|---|
| **운영상태** | 운영자가 정한 값(임시저장/발행/숨김) |
| **노출상태** | 기간에서 자동 계산된 값(예정/진행중/마감임박/종료) |
| **공동구매가** | 대표 상품의 공동구매가 |
| **참여/목표** | 누적 참여 수량 / 목표 수량 |
| **달성률** | 목표 대비 %. 목표 수량을 안 쓰면 `-` |
| **주문 · 매출** | **결제가 완료된 주문만** 집계합니다. 취소·환불은 빠집니다. |

상품을 하나도 담지 않은 공동구매는 목록에 **"상품 미연결 — 고객 화면에 구매 버튼이 없습니다"** 라고 빨간 글씨로 표시됩니다.

**삭제**는 참여(결제) 기록이 하나도 없을 때만 됩니다. 결제된 주문이 있으면 `참여(결제) 기록 N건이 있어 삭제할 수 없습니다.` 라는 오류가 뜹니다. 이때는 **운영 상태를 `숨김`으로 바꾸세요.**

> **삭제하면 담아 둔 상품 설정(가격·수량 제한)도 함께 사라집니다.** 되돌릴 수 없습니다.

## 9. 자주 막히는 상황

**Q. 공동구매를 만들었는데 고객 화면에 안 보입니다.**
아래를 순서대로 확인하세요.
1. **운영 상태**가 `발행`인가? (`임시저장`이면 안 보입니다)
2. **공동구매 목록에 노출**이 켜져 있는가?
3. **기간**이 오늘을 포함하는가? (시작 일시가 미래면 `예정`입니다)
4. **편집 중인 몰**이 맞는가? (다른 몰에 만들었을 수 있습니다)

**Q. 화면은 보이는데 구매 버튼이 없습니다.**
상품을 하나도 담지 않았을 가능성이 큽니다. 목록에 "상품 미연결" 이라고 표시되는지 확인하세요. 담았다면 그 줄의 **구매**·**노출** 체크가 켜져 있는지, 상품 자체가 판매중지·품절이 아닌지 보세요.

**Q. 저장을 눌렀는데 "종료일시를 입력하세요" 라고 나옵니다.**
공동구매는 종료 일시가 **필수**입니다. 종료 없이 상시 운영하려면 공동구매가 아니라 기획전·전문관을 쓰세요.

**Q. 상품 가격을 고쳤는데 반영이 안 됩니다.**
표 아래 **`상품 설정 저장`** 버튼을 눌러야 합니다. 위쪽의 **저장** 버튼은 기본 정보만 저장하며, 상품 표와는 별개입니다.

**Q. 공동구매가를 비워 두고 저장했더니 그 줄만 안 바뀝니다.**
공동구매가가 없으면 결제 금액을 정할 수 없어 그 줄은 건너뜁니다. 반드시 1 이상의 숫자를 넣으세요.

**Q. 삭제가 안 됩니다.**
결제된 참여 기록이 있으면 삭제할 수 없습니다. `숨김`으로 바꾸세요. 주문 출처 추적을 위해 기록을 남깁니다.

**Q. 달성률이 안 보입니다.**
`목표 수량 사용`이 꺼져 있거나 목표 수량이 비어 있습니다. 둘 다 채우고 `달성률 막대 표시`도 켜세요.

**Q. 참여 수량이 안 올라갑니다.**
**결제가 완료된 주문만** 집계합니다. 결제 대기 중이거나 취소된 주문은 세지 않습니다.

## 10. 아직 지원하지 않는 것

- **목표 달성형** — 목표에 도달해야 결제가 확정되고 미달 시 자동 환불되는 방식
- **단계별 가격** — 참여 인원이 늘수록 가격이 내려가는 방식
- **1인 구매 제한의 실제 적용** — 값은 저장되지만 주문 단계에서 막지 않습니다
- 엑셀/CSV로 상품 일괄 담기
