# Header 설정

고객 화면 **맨 위 헤더의 틀**을 정하는 메뉴입니다. 헤더를 어떤 모양으로 짤지(레이아웃)와, 상단 메뉴 줄에 **메뉴를 몇 개까지 보여 줄지**를 정합니다.

> **이 화면은 헤더의 "틀"만 정합니다.** 어떤 메뉴를 켜고 끌지, 메뉴 이름을 뭐라고 할지는 **메뉴 관리** 소관입니다.

## 1. 헷갈리기 쉬운 메뉴 구분

| 하고 싶은 일 | 가야 할 화면 |
|---|---|
| 헤더 모양 바꾸기, 메뉴 **개수** 제한 | **Header 설정** (이 화면) |
| 개별 메뉴 켜기/끄기, 이름·순서 바꾸기 | 일반 메뉴 관리 · 시스템 메뉴 설정 |
| 헤더 위쪽 띠 배너·공지 문구 | 배너 관리 |
| 색상·로고·상호 | 사이트 설정 |
| 모서리 둥글기·글꼴·간격 | 디자인 스타일 |
| 히어로 배치(테마 1·2·3) | 페이지 빌더 → 테마 설정 탭 |

## 2. 헤더 레이아웃

헤더의 전체 구조를 셋 중에 고릅니다. 고르면 바로 아래에 그 레이아웃의 설명이 나옵니다.

| 레이아웃 | 고객 화면 모양 | 어울리는 몰 |
|---|---|---|
| **기본형 — 카테고리 버튼 + 평면 GNB (3단 헤더)** | 상단 유틸바 + 로고/검색 + 메뉴 줄의 3단 구조. 카테고리는 **[☰ 카테고리]** 버튼을 누르면 3단 드롭다운 패널로 열리고, 일반 메뉴가 그 옆에 한 줄로 놓입니다. | 대부분의 일반 몰 |
| **드로어형 — 햄버거 전체메뉴 + 아코디언 카테고리** | 헤더에는 **[☰]·로고·장바구니**만 두고 메뉴 전체를 좌측 슬라이드 드로어에 담습니다. 카테고리 1단계가 일반 메뉴와 같은 목록에 놓이고, 하위는 **[+]** 로 펼칩니다. **검색창도 드로어 안에** 들어갑니다. | 메뉴·카테고리가 많은 종합몰, 모바일 중심 |
| **에디토리얼형 — 투명 오버레이 헤더 (풀블리드 히어로용)** | 투명 헤더가 큰 히어로 이미지 위에 겹쳐 보이고, 스크롤이 히어로를 지나면 흰 배경으로 굳습니다. 로고·중앙 메뉴·우측 아이콘 구성. | 화보 느낌의 라이프스타일 몰 (**테마 3 풀블리드**와 짝) |

> **레이아웃을 바꾸면 메뉴 조립 방식도 자동으로 함께 맞춰집니다.** 예를 들어 드로어형을 고르면 카테고리가 일반 메뉴 목록에 합쳐지도록 알아서 전환됩니다. 운영자가 이 둘을 따로 만질 수 없게 막아 둔 것인데, **"드로어 헤더인데 카테고리가 어디에도 없는"** 같은 깨진 조합을 방지하기 위해서입니다.

> **에디토리얼형은 테마 3(풀블리드)과 함께 쓸 때 의도한 모습이 나옵니다.** 히어로가 작은 테마 1·2에 이 레이아웃을 씌우면 투명 헤더가 어색하게 뜹니다.

## 3. 메뉴 슬롯

상단 메뉴 줄(GNB)은 **[카테고리 버튼] → [기능 메뉴] → [커스텀 메뉴]** 순서로 채워지고, **총량을 넘으면 뒤에서부터 잘려 나갑니다.**

| 입력란 | 범위 | 설명 | 기본값 예 |
|---|---|---|---|
| **GNB 최대 노출 수** | 1~20 | 상단에 보여 줄 메뉴의 최대 개수입니다. **카테고리 버튼은 이 수에 포함되지 않습니다.** | 12 |
| **커스텀 메뉴 슬롯** | 0~10 | 직접 만든 커스텀 메뉴가 차지할 자리 수입니다. **GNB 총량 안에 포함**되며, 총량보다 크게 넣으면 저장 시 총량에 맞춰 줄어듭니다. | 3 |
| **카테고리 최대 뎁스** | 1~3 | 고객 화면에 보여 줄 카테고리 단계 수입니다. | 3 |
| **검색창** | 체크박스 | 헤더에 검색창 사용 | 켜짐 |

### 메뉴를 켰는데 상단에 안 보인다면

원인은 대개 **GNB 최대 노출 수**입니다. 켠 메뉴 수가 이 값보다 많으면 **뒤쪽 메뉴부터 잘립니다.**

두 가지 중에 고르세요.

1. **GNB 최대 노출 수를 늘린다** — 다만 너무 늘리면 메뉴가 두 줄로 넘어가 헤더가 지저분해집니다.
2. **메뉴 관리에서 덜 중요한 메뉴를 끄거나, 순서를 앞으로 올린다** — 이쪽을 권장합니다.

## 4. 카테고리 최대 뎁스 — 낮출 때 주의

이 값은 **지금 실제로 쓰고 있는 카테고리 단계보다 낮게 내릴 수 없습니다.** 시도하면 저장이 거부되고 이런 안내가 뜹니다.

```
카테고리 최대 뎁스를 2 로 낮출 수 없습니다.
현재 3뎁스 카테고리가 있어 스토어프론트에서 사라집니다.
```

입력칸 아래에는 **현재 데이터의 최대 뎁스**가 항상 표시되어 있으니 참고하세요.

> **이것은 안전장치입니다.** 뎁스를 낮추면 그보다 깊은 카테고리가 고객 화면에서 **조용히 사라집니다.** 상품이 그 카테고리에 붙어 있으면 고객이 찾아갈 길이 없어집니다. 정말 낮추려면 **카테고리 관리에서 깊은 카테고리를 먼저 정리**한 뒤 다시 시도하세요.

**상한이 3인 이유** — 고객 화면의 카테고리 드롭다운이 3단계까지만 그리도록 만들어져 있습니다. 4 이상은 넣을 수 없습니다.

## 5. 검색창 체크박스에 대하여

체크박스는 화면에 보이고 값도 저장되지만, **실제 검색창 노출은 이 값이 아니라 시스템 메뉴 설정의 `HEADER_SEARCH` 메뉴가 결정합니다.** 입력칸 아래에도 그렇게 적혀 있습니다.

게다가 검색은 **필수 메뉴**로 지정되어 있어 현재는 끌 수 없습니다.

> **이 체크박스를 꺼도 검색창은 사라지지 않습니다.** 검색창을 조절하려면 시스템 메뉴 설정으로 가세요.

## 6. 저장하기

- **저장** 버튼을 누르면 `저장되었습니다. 스토어프론트에 즉시 반영됩니다.` 가 뜨고, 고객 화면에 바로 적용됩니다.
- 잘못 건드렸다면 저장 전에 **되돌리기**를 눌러 저장된 값으로 화면을 다시 불러올 수 있습니다.
- 저장이 거부되면 빨간 안내로 사유가 표시됩니다. **이때는 아무것도 저장되지 않습니다.**

저장한 뒤에는 결과를 확인하세요.

## 7. 자주 막히는 상황

**Q. 헤더 레이아웃을 바꿨는데 고객 화면이 그대로입니다.**
① 저장했는지, ② **편집 중인 몰**이 맞는지 확인하세요. 그래도 그대로면 브라우저 새로고침(Ctrl+F5)을 해 보세요.

**Q. 메뉴를 켰는데 상단에 안 나옵니다.**
**GNB 최대 노출 수**를 확인하세요. 켠 메뉴 수가 이 값보다 많으면 뒤쪽이 잘립니다. (위 3장)

**Q. 커스텀 메뉴를 5개 만들었는데 3개만 보입니다.**
**커스텀 메뉴 슬롯**이 3으로 설정되어 있을 것입니다. 슬롯을 늘리거나 커스텀 메뉴를 줄이세요. 단, 이 값을 늘리면 그만큼 일반 메뉴가 밀려납니다(총량이 같으므로).

**Q. 카테고리 뎁스를 낮추려는데 저장이 안 됩니다.**
지금 그보다 깊은 카테고리가 있습니다. 안내 문구에 현재 뎁스가 적혀 있으니, 카테고리 관리에서 정리한 뒤 다시 시도하세요.

**Q. 드로어형으로 바꿨더니 헤더에서 검색창이 사라졌습니다.**
정상입니다. 드로어형에서는 **검색창이 드로어 안**에 들어갑니다. [☰]를 누르면 보입니다.

**Q. 에디토리얼형으로 바꿨더니 헤더 글자가 잘 안 보입니다.**
투명 헤더가 배경 이미지 위에 겹치기 때문입니다. 히어로 이미지가 밝으면 글자가 묻힙니다. 어두운 이미지를 쓰거나 다른 레이아웃을 고르세요.

**Q. 메가 메뉴(넓게 펼쳐지는 메뉴)를 쓰고 싶습니다.**
지원하지 않습니다.

## 8. 아직 지원하지 않는 것

- **메가 메뉴** — 화면에 항목이 없고 동작하지 않습니다
- **카테고리 표시 방식을 따로 고르기** — 카테고리가 어떻게 열리는지는 **헤더 레이아웃에 따라 자동으로** 정해집니다
- **이 화면 안에서의 헤더 미리보기** — 메뉴 미리보기나 실제 고객 화면으로 확인하세요
- 카테고리 4단계 이상
