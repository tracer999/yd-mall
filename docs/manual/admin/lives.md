# 쇼핑라이브 관리

> **이 시스템이 방송을 송출하지는 않습니다.** 쇼핑라이브는 **YouTube 또는 Vimeo 에 이미 올라간(또는 방송 중인) 영상을 화면에 불러와 보여 주고**, 그 아래에 상품을 진열해 파는 기능입니다. 방송 자체는 별도 방송 도구로 진행하고, **그 영상 주소만 여기에 등록**합니다.

## 1. 시작하기 전에 — 준비물 4가지

| 준비물 | 왜 필요한가 | 어디서 준비하나 |
|---|---|---|
| **방송 영상 주소** | YouTube 또는 Vimeo 링크가 있어야 화면에 영상이 뜹니다. **방송을 켜려면 필수**입니다. | YouTube / Vimeo |
| **팔 상품** | 이미 등록된 상품을 담아 파는 방식입니다. 여기서 상품을 새로 만들 수는 없습니다. | 상품 관리 |
| **쿠폰** (쓸 경우) | 라이브는 쿠폰을 **만들지 않고 연결만** 합니다. 미리 만들어 두어야 합니다. | 쿠폰 관리 |
| **이미지 3종** | 목록 카드 썸네일 / 상세 PC 대표 / 상세 모바일 대표. 방송 예정·종료 상태에서 영상 자리에 보입니다. | 이미지 편집 도구 |

## 2. 전체 흐름 한눈에 보기

```
① 기본 정보 + 영상 주소 입력 → 등록    (여기까지 해야 라이브가 "생깁니다")
        ↓
② 판매 상품 담기 → 라이브가 입력 → 상품 설정 저장
        ↓
③ 쿠폰 연결 (선택) · 공지 등록 (선택)
        ↓
④ 방송 시각이 되면 목록에서 [방송 시작] 클릭
        ↓
⑤ 방송이 끝나면 목록에서 [방송 종료] 클릭 → 다시보기로 전환
```

**①번 등록을 먼저 해야 ②③이 가능합니다.** 등록 화면에는 상품·쿠폰·공지 영역이 아직 나타나지 않습니다.

> **방송 상태는 절대 자동으로 바뀌지 않습니다.** 외부 영상을 불러오는 방식이라 시스템은 실제 방송이 시작됐는지 끝났는지 알 수 없습니다. **시작 시각이 지나도 저절로 "방송 중"이 되지 않습니다.** 운영자가 목록에서 직접 눌러야 합니다.

### 3-1. 기본 정보

| 입력란 | 필수 | 설명 | 예시 |
|---|---|---|---|
| **방송명** | 필수 | 고객에게 보이는 제목입니다. 최대 200자. | `홍삼 특집 라이브` |
| **슬러그 (URL)** | 선택 | 주소 `/live/` 뒤에 붙는 이름입니다. **비워 두면 방송명에서 자동 생성**됩니다. | `hongsam-live` |
| **한 줄 요약** | 선택 | 목록 카드에 노출됩니다. 최대 500자. | `방송 중 주문 시 무료배송` |
| **방송 소개** | 선택 | 상세 화면 소개글. HTML 을 쓸 수 있습니다. | 여러 줄 입력 가능 |
| **대표 공지** | 선택 | **영상 바로 아래에 고정**되는 문구. 배송 안내·쿠폰 기한처럼 방송 중 꼭 보여야 할 내용. | `오늘 주문분만 당일 발송` |

> **슬러그를 나중에 바꾸면 이미 뿌린 링크가 전부 깨집니다.** 방송 예고를 문자·SNS로 보낸 뒤에는 건드리지 마세요.

> **방송 소개·대표 공지의 HTML 은 저장 시 위험한 태그가 자동 제거됩니다.**

### 3-2. 영상 (가장 중요)

> **iframe 코드를 붙여넣지 마세요.** 영상 **주소(URL)** 또는 **영상 ID** 만 넣습니다. `<iframe ...>` 같은 HTML 을 넣으면 저장이 거부되고 `iframe/HTML 은 넣을 수 없습니다.` 오류가 뜹니다.

| 입력란 | 설명 |
|---|---|
| **플랫폼** | `YouTube` 또는 `Vimeo` |
| **방송 영상 URL 또는 ID** | 실제 방송을 내보낼 영상. 예: `https://youtu.be/xxxxxxxxxxx` / `https://vimeo.com/123456789` |
| **다시보기 플랫폼** | 다시보기 영상의 플랫폼. 보통 방송과 같게 둡니다. |
| **다시보기 영상 URL 또는 ID** | 방송 후 보여 줄 영상. **비우면 방송 영상을 그대로 다시보기로 씁니다.** |

넣을 수 있는 형태는 다음과 같습니다.

| 플랫폼 | 허용되는 형태 |
|---|---|
| **YouTube** | `youtu.be/…`, `youtube.com/watch?v=…`, `/live/…`, `/embed/…`, `/shorts/…`, 또는 11자리 영상 ID |
| **Vimeo** | `vimeo.com/123456789`, `player.vimeo.com/video/123456789`, 또는 6~12자리 숫자 ID |

이 형태가 아니면 `영상 ID 를 찾지 못했습니다.` 또는 `YouTube 주소가 아닙니다.` 라는 오류가 뜹니다.

수정 화면에는 다음 두 가지가 더 나타납니다.

- **미리보기** — 저장된 영상이 실제로 어떻게 나오는지 관리자 화면에서 바로 확인할 수 있습니다.
- **영상 삭제 / 삭제 체크박스** — 등록된 영상을 지웁니다.

> **영상 칸을 비운 채 저장해도 기존 영상은 지워지지 않습니다.** 방송 중에 실수로 폼을 저장해 영상이 날아가는 사고를 막기 위한 장치입니다. 정말 지우려면 **영상 삭제** 체크박스를 켜고 저장하세요.

### 3-3. 이미지 3종

방송 **예정·종료** 상태일 때 영상 자리에 보여 줄 이미지입니다.

| 이미지 | 어디에 보이나 |
|---|---|
| **목록 카드 썸네일** | 라이브 목록의 카드 이미지 |
| **상세 PC 대표 이미지** | 상세 화면 상단(PC) |
| **상세 모바일 대표 이미지** | 상세 화면 상단(모바일) |

이미 올린 이미지 옆의 **삭제** 체크박스를 켜고 저장하면 그 이미지가 지워집니다.

### 3-4. 방송 (상태 · 시각)

| 입력란 | 필수 | 설명 |
|---|---|---|
| **방송 상태** | — | 아래 표 참고. 새로 만들면 `임시저장`입니다. |
| **방송 시작** | 필수 | 방송 예정 시각. 고객 화면에 "언제 시작" 으로 표시됩니다. |
| **방송 종료** | 선택 | 종료 예정 시각. 비워 둘 수 있습니다. |

**방송 상태 5가지**

| 값 | 고객에게 | 의미 |
|---|---|---|
| **임시저장** | 안 보임 | 준비 중. 기본값입니다. |
| **방송 예정** | 보임 | 아직 시작 전. 영상 대신 이미지가 보입니다. |
| **방송 중** | 보임 | 영상이 재생되고 구매가 열립니다. |
| **방송 종료** | 보임 | 다시보기 상태. |
| **방송 취소** | 안 보임 | 취소된 방송. |

> **`방송 중`으로 바꾸려면 방송 영상이 반드시 있어야 합니다.** 영상 없이 시도하면 `방송 중으로 두려면 방송 영상 URL(또는 ID)이 필요합니다.` 라며 저장이 막힙니다. 빈 영상 화면을 고객에게 보이지 않기 위한 장치입니다.

> **종료 시각은 시작 시각보다 뒤여야 합니다.** 앞서면 저장되지 않습니다.

### 3-5. 판매 · 노출

| 항목 | 켜면 |
|---|---|
| **구매 허용** | 고객이 라이브 화면에서 살 수 있습니다. 끄면 보여만 줍니다. |
| **다시보기 노출** | 방송 종료 후 다시보기 영상을 보여 줍니다. |
| **목록에 노출** | 라이브 목록 화면에 나타납니다. |
| **검색에 노출** | 검색 결과에 잡히도록 허용합니다. |
| **공유 버튼 노출** | 고객 화면에 SNS 공유 버튼이 표시됩니다. |

**방송 종료 후 구매**

| 값 | 동작 |
|---|---|
| **구매 차단 (방송 중에만 판매)** | 방송이 끝나면 살 수 없습니다. **기본값**입니다. |
| **구매 허용 (라이브가 유지)** | 종료 후에도 **라이브 특가 그대로** 계속 팔립니다. |

> **라이브 특가는 "방송 중"이 조건입니다.** `구매 허용`을 켜 두면 방송이 끝난 뒤에도 할인가로 무기한 팔립니다. 특별한 이유가 없으면 기본값(`구매 차단`)을 두세요.

**방송 종료 후 접근**

| 값 | 동작 |
|---|---|
| **접근 허용 (다시보기)** | 종료 후에도 화면이 열립니다. **기본값**입니다. |
| **접근 차단 (404)** | 상세 화면 자체가 열리지 않습니다. |

> **링크를 많이 뿌린 방송이라면 `접근 허용` + `구매 차단` 조합을 권장합니다.** 접근까지 차단하면 나중에 들어온 고객이 빈 화면을 보게 되어 문의가 몰립니다.

여기까지 입력했으면 **등록**(수정 화면에서는 **기본 정보 저장**) 버튼을 누릅니다.

## 4. 2단계 — 판매 상품 담기

수정 화면의 **판매 상품** 영역을 사용합니다.

1. 우측의 **상품 담기** 버튼을 누릅니다.
2. **상품명**을 입력하면 검색 결과가 나타납니다(최대 20건).
3. 상품을 클릭하면 바로 담깁니다.

- **첫 번째로 담은 상품이 `대표 상품`** 이 됩니다. 이후는 `함께 판매` 입니다.
- 같은 상품을 두 번 담으면 `이미 담긴 상품입니다.` 오류가 납니다.
- **다른 몰의 상품은 담을 수 없습니다.**

담긴 상품은 표로 나열되며 각 줄마다 다음을 지정합니다.

| 항목 | 설명 |
|---|---|
| **역할** | `대표 상품` = 화면 맨 위 구매 카드가 됩니다. `함께 판매` = 아래 목록에 나열됩니다. |
| **뱃지** | 상품 카드에 붙는 짧은 문구. 최대 100자. 예: `방송 한정 특가` |
| **정상가** | 할인 전 비교 가격. 비우면 상품의 판매가가 쓰입니다. |
| **라이브가** | **방송 중 판매 단가.** **비우면 상품 원가로 팝니다.** |
| **할인율** | 정상가 대비 **자동 계산**됩니다. 직접 입력할 수 없습니다. |
| **최소** | 한 번에 살 수 있는 최소 수량. 비우면 1. |
| **최대** | 한 번에 살 수 있는 최대 수량. 비우면 재고까지. |
| **구매** | 끄면 보여만 주고 구매 버튼이 막힙니다. |
| **노출** | 끄면 이 라이브에서만 상품이 숨겨집니다. |
| **삭제** | 이 라이브에서 상품을 빼냅니다. (상품 자체는 삭제되지 않습니다.) |

**표를 고친 뒤에는 반드시 `상품 설정 저장` 버튼을 누르세요.**

> **대표 상품은 하나뿐입니다.** 두 줄 이상을 `대표 상품`으로 지정하면 위에서부터 첫 번째만 대표가 되고 나머지는 자동으로 `함께 판매`로 내려갑니다.

> **결제 금액은 서버가 다시 계산합니다.** 화면에 보이는 값이 아니라 저장된 **라이브가**가 최종 결제 단가입니다. 라이브가를 비우면 **상품 원가로 팔리므로**, 할인 방송인데 라이브가를 빠뜨리면 정가로 팔립니다.

> **`최대`를 `최소`보다 작게 넣으면 그 값은 무시되고 최대 제한이 없어집니다.**

**이 몰에는 상품 옵션이 없습니다.** 고객은 **수량만** 고릅니다.

**상품 원본이 우선입니다.** 표에서 켜 두어도 상품 관리에서 판매중지·품절이면 팔 수 없습니다. 표의 상품명 아래에 빨간 글씨로 표시됩니다.

## 5. 3단계 — 쿠폰 연결하기 (선택)

**라이브는 쿠폰을 만들지 않습니다. 연결만 합니다.** 쿠폰은 쿠폰 관리에서 먼저 만들어야 합니다.

<div class="manual-goto-bar">
  <span class="manual-goto-label">쿠폰이 없다면</span>
  <a class="manual-goto" href="/admin/coupons" target="_blank">쿠폰 관리에서 쿠폰 만들기</a>
</div>

**연결 쿠폰** 영역의 선택 목록에서 쿠폰을 고르고 **연결** 버튼을 누릅니다. 목록에는 이 몰의 쿠폰과 전 몰 공용 쿠폰이 함께 나옵니다.

연결된 쿠폰 표에서 다음을 조정합니다.

| 항목 | 설명 |
|---|---|
| **대표** | 화면에 강조해 보여 줄 쿠폰 하나를 라디오 버튼으로 고릅니다. |
| **발급방식** | 쿠폰이 어떻게 발급되는지. `다운로드` 는 초록색, 그 외는 노란색으로 표시됩니다. |
| **발급/한도** | 지금까지 발급된 수 / 최대 발급 한도 |
| **상태** | 쿠폰 자체의 상태. `ACTIVE` 가 아니면 빨간 글씨입니다. |
| **노출** | 끄면 이 라이브 화면에서 그 쿠폰이 숨겨집니다. |
| **해제** | 연결만 끊습니다. **쿠폰 자체는 지워지지 않습니다.** |

조정 후 **`쿠폰 설정 저장`** 버튼을 누릅니다.

> **고객이 라이브 화면에서 쿠폰을 받아 가려면 발급방식이 `다운로드`여야 합니다.** 다른 발급방식(자동지급·관리자지급 등)의 쿠폰은 화면에 표시되기는 하지만 **'받기' 버튼이 나오지 않습니다.**

## 6. 4단계 — 공지 등록하기 (선택)

**실시간 채팅이 없으므로 공지가 유일한 방송 중 커뮤니케이션 수단입니다.** 배송·쿠폰 사용기한·사은품 조건을 여기에 적으세요.

**공지** 영역의 입력 폼에 다음을 넣고 **공지 추가** 버튼을 누릅니다.

| 입력란 | 필수 | 설명 |
|---|---|---|
| **제목** | 필수 | 최대 200자. 예: `방송 중 주문 시 무료배송` |
| **중요도** | — | `일반` 또는 `중요`. `중요`가 위로 올라옵니다. |
| **노출 위치** | — | 아래 표 참고. |
| **내용** | 필수 | 공지 본문. |

**노출 위치 3가지**

| 값 | 어디에 보이나 |
|---|---|
| **공지 탭** | 상세 화면의 공지 탭 안 |
| **영상 아래 고정** | 영상 바로 아래에 항상 표시 |
| **구매 패널 하단** | 구매 버튼 영역 아래 |

등록된 공지는 아래에 카드로 나열되며 제목·중요도·위치·내용을 고칠 수 있고, 다음도 지정합니다.

| 항목 | 설명 |
|---|---|
| **노출 시작 / 종료** | 이 기간에만 공지가 보입니다. **비우면 제한 없이 계속 보입니다.** |
| **사용** | 끄면 공지가 화면에서 빠집니다. |
| **삭제** | 공지를 지웁니다. |

고친 뒤에는 **`공지 저장`** 버튼을 누르세요.

> **제목이나 내용을 비운 채 저장하면 그 공지 줄은 저장되지 않고 그대로 남습니다.** 지우려면 **삭제** 버튼을 쓰세요.

## 7. 방송 진행하기 — 목록 화면의 방송 제어

<div class="manual-goto-bar">
  <span class="manual-goto-label">방송 제어 화면</span>
  <a class="manual-goto" href="/admin/lives" target="_blank">쇼핑라이브 목록 열기</a>
</div>

목록 각 줄의 **방송 제어** 칸에 상태에 따라 버튼 하나가 나타납니다.

| 현재 상태 | 나타나는 버튼 | 누르면 |
|---|---|---|
| 임시저장 · 방송 예정 · 방송 종료 | **방송 시작** (빨간 버튼) | 상태가 `방송 중`으로 바뀌고 고객 화면에 즉시 LIVE 로 표시됩니다. |
| 방송 중 | **방송 종료** (검은 버튼) | 상태가 `방송 종료`로 바뀌고 고객 화면이 다시보기로 바뀝니다. |
| 방송 취소 | 없음 (`-`) | — |

> **방송 영상이 없으면 `방송 시작` 버튼이 회색으로 비활성화됩니다.** 먼저 수정 화면에서 영상 주소를 등록하세요.

**방송 제어 버튼은 상태만 바꿉니다.** 폼 전체를 다시 저장하지 않으므로, 방송 중에 다른 설정이 함께 바뀌는 사고가 나지 않습니다.

목록에는 다음 경고도 표시됩니다.

- **"종료 시각이 지났습니다"** — 방송 중인데 예정 종료 시각을 넘김. 종료 버튼을 눌러 주세요.
- **"시작 시각이 지났습니다"** — 방송 예정인데 시작 시각을 넘김. 시작 버튼을 눌러 주세요.
- **"상품 미연결 — 고객 화면에 구매 버튼이 없습니다"** — 상품을 하나도 담지 않음.
- **"영상 없음"** — 영상 주소가 등록되지 않음.

## 8. 목록 화면 읽는 법 · 삭제

목록 상단에서 **상태**로 거르거나 **방송명/슬러그**로 검색할 수 있습니다. 정렬은 `방송 중 → 방송 예정 → 임시저장 → 방송 종료 → 방송 취소` 순입니다.

| 열 | 의미 |
|---|---|
| **영상** | 등록된 플랫폼과 영상 ID. 없으면 `영상 없음` |
| **쿠폰 · 공지** | 연결된 건수 |
| **주문 · 매출** | **이 라이브를 통해 결제된 주문만** 집계합니다. 취소·환불 주문은 제외됩니다. |

**삭제**는 이 라이브에서 나온 주문이 하나도 없을 때만 됩니다. 주문이 있으면 `이 라이브에서 발생한 주문 N건이 있어 삭제할 수 없습니다.` 라는 오류가 뜹니다. 이때는 **`방송 취소`로 바꾸거나 `목록에 노출`을 끄세요.**

> **삭제하면 연결한 상품·쿠폰·공지 설정도 함께 지워집니다.** 되돌릴 수 없습니다.

## 9. 자주 막히는 상황

**Q. 시작 시각이 됐는데 고객 화면에 방송이 안 뜹니다.**
정상입니다. **상태는 자동으로 바뀌지 않습니다.** 목록에서 **방송 시작** 버튼을 눌러야 `방송 중`이 됩니다.

**Q. "방송 중으로 두려면 방송 영상 URL이 필요합니다" 라고 나옵니다.**
영상 주소가 등록되지 않았습니다. 수정 화면의 **영상** 영역에서 URL 또는 ID 를 넣고 저장한 뒤 다시 시도하세요.

**Q. iframe 코드를 넣었더니 저장이 안 됩니다.**
의도된 동작입니다. `<iframe ...>` 같은 HTML 은 넣을 수 없습니다. **주소(URL) 또는 영상 ID만** 넣으세요. 재생용 주소는 시스템이 알아서 만듭니다.

**Q. 라이브를 만들었는데 고객 목록에 안 보입니다.**
1. **방송 상태**가 `임시저장`·`방송 취소`가 아닌가? (이 둘은 고객에게 안 보입니다)
2. **목록에 노출**이 켜져 있는가?
3. **편집 중인 몰**이 맞는가?

**Q. 화면은 보이는데 구매 버튼이 없습니다.**
① 상품을 담았는지, ② 그 줄의 **구매**·**노출** 체크가 켜져 있는지, ③ 기본 정보의 **구매 허용**이 켜져 있는지, ④ 상태가 `방송 중`인지 확인하세요. `방송 예정` 상태에서는 팔리지 않습니다.

**Q. 할인 방송인데 정가로 팔립니다.**
상품 표의 **라이브가**를 비워 두면 상품 원가로 팔립니다. 할인 금액을 직접 넣고 `상품 설정 저장`을 누르세요.

**Q. 방송이 끝났는데 아직 할인가로 팔립니다.**
**방송 종료 후 구매**가 `구매 허용 (라이브가 유지)`으로 되어 있습니다. `구매 차단`으로 바꾸세요.

**Q. 쿠폰을 연결했는데 고객이 못 받습니다.**
쿠폰의 **발급방식**이 `다운로드`가 아닙니다. 쿠폰 관리에서 발급방식을 확인하세요.

**Q. 영상 칸을 비우고 저장했는데 기존 영상이 그대로입니다.**
정상입니다. 실수로 영상이 날아가는 것을 막는 장치입니다. 지우려면 **영상 삭제** 체크박스를 켜세요.

**Q. 라이브를 삭제할 수 없습니다.**
이 라이브에서 나온 주문이 있습니다. `방송 취소`로 바꾸거나 목록 노출을 끄세요.

## 10. 아직 지원하지 않는 것

- **실시간 채팅** — 공지로 대신합니다
- **방송 송출** — 영상은 YouTube/Vimeo 에서 진행하고 주소만 등록합니다
- **상태 자동 전환** — 시간이 지나도 방송 시작·종료가 자동으로 되지 않습니다
- **YouTube·Vimeo 외 다른 영상 플랫폼**
- **1인 구매 제한의 실제 적용**
