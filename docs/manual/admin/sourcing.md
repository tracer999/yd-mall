# 외부몰 연동

**도매꾹·도매매·온채널** 같은 공급처에서 상품을 가져와, 우리 몰을 거쳐 **네이버 스마트스토어**에 판매하기 위한 메뉴 묶음입니다.

---

## 지금 어디까지 되나요? (먼저 읽어주세요)

이 메뉴는 **아직 만드는 중**입니다. 7개 화면 중 **실제로 동작하는 것은 2개**뿐입니다.

| 화면 | 상태 |
|------|------|
| **공급처/채널 연결** (`/admin/sourcing/connections`) | ✅ **사용 가능** |
| 상품 가져오기 (`/admin/sourcing/import`) | 🚧 준비 중 (안내 문구만 표시) |
| 가져온 상품(중간) (`/admin/sourcing/staging`) | 🚧 준비 중 (안내 문구만 표시) |
| 스마트스토어 등록 (`/admin/sourcing/publish`) | 🚧 준비 중 (안내 문구만 표시) |
| 스토어 상품 가져오기 (`/admin/sourcing/channel-import`) | 🚧 준비 중 (안내 문구만 표시) |
| 재고·주문 가져오기 (`/admin/sourcing/sync`) | 🚧 준비 중 (안내 문구만 표시) |
| **네이버 카테고리 리소스** (`/admin/sourcing/naver-taxonomy`) | ✅ **사용 가능** |

준비 중인 화면에 들어가면 **공사중 표시와 함께 "어떤 단계에서 구현됩니다"** 라는 안내만 나옵니다. 버튼도 목록도 없습니다.

---

## 이 메뉴가 안 보인다면

외부몰 연동은 **몰별로 켜고 끄는 유료 기능**입니다.

- **꺼져 있는 몰에서는 대메뉴 자체가 사이드바에 나타나지 않습니다.**
- 주소를 직접 입력해 들어가도 **"외부몰 연동은 유료 기능입니다"** 안내가 뜨고 막힙니다.
- **켜는 것은 제공자(super_admin)만 할 수 있습니다.** 일반 관리자에게는 사용 여부 체크박스가 아예 비활성으로 보입니다.

---

## 작업 순서

지금 할 수 있는 일은 **1번 → 2번**까지입니다. 3번 이후는 화면이 준비되면 이어집니다.

1. **공급처/채널 연결** — 연동을 켜고, 계정을 등록하고, **검증**을 눌러 상태를 **ACTIVE** 로 만듭니다.
2. **네이버 카테고리 리소스** — **[지금 수집]** 으로 네이버 카테고리를 받아옵니다. *(1번에서 네이버 검증이 ACTIVE 여야 동작)*
3. *(준비 중)* **상품 가져오기** — 공급처에서 상품을 검색해 가져옵니다.
4. *(준비 중)* **가져온 상품(중간)** — 가져온 목록을 확인하고 골라서 상품으로 등록합니다.
5. *(준비 중)* **스마트스토어 등록** — 편집한 상품을 네이버에 올리고 검수 상태를 봅니다.
6. *(준비 중)* **재고·주문 가져오기** — 재고와 주문을 가져옵니다.

> **2번까지만 해두어도 얻는 게 있습니다.** 네이버 카테고리를 수집해 두면 **상품 관리**의 상품 등록·수정 화면에서 **"🔎 네이버 카테고리에서 찾기"** 검색이 동작합니다. (아래 참고)

---

## 1. 공급처/채널 연결

화면 맨 위에 **현재 편집 몰**이 표시됩니다. **여기서 저장하는 계정·설정은 모두 그 몰의 것**입니다. 다른 몰 것을 만지고 있지 않은지 항상 확인하세요.

### 연동 사용 설정

- **이 몰에서 외부몰 연동 사용** — 제공자(super_admin)만 변경할 수 있습니다.
- **기본 목표 마진율(%)** / **기본 채널 수수료율(%)** — 저장은 되지만, **이 값을 쓰는 화면(가격 계산)이 아직 없습니다.** 미리 적어두는 용도입니다.
- **[설정 저장]**

### 연결할 수 있는 채널

| 채널 | 구분 | 입력하는 것 |
|------|------|------------|
| **도매꾹·도매매** | 공급처 | **Open API Key(aid) 하나만** (시크릿 없음) |
| **온채널** | 공급처 | **없음** — 수동·CSV 방식이라 계정이 필요 없습니다 |
| **네이버 스마트스토어** | 판매채널 | client_id + client_secret **둘 다 필수** |

> **도매매는 따로 등록하지 않습니다.** 도매꾹과 도매매는 **아이디를 공유**해서 **API Key 하나로 양쪽 상품이 다 조회**됩니다. 그래서 채널 목록에 "도매꾹·도매매" 한 줄만 있습니다. 어느 쪽 상품을 가져올지는 나중에 **상품 가져오기 화면에서 고르는 것**이지, 계정을 나누는 게 아닙니다.

### 도매꾹 API Key 받는 법

1. 도매꾹에 **로그인**합니다. (일부 API는 사업자만 쓸 수 있으니 **사업자 인증**을 먼저 해두세요)
2. **API 키 발급/관리** (`https://mobile.domeggook.com/APIs/gate`) 에서 발급합니다. 심사 없이 바로 나옵니다.
3. 아이디 1개당 **최대 5개**까지 만들 수 있습니다.
4. 받은 키를 아래 **Open API Key (aid)** 칸에 넣습니다.

> **호출 제한이 있습니다 — 분당 180회, 하루 15,000회.** 넘기면 자동 차단됩니다. 상품을 대량으로 가져올 때 걸릴 수 있는 한도입니다.

### 계정 추가 / 수정

- **채널을 고르면 그 채널에 필요한 칸만 나타납니다.** 도매꾹은 API Key 한 칸, 온채널은 아예 없고, 네이버만 시크릿까지 받습니다.
- 필요한 값을 넣고 **[계정 저장]**.
- **같은 채널 + 같은 계정 이름**으로 저장하면 **기존 값이 갱신**됩니다. (새로 생기지 않습니다)
- **시크릿을 비워두고 저장하면 기존 시크릿이 그대로 유지**됩니다. 시크릿은 **바꿀 때만** 입력하세요.
- **부가 설정(JSON, 선택)** — 스토어채널ID·반품지코드 같은 값을 넣는 칸입니다. 형식이 틀리면 저장이 거부됩니다.

### 연결된 계정 목록

채널·계정·client_id(일부 가려짐)·시크릿 설정 여부·**상태**·최근 확인 시각이 보이고, 줄마다 **[검증]** 과 **[삭제]** 가 있습니다.

상태는 **ACTIVE(정상) / INVALID(문제) / EXPIRED(만료) / DISABLED(사용 안 함)** 로 표시됩니다.

### 다른 몰의 자격증명 복사

편집 몰 외에 다른 몰이 있으면 **원본 몰**을 골라 **[복사]** 할 수 있습니다. 여러 서브몰이 같은 계정을 쓸 때 편합니다.

- **값을 복제하는 것이지 공유가 아닙니다.** 나중에 원본 몰에서 계정을 바꿔도 복사본은 따라 바뀌지 않습니다.
- **이 몰의 같은 채널 설정을 덮어씁니다.**

## 2. 네이버 카테고리 리소스

네이버 스마트스토어의 **전체 카테고리 목록을 통째로 받아와 보관**하는 화면입니다. 상품을 등록할 때 **참고용**으로 씁니다.

### 화면 구성

- **통계 4칸** — 전체 적재 / 활성(현재 수집분) / **활성 리프(선택가능)** / 마지막 수집 시각
- **[지금 수집]** — 네이버에서 전체 카테고리를 즉시 다시 받아옵니다.
- **자동 수집 주기** — **자동 수집 사용** 체크 + **주기(시간)** 입력. 기본 24시간(하루 1회)이며 **최소 1시간, 최대 720시간(30일)** 입니다.
- **최근 수집 로그** — 시작 시각·트리거(MANUAL/CRON)·상태(SUCCESS/FAILED/SKIPPED)·수집 건수·비활성 건수·메시지

### 알아둘 점

- **상품 등록 화면에서 고를 수 있는 건 "활성 리프" 카테고리뿐입니다.** (가장 끝단 분류만 선택 가능)
- 수집할 때마다 **이번에 안 내려온 카테고리는 자동으로 비활성 처리**됩니다. 네이버에서 사라진 분류가 목록에 남지 않게 하는 정리 작업입니다.
- 네이버 카테고리는 **판매자 공통**이라 **몰마다 따로 수집할 필요가 없습니다.** ACTIVE 인 네이버 계정이 **어느 몰에든 하나만 있으면** 수집됩니다.

### 상품 등록에서 쓰는 법

**상품 관리 → 상품 등록/수정** 화면의 카테고리 영역에 **"🔎 네이버 카테고리에서 찾기"** 검색창이 있습니다.

1. 검색어를 입력합니다. (예: `홍삼`)
2. 전체 경로가 뜨는 목록에서 하나를 고릅니다.
3. 고른 **네이버 분류 이름이 카테고리 직접 입력 칸에 자동으로 채워집니다.**
4. 저장하면 **비슷한 몰 카테고리가 있으면 연결하고, 없으면 새로 만듭니다.**

## ⚠️ 주의사항

**"검증" 이 통과해도 실제 연결이 확인된 게 아닐 수 있습니다.**

> 지금 **실제로 접속해서 확인하는 채널은 네이버 스마트스토어뿐**입니다. **도매꾹·도매매**는 **"자격증명 형식 확인됨"** — 즉 **빈칸이 아닌지만 봅니다.** **온채널**은 자격증명이 필요 없어 **항상 통과**합니다. 이 채널들이 ACTIVE 로 보인다고 해서 키가 맞다는 뜻은 아닙니다.

**네이버 카테고리는 몰 카테고리에 자동 반영되지 않습니다.**

> 5천 건 넘게 수집해도 **카테고리 관리 화면에는 아무것도 늘지 않습니다.** 이건 **참조용 목록**일 뿐이고, 실제 몰 카테고리는 **상품 등록 때 고른 것만** 만들어집니다. 정상 동작입니다.

**네이버 카테고리를 수집하려면 네이버 계정이 먼저 ACTIVE 여야 합니다.**

> 자격증명이 없으면 수집은 **실패가 아니라 SKIPPED(건너뜀)** 로 조용히 기록됩니다. 로그에 SKIPPED 만 쌓인다면 **공급처/채널 연결에서 [검증]을 눌러 ACTIVE 로 만들었는지** 확인하세요.

**[지금 수집]은 눌러도 화면이 바로 바뀌지 않습니다.**

> 수집은 **뒤에서 돌아갑니다.** "수집을 시작했습니다" 안내가 뜨면 **잠시 후 새로고침**해서 통계와 로그를 확인하세요. 버튼을 여러 번 누를 필요 없습니다.

**자동 수집은 서버에 자동 실행이 등록되어 있어야 동작합니다.**

> 화면에서 **자동 수집 사용**을 켜는 것만으로는 부족합니다. 서버 쪽 자동 실행이 준비되지 않았다면 주기가 지나도 아무 일도 일어나지 않습니다. **마지막 실행이 계속 "없음"** 이라면 이 경우를 의심하세요. 그동안에는 **[지금 수집]** 으로 수동 수집하면 됩니다.

**계정 삭제와 자격증명 복사는 되돌릴 수 없습니다.**

> **[삭제]** 는 확인창 한 번 뒤에 바로 지웁니다. **[복사]** 는 이 몰의 같은 채널 설정을 **덮어씁니다.** 시크릿은 화면에 다시 보여주지 않으므로, 지우면 **원래 키를 발급처에서 다시 받아와야** 합니다.

**Shopify 연동과는 완전히 별개입니다.**

> 이 메뉴는 **네이버 스마트스토어**용입니다. 예전 해외몰(Shopify) 연동과는 **아무 관련이 없고, 설정도 공유하지 않습니다.**

## 아직 안 되는 것

- **상품 가져오기 · 가져온 상품(중간) · 스마트스토어 등록 · 스토어 상품 가져오기 · 재고·주문 가져오기** — **5개 화면 모두 안내 문구만 있는 빈 화면**입니다. 실제 기능은 없습니다.
- **도매꾹 · 도매매 · 온채널의 실제 API 연결 검증** — 형식만 확인합니다. 진짜 연결 확인은 각 공급처 연동이 붙은 뒤에 가능합니다.
- **온채널 자동 연동** — 제휴 API가 확인되기 전까지는 **수동(CSV) 방식**만 계획되어 있습니다.
- **네이버 브랜드 목록 수집** — **카테고리만 수집합니다.** 브랜드는 아직 수집하지 않습니다.
- **기본 목표 마진율 · 기본 채널 수수료율** — 저장만 되고 **아직 어떤 계산에도 쓰이지 않습니다.**
- **재고·주문 자동 동기화(배치)** — 계획상으로도 1차에는 **버튼을 눌러 그 시점에 가져오는 방식**이며, 자동 배치는 없습니다.
- **몰 간 계정 공유** — 계정은 **몰마다 따로**입니다. 공유는 안 되고 **복사만** 됩니다.
- **한 채널에 여러 계정 동시 사용** — 채널당 계정 이름별로 저장은 되지만, 실제 작업에 **어느 계정을 쓸지 고르는 화면은 없습니다.**
