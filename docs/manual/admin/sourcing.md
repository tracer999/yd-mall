# 외부몰 연동

**도매꾹·도매매** 같은 공급처에서 상품을 검색해 가져오고, 그것을 **우리 몰 상품으로 등록**하는 메뉴 묶음입니다. 앞으로 **네이버 스마트스토어**에 그 상품을 올려 파는 것까지가 목표이며, 그중 일부는 아직 준비 중입니다.

> **"가져오기"와 "상품 등록"은 다른 일입니다.** 가져오기는 공급처 정보를 **창고(중간 목록)에 복사해 두는 것**이고, 상품 등록은 그것을 **우리 몰에서 실제로 파는 상품으로 만드는 것**입니다. 두 단계를 반드시 구분하세요.

---

## 1. 이 메뉴가 안 보인다면

외부몰 연동은 **몰마다 켜고 끄는 기능**입니다.

- 꺼져 있는 몰에서는 **사이드바에 대메뉴 자체가 나타나지 않습니다.**
- 주소를 직접 입력해도 `이 몰은 외부몰 연동 기능이 활성화되어 있지 않습니다.` 안내가 뜨고 막힙니다.
- **켜는 것은 제공자(최고관리자)만 할 수 있습니다.** 일반 관리자에게는 사용 여부 체크박스가 회색으로 잠겨 보입니다.

## 3. 전체 흐름 한눈에 보기

```
① 공급처/채널 연결          도매꾹 API 키 등록 → [검증] → 상태 ACTIVE
        ↓
② 상품 가져오기             검색 → 원하는 상품 체크 → [선택 상품 가져오기]
        ↓                   (공급처 원본이 '가져온 상품' 목록에 쌓입니다)
③ 가져온 상품               내용 확인 · 재수집 · 삭제
        ↓
④ 우리 몰에 등록            카테고리 + 마진율 지정 → [우리 몰에 등록]
        ↓                   (판매중지 · 비노출 상태로 만들어집니다)
⑤ 상품 관리                 내용 확인 후 판매 상태와 노출을 켠다
        ↓
⑥ 네이버에 올리기(선택)      리소스 수집 → 카테고리에 네이버 분류 연결
                            → 등록 기본값 작성 → [선택 상품을 네이버에 등록]
```

> **③에서 [스마트스토어로 바로 전송]을 누르면 ④·⑥을 한 번에 처리합니다.** 우리 몰 상품은 그대로 만들어지고, 이어서 네이버로 전송됩니다(한 번에 최대 10건). 네이버 등록 기본값과 자격증명이 먼저 설정돼 있어야 하며, 상태를 판매중지로 두면 네이버에서도 전시중지로 올라갑니다.

> **④에서 만들어지는 상품은 반드시 "판매중지 · 비노출" 상태입니다.** 확인 없이 고객에게 노출되지 않도록 하는 안전장치입니다. 상품 관리에서 **판매 상태와 노출 설정을 둘 다** 켜야 팔립니다.

> **모든 작업은 지금 편집 중인 몰 기준입니다.** 각 화면 맨 위에 **현재 편집 몰**이 파란 띠로 표시됩니다. 계정도 가져온 상품도 몰마다 따로입니다.

## 4. 1단계 — 공급처/채널 연결

<div class="manual-goto-bar">
  <span class="manual-goto-label">이 단계 화면</span>
  <a class="manual-goto" href="/admin/sourcing/connections" target="_blank">공급처/채널 연결 열기</a>
</div>

### 4-1. 연동 사용 설정

| 입력란 | 설명 |
|---|---|
| **이 몰에서 외부몰 연동 사용** | 제공자(최고관리자)만 바꿀 수 있습니다. |
| **기본 목표 마진율(%)** | **실제로 쓰입니다.** 판매가 = 공급가 × (1 + 마진율), 10원 단위 반올림. 상품 등록 때 마진율을 비우면 이 값이 적용됩니다. |
| **기본 채널 수수료율(%)** | 스마트스토어 등 외부 채널이 떼가는 수수료율. **지금은 저장만 되고 어떤 가격에도 반영되지 않습니다.** |

입력 후 **[설정 저장]**을 누릅니다.

> **기본 목표 마진율을 먼저 정해 두세요.** 비워 두면 상품을 등록할 때마다 마진율을 직접 입력해야 하고, 안 넣으면 `기본값이 없으니 직접 입력하세요` 경고가 뜹니다.

### 4-2. 연결할 수 있는 채널

| 채널 | 구분 | 입력하는 것 |
|---|---|---|
| **도매꾹·도매매** | 공급처 | **Open API Key (aid) 하나만**. 시크릿 없음. |
| **온채널** | 공급처 | **없음** — 수동·CSV 방식이라 자격증명이 필요 없습니다. |
| **네이버 스마트스토어** | 판매채널 | **client_id + client_secret 둘 다** 필요합니다. |

> **도매매는 따로 등록하지 마세요.** 도매꾹과 도매매는 아이디를 공유해 **키 하나로 양쪽 상품이 다 조회**됩니다. 그래서 채널 목록에 "도매꾹·도매매" 한 줄만 있습니다. 어느 쪽 상품을 가져올지는 **상품 가져오기 화면에서 고르는 것**이지 계정을 나누는 게 아닙니다.

**도매꾹 API 키 받는 법**

도매꾹에 로그인한 뒤 **API 키 관리**에서 발급합니다(심사 없음, 시크릿 없음). 아이디당 **최대 5개**까지 만들 수 있습니다.

> **호출 제한이 있습니다 — 분당 180회, 하루 15,000회.** 상품을 대량으로 가져올 때 걸릴 수 있는 한도입니다.

### 4-3. 계정 추가 / 수정

채널을 고르면 **그 채널에 필요한 칸만** 나타납니다.

| 입력란 | 설명 |
|---|---|
| **채널** | 위 표의 3개 중 선택 |
| **계정 이름** | 기본값은 `기본` |
| **client_id / API Key** | 채널에 따라 라벨이 바뀝니다 (도매꾹은 `Open API Key (aid)`) |
| **client_secret / 시크릿** | 네이버에만 나타납니다 |

> **같은 채널 + 같은 계정 이름으로 저장하면 기존 값이 갱신됩니다.** 새로 생기지 않습니다.

> **시크릿을 비워 두고 저장하면 기존 시크릿이 그대로 유지됩니다.** 시크릿은 **바꿀 때만** 입력하세요.

### 4-4. 연결된 계정 목록과 [검증]

채널 · 계정 · client_id(일부 가려짐) · 시크릿 설정 여부 · **상태** · 최근 확인 시각이 보이고, 줄마다 **[검증]** 과 **[삭제]** 가 있습니다.

| 상태 | 뜻 |
|---|---|
| **ACTIVE** | 정상 |
| **INVALID** | 문제 있음 (메시지가 함께 표시됩니다) |
| **EXPIRED** | 만료 |
| **DISABLED** | 사용 안 함 |

**[검증]**은 각 채널을 실제로 호출해 확인합니다.

- **도매꾹·도매매** — Open API 로 1건 조회해 **키가 진짜 맞는지** 확인합니다.
- **네이버 스마트스토어** — 실제 토큰을 발급받아 확인합니다.
- **온채널** — 확인할 자격증명이 없어 항상 통과합니다.

> **가져오기를 하기 전에 반드시 [검증]으로 ACTIVE 를 확인하세요.** 키가 틀리면 상품 가져오기 화면에서 카테고리 목록조차 불러오지 못합니다.

### 4-5. 다른 몰의 자격증명 복사

다른 몰이 있으면 **원본 몰**을 골라 **[복사]** 할 수 있습니다. 여러 서브몰이 같은 계정을 쓸 때 편합니다.

> **값을 복제하는 것이지 공유가 아닙니다.** 나중에 원본 몰에서 계정을 바꿔도 복사본은 따라 바뀌지 않습니다. 그리고 **이 몰의 같은 채널 설정을 덮어씁니다.**

## 5. 2단계 — 상품 가져오기

<div class="manual-goto-bar">
  <span class="manual-goto-label">이 단계 화면</span>
  <a class="manual-goto" href="/admin/sourcing/import" target="_blank">상품 가져오기 열기</a>
</div>

### 5-1. 검색하기

| 입력란 | 설명 |
|---|---|
| **공급처** | `도매꾹` 또는 `도매매` |
| **검색어** | 예: `홍삼` |
| **개수** | 20 / 50 / 100 |
| **카테고리로 찾기** | 대분류 → 중분류 → 소분류를 이름으로 골라 갑니다. 코드를 외울 필요가 없습니다. |

> **도매꾹과 도매매는 키를 공유하지만 노출 상품과 공급가가 다릅니다.** 가져올 마켓을 정확히 고르세요.

> **검색어 또는 카테고리 중 하나는 반드시 있어야 합니다.** 또 **대분류만으로는 검색되지 않습니다** — 중분류 이상을 고르거나 검색어를 함께 넣으세요. (공급처 API 제약입니다.)

**검색만으로는 아무것도 저장되지 않습니다.**

### 5-2. 결과에서 골라 가져오기

결과 표에는 이미지 · 상품명 · 상품번호 · **공급가** · 최소수량 · 판매자 · 배송비 · 상태가 나옵니다. 상품명 아래의 **[공급처에서 보기]** 로 원본 페이지를 새 창에서 확인할 수 있습니다.

- `성인` · `사업자전용` 배지가 붙은 상품은 판매 조건을 반드시 확인하세요.
- `가져옴` 표시는 **이미 가져온 상품**이라는 뜻입니다. 다시 선택하면 **최신 정보로 갱신**되며 중복으로 만들어지지 않습니다.

담을 상품을 체크하고 **[선택 상품 가져오기]**를 누릅니다.

> **한 번에 최대 50건까지만 가져올 수 있습니다.** 공급처 호출 간격 때문에 50건이면 20초 가까이 걸립니다. 맨 위 전체선택을 눌러도 앞에서 50건만 선택되며 안내가 뜹니다.

완료되면 **가져온 상품** 화면으로 넘어가면서 결과가 요약됩니다. 예: `12건 가져오기 완료 · 1건 실패 · 재판매 금지 2건 포함(등록 전 확인 필요)`

## 6. 3단계 — 가져온 상품 확인하기

<div class="manual-goto-bar">
  <span class="manual-goto-label">이 단계 화면</span>
  <a class="manual-goto" href="/admin/sourcing/staging" target="_blank">가져온 상품 열기</a>
</div>

공급처에서 복사해 온 **원본 스냅샷 목록**입니다. 아직 우리 몰 상품이 아닙니다.

### 6-1. 요약 카드 5개

| 카드 | 뜻 |
|---|---|
| **전체 적재** | 이 몰에 가져온 총 건수 |
| **수집 실패** | 상세 정보를 못 받아온 건수 |
| **재판매 금지** | 공급처가 재판매를 막은 상품 수 |
| **우리 몰 등록됨** | 이미 상품으로 만든 건수 (눌러서 바로 필터링) |
| **미등록** | 아직 상품으로 만들지 않은 건수 (눌러서 바로 필터링) |

### 6-2. 필터

| 항목 | 값 |
|---|---|
| **공급처** | 전체 / 도매꾹 / 도매매 / 온채널 |
| **수집 상태** | 전체 / 상세 수집 완료 / 수집 실패 / 목록만 |
| **우리 몰 등록 여부** | 전체 / 미등록만 / 등록됨만 |
| **상품명 / 상품번호** | 검색어 |

**[미등록만 빠르게 보기]** 버튼으로 아직 처리하지 않은 것만 골라 볼 수 있습니다.

### 6-3. 상세 화면에서 할 수 있는 일

상품명을 누르면 상세 화면이 열립니다. 공급처 원본(가격·재고·옵션·이미지·상세 내용)을 그대로 확인할 수 있고, 아래 두 버튼이 있습니다.

- **[공급처에서 다시 가져오기]** — 공급처의 **현재** 가격·재고·옵션으로 스냅샷을 갱신합니다.
- **[삭제]** — 이 목록에서만 지웁니다. **공급처 원본에는 영향이 없습니다.**

> **가격과 재고는 "가져온 시점"의 값입니다.** 시간이 지났다면 등록 전에 **[공급처에서 다시 가져오기]**로 최신화하세요. 오래된 공급가로 판매가를 계산하면 손해를 볼 수 있습니다.

> **수량별 가격(예: 1개 17,000원 / 2개부터 16,800원) 상품은 최소 수량 기준 단가를 공급가로 씁니다.** 목록·상세의 공급가와 예상 판매가 모두 이 값 기준입니다. 대량 매입 단가로 팔 계획이라면 등록 후 상품 관리에서 매입가·판매가를 직접 조정하세요.

> **`재판매 금지` 배지를 반드시 확인하세요.** 공급처가 온라인 재판매를 막은 상품입니다. 그대로 팔면 제재 대상이 될 수 있습니다.

## 7. 4단계 — 우리 몰 상품으로 등록하기

**여러 건을 한 번에** 등록하거나, **한 건씩** 등록할 수 있습니다.

### 7-1. 여러 건 한 번에 (목록 화면)

목록 위쪽 초록 영역에서 진행합니다.

| 입력란 | 필수 | 설명 |
|---|---|---|
| **우리 몰 카테고리** | 필수 | 선택기에서 고릅니다. 공급처 분류와 우리 몰 분류 체계가 다르므로 직접 골라야 합니다. |
| **마진율(%)** | 선택 | 비우면 **몰 기본 목표 마진율**이 적용됩니다. |

상품을 체크하고 **[선택 상품을 우리 몰에 등록]**을 누릅니다.

> **선택한 상품 전부에 같은 카테고리·마진율이 적용됩니다.** 상품마다 다르게 하려면 상세 화면에서 한 건씩 등록하세요.

> **이미 등록된 상품은 자동으로 제외됩니다.** 확인창에 `※ 이미 등록된 N건은 제외됩니다` 로 안내되며 중복 생성되지 않습니다.

### 7-1-1. 스마트스토어로 바로 전송

우리 몰 등록과 네이버 등록을 **한 번에** 하고 싶다면 같은 자리의 **[스마트스토어로 바로 전송]** 을 누르세요.

1. 위쪽에서 **우리 몰 카테고리·마진율**을 먼저 고릅니다(일반 등록과 같습니다).
2. 아래 **네이버 카테고리(리프)** 는 두 글자 이상 입력해 검색·선택합니다. 비워 두면 우리 몰 카테고리에 연결된 네이버 분류를 자동으로 씁니다.
3. 버튼을 누르면 확인창이 뜹니다 — **스마트스토어에 실제로 등록되는 작업**입니다.

- 한 번에 **최대 10건**입니다(건당 2~4초 걸립니다).
- **[공급처/채널 연결]의 네이버 계정과 [네이버 등록 기본값]이 먼저 설정돼 있어야 합니다.** 안 되어 있으면 *"우리 몰에는 N건이 등록됐지만 네이버 전송은 시작하지 못했습니다"* 라고 알려 드립니다. 이 경우 **우리 몰 상품은 이미 만들어져 있으니**, 설정을 마친 뒤 [스마트스토어 등록] 화면에서 그 상품을 다시 보내면 됩니다.
- 상태를 **판매중지**로 두면 네이버에서도 **전시중지**로 올라갑니다. 바로 팔려면 판매중으로 바꿔 등록하세요.

### 7-2. 한 건씩 (상세 화면)

상세 화면 위쪽의 **우리 몰 상품으로 등록** 영역에서 카테고리와 마진율을 넣습니다. **예상 판매가가 즉시 미리보기로 표시**되므로 등록 전에 금액을 확인할 수 있습니다.

```
공급가 12,000원  →  예상 판매가 15,600원   (마진율 30%, 10원 단위 반올림)
```

이미 등록한 상품이면 폼 대신 `이미 등록되었습니다 — 상품번호 N` 과 **[상품 수정으로 이동]** 링크가 나옵니다.

### 7-3. 등록하면 무엇이 만들어지나

| 공급처 정보 | 우리 몰 상품 |
|---|---|
| 공급가 | **매입가** |
| 공급가 × (1 + 마진율) | **판매가** (10원 단위 반올림) |
| 옵션 목록 | 옵션·SKU (옵션별 추가금 반영) |
| 대표·상세 이미지 | 내려받아 우리 서버에 저장 |
| 상세 설명 | 위험한 태그를 걸러 저장 |
| 어느 공급처에서 가져왔는지 | **상품 출처** (`도매꾹` / `도매매` / `온채널`) |

> **등록된 상품에는 출처 표시가 붙습니다.** 상품 관리 목록에서 상품명 아래에 `도매꾹` 처럼 색이 있는 표시가 나오고, 직접 등록한 상품은 회색 `몰상품` 으로 나옵니다.
> 목록 위 **상품 출처** 필터로 "가져온 상품만" 골라 볼 수도 있습니다. 등록 직후 **판매 상태·노출 설정을 켜러 갈 때** 이 필터를 쓰면 방금 등록한 상품만 모아서 처리할 수 있습니다.
>
> 출처는 등록 시점에 자동으로 정해지며 **나중에 바꿀 수 없습니다.** 상품을 지웠다가 직접 등록하면 그때는 `몰상품` 이 됩니다.

> **상품은 반드시 "판매중지 · 비노출"로 만들어집니다.** 확인 없이 팔리는 사고를 막기 위한 안전장치입니다. 내용을 검토한 뒤 **상품 관리에서 판매 상태와 노출 설정을 켜세요.**

> **옵션(SKU)은 처음부터 `판매` 상태로 만들어집니다.** 막는 일은 상품 쪽에서 하기 때문에 옵션까지 내리지 않습니다. 옵션을 함께 내려 두면 나중에 상품 판매를 켰을 때 **고객이 고를 수 있는 옵션이 하나도 없는 상태**가 됩니다. 특정 옵션만 안 팔고 싶다면 **옵션·SKU 관리** 화면에서 그 줄만 `중지` 로 두세요.

> **상세 수집이 완료된 상품만 등록할 수 있습니다.** 상태가 `FAILED` 인 상품은 `상세 수집이 완료된 상품만 등록할 수 있습니다` 로 거부됩니다. 먼저 **[공급처에서 다시 가져오기]**를 시도하세요.

> **공급가가 비정상(1억 원 초과)이면 등록이 막힙니다.**
> `공급가가 비정상입니다(공급가 … / 판매가 …). 공급처 원본 가격을 확인한 뒤 [상세 다시 가져오기] 로 갱신하세요.` 라는 메시지가 뜹니다.
> 공급처가 **수량별 가격**(예: 1개 17,000원 / 2개부터 16,800원)을 준 상품을 예전 방식으로 읽으면 두 금액이 하나로 붙어 **억 단위 가격**이 되고, 그대로 등록되면 21억짜리 상품이 몰에 노출됩니다. 실제로 그런 사고가 있어 막아 둔 것입니다.
> 지금은 수량별 가격을 제대로 읽어 **최소 수량 기준 단가**를 공급가로 씁니다. 그래도 이 메시지가 뜬다면 **[공급처에서 다시 가져오기]** 로 스냅샷을 갱신한 뒤 다시 시도하고, 그래도 같으면 공급처 원본 가격 자체를 확인하세요.

이미지 일부를 못 받아오면 `이미지 N장은 가져오지 못했습니다` 안내가 붙습니다. 등록 자체는 진행되므로 상품 관리에서 직접 채워 주세요.

## 8. 네이버 리소스 관리

<div class="manual-goto-bar">
  <span class="manual-goto-label">이 단계 화면</span>
  <a class="manual-goto" href="/admin/sourcing/naver-taxonomy" target="_blank">네이버 리소스 관리 열기</a>
</div>

네이버가 정해 둔 **코드값 목록을 미리 받아 보관**하는 화면입니다. 카테고리 ID·원산지 코드 같은 것은 사람이 알 수 없는 값이라, 여기에 받아 두고 다른 화면에서는 **검색해서 고르기만** 하면 되게 만들어 둔 것입니다.

> **수집된 네이버 리소스를 보는 곳은 이 화면 하나입니다.** 예전에는 `몰 관리 → 리소스 관리`에도 같은 내용이 절반쯤 겹쳐 있었지만, 그 메뉴는 **제거되고 이 화면으로 통합**되었습니다.

### 8-1. 받아오는 목록 4가지

화면 맨 위에 **카드 4장**이 나란히 있습니다. 각 카드에 건수·마지막 수집 시각·**[지금 수집]** 버튼이 함께 있습니다.

| 카드 | 무엇 | 어디에 쓰이나 |
|---|---|---|
| **카테고리** | 네이버 상품 분류 전체 | 상품을 네이버 어느 분류에 올릴지 |
| **원산지 코드** | 국산·수입산·시군구까지 | 등록 기본값의 **원산지** 칸 |
| **고시 상품군** | 품목군별로 적어야 하는 항목 묶음 | 등록 기본값의 **상품정보제공고시** |
| **브랜드** | (미구현) | — |

> **고시 상품군은 처음부터 채워져 있습니다.** 네이버 계정을 연결하지 않아도 바로 쓸 수 있도록 기본 제공됩니다. [지금 수집]은 최신 내용으로 덮어쓸 때만 누르면 됩니다.

> **브랜드 카드는 비어 있는 것이 정상입니다.** 네이버 브랜드 목록을 받아오는 기능은 **아직 만들어지지 않았습니다.** 브랜드를 지정하지 않아도 상품 등록은 정상 동작합니다.

이 밖에 화면 아래쪽에 다음이 있습니다.

- **자동 수집 주기** — `자동 수집 사용` 체크 + `주기(시간)`. 기본 24시간, **최소 1시간 · 최대 720시간(30일)**.
- **수집된 리소스 목록** — `카테고리` / `원산지 코드` / `고시 상품군` / `브랜드` **네 탭**.
- **최근 수집 로그** — 시작 시각 · 종류 · 트리거(MANUAL/CRON) · 상태(SUCCESS/FAILED) · 수집 건수 · 비활성 건수 · 메시지

### 8-2. 목록에서 찾아보기

탭을 고르고 검색창에 **이름 일부**를 넣으면 됩니다.

| 탭 | 이렇게 찾습니다 |
|---|---|
| 카테고리 | 전체 경로(대>중>소>세) 또는 카테고리 ID |
| 원산지 코드 | 원산지명(예: `강원도`) 또는 코드 |
| 고시 상품군 | 상품군명(예: `건강기능식품`) 또는 코드 |
| 브랜드 | 브랜드명·영문명·브랜드 ID |

| 필터 | 뜻 |
|---|---|
| **활성만** | 마지막 수집분에 들어 있던 것만 봅니다. **처음 들어오면 켜져 있습니다.** |
| **리프만** (카테고리 탭) | 상품 등록에서 실제로 고를 수 있는 **끝단 분류**만 봅니다. |

> **비활성 항목도 볼 수 있습니다.** `활성만` 체크를 풀면 네이버에서 사라져 비활성 처리된 과거 분류까지 나옵니다. 왜 특정 카테고리가 상품 등록 화면에서 안 보이는지 확인할 때 쓰세요.

### 8-3. 카테고리마다 붙는 "고시 유형"

카테고리 탭의 표에는 **고시 유형** 칸이 있습니다. 상품정보제공고시는 품목군마다 적어야 할 항목이 전혀 달라서(의류는 소재·치수, 건강기능식품은 원료명·기능정보), 어느 유형인지가 먼저 정해져야 합니다.

- **카테고리를 수집하면 자동으로 배정됩니다.** 따로 누를 것이 없습니다.
- 맞지 않는 것만 표에서 **드롭다운으로 직접 고치고 [저장]** 하면 됩니다.
- 직접 고친 것에는 **`직접`** 표시가 붙고, **다시 수집하거나 [규칙으로 일괄 배정]을 눌러도 유지**됩니다.
- 위쪽 노란 띠에서 `배정 4,999 / 전체 리프 4,999` 처럼 현황을 볼 수 있습니다. **미지정이 있으면 빨간색**으로 표시됩니다.

> **고시 유형을 상품마다 고를 필요는 없습니다.** 상품에 네이버 카테고리만 지정하면 유형은 자동으로 따라옵니다.

### 8-4. 알아둘 점

- 상품 등록에서 고를 수 있는 것은 **"활성 리프"(가장 끝단 분류)뿐**입니다.
- 수집할 때마다 **이번에 안 내려온 카테고리는 자동으로 비활성 처리**됩니다. 네이버에서 사라진 분류를 정리하는 작업입니다.
- 네이버 카테고리는 **판매자 공통**이라 몰마다 따로 수집할 필요가 없습니다.

> **수집하려면 네이버 계정이 먼저 ACTIVE 여야 합니다.** 화면 위에 `ACTIVE 상태의 네이버 스마트스토어 자격증명이 없습니다` 경고가 뜬다면, 공급처/채널 연결에서 **[검증]**을 먼저 누르세요.

> **[지금 수집]은 눌러도 화면이 바로 바뀌지 않습니다.** 수집은 뒤에서 돌아갑니다. `수집을 시작했습니다` 안내가 뜨면 **잠시 후 새로고침**해 통계와 로그를 확인하세요. 여러 번 누를 필요 없습니다.

> **수집한 네이버 카테고리는 우리 몰 카테고리에 자동 반영되지 않습니다.** 수천 건을 받아와도 카테고리 관리 화면에는 아무것도 늘지 않습니다. **참조용 목록**일 뿐이며, 상품 등록에서 고른 것만 실제 카테고리로 연결·생성됩니다. 정상 동작입니다.

> **자동 수집은 서버 쪽 자동 실행이 준비돼야 동작합니다.** 화면에서 체크만 해서는 부족합니다. **마지막 실행이 계속 "없음"** 이라면 이 경우이니, 그동안은 **[지금 수집]**으로 수동 수집하세요.

## 9. 스마트스토어 등록 — 우리 몰 상품을 네이버에 올리기

<div class="manual-goto-bar">
  <span class="manual-goto-label">이 단계 화면</span>
  <a class="manual-goto" href="/admin/sourcing/publish" target="_blank">스마트스토어 등록 열기</a>
  <a class="manual-goto is-sub" href="/admin/categories" target="_blank">카테고리 관리</a>
</div>

> **방향에 주의하세요.** 이 화면은 **우리 몰 → 네이버** 입니다. `가져온 상품`의 [등록]은 **공급처 → 우리 몰** 이라 이름만 비슷하고 정반대입니다.

### 9-1. 준비물 세 가지

이 세 가지가 갖춰지지 않으면 등록 버튼이 잠깁니다.

| 준비물 | 어디서 |
|---|---|
| ① 네이버 계정 연결 | 공급처/채널 연결에서 [검증] → ACTIVE |
| ② 리소스 수집 | 네이버 리소스 관리에서 카테고리·원산지 [지금 수집] |
| ③ 등록 기본값 | 이 화면의 **[네이버 등록 기본값]** (몰당 한 번만) |

### 9-2. 네이버 등록 기본값 (몰당 한 번)

네이버는 상품 한 건을 올릴 때도 A/S·원산지·반품비 같은 **판매자 정보를 매번 함께** 요구합니다. 상품마다 적을 수 없으니 몰에 한 벌 저장해 두고 자동으로 붙입니다.

- **A/S 전화번호·안내** — 비워 두면 `사이트 설정`의 대표 전화번호와 고객센터 운영시간을 **자동으로 가져옵니다.** 자동으로 채운 항목에는 초록색으로 `사이트 설정에서 가져옴` 이 표시됩니다.
- **원산지** — 검색창에 `국산`, `강원도` 처럼 입력해 고릅니다. **코드를 직접 적지 않습니다.**
- **기본 배송비·무료배송 기준** — `배송 정책`에서 자동으로 가져옵니다.
- **반품·교환 배송비** — 직접 입력해야 합니다.
- **기본 택배사** — 목록에서 고릅니다.
- **출고지·반품지** — 아래 9-5 참고.
- **상품정보제공고시** — 유형을 고르면 **그 유형에 맞는 입력칸이 자동으로 나타납니다.**

> **"기본 택배사"는 실제 송장과 다릅니다.** 이 값은 **상품에 표기되는 기본 배송 정보**일 뿐이고, 실제로 물건을 보낼 때 쓰는 택배사·운송장번호는 주문마다 따로 등록합니다. 달라도 괜찮습니다.

### 9-3. 상품에 네이버 카테고리 지정하기 — **카테고리 관리에서 하세요**

네이버는 **끝단 분류(리프)** 를 요구합니다. 상품이 수천 건이면 하나씩 지정할 수 없으므로, **카테고리에 한 번 지정하고 상품이 물려받게** 합니다.

1. `메뉴/카테고리 관리 → 카테고리` 에서 **카테고리 이름을 클릭**해 상세로 들어갑니다.
2. 왼쪽 **[네이버 스마트스토어 연결]** 카드의 검색창에 두 글자 이상 입력하고 목록에서 고릅니다.
3. **[이 카테고리 상품에 바로 반영]** 을 체크하면 소속 상품에 즉시 채워집니다.
4. **[네이버 카테고리 저장]**

> **이미 지정된 상품은 덮어쓰지 않습니다.** 개별로 다르게 지정해 둔 상품은 그대로 보존됩니다.

> **끝단 분류가 아니면 저장되지 않습니다.** `네이버는 리프(최하위)만 받습니다` 라고 뜨면 더 깊은 분류를 고르세요. 여기서 막는 이유는, 잘못 지정하면 **상품 수백 건이 등록 단계에서 한꺼번에 실패**하기 때문입니다.

> **고시 유형은 고르지 않습니다.** 연결한 네이버 카테고리가 정하며, 카드에 무엇으로 정해졌는지 표시만 됩니다.

이 화면에서도 보조 수단을 제공합니다.

- **[카테고리에서 일괄 지정]** — 카테고리에 연결해 둔 값을 상품에 한 번에 내려 줍니다. 끝단 분류가 아닌 카테고리는 건너뛰고, 몇 건이 왜 건너뛰어졌는지 숫자로 알려 줍니다.
- **선택 상품에 지정** — 필터 `카테고리 미지정만` 으로 좁힌 뒤 전체선택 → 카테고리 검색 → **[선택 상품에 지정]**. 이쪽은 **이미 지정된 것도 덮어씁니다**(다시 지정하는 것이 목적이므로).

### 9-4. 등록하기

상품을 체크하고 **[선택 상품을 네이버에 등록]** 을 누릅니다.

> **네이버에는 대량 등록 기능이 없습니다.** 서버가 한 건씩 순서대로 보내며 **건당 2~4초** 걸립니다. **1회 최대 30건**이고, 완료될 때까지 창을 닫지 마세요.

등록 후에는 **[상태 확인]** 을 눌러 실제 상태를 확인하세요. 네이버는 등록을 받아 준 **뒤에** 분류를 강제로 옮기거나 판매금지로 바꿀 수 있습니다. 강제로 옮겨졌다면 화면에 표시됩니다.

> **등록 후에는 대분류를 바꿀 수 없습니다.** 잘못 올렸다면 삭제하고 다시 등록해야 합니다. 그래서 대량으로 돌리기 전에 **한두 건만 먼저** 해 보시길 권합니다.

### 9-5. 출고지·반품지 고르기

**네이버 등록 기본값**에는 출고지·반품지를 지정하는 칸이 있습니다. 네이버가 매긴 **주소록 번호**라 사람이 알 수 없는 값이어서, 직접 적지 않고 목록에서 고릅니다.

1. **[주소록 불러오기]** 를 누릅니다. 네이버 판매자센터에 등록된 주소록을 그대로 받아 옵니다.
2. **출고지**·**반품지** 드롭다운에서 각각 고릅니다. 주소록 이름과 주소가 함께 보이므로 번호를 몰라도 됩니다.
3. **[프로필 저장]** 을 눌러야 반영됩니다.

> **목록은 저장해 두지 않습니다.** 누를 때마다 네이버에서 새로 받아 옵니다. 판매자센터에서 주소를 바꿨다면 다시 눌러 주세요.

> **"현재 설정 유지 (번호 ○○ — 주소록에 없음)"** 이 보인다면, 예전에 지정한 주소록이 판매자센터에서 지워졌다는 뜻입니다. 그대로 두면 유지되지만 등록이 실패할 수 있으니 목록에서 다시 고르세요.

> **주소록을 불러오지 못했다는 오류가 뜬다면** — 네이버 API 는 허용된 IP 에서만 응답합니다. 사내망·서버 환경 문제일 수 있으니 개발 담당자에게 알려 주세요. 이때도 **이미 저장된 번호는 그대로 유지**됩니다.

## 10. 자주 막히는 상황

**Q. 사이드바에 외부몰 연동이 없습니다.**
이 몰에서 기능이 꺼져 있습니다. 제공자(최고관리자)에게 켜 달라고 요청하세요.

**Q. 상품 가져오기 화면에서 카테고리 목록이 안 뜹니다.**
`카테고리 목록을 불러오지 못했습니다` 가 뜬다면 **도매꾹 자격증명 문제**입니다. 공급처/채널 연결에서 API 키를 확인하고 **[검증]**을 눌러 ACTIVE 로 만드세요.

**Q. 검색 버튼을 눌러도 아무 반응이 없습니다.**
`검색어 또는 카테고리 코드를 입력한 뒤 [검색]을 눌러주세요` 안내를 확인하세요. **대분류만 고른 경우에도** 검색되지 않습니다 — 중분류 이상을 고르거나 검색어를 함께 넣으세요.

**Q. 상품을 가져왔는데 상품 관리에 없습니다.**
정상입니다. 가져오기는 **중간 목록에 담는 것**까지입니다. **가져온 상품** 화면에서 **[우리 몰에 등록]**을 해야 상품이 됩니다.

**Q. 등록했는데 고객 화면에 안 보입니다.**
정상입니다. 등록된 상품은 **판매중지 · 비노출** 상태로 만들어집니다. 상품 관리에서 **판매 상태**와 **노출 설정**을 둘 다 켜세요. 둘 중 하나만 켜면 여전히 보이지 않습니다.

**Q. 같은 상품을 실수로 두 번 가져왔습니다.**
중복으로 생기지 않습니다. 같은 상품을 다시 가져오면 **최신 정보로 갱신**될 뿐입니다.

**Q. 방금 등록한 상품들만 상품 관리에서 모아 보고 싶습니다.**
상품 관리 목록 위 **상품 출처** 필터에서 `도매꾹`(또는 가져온 공급처)을 고르세요. 직접 등록한 `몰상품` 은 빠지고 **그 공급처에서 가져온 상품만** 나옵니다. 전체 선택 → **[판매중 처리]** 로 한 번에 판매를 켤 수 있습니다.

**Q. 어떤 상품이 직접 등록한 것이고 어떤 게 가져온 것인지 헷갈립니다.**
상품 관리 목록에서 **상품명 아래 첫 번째 표시**를 보세요. 회색 `몰상품` 이면 직접 등록한 것, `도매꾹`·`도매매`·`온채널` 처럼 색이 있으면 여기서 가져온 상품입니다. 상세 보기 화면 맨 위에서도 같은 값을 확인할 수 있습니다.

**Q. 등록 버튼을 눌렀는데 카테고리를 고르라는 경고만 나옵니다.**
**우리 몰 카테고리는 필수**입니다. 선택기에서 끝단 분류까지 고르세요.

**Q. 마진율을 비웠더니 경고가 나옵니다.**
몰 **기본 목표 마진율**이 설정되지 않았습니다. 공급처/채널 연결의 연동 사용 설정에서 먼저 정하거나, 이번만 직접 입력하세요.

**Q. 상세 수집 실패(FAILED) 상품을 등록할 수 없습니다.**
정상입니다. **[공급처에서 다시 가져오기]**로 재수집한 뒤 다시 시도하세요.

**Q. 가져온 상품을 삭제하면 공급처 상품도 지워지나요?**
아닙니다. **우리 쪽 중간 목록에서만** 지워집니다.

**Q. 등록한 상품 가격을 나중에 바꿔도 되나요?**
됩니다. 등록 이후에는 **일반 상품과 똑같이** 상품 관리에서 수정합니다. 가져온 상품 목록의 값을 고쳐도 이미 등록된 상품에는 반영되지 않습니다.

**Q. 예전 해외몰(Shopify) 연동과 관련이 있나요?**
없습니다. 이 메뉴는 국내 공급처와 네이버 스마트스토어용이며 설정도 공유하지 않습니다.

**Q. 네이버 등록에서 `등록권한이 있어야만 판매가 가능합니다` 가 뜹니다.**
관리자 화면 문제가 아닙니다. 건강기능식품처럼 **판매 권한이 필요한 분류**입니다. **스마트스토어센터에서 해당 카테고리 판매 권한을 먼저 신청**하세요. 권한이 나오기 전에는 몇 번을 눌러도 같은 오류가 납니다.

**Q. 등록 대상 상품이 하나도 없다고 나옵니다.**
상품에 **네이버 카테고리가 지정되지 않아서**입니다. `카테고리 관리`에서 카테고리에 네이버 분류를 연결하고 [이 카테고리 상품에 바로 반영]을 체크하세요(9-3).

**Q. [카테고리에서 일괄 지정]을 눌렀는데 상당수가 건너뛰어집니다.**
그 상품들의 카테고리가 네이버 **대분류·중분류**에 연결돼 있어서입니다. 네이버는 **끝단 분류만** 받습니다. 카테고리 관리에서 더 깊은 분류로 다시 연결하세요.

**Q. 원산지 검색창에 아무것도 안 나옵니다.**
`네이버 리소스 관리`에서 **원산지 코드**를 아직 수집하지 않은 것입니다. 해당 카드의 [지금 수집]을 누르세요.

**Q. 고시 유형이 상품과 안 맞습니다.**
`네이버 리소스 관리 → 카테고리` 탭에서 해당 분류를 검색해 **고시 유형을 직접 고르고 [저장]** 하세요. 직접 고친 값은 다시 수집해도 유지됩니다.

**Q. 등록은 성공했다는데 스토어에서 안 보입니다.**
**[상태 확인]** 을 눌러 보세요. 네이버가 등록을 받은 뒤 심사에서 판매금지·보류로 바꿨을 수 있습니다.

## 11. 스토어 상품 가져오기 — 네이버 상품을 우리 몰로

`/admin/sourcing/channel-import`

9장(스마트스토어 등록)과 **방향이 반대**입니다. 스마트스토어에 **이미 직접 올려 둔 상품**을 우리 몰로 가져와 한 곳에서 관리하는 화면입니다.

### 11-1. 조회하기

1. [스토어 상품 조회]를 누릅니다. 메뉴에 들어오기만 해서는 조회하지 않습니다(외부 호출이라 그렇습니다).
2. 검색창은 **판매자관리코드 또는 채널상품번호**만 받습니다. 비우고 조회하면 전체가 나옵니다.
3. **상품명으로는 네이버가 검색을 지원하지 않습니다.** 전체를 불러온 뒤, 목록 위의 **[목록에서 이름으로 찾기]** 로 걸러 보세요. 이 입력은 **지금 보고 있는 페이지 안에서만** 걸러 냅니다.

### 11-2. 목록의 "우리 몰 연결" 표시

| 표시 | 뜻 | 선택 |
|---|---|---|
| **새 상품** | 우리 몰에 없습니다 | 가져올 수 있습니다 |
| **우리가 등록함** | 우리 몰 상품을 9장에서 네이버에 올린 것입니다 | 선택 불가 — 가져오면 상품이 두 개가 됩니다 |
| **이미 가져옴** | 전에 가져왔습니다 | 선택 불가 — 상품 번호를 눌러 확인하세요 |

### 11-3. 가져오기

1. **새 상품**을 체크합니다. 한 번에 최대 **20건**입니다.
2. 오른쪽 위에서 세 가지를 정합니다.
   - **카테고리 자동 매핑 실패 시** — 네이버 분류에 연결된 우리 카테고리가 없을 때 넣을 카테고리. 그냥 두면 **미분류**로 들어옵니다.
   - **가져온 뒤 상태 / 노출** — 기본은 **판매중지 · 비노출**(권장)입니다.
3. [선택 상품 가져오기]를 누릅니다. 건당 상세 조회 + 이미지 내려받기가 있어 시간이 걸립니다.

가져오면 이런 것이 만들어집니다.

- 상품 1건(대표 이미지·추가 이미지는 **우리 서버로 내려받아** 저장됩니다)
- 옵션이 있으면 옵션·옵션값·SKU. 네이버에서 **판매 중지된 옵션은 SKU 도 꺼진 상태**로 들어옵니다.
- 가격은 네이버 판매가 기준이며, 옵션 추가금액은 각 SKU 가격에 더해집니다.

> **판매중지·비노출로 들어옵니다.** 배송·고시 정보가 우리 몰 정책과 다를 수 있어서입니다. 상품 관리에서 내용을 확인한 뒤 **판매 상태와 노출을 둘 다** 켜세요.

> **같은 상품을 다시 선택해도 중복으로 생기지 않습니다.** "이미 가져옴"으로 건너뜁니다.

### 11-4. 카테고리는 이렇게 연결됩니다

네이버 분류는 **4단계**(예: `스포츠/레저>등산>등산의류>반팔티셔츠`)까지 있지만, 우리 몰 카테고리는 **3단계**까지입니다. 그래서 네이버 4단계 상품은 **바로 위 3단계 카테고리**(위 예에서는 `등산의류`)에 연결됩니다.

- 3단계 분류의 상품은 같은 이름의 우리 카테고리에 **그대로** 연결됩니다.
- 상품에는 **네이버 분류 원본(4단계)도 함께 저장**됩니다. 나중에 스마트스토어에 다시 올릴 때 이 값을 씁니다. 상품 수정 화면의 `네이버 카테고리` 칸에서 확인할 수 있습니다.
- 더 세분해 관리하고 싶으면 카테고리를 직접 만들어 상품을 옮기면 됩니다. 가져오기가 다시 덮어쓰지 않습니다.

## 12. 재고 연동 — 우리 재고를 스마트스토어로

`/admin/sourcing/sync`

**재고의 기준은 이 몰입니다.** 우리 몰의 **판매가능재고**(판매중이고 재고를 직접 관리하는 SKU 재고의 합)를 스마트스토어로 보냅니다. 네이버 값을 우리 몰에 덮어쓰지 않습니다.

### 12-1. 목록 보기

네이버에 올라가 있는 상품(9장으로 올린 것 + 11장으로 가져온 것)이 모두 나옵니다.

| 칸 | 뜻 |
|---|---|
| **우리 재고** | 지금 우리 몰의 판매가능재고 |
| **마지막 전송** | 최근에 네이버로 보낸 값. `-` 면 아직 보낸 적이 없습니다 |
| **유형** | `우리 등록`(9장) / `역수집`(11장) |

[마지막 전송과 다른 것만] 을 체크하면 보낼 필요가 있는 것만 추립니다.

### 12-2. 전송하기

1. 보낼 상품을 체크합니다(한 번에 최대 **30건**).
2. [선택 상품 재고 전송]을 누르면 확인창이 뜹니다 — **스마트스토어에 실제로 반영되는 작업**입니다.
3. 마지막 전송과 값이 같으면 **보내지 않고 건너뜁니다**(불필요한 호출을 막습니다). 그래도 보내야 하면 [변경이 없어도 강제 전송]을 체크하세요.

> ⚠ **옵션이 있는 상품은 재고와 함께 판매가도 전송됩니다.** 네이버가 옵션 재고를 고칠 때 판매가를 반드시 같이 받도록 되어 있기 때문입니다(옵션 가격이 "판매가 + 추가금액" 구조라서 기준 가격 없이는 계산되지 않습니다).
> 그래서 우리 몰에서 **가격을 바꾼 뒤 재고를 전송하면 네이버 판매가도 함께 바뀝니다.** 가격을 그대로 두고 싶다면 재고를 보내기 전에 우리 몰 가격이 맞는지 확인하세요.
> 옵션이 없는 단일 상품은 재고만 전송되며 가격은 건드리지 않습니다.

### 12-3. 전송 뒤에는 확인하세요

**전송 성공(200)이 곧 반영은 아닙니다.** 각 행의 [네이버 재고 확인]을 누르면 네이버의 **현재 재고**를 조회해 우리 값과 나란히 보여 줍니다. 조회만 하며 아무것도 바꾸지 않습니다.

- **일치** — 정상입니다.
- **불일치** — 다시 전송하거나, 스마트스토어센터에서 상품 상태(판매중지·검수중 등)를 확인하세요.

> **건강기능식품처럼 판매 권한이 필요한 카테고리는 전송 뒤 잠시 있다가 한 번 더 확인하세요.** 옵션이 없는 상품은 재고를 보낼 때 네이버가 상품을 다시 검수할 수 있고, 검수 결과는 바로 나오지 않습니다.

옵션상품은 옵션별 재고까지 함께 보여 줍니다.

## 13. 아직 지원하지 않는 것

- **주문 수집 · 공급처 발주 · 송장 반영** — 네이버에서 들어온 주문을 가져오는 기능은 아직 없습니다.
- **온채널 상품 가져오기** — 수동·CSV 방식이라 API 가져오기를 지원하지 않습니다. 시도하면 `온채널은 API 연동이 아닌 수동·CSV(L1) 방식입니다` 안내가 나옵니다.
- **기본 채널 수수료율 반영** — 저장만 되고 아직 어떤 가격 계산에도 쓰이지 않습니다.
- **재고 자동 동기화(배치)** — 재고 전송은 **버튼을 눌러 그 시점에** 보냅니다. 자동 반복은 아직 없습니다.
- **몰 간 계정 공유** — 계정은 몰마다 따로입니다. **복사만** 가능합니다.
- **한 채널에 여러 계정 골라 쓰기** — 계정 이름별로 저장은 되지만 작업 시 어느 계정을 쓸지 고르는 화면은 없습니다.
- **네이버 브랜드 목록 수집** — 브랜드 카드는 항상 0건입니다. 브랜드 없이도 상품 등록은 정상 동작합니다.
- **한 번에 50건 초과 가져오기** — 나눠서 진행해야 합니다.
- **네이버에 한 번에 30건 초과 등록** — 네이버에 대량 등록 기능이 없어 한 건씩 순차 전송합니다.
- **네이버 상품 수정·삭제** — 지금은 **등록**과 **상태 확인**만 됩니다. 올린 상품을 고치거나 내리는 것은 스마트스토어센터에서 하세요.
- **네이버 판매분 자동 차감** — 네이버에서 팔린 만큼 우리 재고가 저절로 줄지는 않습니다. 우리 재고를 고친 뒤 [재고 전송]을 눌러 맞춥니다.
- **스토어 상품을 이름으로 검색** — 네이버가 상품명 검색을 제공하지 않습니다. 전체를 불러온 뒤 목록에서 걸러 보세요.
- **가져온 상품을 최신 내용으로 다시 받기** — 한 번 가져온 상품은 다시 선택해도 **건너뜁니다**. 가져온 뒤 손본 내용이 지워지지 않게 하려는 것입니다. 네이버 쪽이 바뀌었으면 상품 관리에서 직접 고치세요.
- **상세 설명 안의 이미지** — 대표·추가 이미지는 우리 서버로 내려받지만, **상세 설명 본문의 이미지는 네이버 것을 그대로 불러옵니다.** 네이버에서 해당 상품을 지우면 상세 이미지가 깨질 수 있습니다.
