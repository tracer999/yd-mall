# 쇼핑특가 관리

정해진 **기간·시간·요일**에만 특별 가격으로 파는 행사를 만드는 화면입니다. "오늘의 특가", "저녁 8시 타임특가", "주말 한정 특가" 같은 행사가 여기서 만들어집니다.

특가는 **특가 카테고리**(고객 화면의 탭)와 **특가**(개별 행사) 두 층으로 되어 있습니다. 카테고리를 먼저 만들고, 그 안에 특가를 만들고, 특가에 상품을 담는 순서입니다.

> ## ⚠️ 가장 먼저 읽어야 할 것 — 이 화면은 **실제 결제 금액**을 바꿉니다
>
> 여기서 넣는 **특가**는 화면에 표시만 하는 숫자가 아닙니다. **장바구니·주문서·결제 금액이 전부 이 값으로 계산됩니다.**
>
> - **`10000` 을 `1000` 으로 잘못 넣으면, 그 금액 그대로 팔립니다.**
> - **승인 단계도, 예약 발행도 없습니다.** 조건(기간·시간창·요일)을 만족하는 **그 순간부터 곧바로** 적용됩니다.
> - **되돌리는 버튼이 없습니다.** 이미 팔린 주문은 그 가격으로 남습니다.
>
> 특가를 저장하기 전에 **정가와 특가를 한 번 더 눈으로 확인**하세요. 실수가 걱정되면 **시작일시를 미래로 잡아 두고** 상품·가격을 다 채운 뒤 시각을 앞당기는 방식을 권합니다.

## 2. 전체 흐름 한눈에 보기

```
① 특가 카테고리 만들기        (탭 하나. 한 번 만들면 계속 씁니다)
        ↓
② 특가 등록 — 기본 정보 · 스케줄 · 노출 입력
        ↓
③ [저장]                      ← 여기까지 해야 상품을 담을 수 있습니다
        ↓
④ [상품 추가] 로 상품 담기     (이때 특가 = 정가, 즉 할인 0%)
        ↓
⑤ 줄마다 특가 금액 입력 → [상품 저장]   ← 여기서 진짜 할인이 걸립니다
        ↓
⑥ 목록에서 상태가 「진행중」인지 확인
```

**③번 저장 전에는 상품을 담을 수 없습니다.** 등록 화면에는 `특가를 먼저 저장하면 대상 상품을 담을 수 있습니다.` 라는 안내만 나옵니다.

> **④번에서 담은 직후에는 할인이 0% 입니다.** 담긴 상품의 특가는 **정가와 같은 값**으로 들어갑니다. 일부러 그렇게 되어 있습니다 — 실수해도 "의도 없이 싸게 팔리는" 사고가 나지 않도록 하기 위해서입니다. **⑤번에서 특가를 직접 넣어야** 할인이 시작됩니다.

## 3. 1단계 — 특가 카테고리 만들기

특가 카테고리는 고객 화면 쇼핑특가 페이지의 **탭**이 됩니다. "오늘의 특가", "타임특가", "시즌특가" 같은 것입니다.

<div class="manual-goto-bar">
  <span class="manual-goto-label">이 단계 화면</span>
  <a class="manual-goto" href="/admin/deal-categories" target="_blank">특가 카테고리 목록</a>
  <a class="manual-goto" href="/admin/deal-categories/new" target="_blank">카테고리 등록하기</a>
</div>

| 입력란 | 필수 | 설명 | 예시 |
|---|---|---|---|
| **코드** | 필수 | 내부 구분용 값. **영문 대문자·숫자·언더스코어**만 쓸 수 있고, 다른 글자는 자동으로 `_` 로 바뀝니다. 최대 40자. | `TODAY`, `TIME`, `SEASON` |
| **카테고리명** | 필수 | 고객에게 보이는 탭 이름. 최대 60자. | `오늘의 특가` |
| **설명** | 선택 | 최대 200자. | |
| **스케줄 유형** | 필수 | 아래 표 참고. | |
| **뱃지 문구** | 선택 | 상품에 붙는 짧은 문구. 최대 20자. **비우면 카테고리명이 쓰입니다.** | `타임특가` |
| **뱃지 색상** | 선택 | 뱃지 색을 고릅니다. | `rose` |
| **정렬 순서** | 선택 | 작을수록 앞 탭. | `0` |
| **사용** | 선택 | 체크하면 사용합니다. | |

**스케줄 유형 2가지**

| 값 | 동작 |
|---|---|
| **기간 특가** | 시작~종료 기간 내내 상시 적용됩니다. (오늘의 특가·시즌특가) |
| **타임 특가** | 기간 중 **매일 특정 시간대에만** 적용됩니다. 이 카테고리를 고르면 **특가 등록 폼에서 시간창 입력이 필수**가 됩니다. |

> **코드는 나중에 바꾸지 마세요.** 고객 화면에서 탭을 구분하는 값이라, 바꾸면 그 탭을 가리키던 링크·연결이 어긋날 수 있습니다.

### 3-1. 카테고리 끄기 · 지우기

목록에서 **사용** 을 끄면 그 카테고리에 속한 **특가가 전부 즉시 중단**됩니다. 데이터는 그대로 남습니다.

> **급하게 특가를 전부 내려야 한다면 카테고리의 `사용` 을 끄는 것이 가장 빠르고 안전합니다.** 특가를 하나씩 지울 필요가 없습니다.

**삭제**는 그 카테고리를 쓰는 특가가 하나도 없을 때만 가능합니다. 특가가 있으면 `사용 중인 특가가 N건 있습니다...` 안내가 뜨고 막힙니다.

## 4. 2단계 — 특가 등록하기 (기본 정보 · 스케줄 · 노출)

<div class="manual-goto-bar">
  <span class="manual-goto-label">이 단계 화면</span>
  <a class="manual-goto" href="/admin/deals/new" target="_blank">새 특가 등록 화면 열기</a>

### 4-1. 기본 정보

| 입력란 | 필수 | 설명 | 예시 |
|---|---|---|---|
| **특가 카테고리** | 필수 | 위에서 만든 카테고리를 고릅니다. 옆에 `(타임)` / `(기간)` 이 표시되고, 꺼진 카테고리는 `— 사용 꺼짐` 이 붙습니다. | `오늘의 특가 (기간)` |
| **특가 제목** | 필수 | 최대 100자. | `주말 한정 타임특가` |
| **부제목** | 선택 | 최대 200자. | |

### 4-2. 스케줄 — 세 조건을 **모두** 만족해야 적용됩니다

특가가 실제로 걸리려면 **기간 · 시간창 · 요일**이 전부 맞아야 합니다. 하나라도 어긋나면 정가로 팔립니다.

| 입력란 | 필수 | 설명 |
|---|---|---|
| **시작일시** | 필수 | 특가 캠페인이 시작되는 날짜와 시각. |
| **종료일시** | 필수 | 끝나는 날짜와 시각. **시작일시보다 뒤여야** 합니다. |
| **매일 시작시각** | 조건부 | 매일 몇 시부터 적용할지. |
| **매일 종료시각** | 조건부 | 매일 몇 시까지 적용할지. |
| **적용 요일** | 선택 | 월~일 체크박스. |

**시간창(매일 시작·종료시각) 규칙**

- **둘 다 비우면** 기간 내내 **상시** 적용입니다.
- **한쪽만 채울 수 없습니다.** 저장할 때 `시간창은 시작시각과 종료시각을 둘 다 입력하거나 둘 다 비워야 합니다.` 로 막힙니다.
- **종료시각이 시작시각보다 뒤여야** 합니다.
- 카테고리가 **타임 특가**면 시간창이 **필수**입니다. 비우면 `"카테고리명" 은(는) 타임 특가 카테고리입니다. 시간창(시작·종료 시각)을 입력하세요.` 로 막힙니다.

> **자정을 넘는 시간창(예: 22:00 ~ 02:00)은 지원하지 않습니다.** 심야 특가를 하려면 `22:00 ~ 23:59` 처럼 자정 앞에서 끊어야 합니다.

**적용 요일 규칙**

- **전부 체크하거나 전부 해제하면 = 매일** 적용입니다. 두 경우 결과가 같습니다.
- 일부만 체크하면 그 요일에만 적용됩니다.

### 4-3. 노출

| 입력란 | 설명 |
|---|---|
| **우선순위** | 한 상품에 특가가 여러 개 겹치면 **숫자가 큰 쪽이 이깁니다.** |
| **정렬 순서** | 목록에서의 순서. 작을수록 앞. |
| **사용** | 체크해야 특가가 동작합니다. 끄면 즉시 중단됩니다. |

> **한 상품에 특가가 겹치면 하나만 적용됩니다.** 우선순위가 큰 것 → 그다음은 더 싼 것 → 그다음은 먼저 만든 것 순으로 이깁니다. 여러 특가에 같은 상품을 넣었다면 어떤 가격으로 팔릴지 반드시 확인하세요.

입력을 마쳤으면 **[저장]** 을 누릅니다. 저장하면 수정 화면으로 넘어가고, 이제 상품을 담을 수 있습니다.

## 5. 3단계 — 상품 담고 특가 넣기 (가장 조심할 단계)

### 5-1. 상품 담기

오른쪽 위 **[상품 추가]** 를 누르면 **상품 조회** 창이 열립니다.

| 필터 | 설명 |
|---|---|
| **검색어** | 상품명·상품코드 |
| **카테고리** | 상품 카테고리 |
| **브랜드** | 브랜드 |
| **재고** | `전체` / `재고 있음` / `재고 없음` |
| **노출** | `전체` / `노출` / `숨김` / `회원전용` |

**[검색]** 을 누르고, 담을 상품을 체크한 뒤 **[선택 담기]** 를 누릅니다. 이미 담긴 상품은 검색 결과에 나오지 않습니다.

> **담긴 직후 특가는 정가와 같습니다(할인 0%).** 창 아래에도 `담긴 상품의 특가는 정가로 시작합니다` 라고 안내됩니다. **아직 아무 할인도 걸리지 않은 상태**이며, 목록에서 그 줄은 노란색 배경으로 표시됩니다.

### 5-2. 특가 금액 넣기

**특가 상품** 표에서 줄마다 값을 넣습니다.

| 칸 | 설명 |
|---|---|
| **정가** | 상품의 현재 판매가 (읽기 전용) |
| **특가 (원)** | **실제 결제 금액.** 여기에 넣는 값이 고객이 내는 돈입니다. |
| **할인율** | 자동 계산되어 표시됩니다. |
| **한정 수량** | 이 특가로 팔 최대 개수. **비우면 무제한**입니다. |
| **판매** | 지금까지 특가로 팔린 수량. 한정 수량이 있으면 `판매수 / 한정수량` 으로 보이고, 다 팔리면 `소진` 이 붙습니다. |
| **순서 / 제거** | ↑ ↓ 로 순서를 바꾸고, **제거**로 특가에서 뺍니다. |

값을 다 넣었으면 **[상품 저장]** 을 누릅니다.

> **[상품 저장]을 누르지 않으면 아무것도 반영되지 않습니다.** 특가 금액을 넣고 화면을 벗어나면 그대로 사라지고, 상품은 **여전히 정가**로 팔립니다.

### 5-3. 저장이 거부되는 경우

**한 줄이라도 아래 조건에 걸리면 전체가 저장되지 않습니다.** 일부만 저장되는 일은 없습니다.

| 상황 | 안내 메시지 |
|---|---|
| 특가가 비어 있거나 0 이하 | `"상품명" 의 특가를 1원 이상으로 입력하세요.` |
| 특가가 정가와 같거나 더 비쌈 | `"상품명" 의 특가(...원)가 정가(...원) 이상입니다. 정가보다 싸게 입력하세요.` |
| 한정 수량이 0 이하 | `"상품명" 의 한정 수량은 1 이상이거나 비워 두어야 합니다(비우면 무제한).` |

> **정가는 저장할 때 상품 정보에서 다시 읽어 검증합니다.** 화면에 표시된 정가만 믿지 않습니다. 그래서 화면을 열어 둔 사이 상품 가격이 바뀌었다면 저장이 거부될 수 있습니다. 그때는 화면을 새로고침하고 다시 넣으세요.

> **특가 저장 뒤에 상품의 판매가를 특가보다 낮게 내리면, 그 특가는 자동으로 무시됩니다.** 가격이 올라가는 일은 절대 없도록 되어 있습니다. 다만 그 순간부터 **특가가 아니라 새 판매가로 팔립니다.** 상품 가격을 내릴 때는 걸려 있는 특가가 있는지 함께 확인하세요.

### 5-4. 상품 빼기

각 줄의 **제거**를 누르면 확인 창이 뜨고 특가에서 빠집니다. **상품 자체는 삭제되지 않습니다.**

| **시간창** | 매일 적용 시간. 없으면 `상시` |
| **요일** | 적용 요일. 전부면 `매일` |
| **상품 / 판매** | 담긴 상품 수와 누적 판매 수 |
| **우선순위** | 겹칠 때 이기는 기준 |

> **상태 뱃지는 "기간" 기준일 뿐입니다.** `진행중` 이라고 해서 지금 이 순간 특가가 걸려 있다는 뜻은 아닙니다. 타임특가라면 **시간창·요일을 벗어난 시각에는 정가로 팔립니다.** 뱃지는 캠페인의 생애주기만 말합니다.

> **`진행중` 아래에 `카테고리 OFF` 가 보이면 실제로는 적용되지 않고 있습니다.** 소속 카테고리의 사용이 꺼져 있다는 뜻입니다.

### 6-1. 특가 삭제

목록의 **삭제**를 누르면 확인 창이 뜹니다.

> **특가를 삭제하면 담긴 상품 목록도 함께 지워집니다.** 되돌릴 수 없습니다. 잠시 멈추려는 것이라면 수정 화면에서 **사용** 체크를 끄세요.

## 7. 특가를 급히 멈추는 3가지 방법

가격을 잘못 넣었을 때 **가장 빨리 멈추는 순서**입니다.

| 방법 | 범위 | 어디서 |
|---|---|---|
| **특가의 `사용` 끄기** | 이 특가 하나만 즉시 중단 | 특가 수정 화면 → 노출 → 사용 |
| **카테고리의 `사용` 끄기** | 그 카테고리의 **모든 특가**가 즉시 중단 | 특가 카테고리 목록 |
| **문제 상품만 제거** | 그 상품만 정가로 복귀 | 특가 수정 화면 → 특가 상품 → 제거 |

## 8. 자주 막히는 상황

**Q. 특가를 만들었는데 고객 화면 가격이 그대로입니다.**
아래를 순서대로 확인하세요.
1. **[상품 저장]** 을 눌렀는가? (담기만 하면 특가 = 정가라 할인이 0% 입니다)
2. 특가의 **사용**이 켜져 있는가?
3. 소속 **카테고리의 사용**이 켜져 있는가? (목록에 `카테고리 OFF` 표시)
4. 지금이 **기간 · 시간창 · 요일**을 모두 만족하는가?
5. 그 상품의 **한정 수량이 소진**되지 않았는가?
6. **편집 중인 몰**이 맞는가?

**Q. 저장을 눌렀는데 "특가가 정가 이상입니다" 라고 나옵니다.**
특가가 정가와 같거나 더 비쌉니다. **정가보다 싼 금액**을 넣어야 합니다. 담은 직후에는 특가가 정가와 같으므로 반드시 값을 고쳐야 저장됩니다.

**Q. 시간창을 넣었는데 저장이 안 됩니다.**
시작시각과 종료시각을 **둘 다** 넣었는지, 종료시각이 시작시각보다 **뒤인지** 확인하세요. 자정을 넘는 시간대는 지원하지 않습니다.

**Q. 상품 조회 창에서 담고 싶은 상품이 안 나옵니다.**
이미 이 특가에 담겨 있거나, **다른 몰의 상품**입니다. 이미 담긴 상품은 검색 결과에서 제외됩니다.

**Q. 한정 수량을 다 팔았는데 특가가 아직 목록에 있습니다.**
정상입니다. **그 상품만** 특가에서 빠지고 정가로 돌아갑니다. 특가 자체는 다른 상품에 계속 적용됩니다.

**Q. 같은 상품에 특가를 두 개 걸었습니다. 어느 쪽이 적용되나요?**
**우선순위가 큰 것**이 이깁니다. 우선순위가 같으면 **더 싼 것**, 그것도 같으면 **먼저 만든 것**이 이깁니다.

**Q. 기획전에 적어 둔 할인율과 실제 가격이 다릅니다.**
그 상품에 쇼핑특가가 걸려 있으면 기획전 화면에도 특가 금액이 보입니다. 기획전 배너 문구를 쓰기 전에 진행 중인 특가를 확인하세요.

**Q. 이미 잘못된 가격으로 주문이 들어왔습니다.**
특가를 끄더라도 **이미 결제된 주문은 그대로 남습니다.** 주문 관리에서 개별로 처리해야 합니다.

**Q. 특가 카테고리를 지우려는데 안 됩니다.**
그 카테고리를 쓰는 특가가 남아 있습니다. 특가를 다른 카테고리로 옮기거나 지운 뒤 삭제하세요. 노출만 멈추면 되는 상황이라면 **사용**을 끄는 것으로 충분합니다.

## 9. 아직 지원하지 않는 것

- 자정을 넘는 시간창 (예: 22:00 ~ 02:00)
- 정가 대비 **비율(%)** 로 특가를 일괄 지정하는 기능 (금액을 직접 넣어야 합니다)
- 특가 승인·검토 단계 (저장하면 조건 충족 즉시 적용됩니다)
- 엑셀/CSV 로 특가 상품·가격 일괄 등록
- 특가별 매출 집계 리포트
