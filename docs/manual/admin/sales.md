# 판매(주문) 관리

고객이 결제한 **주문을 확인하고 상태를 바꾸는 화면**입니다. 무엇이 몇 개 팔렸는지, 얼마가 결제됐는지, 쿠폰·포인트가 얼마나 쓰였는지, 배송이 어디까지 갔는지를 이 한 곳에서 봅니다. 주문 한 건 한 건의 **변경 이력**도 여기에 남습니다.

## 1. 주문 목록 보기

목록에는 **모든 몰의 주문이 함께** 나옵니다. 주문은 몰별로 나눠 관리하지 않고 통합해서 봅니다. 어느 몰에서 들어온 주문인지는 각 줄의 **몰 배지**로 구분합니다.

| 열 | 내용 |
|---|---|
| **주문번호** | 클릭하면 주문 상세로 갑니다. |
| **몰** | 고객이 결제한 몰 이름. 기본몰이면 `· 기본` 이 붙습니다. 확인이 안 되는 과거 주문은 `미확인`. |
| **주문일시** | 주문이 들어온 날짜와 시각. **최신 주문이 맨 위**입니다. |
| **주문자** | 회원 이름(비회원이면 `비회원`)과 가입 경로 배지(Google·Kakao). 아래 작은 글씨는 **받는 분 이름과 연락처**입니다. |
| **결제금액** | 고객이 실제로 결제한 최종 금액. |
| **상태** | 아래 2장의 7가지 중 하나. |
| **관리** | `상세보기` 버튼. |

### 목록 좁히기

- **몰 셀렉트** — 상단 `몰` 선택에서 특정 몰을 고르면 그 몰의 주문만 봅니다. 기본값은 `전체 몰`이고, 고르는 즉시 새로고침됩니다.
- **상태 필터** — 상태로 걸러 볼 수 있습니다. 필터가 걸리면 상단에 파란 배지가 뜨고, 옆의 `×` 를 누르면 해제됩니다.

두 필터는 **함께** 걸립니다. (예: 건강식품관의 배송준비 주문만 보기)

> **주문은 관리자에서 만들 수 없습니다.** 목록에 있는 주문은 전부 고객이 실제로 결제한 것입니다. 테스트가 필요하면 고객 화면에서 직접 주문해 보세요.

## 2. 주문 상태 7가지

| 상태 | 뜻 | 이 상태에서 하는 일 |
|---|---|---|
| **대기** | 결제창은 떴지만 **결제가 확정되지 않은** 주문입니다. 재고도 아직 빠지지 않았습니다. | 그대로 둡니다. 고객이 결제를 포기한 주문입니다. |
| **결제완료** | 돈이 들어왔습니다. 재고가 차감된 상태입니다. | 상품을 준비합니다. |
| **배송준비** | 포장·출고 준비 중입니다. | 송장이 나오면 배송 관리에서 등록합니다. |
| **배송중** | 송장이 등록되어 택배가 출발했습니다. | 배송이 끝나면 배송완료로 바꿉니다. |
| **배송완료** | 고객이 물건을 받았습니다. | **이 시점부터 고객이 반품을 신청할 수 있습니다**(수령 후 7일). |
| **취소** | 취소된 주문입니다. 재고·쿠폰·포인트가 복원된 상태입니다. | 더 할 일이 없습니다. |
| **환불** | 환불까지 마무리된 주문입니다. | 더 할 일이 없습니다. |

> **`대기`를 손으로 `결제완료`로 올리지 마세요.** `대기`는 돈이 안 들어온 주문입니다. 상태만 바꾼다고 돈이 들어오지 않습니다. 매출 집계와 재고가 함께 어긋납니다.

> 📧 **고객이 결제를 마치는 순간 주문 완료 안내 메일이 자동으로 나갑니다.** 관리자가 따로 보낼 필요는 없습니다. 다만 **여기서 상태를 손으로 바꿔도 메일은 나가지 않습니다** — 메일은 고객의 실제 결제, 배송 관리의 송장 등록·배송완료 같은 **실제 처리 시점**에만 발송됩니다. 문구 변경·발송 중지는 [이메일 템플릿 관리](/manual/admin/email_templates)에서 합니다.

## 3. 주문 상세 화면

목록에서 주문번호나 `상세보기`를 누르면 열립니다.

### 3-1. 주문 상품과 금액

상품별로 **상품명 / 수량 / 가격 / 합계**가 나오고, 그 아래에 금액이 단계별로 쌓입니다.

| 줄 | 뜻 |
|---|---|
| **상품 금액 합계** | 할인 전 상품값 총액입니다. |
| **쿠폰 할인** | 고객이 쓴 주문 쿠폰의 할인액. 괄호 안에 쿠폰 이름이 나옵니다. |
| **포인트 사용** | 고객이 쓴 적립금입니다. |
| **배송비** | 붙은 배송비. 0원이면 `무료`로 표시됩니다. |
| **배송비 할인** | 배송비 쿠폰을 썼을 때만 나옵니다. |
| **최종 결제 금액** | 고객이 실제로 낸 돈입니다. |

상품명 옆에 `(삭제된 상품)`이 붙어 있다면, 그 상품이 상품 관리에서 지워진 것입니다. 주문 기록 자체는 그대로 남습니다.

### 3-2. 배송 정보

받는 분 · 연락처 · 주소 · 배송 메시지가 나오고, **송장이 등록돼 있으면** 택배사 · 운송장 번호 · 배송 시작일이 함께 표시됩니다. 오른쪽 위에는 배송 상태(`준비` / `배송중` / `배송완료`)가 배지로 붙습니다.

아직 등록 전이면 "아직 배송이 시작되지 않았습니다"라는 안내와 **배송 관리 바로가기** 링크가 나옵니다.

### 3-3. 주문자 정보 · 시스템 정보

- **주문자 정보** — 이름·이메일·가입 경로. 회원 주문이면 `회원 정보 보기` 링크로 회원 상세로 넘어갑니다.
- **시스템 정보** — 주문 번호, 주문 일시, 결제 수단, 결제 일시. 그 아래에 **상태가 네 갈래로 따로** 표시됩니다.

| 항목 | 나올 수 있는 값 |
|---|---|
| **결제 상태** | 결제대기 / 결제완료 / 결제취소 / 환불완료 / 부분환불 |
| **클레임** | 접수 / 승인 / 완료 / 거절 — 클레임이 있을 때만 표시 |
| **환불** | 요청 / 완료 / 실패 — 환불이 있었을 때만 표시 |

> **주문 상태와 결제 상태는 별개입니다.** "배송준비 + 결제완료"처럼 두 값이 동시에 성립합니다. 특히 **환불이 `실패`로 남아 있으면 돈이 아직 고객에게 돌아가지 않은 것**입니다. 반드시 클레임 관리에서 수동으로 마감하세요.

### 3-4. 클레임 · 주문 변경 이력

- **클레임** — 이 주문에 취소·반품 신청이 있으면 목록으로 나옵니다. 유형(취소·반품·교환)과 사유(단순변심·불량·오배송·기타), 신청일이 보이고 `보기`로 클레임 상세로 갑니다.
- **주문 변경 이력** — 어떤 값이 무엇에서 무엇으로 바뀌었는지, 누가(고객·관리자·시스템) 언제 바꿨는지가 전부 남습니다. **"이 주문이 왜 취소됐지?"를 확인하는 가장 확실한 자리입니다.**

## 4. 주문 상태 바꾸기

상세 화면 오른쪽 위의 셀렉트에서 상태를 고르고 **`상태 변경`** 버튼을 누릅니다.

셀렉트에 나오는 값은 **6가지**입니다.

```
대기 · 결제완료 · 배송준비 · 배송중 · 배송완료 · 취소
```

`환불`은 셀렉트에 없습니다. 환불은 클레임 관리의 승인 처리로 진행됩니다.

### 4-1. 상태 전이 표 — 어디서 어디로 갈 수 있나

시스템은 위 6개 값 사이의 이동을 **기술적으로 막지 않습니다.** 어디서 어디로든 바뀝니다. 하지만 **업무적으로 올바른 경로는 정해져 있습니다.** 아래 표를 기준으로 움직이세요.

| 현재 상태 | 정상 경로 | 하면 안 되는 것 |
|---|---|---|
| **대기** | → 취소 | **→ 결제완료 / 배송준비 / 배송중** — 돈이 안 들어온 주문을 팔린 것으로 만듭니다 |
| **결제완료** | → 배송준비 / → 취소 | → 대기 (되돌리기가 아니라 기록만 꼬입니다) |
| **배송준비** | → 배송중 *(배송 관리에서 자동)* / → 취소 | → 대기 |
| **배송중** | → 배송완료 *(배송 관리에서 자동)* | **→ 취소** — 물건은 이미 나갔습니다. **반품 클레임으로 처리하세요** |
| **배송완료** | (정상 종료) | **→ 취소** — 반품 클레임으로 처리하세요 |
| **취소** | (되돌릴 수 없음) | 다른 상태로 되돌리기 — 돈은 이미 환불됐고 재고는 이미 복구됐습니다 |
| **환불** | (되돌릴 수 없음) | 위와 같음 |

**`배송중`·`배송완료`는 이 화면이 아니라 배송 관리에서 자동으로 바뀌는 것이 정상입니다.** 송장을 등록하면 `배송중`, 배송완료 버튼을 누르면 `배송완료`가 됩니다. 여기서 손으로 바꾸면 **송장 정보가 없는 채 상태만 배송중이 되어** 고객이 배송조회를 할 수 없습니다.

### 4-2. `취소`로 바꾸면 실제로 무슨 일이 일어나나

> ⚠️ **되돌릴 수 없는 작업입니다.** 확인 창이 뜨지 않고 버튼을 누른 즉시 실행됩니다.

`취소`로 바꾸는 순간 아래가 한꺼번에 진행됩니다.

1. **재고 복원** — 팔린 수량이 상품 재고로 되돌아갑니다. (`대기` 주문은 차감 전이라 복원할 것이 없습니다.)
2. **쿠폰 복원** — 이 주문에 쓴 쿠폰이 다시 사용 가능해집니다.
3. **포인트 복원** — 고객이 쓴 적립금은 돌려주고, 이 주문으로 지급했던 구매적립은 회수합니다. (이미 써 버린 적립금은 잔액이 음수가 되지 않는 만큼만 회수합니다.)
4. **특가 수량 복원** — 선착순 특가로 소진된 수량이 되돌아갑니다.
5. **카드 결제 취소** — 결제사(토스)로 취소 요청이 나갑니다. **실제로 고객에게 돈이 돌아갑니다.**
6. **상태 기록** — 결제 상태·클레임 상태·환불 상태가 함께 갱신되고 변경 이력에 남습니다.

**결제 내역이 없는 주문**(테스트 결제, 전액 포인트·쿠폰으로 0원 결제 등)은 5번이 생략되고 나머지만 복원됩니다.

같은 주문을 두 번 취소해도 **재고가 두 번 늘어나지는 않습니다.** 복원은 한 번만 실행되도록 잠금이 걸려 있습니다.

> **취소는 원칙적으로 클레임 관리에서 하세요.** 이 화면의 상태 변경은 **사유도, 귀책도, 반품 배송비도 남기지 않습니다.** 고객이 요청한 취소라면 클레임으로 처리해야 나중에 "왜 취소됐는지"를 설명할 수 있습니다.

## 5. 자주 막히는 상황

**Q. 결제는 됐다는데 목록에 `대기`로 떠 있습니다.**
결제 승인이 중간에 끊긴 주문입니다. 손으로 `결제완료`로 올리지 마세요. 실제로 돈이 들어왔는지 결제사 관리자에서 먼저 확인하고, 들어왔다면 환불 후 재주문을 안내하는 편이 안전합니다.

**Q. 상태를 `취소`로 바꿨는데 고객이 "환불이 안 됐다"고 합니다.**
주문 상세의 **시스템 정보 → 환불** 항목을 보세요. `실패`라면 결제사 취소 요청이 실패한 것입니다. 상태만 취소가 되고 돈은 그대로입니다. 계좌로 직접 환불한 뒤 클레임 관리 상세에서 **수동 완료**로 마감하세요.

**Q. 상태를 `배송중`으로 바꿨는데 고객 화면에 운송장이 안 보입니다.**
이 화면에서 손으로 바꾸면 송장 정보가 만들어지지 않습니다. 배송 관리에서 택배사와 운송장 번호를 등록하세요. 등록하면 상태도 자동으로 `배송중`이 됩니다.

**Q. 고객이 취소를 요청했는데 이미 `배송중`입니다.**
취소가 아니라 **반품**입니다. 억지로 `취소`로 바꾸면 물건은 고객 손에 있는데 환불만 나갑니다. 배송완료 처리 후 클레임(반품)으로 진행하세요.

**Q. 결제금액과 상품 금액이 다릅니다.**
쿠폰 할인·포인트 사용·배송비가 반영된 결과입니다. 주문 상세의 금액 표를 위에서 아래로 따라 읽으면 계산이 맞습니다.

**Q. 취소한 주문을 되살릴 수 있나요?**
없습니다. 결제가 이미 취소됐으므로 고객이 새로 주문해야 합니다.

**Q. 특정 몰의 주문만 보고 싶습니다.**
상단 `몰` 셀렉트에서 그 몰을 고르세요. 상태 필터와 함께 걸 수 있습니다.

---

## 6. 주문 찾기

고객이 전화로 "제 주문 어떻게 됐나요"라고 물으면 **목록 위 검색창**에 아는 것을 아무거나 넣으세요.
한 칸에서 아래를 모두 훑습니다.

- 주문번호
- 주문자 이름 · 받는 분 이름
- 연락처 (주문자 · 받는 분) — **뒷자리만 넣어도 찾습니다**
- 이메일

몰·상태 필터와 함께 걸 수 있고, 검색 결과가 많으면 **50건씩 나눠** 보여 줍니다.
검색을 풀려면 검색어 옆 `×` 를 누르세요.

---

## 7. 주문 내려받기 (엑셀)

목록 위 **[엑셀 내려받기]** 를 누르면 **지금 화면에 걸린 조건 그대로** 파일이 만들어집니다.
검색어·몰·상태를 걸어 두고 눌렀다면 그 결과만 담깁니다 — 화면과 파일이 어긋나지 않습니다.

담기는 값: 주문번호 · 주문일시 · 몰 · 주문/결제상태 · 주문자 · 받는 분 · 주소 · 상품 요약 · 금액 내역(상품·배송비·할인·포인트·결제) · 결제수단 · **택배사 · 송장번호 · 발송/배송완료 일시**

> **엑셀에서 바로 열립니다.** 한글이 깨지지 않도록 저장되며, 송장번호·전화번호처럼 긴 숫자가
> `1.23457E+11` 같은 지수 표기로 뭉개지지 않습니다.

> 택배사 접수 파일을 만들 때는 이 파일을 받아 필요한 열만 남겨 쓰세요.

---

## 8. 주문서 · 거래명세서 출력

주문 상세 오른쪽 위의 **[주문서]** · **[거래명세서]** 를 누르면 인쇄용 화면이 새 탭으로 열립니다.
화면의 **[인쇄 / PDF 저장]** 버튼으로 종이에 뽑거나 PDF 로 저장합니다.

| 문서 | 언제 쓰나 | 담기는 내용 |
|------|-----------|-------------|
| **주문서** | 상품과 함께 동봉 | 주문 정보 · 받는 분 · 품목 · 금액 |
| **거래명세서** | 사업자 고객이 요구할 때 | 위 내용 + **공급자 정보 · 공급가액 · 부가세** |

- 발행처(상호·사업자등록번호·주소·연락처)는 **그 주문이 속한 몰의 사이트 설정**에서 자동으로 가져옵니다. 따로 입력하지 않습니다.
- 사업자등록번호가 비어 있으면 그 자리가 `-` 로 나옵니다. [사이트 설정](/admin/settings)에서 채우세요.
- 공급가액·부가세는 주문에 기록된 값을 쓰고, 없으면 결제금액에서 역산(÷1.1)해 보여 줍니다.

## 9. 정산 · 매출 리포트

목록 위 **[정산 리포트]** 또는 `/admin/sales/settlement` 로 들어갑니다.
기간·몰·구분(일반/기업)을 골라 조회합니다. 기본은 **이번 달 1일 ~ 오늘**입니다.

<div class="manual-goto-bar">
  <a class="manual-goto is-sub" href="/admin/sales/settlement" target="_blank">정산 리포트 열기</a>
</div>

**위쪽 요약 카드 4개**

| 카드 | 뜻 |
|------|-----|
| **매출** | 결제가 살아 있는 주문의 결제금액 합계 · 주문 수 · 객단가 |
| **환불** | 이 기간에 **실제로 나간** 환불액 · 건수 · 반품비 회수액 |
| **순매출** | 매출 − 환불. **정산의 기준 숫자입니다** |
| **배송비 수취** | 청구한 배송비 − 배송비 할인 |

**아래쪽**은 할인·포인트 구성, 결제수단별 금액, **일자별 표**(매출/환불/순매출), **상품별 매출 상위 50**입니다.
**[엑셀 내려받기]** 는 일자별 표를 그대로 내려받습니다.

> ### 집계 기준 — 이걸 알아야 숫자를 읽을 수 있습니다
> - **매출로 세는 주문**은 `결제완료 · 배송준비 · 배송중 · 배송완료` 입니다. **대기(미결제)·취소·환불은 제외**합니다.
> - 기준 시각은 주문일이 아니라 **결제일**입니다. 어제 주문하고 오늘 결제됐다면 오늘 매출입니다.
> - **환불은 실제로 환불이 끝난 날**에 잡습니다. 지난달 주문이 이번 달 환불되면 **이번 달 순매출에서** 빠집니다.
> - 그래서 어떤 달은 순매출이 매출보다 작을 수 있습니다. 고장이 아니라 정산의 실제 모습입니다.
> - 부분 환불도 그대로 반영됩니다(주문은 살아 있고 일부 금액만 빠짐).

## 10. 아직 지원하지 않는 것

- 관리자가 직접 주문 생성(대리 주문)
- 주문 목록에서 여러 건을 골라 한 번에 상태 바꾸기
- 세금계산서 발행 · 전자계산서 연동
- 정산 리포트의 상품별 표 내려받기 (일자별 표만 내려받습니다)
