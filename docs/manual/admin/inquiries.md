# 문의 · 고객센터 관리

고객이 보낸 **1:1 문의에 답변**하고, 고객센터 화면에 뜨는 **자주 묻는 질문(FAQ)** 을 관리하는 곳입니다. 두 화면은 목적이 다릅니다. 문의 관리는 **이미 들어온 질문 하나하나에 답하는 곳**이고, 고객센터 관리는 **미리 답을 적어 두어 문의 자체를 줄이는 곳**입니다.

> **문의 관리와 고객센터 관리는 몰을 다루는 방식이 다릅니다.** 문의는 **모든 몰의 문의가 한 목록에 나오고 필요할 때 몰로 걸러 보는** 방식이고, FAQ는 **지금 편집 중인 몰에만 저장**됩니다. 이 차이를 모르면 "FAQ를 만들었는데 안 보인다"는 상황이 생깁니다.

## 1. 두 화면의 역할 구분

| | 문의 관리 | 고객센터 관리(FAQ) |
|---|---|---|
| 무엇을 다루나 | 고객이 직접 보낸 질문 | 운영자가 미리 적어 둔 질문·답변 |
| 누가 먼저 시작하나 | 고객 | 운영자 |
| 고객이 보는 곳 | 자기 문의 내역 | 고객센터 화면 |
| 몰 구분 | **전체 몰이 한 목록** (몰 필터로 골라 봄) | **편집 중인 몰에만** 저장 |
| 등록·삭제 | 운영자가 새로 만들 수 없음 · 삭제 없음 | 등록·수정·삭제 모두 가능 |

**같은 문의가 반복해서 들어온다면 그 내용을 FAQ로 옮기세요.** 매번 답하는 일이 줄어듭니다.

### 2-1. 목록에서 무엇을 보나

목록에는 **최신 문의가 맨 위**에 오도록 정렬되어 나옵니다.

| 칸 | 내용 |
|---|---|
| **번호** | 문의 고유 번호입니다. 고객과 통화할 때 이 번호로 특정하면 편합니다. |
| **제목** | 고객이 적은 문의 제목입니다. |
| **몰** | 이 문의가 **어느 몰에서 들어왔는지**입니다. 고객이 문의를 남긴 시점의 몰이 기록됩니다. |
| **작성자** | 회원 이름과 이메일이 함께 나옵니다. 문의는 **회원만** 남길 수 있습니다. |
| **상태** | `답변대기`(노란색) 또는 `답변완료`(초록색) |
| **작성일** | 문의가 들어온 날짜 |
| **관리** | `상세보기` 를 누르면 답변 화면으로 들어갑니다. |

### 2-2. 몰로 걸러 보기

목록 오른쪽 위의 **몰** 선택 상자에서 몰을 고르면 그 몰의 문의만 보입니다. 기본값은 **`전체 몰`** 이라 아무것도 고르지 않으면 모든 몰의 문의가 함께 나옵니다.

> **문의 관리는 다른 화면들과 달리 "편집 중인 몰"을 따라가지 않습니다.** 상단의 편집 몰을 A몰로 바꿔도 문의 목록은 그대로 전체가 나옵니다. 담당 몰이 정해져 있다면 **목록에 들어올 때마다 몰 필터를 직접 걸어야** 합니다. 이 선택은 저장되지 않습니다.

### 2-3. 답변 작성

`상세보기` 를 누르면 위쪽에 고객이 보낸 제목·작성자·몰·내용이 나오고, 아래쪽에 **답변 작성** 영역이 나옵니다.

| 입력란 | 필수 | 설명 |
|---|---|---|
| **답변 내용** | 필수 | 고객에게 보여 줄 답변입니다. 여러 줄로 쓸 수 있고 줄바꿈이 그대로 유지됩니다. 비워 두고는 등록할 수 없습니다. |

버튼 문구는 상황에 따라 달라집니다.

- 아직 답변이 없으면 → **답변 등록**
- 이미 답변한 문의를 다시 열면 → **답변 수정** (기존 답변이 미리 채워져 나옵니다)

**답변을 등록하면 그 문의의 상태가 곧바로 `답변완료`로 바뀝니다.** 답변을 지우고 다시 `답변대기`로 되돌리는 기능은 없습니다. 나중에 보완하려면 답변 내용을 고쳐 다시 저장하세요.

> **답변을 저장해도 고객에게 문자·이메일이 자동으로 나가지 않습니다.** 고객은 사이트에 다시 들어와 자기 문의 내역을 확인해야 답변을 봅니다. 급한 건은 별도로 연락하세요.

### 2-4. 답변 작성 요령

- **답변에는 서식(굵게·색·이미지)을 넣을 수 없습니다.** 글자만 들어가므로 문단을 나누어 읽기 쉽게 쓰는 것이 최선입니다.
- 링크를 안내해야 한다면 주소를 그대로 적어 주세요.
- 개인정보(주민번호·카드번호 등)는 답변에 남기지 마세요. 문의 내역은 지워지지 않고 계속 보관됩니다.
- 배송·환불처럼 주문과 얽힌 문의는 답변 전에 실제 주문 상태를 먼저 확인하세요.

## 3. 고객센터(FAQ) 관리하기

<div class="manual-goto-bar">
  <span class="manual-goto-label">이 단계 화면</span>
  <a class="manual-goto" href="/admin/faqs" target="_blank">고객센터 관리 열기</a>
  <a class="manual-goto" href="/admin/faqs/new" target="_blank">FAQ 등록 화면 열기</a>
</div>

여기서 등록한 질문·답변이 고객 화면의 **고객센터**에 그대로 나옵니다.

### 3-1. 목록 화면 읽기

목록 위쪽에는 **분류 버튼 줄**이 있습니다. `전체` 를 포함해 분류별로 걸러 볼 수 있습니다.

| 칸 | 내용 |
|---|---|
| **ID** | FAQ 고유 번호 |
| **분류** | 지정한 분류 이름. 지정하지 않았으면 `미분류` |
| **질문** | 누르면 수정 화면으로 갑니다. |
| **순서** | 숫자가 작을수록 위에 나옵니다. |
| **조회** | 고객이 이 FAQ를 몇 번 열어 봤는지입니다. |
| **상태** | `BEST` 배지 · `ON`(노출) / `OFF`(숨김) |
| **관리** | `수정` · `삭제` |

**노출을 끈(OFF) 줄은 회색으로 흐리게 표시**되므로 한눈에 구분됩니다.

### 3-2. FAQ 등록·수정 입력란

오른쪽 위 **FAQ 등록** 버튼을 누르면 등록 화면이 열립니다.

| 입력란 | 필수 | 설명 | 예시 |
|---|---|---|---|
| **분류** | 선택 | 고객센터에서 묶어 보여 줄 분류입니다. 고르지 않으면 `미분류`로 들어갑니다. | `배송` |
| **노출 순서** | 선택 | 0~9999 사이 숫자. **작을수록 위**입니다. 기본값은 0. | `10` |
| **질문** | 필수 | 고객에게 보이는 질문 문장입니다. 최대 255자. | `배송은 며칠 걸리나요?` |
| **답변** | 필수 | 서식 편집기로 작성합니다. 굵게·목록·링크·표를 넣을 수 있습니다. | 여러 문단 가능 |
| **노출** | 체크박스 | 켜야 고객센터에 나옵니다. | |
| **자주 묻는 질문(BEST) 에 노출** | 체크박스 | 켜면 고객센터 **상단의 "자주 묻는 질문" 영역**에도 함께 나옵니다. | |

> **질문과 답변 둘 중 하나라도 비어 있으면 저장되지 않습니다.** 저장 버튼을 눌렀는데 아무 안내 없이 화면이 되돌아왔다면 둘 중 하나가 비어 있는 것입니다.

> **`노출` 체크를 켜야 고객에게 보입니다.** 등록만 하고 체크를 빠뜨리면 관리자 목록에는 있는데 고객센터에는 없습니다. 가장 흔한 실수입니다.

### 3-3. 답변 편집기 사용 시 주의

답변 편집기는 서식을 지원하지만, **저장할 때 위험한 태그와 속성(스크립트, 이벤트 동작 등)이 자동으로 제거**됩니다. 화면에도 "허용되지 않는 태그·속성(스크립트, 이벤트 핸들러 등)은 저장 시 자동으로 제거됩니다"라고 안내되어 있습니다.

- 외부 페이지 내용을 그대로 붙여넣으면 **일부가 사라진 채 저장**될 수 있습니다.
- 저장 후 반드시 다시 열어 내용이 온전한지 확인하세요.
- 문단·굵게·목록·링크·표 정도로 구성하는 것이 가장 안전합니다.

### 3-4. BEST 지정은 몇 개가 적당한가

`BEST`는 고객센터 맨 위에 뜨는 자리입니다. **정말 자주 들어오는 질문 5~8개 정도**로 제한하세요. 전부 BEST로 켜면 상단이 길어져 오히려 아무것도 눈에 띄지 않습니다.

BEST 선정은 목록의 **조회** 수치를 참고하면 됩니다. 조회가 높은데 BEST가 아닌 항목은 올리고, 오래 방치되어 조회가 낮은 BEST는 내리세요. 문의 관리에 같은 질문이 반복해서 들어오는지도 함께 보세요.

### 3-5. 삭제

목록의 **삭제** 버튼을 누르면 확인 창이 뜨고, 확인하면 **완전히 지워집니다.** 조회 기록도 함께 사라지고 되돌릴 수 없습니다.

> **당분간 안 쓸 FAQ는 삭제하지 말고 `노출` 체크를 끄세요.** 내용과 조회 기록이 그대로 남아 나중에 되살릴 수 있습니다.

## 4. 몰 스코프 — 어디에 저장되는가

이 앱은 몰을 여러 개 만들 수 있는 빌더입니다. 두 화면의 동작이 다릅니다.

**FAQ는 지금 편집 중인 몰에만 저장됩니다.**

- 편집 몰이 A몰인 상태에서 만든 FAQ는 **A몰 고객센터에만** 나옵니다.
- B몰에도 같은 FAQ가 필요하면 **편집 몰을 B몰로 바꾼 뒤 다시 등록**해야 합니다. 복사 기능은 없습니다.
- 목록 위의 분류 버튼도 그 몰의 분류만 보입니다.
- 다른 몰의 FAQ는 주소를 직접 입력해도 열리지 않습니다.

**문의는 몰 구분 없이 모두 나옵니다.** 다만 목록에 어느 몰에서 들어온 문의인지 표시되고, 몰 필터로 걸러 볼 수 있습니다.

> **"FAQ를 등록했는데 목록에 없다"면 십중팔구 다른 몰에 만든 것입니다.** 관리자 화면 상단의 편집 몰을 먼저 확인하세요.

## 6. 자주 막히는 상황

**Q. 문의 목록에 다른 몰 문의까지 섞여 나옵니다.**
정상입니다. 문의 관리는 기본이 전체 조회입니다. 목록 오른쪽 위 **몰** 선택 상자에서 담당 몰을 고르세요. 다만 이 선택은 **저장되지 않아** 화면에 다시 들어오면 전체로 돌아갑니다.

**Q. 답변을 잘못 등록했습니다. 취소할 수 있나요?**
답변을 지워 `답변대기`로 되돌리는 기능은 없습니다. 상세 화면에서 답변 내용을 고쳐 **답변 수정**으로 다시 저장하세요.

**Q. 답변했는데 고객이 모릅니다.**
답변 등록 시 자동 알림이 나가지 않습니다. 고객이 사이트에 다시 들어와야 확인합니다. 급한 건은 전화·문자로 따로 알리세요.

**Q. FAQ를 등록했는데 고객센터에 안 보입니다.**
순서대로 확인하세요.
1. **노출** 체크가 켜져 있는가?
2. **편집 중인 몰**이 고객이 보는 몰과 같은가?
3. 저장 후 목록에 실제로 나타났는가? (질문·답변 중 하나가 비면 저장이 안 됩니다)

**Q. FAQ 상단의 "자주 묻는 질문"에 안 나옵니다.**
`자주 묻는 질문(BEST) 에 노출` 체크를 켜야 합니다. `노출` 체크와는 별개이며, **두 개 모두 켜져 있어야** 합니다.

**Q. 답변에 붙여넣은 내용이 저장 후 사라졌습니다.**
허용되지 않는 태그가 저장 시 제거된 것입니다. 외부 페이지에서 그대로 복사하지 말고, 편집기 안에서 문단·목록·링크로 다시 구성하세요.

**Q. FAQ 분류를 새로 만들고 싶습니다.**
현재 관리자 화면에는 **분류를 추가·수정·삭제하는 기능이 없습니다.** 몰을 만들 때 함께 생성된 분류만 사용할 수 있습니다. 마땅한 분류가 없으면 `미분류`로 두고 질문 문장 앞에 구분어를 붙이는 방식으로 대응하세요. (예: `[배송] 며칠 걸리나요?`)

**Q. 순서를 바꿨는데 목록에서 위치가 그대로입니다.**
목록은 **분류 순서 → FAQ 순서 → 번호** 차례로 정렬됩니다. 분류가 다르면 순서 값을 아무리 낮춰도 다른 분류 위로 올라가지 않습니다. 같은 분류 안에서만 순서가 의미를 가집니다.

**Q. 문의를 삭제하고 싶습니다.**
문의를 지우는 기능은 없습니다. 고객 문의 기록은 그대로 보관됩니다.

**Q. 비회원이 문의를 남길 수 없나요?**
남길 수 없습니다. 문의는 회원만 작성할 수 있으므로 목록에는 항상 회원 정보가 붙습니다.

---

## 7. 아직 지원하지 않는 것

- 문의 삭제, `답변대기`로 되돌리기
- 답변 등록 시 고객에게 자동 알림(문자·이메일) 발송
- 문의 검색·상태별 필터 (몰 필터만 있습니다)
- 문의 답변에 이미지·서식 넣기 (FAQ 답변에는 가능합니다)
- FAQ 분류 추가·수정·삭제
- FAQ를 다른 몰로 복사
- 비회원 문의 접수
