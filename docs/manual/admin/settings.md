# 사이트 설정 · 시스템 설정

쇼핑몰의 **회사 정보·로고·브랜드 색상**과, **결제·소셜 로그인·메일 발송 같은 외부 서비스 연동 키**를 관리하는 메뉴입니다. 몰을 처음 만든 뒤 **가장 먼저 한 번씩 채워 두어야 하는 화면**입니다.

화면 위치: <span class="manual-path">관리자 &gt; 사이트 설정</span> (`/admin/site-settings`) · <span class="manual-path">관리자 &gt; 시스템 설정</span> (`/admin/sys-settings`)

> **이 두 화면은 성격이 완전히 다릅니다.** 사이트 설정은 **몰마다 따로** 저장되고, 시스템 설정은 **모든 몰이 함께 쓰는 공용 값**입니다. 아래 1장에서 이 차이를 반드시 확인하세요.

---

## 1. 두 화면의 차이

| | **사이트 설정** (`/admin/site-settings`) | **시스템 설정** (`/admin/sys-settings`) |
|---|---|---|
| 무엇을 담나 | 회사명·로고·연락처·주소·SNS·브랜드 색상·파비콘 | 결제 키·소셜 로그인 키·메일 서버·AI 키·포인트 규칙 |
| **저장 범위** | **지금 편집 중인 몰에만** | **전 몰 공용** |
| 누가 채우나 | 운영자 본인 | 개발자·시스템 담당자가 알려 준 값 |
| 잘못 넣으면 | 화면에 엉뚱한 정보가 나옴 | **결제·로그인·메일이 동작하지 않음** |

`/admin/settings` 로 들어가면 **기본 정보관리 / 시스템 설정** 두 탭이 한 화면에 함께 나옵니다. 내용은 위 두 화면과 같습니다.

> **사이트 설정은 몰마다 따로 저장됩니다.** 오른쪽 위의 편집 몰이 A몰인데 B몰의 로고를 바꿀 생각이었다면, 저장해도 B몰은 그대로입니다. 저장 전에 편집 몰을 확인하세요.

## 2. 사이트 설정 — 회사 정보

법으로 쇼핑몰 하단(푸터)에 표시해야 하는 항목들입니다.

| 입력란 | 필수 | 설명 | 예시 |
|---|---|---|---|
| **회사명(상호)** | 필수 | 쇼핑몰 상단·푸터·로그인 화면에 쓰이는 이름입니다. | `와이디데이터` |
| **대표자 성명** | 선택 | 푸터에 표시됩니다. | `홍길동` |
| **사업자 등록번호** | 선택 | **입력하면 푸터에 '사업자정보확인'(공정위 조회) 링크가 자동으로 붙습니다.** | `357-86-02849` |
| **통신판매업 신고번호** | 선택 | 푸터에 표시됩니다. | `2023-서울강남-00428` |
| **주소(사업장 소재지)** | 선택 | 푸터에 표시됩니다. | `06170 서울특별시 강남구 …` |
| **대표 이메일** | 선택 | 고객 문의용으로 푸터에 표시됩니다. | `help@example.com` |
| **고객센터 전화번호** | 선택 | 푸터·고객센터에 표시됩니다. | `080-344-5575` |
| **고객센터 상담시간 안내** | 선택 | **여러 줄 입력 가능**하며, 줄바꿈 그대로 푸터에 나옵니다. | `평일 10:00 - 17:00`<br>`주말 및 공휴일 휴무` |
| **개인정보 보호책임자 성명** | 선택 | 개인정보보호법상 표시 항목입니다. | `홍길동` |
| **개인정보 보호책임자 이메일** | 선택 | 위와 같습니다. | `privacy@example.com` |

> **전자상거래법·개인정보보호법상 상호·대표자·사업자등록번호·통신판매업 신고번호·주소·연락처·개인정보 보호책임자는 쇼핑몰 하단에 표시해야 하는 항목입니다.** 테스트 몰이 아니라면 비워 두지 마세요.

## 3. 사이트 설정 — 회사 소개 & 푸터

| 입력란 | 어디에 보이나 |
|---|---|
| **헤더 슬로건** | 상단 헤더 바에 노출되는 한 줄 문구. **관리자 로그인 화면에도 표시**됩니다. (예: `건강한 하루의 시작 | 와이디몰`) |
| **푸터 슬로건** | 푸터에 짧게 노출되는 한 줄 문구. 간단한 서식 편집기가 붙어 있습니다. (예: `자연이 주는 건강한 선물`) |
| **회사 소개 페이지 내용** | 별도의 **회사 소개 페이지(`/about`)** 에 들어갈 본문. 이미지와 표를 넣을 수 있는 전체 기능 편집기입니다. |

**SNS 링크** — 인스타그램 · 페이스북 · 유튜브 · 카카오채널 네 가지입니다.

> **체크박스를 켜고 URL도 넣어야 아이콘이 나옵니다.** URL만 넣고 체크를 안 하면 표시되지 않습니다. 반대로 체크만 하고 URL이 비어 있으면 눌러도 아무 데도 가지 않습니다.

## 4. 사이트 설정 — 분석 & 브랜드 색상

### 4-1. Google Analytics 4 측정 ID

GA4에서 발급받은 측정 ID(`G-XXXXXXXXXX` 형식)를 넣으면 고객 화면에 자동으로 적용됩니다. 비워 두면 아무것도 붙지 않습니다.

### 4-2. 브랜드 키 컬러 (3색)

| 색상 | 어디에 쓰이나 |
|---|---|
| **브랜드 기본색** | 주요 버튼, 강조 텍스트 |
| **브랜드 진한색** | 마우스를 올렸을 때, 강조 라인, 텍스트 |
| **브랜드 연한색** | 히어로 배경, 알림 박스처럼 넓은 면적 |

색은 세 가지 방법으로 정할 수 있습니다.

1. **색상 상자를 눌러 팔레트에서 고르기**
2. **`#76A764` 같은 코드를 직접 입력하기**
3. **아래 준비된 팔레트 버튼**(Mall Green · Emerald · Blue · Purple · Rose · Amber · Teal · Slate · Lime · Orange · Indigo) **누르기** — 세 톤이 한 번에 채워집니다.

**기본색 기준으로 진/연톤 자동 생성** 버튼을 누르면 기본색만 정하고 나머지 두 톤을 자동으로 만들어 줍니다.

> **색상은 여기(사이트 설정)에서 정합니다.** 디자인 스타일 화면에는 색상 항목이 없습니다. 많이 헷갈리는 부분입니다.

## 5. 사이트 설정 — 이미지 3종

| 항목 | 용도 | 권장 규격 | 없으면 |
|---|---|---|---|
| **로고** | 헤더와 관리자 로그인 화면 | **800×280 px**, 투명 배경 PNG 권장 | 회사명이 글자로 표시됩니다 |
| **카카오톡 링크 공유 썸네일** | 카카오톡·SNS로 링크를 보냈을 때 뜨는 미리보기 | **1200×630** 비율 | **로고 이미지가 대신 쓰입니다** |
| **파비콘** | 브라우저 탭에 뜨는 작은 아이콘 | **32×32 또는 48×48**, PNG/ICO/SVG | 기본 아이콘이 나옵니다 |

세 항목 모두 파일을 고르면 **바로 아래에 미리보기**가 나타납니다. 다만 미리보기가 바뀌었다고 저장된 것은 아닙니다.

> **파일을 고른 뒤 반드시 화면 맨 아래 [저장하기]를 눌러야 반영됩니다.** 미리보기만 보고 화면을 벗어나면 원래 이미지가 그대로 남습니다.

파비콘은 업로드하면 자동으로 아이콘 형식으로 변환됩니다.

## 6. 시스템 설정 — 무엇이 있나

**여기 있는 값은 전 몰이 함께 씁니다.** 몰마다 다른 결제 키·소셜 로그인 키를 쓸 수는 없습니다.

### 6-1. 에디터 / AI 설정

| 항목 | 설명 |
|---|---|
| **사이트 기본 도메인** | 검색엔진용 주소와 SNS 공유 정보에 쓰이는 기본 주소입니다. (예: `https://mall.co.kr`) |
| **TinyMCE Key** | 상품 설명·회사 소개를 쓰는 **글 편집기의 라이선스 키**입니다. **비어 있으면 편집기가 뜨지 않고 밋밋한 입력칸만 나옵니다.** |
| **AI 사용** | `사용 (Y)` / `미사용 (N)`. AI 추천 문구·SEO 메타 생성 버튼 전체를 켜고 끕니다. |
| **OpenAI API Key** | AI 기능에 쓰는 키. **미사용이거나 이 키가 비어 있으면 AI 버튼이 비활성화됩니다.** |
| **OpenAI 모델 / 타임아웃(ms)** | 사용할 모델 이름과 응답 대기 시간입니다. |

### 6-2. Shopify 연동

**Shopify 동기화 사용** — `사용` / `미사용`. 상품 저장·삭제 시 Shopify로 자동 동기화할지 정합니다. 현재는 **미사용**이 정상입니다.

### 6-3. 소셜 로그인 (Google · Kakao · Naver)

세 서비스 모두 같은 구조입니다.

| 항목 | 설명 |
|---|---|
| **Client ID** | 각 서비스 개발자센터에서 발급받은 앱 ID |
| **Client Secret** | 위와 짝이 되는 비밀 값 |
| **Dev Callback URL** | 개발용 되돌아올 주소 |
| **Prod Callback URL** | 운영용 되돌아올 주소 |
| **공통 Callback URL** (Google만) | 위 둘 대신 쓰는 단일 주소 |

> **키와 Callback URL이 둘 다 채워져야 고객 화면에 그 소셜 로그인 버튼이 나옵니다.** 하나만 넣으면 버튼 자체가 표시되지 않습니다. 또한 **각 서비스 개발자센터에도 같은 Callback URL을 등록**해 두어야 로그인이 완료됩니다.

**Kakao 공유 설정 — Kakao JavaScript 키**는 로그인이 아니라 **카카오톡 공유 버튼**에 쓰입니다. 별도 항목입니다.

### 6-4. 신상품 · 신규 입점 브랜드

| 항목 | 기본값 | 설명 |
|---|---|---|
| **신상품 노출 기간 (일)** | 100 | 상품의 **판매 시작일**부터 이 기간 안에 있는 상품을 신상품으로 봅니다. 1~3650 |
| **신규 입점 브랜드 노출 기간 (일)** | 180 | 브랜드의 **입점일**부터 이 기간 안에 있는 브랜드를 신규 입점으로 봅니다. 1~3650 |

기간이 지나면 자동으로 빠집니다. 단, **상품에 NEW 뱃지를 직접 체크해 두면 기간과 무관하게 계속 노출**됩니다.

### 6-5. 포인트 설정

| 항목 | 기본값 | 설명 |
|---|---|---|
| **구매 적립률 (%)** | 5 | 결제 금액 대비 적립되는 포인트 비율입니다. `5` 를 넣으면 5%입니다. 0~100 |
| **포인트 최소 사용 단위 (원)** | 1000 | 결제 시 사용할 수 있는 최소 포인트 단위입니다. |

> **이 값은 고객이 실제로 받는 적립금에 직접 영향을 줍니다.** `5`를 `50`으로 잘못 넣으면 결제 금액의 절반이 포인트로 나갑니다. 저장 전에 자릿수를 확인하세요.

### 6-6. 토스페이먼츠 결제 설정

| 항목 | 설명 |
|---|---|
| **클라이언트 키** | 고객에게 뜨는 결제창 연동용입니다. 테스트 키는 `test_ck_` 로 시작합니다. |
| **시크릿 키** | 서버에서 결제를 승인할 때 쓰는 키입니다. 테스트 키는 `test_sk_` 로 시작합니다. |

> **시크릿 키는 절대 외부에 공유하지 마세요.** 화면에서도 가려져 표시됩니다. 또한 **두 키가 짝이 맞지 않으면 결제창은 뜨는데 결제가 실패**합니다. 반드시 같은 상점의 키 쌍을 넣으세요.

### 6-7. SMTP 메일 설정

주문 알림, 배송 알림, **관리자 이중인증 코드** 발송에 모두 이 설정이 쓰입니다.

| 항목 | 설명 |
|---|---|
| **메일 서버 주소** | 예: `smtp.gmail.com` |
| **포트** | 예: `587` |
| **발송자 이메일 주소** | 고객에게 보내는 사람으로 표시될 주소 |
| **메일 서비스 유형** | **지메일(Gmail)** 또는 **기타 SMTP** 중 선택 |
| **지메일 앱 비밀번호** | 지메일을 고른 경우에만 보입니다. Google 계정에서 2단계 인증 후 발급한 앱 비밀번호입니다. 공백이 있어도 됩니다. |
| **SMTP 비밀번호** | 기타 SMTP를 고른 경우에만 보입니다. |

**테스트 발송** — 수신자 이메일을 넣고 버튼을 누르면 확인 메일이 나갑니다. 결과가 버튼 아래에 초록(성공) 또는 빨강(실패)으로 표시됩니다.

> **테스트 발송은 저장된 설정으로 보냅니다.** 값을 고쳤다면 **먼저 [저장하기]를 누른 뒤** 테스트하세요. 저장하지 않고 테스트하면 예전 설정으로 보내집니다.

> **메일 발송이 안 되면 관리자 이중인증 로그인도 막힙니다.** 관리자 계정의 이중인증을 켜기 전에 반드시 테스트 발송으로 확인하세요.

## 7. 저장 방법

- **사이트 설정**과 **시스템 설정**은 **저장 버튼이 각각 따로**입니다. 한쪽을 저장해도 다른 쪽은 저장되지 않습니다.
- `/admin/settings` 에서 탭을 옮길 때도 마찬가지입니다. **탭을 바꾸기 전에 저장하세요.**
- 저장하면 값이 즉시 반영됩니다. 시스템 설정은 저장과 동시에 실행 중인 서비스에도 적용됩니다.

## 8. 자주 막히는 상황

**Q. 로고를 바꿨는데 고객 화면이 그대로입니다.**
① 저장 버튼을 눌렀는지, ② **편집 중인 몰이 맞는지** 확인하세요. 사이트 설정은 몰마다 따로입니다. 그래도 안 바뀌면 브라우저 새로고침(Ctrl+F5)을 해 보세요.

**Q. 상품 설명 편집기가 안 뜨고 그냥 네모칸만 나옵니다.**
시스템 설정의 **TinyMCE Key**가 비어 있거나 잘못된 값입니다.

**Q. 카카오톡으로 링크를 보냈는데 미리보기 이미지가 이상합니다.**
**카카오톡 링크 공유 썸네일**을 등록하세요. 비워 두면 로고가 대신 쓰이는데, 로고는 가로로 길어 잘려 보일 수 있습니다.

**Q. 네이버(또는 구글·카카오) 로그인 버튼이 고객 화면에 안 보입니다.**
Client ID와 Callback URL이 **둘 다** 채워져야 나옵니다. 하나라도 비어 있으면 버튼이 표시되지 않습니다.

**Q. 소셜 로그인 버튼은 보이는데 누르면 오류가 납니다.**
각 서비스 개발자센터에 등록한 Callback URL과 여기 넣은 값이 **글자 하나까지 똑같아야** 합니다.

**Q. 결제창이 안 뜹니다.**
토스페이먼츠 **클라이언트 키**를 확인하세요. 결제창은 뜨는데 승인에서 실패한다면 **시크릿 키** 쪽 문제입니다.

**Q. AI 버튼이 회색으로 눌리지 않습니다.**
**AI 사용**이 `미사용 (N)` 이거나 **OpenAI API Key**가 비어 있습니다.

**Q. 푸터에 '사업자정보확인' 링크가 없습니다.**
**사업자 등록번호**를 넣으면 자동으로 생깁니다.

**Q. 사이트 설정을 열었더니 다른 몰의 정보가 보입니다.**
새로 만든 몰이라 아직 자기 설정이 없는 경우, 기본몰의 값을 보여 줍니다. **저장하는 순간 이 몰 전용 설정이 만들어집니다.**

## 9. 아직 지원하지 않는 것

- **몰마다 다른 결제 키·소셜 로그인 키** — 시스템 설정은 전 몰 공용입니다
- 설정 변경 이력 조회·되돌리기
- 로고의 밝은/어두운 배경용 이미지 분리 등록
