# 페이지 빌더 · 상품 그룹

쇼핑몰의 **메인 화면(홈)** 과 **신상품 같은 랜딩 페이지**를 직접 구성하는 메뉴입니다. 화면을 **섹션**이라는 가로 블록으로 쌓아 만들고, 그중 상품을 보여 주는 섹션에는 **상품 그룹**을 데이터로 연결합니다. 두 화면이 짝을 이루므로 이 문서에서 함께 다룹니다.

> **가장 중요한 규칙: 저장과 발행은 다릅니다.** 섹션을 고쳐 **설정 저장**을 눌러도 고객 화면은 그대로입니다. 상단의 **발행** 버튼을 눌러야 실제 화면에 반영됩니다.

> **지금 편집 중인 몰을 반드시 확인하세요.** 페이지·섹션·상품 그룹은 모두 **몰별로 따로 저장**됩니다.

## 1. 기본 개념 — 섹션을 쌓아 화면을 만듭니다

홈 화면은 위에서 아래로 여러 개의 **섹션**이 쌓인 구조입니다.

```
[ 히어로 (최상단 큰 배너/슬라이더) ]
[ 퀵 메뉴 ]
[ 베스트/랭킹 ]
[ 쇼핑특가 캐러셀 ]
[ 프로모션 배너 ]
[ 카테고리별 상품 ]
```

각 섹션은 **어떤 데이터를 어떻게 보여 줄지**가 정해져 있습니다. 데이터가 오는 곳은 섹션마다 다릅니다.

| 데이터 출처 | 해당 섹션 | 어디서 채우나 |
|---|---|---|
| **상품 그룹** | 상품 그리드 · 상품 캐러셀 · 혜택 벤토 | 상품 그룹 관리 |
| **배너** | 히어로 · 프로모션 배너 | 배너 관리 |
| **랭킹** | 베스트/랭킹 · 랭킹 탭 | 베스트·랭킹 관리 |
| **특가** | 쇼핑특가 캐러셀 | 쇼핑특가 관리 |
| **카테고리·브랜드** | 카테고리별 상품 · 베스트 브랜드 · 신상품 섹션들 | 상품 관리(상품에 카테고리·브랜드 지정) |
| **직접 입력** | 퀵 메뉴 · 커스텀 HTML(개발자용) | 섹션 설정에서 바로 입력 |

## 2. 전체 흐름 한눈에 보기

```
① (필요하면) 상품 그룹을 먼저 만든다
        ↓
② 페이지 빌더에서 편집할 페이지 선택
        ↓
③ 섹션 추가 → 순서 조정
        ↓
④ 섹션을 클릭 → 오른쪽에서 설정 → 설정 저장
        ↓
⑤ 미리보기로 확인
        ↓
⑥ 발행  ← 여기까지 해야 고객 화면에 반영됩니다
```

## 3. 화면 구조 — 두 개의 상위 탭

페이지 빌더는 위쪽에 **테마 설정 / 페이지 설정** 두 탭이 있습니다. (테마 설정 탭은 홈 페이지를 편집할 때만 나타납니다.)

### 3-1. 테마 설정 탭

홈 상단 히어로의 **배치 뼈대**와 **메뉴 구성 방식**을 정합니다.

| 항목 | 설명 |
|---|---|
| **테마 — 히어로 배치** | 홈 상단 히어로를 **어떻게 배치할지**만 고릅니다. 안에 무엇이 들어갈지는 배너 관리의 메인 슬라이더에서 정합니다. |
| **메뉴 구성 방식** | 상단 GNB에서 카테고리와 일반 메뉴를 나눌지, 한 줄로 합칠지 정합니다. |
| **홈 섹션까지 기본 구성으로 교체** | 아래 경고 참고. 기본은 꺼져 있습니다. |

정한 뒤 **적용** 버튼을 누릅니다.

> **"홈 섹션까지 기본 구성으로 교체"는 되돌릴 수 없습니다.** 체크하고 적용하면 히어로 아래의 캐러셀들이 테마 기본 구성(베스트·쇼핑특가·베스트 카테고리·베스트 브랜드)으로 **전부 교체**되고, 지금 꾸며 둔 섹션은 삭제됩니다.

> **체크하지 않아도 히어로 배치와 메뉴 구성은 바뀝니다.** 꾸며 둔 섹션만 그대로 유지됩니다. 테마만 바꿔 보고 싶다면 체크를 켜지 마세요.

### 3-2. 페이지 설정 탭

홈을 편집할 때는 그 안에서 다시 **이지모드 / 직접설정** 으로 나뉩니다.

| 하위 탭 | 무엇을 하나 |
|---|---|
| **이지모드** | 미리 만들어 둔 캐러셀 구성(번들)을 골라 한 번에 적용합니다. |
| **직접설정** | 섹션을 하나씩 추가·삭제·설정합니다. 기본으로 열립니다. |

**이지모드**는 카드마다 어떤 섹션이 몇 번째로 들어가는지 목록으로 보여 주고, **이 구성 적용** 버튼 하나로 끝납니다.

> **이지모드는 "메인 상단 히어로는 그대로 두고, 그 아래 캐러셀만 교체하고 곧바로 발행"합니다.** 지금 히어로 아래에 꾸며 둔 섹션은 사라집니다. 확인 창이 한 번 뜨니 신중히 누르세요.

## 4. 직접설정 — 상단 툴바 읽는 법

| 항목 | 설명 |
|---|---|
| **페이지** 선택 | 편집할 페이지를 고릅니다. `홈`, 그 외 랜딩 페이지가 나오며 각각 섹션 개수와 `미발행` 여부가 함께 표시됩니다. |
| **발행됨 #N / 발행 이력 없음** | 이 페이지의 최신 발행 번호. |
| **미발행 변경사항** | 노란 배지. **작업본이 최신 발행본과 다르다**는 뜻입니다. 이게 떠 있으면 아직 고객 화면에 반영되지 않았습니다. |
| **리비전 선택(롤백)** | 과거 발행 이력을 고릅니다. |
| **롤백** | 고른 리비전 상태로 되돌립니다. |
| **미리보기 새로고침** | 가운데 미리보기를 다시 그립니다. |
| **발행** | 지금 작업본을 고객 화면에 반영합니다. |

> **"미발행 변경사항" 배지는 새로고침해도 사라지지 않습니다.** 서버가 작업본과 최신 발행본을 비교해 판단하기 때문입니다. 배지가 떠 있다면 발행을 잊은 것입니다.

**롤백**은 과거 발행 시점의 구성으로 되돌리는 기능입니다. 잘못 발행했을 때 쓰세요.

## 5. 직접설정 — 섹션 추가하기

화면은 세로로 셋으로 나뉩니다. **왼쪽 = 섹션 목록 / 가운데 = 미리보기 / 오른쪽 = 선택한 섹션 설정.**

### 5-1. 섹션 추가

왼쪽 위의 선택 목록에서 섹션 종류를 고르고 **추가** 버튼을 누릅니다.

섹션 이름만으로 감이 안 온다면 그 아래의 **섹션 둘러보기 — 뭐가 있고 어떻게 보이는지** 버튼을 누르세요. 각 섹션을 **이 몰의 실제 데이터로 그려서** 보여 주고, 카드에서 바로 추가할 수 있습니다.

> **둘러보기에 "예시 데이터" 표시가 붙는 섹션이 있습니다.** 퀵 메뉴·커스텀 HTML·혜택 벤토처럼 운영자가 직접 채워야 하는 섹션은, 비어 보이지 않도록 예시 값으로 그려 줍니다. 실제 데이터가 아닙니다.

### 5-2. 섹션 목록 다루기

각 섹션 카드에는 다음 버튼이 있습니다.

| 버튼 | 동작 |
|---|---|
| **복제** | 같은 설정의 섹션을 하나 더 만듭니다. |
| **삭제** | 섹션을 지웁니다. (확인 창이 뜹니다) |

순서는 목록에서 끌어 옮겨 바꿉니다.

**프론트 미노출 배지** — 그 섹션이 지금 고객 화면에 **나오지 않는다**는 경고입니다. 왜 안 나오는지 이유가 함께 표시되고, 다른 화면에서 데이터를 채워야 하는 경우에는 **그 화면으로 가는 버튼**까지 붙습니다.

| 이유 | 어떻게 고치나 |
|---|---|
| 비활성(숨김) 상태입니다 | 오른쪽 설정에서 **활성**을 켭니다 |
| 노출 시작 시각이 아직 오지 않았습니다 | **노출 시작**을 비우거나 앞당깁니다 |
| 노출 종료 시각이 지났습니다 | **노출 종료**를 비우거나 미룹니다 |
| PC·모바일 노출이 둘 다 꺼져 있습니다 | **PC** 또는 **모바일** 중 하나를 켭니다 |
| 데이터가 없습니다 | 배너·상품 그룹·랭킹 등을 **다른 화면에서 채웁니다** (버튼이 붙습니다) |
| 코드에 없는 섹션 타입입니다 | 삭제하세요 |

## 6. 직접설정 — 섹션 설정하기 (오른쪽 패널)

왼쪽에서 섹션을 클릭하면 오른쪽에 설정이 나타납니다. **모든 섹션에 공통인 항목**은 다음과 같습니다.

| 입력란 | 설명 |
|---|---|
| **제목** | 섹션 위에 표시되는 제목입니다. |
| **상품 그룹** | 상품 그룹을 쓰는 섹션에만 나타납니다. 목록에서 고릅니다. |
| **PC / 모바일** | 각 기기에 노출할지. **둘 다 끄면 아무 데도 안 나옵니다.** |
| **활성** | 끄면 노출되지 않습니다. |
| **노출 시작 / 노출 종료** | 기간을 정합니다. 비우면 제한이 없습니다. |

입력 후 **설정 저장** 버튼을 누릅니다.

> **저장은 작업본에만 반영됩니다. 실서비스 반영은 상단 `발행`.** 화면에도 같은 안내가 적혀 있습니다.

저장하면 그 자리에서 **프론트 미노출** 여부를 다시 판정해 배지를 갱신합니다. 상품 그룹을 붙여 고쳤다면 경고가 바로 사라집니다.

### 6-1. 섹션 종류별 설정 항목

| 섹션 | 데이터 출처 | 주요 설정 |
|---|---|---|
| **히어로** | 배너 관리의 메인 배너 | 없음 |
| **테마 히어로** | 메인 슬라이더 | **배치** (`split_feature` / `full_width` / `full_bleed`) |
| **상품 그리드** | 상품 그룹 | 표시 상품 수(기본 8) · 열 수(PC, 기본 4) · 더보기 링크 |
| **상품 캐러셀** | 상품 그룹 | 표시 상품 수(기본 12) · 뷰당 표시 수(PC, 기본 4) · 더보기 링크 |
| **베스트/랭킹** | 랭킹 | 랭킹 탭 ID(0 = 전체) · 기간 · 표시 상품 수 · 열 수 · 더보기 링크 |
| **랭킹 탭** | 랭킹 그룹 | 탭 개수(기본 6) · 탭당 상품 수(기본 8) · 기간 |
| **쇼핑특가 캐러셀** | 진행 중인 특가 | 특가 카테고리 코드(비우면 전체) · 표시 상품 수 · 뷰당 표시 수 · 더보기 링크 |
| **프로모션 배너** | 배너 그룹 | **배너 그룹 키** · 표시 배너 수 · 레이아웃(`rect`/`vertical`) · 열 수 |
| **베스트 브랜드** | 브랜드 카테고리 | 표시 브랜드 수 · 브랜드당 상품 수 · 더보기 링크 |
| **카테고리별 상품** | 노출 카테고리 | 없음 |
| **혜택 벤토** | 상품 그룹 | 대형 딜 상품 ID · 썸네일 수 · 프로모 블록 |
| **퀵 메뉴** | 직접 입력 | 바로가기 항목(행 단위 편집) · 열 수 |
| **최근 본 상품** | 방문자별 | 표시 상품 수 |
| **커스텀 HTML (개발자용)** | 직접 입력 | HTML 코드 — 아래 설명 참고 |
| **카테고리별 신상품** | 신상품 | 탭별 상품 수 · 탭(카테고리) 수 |

| **브랜드별 신상품** | 신상품 | 브랜드별 상품 수 · 브랜드 수 |
| **신규 입점 브랜드** | 신규 브랜드 | 브랜드 수 · 브랜드별 대표 상품 수 |
| **카카오 상담 CTA** | 사이트 설정 | 없음 |

> **커스텀 HTML 은 이름 그대로 "개발자용"입니다.** 관리자 화면에서 HTML 코드를 직접 쓰는 유일한 섹션이라, HTML 을 아는 사람이 다뤄야 합니다.
> 위 목록의 다른 섹션으로 표현할 수 없을 때만 쓰는 **마지막 수단**으로 남겨 둔 것입니다. 글·이미지·표를 넣고 싶은 정도라면 기획전이나 이벤트 화면의 **편집기**를 쓰는 편이 훨씬 쉽습니다.

**기간 옵션**은 `REALTIME`(실시간) · `DAILY`(일간) · `WEEKLY`(주간) · `MONTHLY`(월간) 입니다.

### 6-2. 퀵 메뉴 항목 입력하기

**바로가기 항목**은 행 단위로 추가합니다. **항목 추가** 버튼을 누르고 다음을 채웁니다.

| 칸 | 설명 |
|---|---|
| **이동할 페이지** | 목록에서 고릅니다. **고르면 링크와 아이콘이 자동으로 채워집니다.** |
| **표시 이름** | 화면에 보일 이름. 예: `오늘특가` |
| **뱃지(선택)** | 작은 표시. 예: `N` |
| **아이콘** | 페이지를 고르면 자동으로 채워집니다. **'직접 입력'을 골랐을 때만** 손으로 넣습니다. |

> **전부 빈 행은 저장되지 않습니다.** 실수로 추가한 빈 행은 그냥 두어도 됩니다.

### 6-3. 커스텀 HTML

다른 섹션으로 표현할 수 없을 때의 마지막 수단입니다.

> **저장·표시 시 위험한 태그가 자동으로 제거됩니다.** 스크립트를 넣어도 사라지니 이미지·문단·표 정도로 구성하세요.

### 6-4. 프로모션 배너 연결

**배너 그룹 키**에 적은 값과 **같은 그룹 키로 등록된 배너들**이 이 자리에 캐러셀로 나옵니다. 배너 쪽 그룹 키가 다르면 아무것도 나오지 않습니다.

## 7. 미리보기와 발행

가운데 영역이 작업본 미리보기입니다. 위쪽 버튼으로 **PC 1440 / 태블릿 820 / 모바일 390** 폭을 전환해 확인합니다. 설정을 바꾼 뒤에는 **미리보기 새로고침**을 눌러 다시 그립니다.

확인이 끝나면 **발행**을 누릅니다. 발행하면 그 시점의 구성이 리비전으로 저장되고, 고객 화면이 그 구성으로 바뀝니다.

> **발행하지 않으면 아무리 저장해도 고객 화면은 바뀌지 않습니다.** 반대로 말하면, 마음 놓고 편집해 두었다가 원하는 때에 발행할 수 있습니다.

## 8. 상품 그룹 관리

<div class="manual-goto-bar">
  <span class="manual-goto-label">이 단계 화면</span>
  <a class="manual-goto" href="/admin/product-groups" target="_blank">상품 그룹 목록 열기</a>
  <a class="manual-goto" href="/admin/product-groups/new" target="_blank">새 상품 그룹 만들기</a>
</div>

**상품 그룹은 "상품을 모아 둔 바구니"** 입니다. 상품 그리드·상품 캐러셀·혜택 벤토 섹션이 이 바구니를 데이터로 씁니다.

### 8-1. 기본 정보

| 입력란 | 필수 | 설명 |
|---|---|---|
| **그룹명** | 필수 | 관리용 이름. 최대 200자. |
| **메뉴 상단 노출** | 선택 | 지정한 메뉴 페이지 **맨 위에 이 그룹이 캐러셀로** 뜹니다. |
| **섹션 제목** | 선택 | 메뉴 상단에 표시될 제목. **비우면 그룹명이 쓰입니다.** 최대 100자. |
| **구성 방식** | 필수 | `수동 선택` 또는 `조건 검색`. 아래 참고. **메뉴 상단 노출을 지정하면 이 항목이 사라집니다**(둘을 함께 쓰기 때문). |
| **사용** | — | 끄면 이 그룹을 쓰는 섹션이 **빈 상태**가 됩니다. |

**구성 방식 2가지**

| 값 | 동작 |
|---|---|
| **수동 선택** | 상품을 직접 담고 순서를 지정합니다. 정한 순서 그대로 노출됩니다. |
| **조건 검색** | 배지·카테고리 등 조건으로 **자동 수집**합니다. 상품이 늘어나면 자동으로 반영됩니다. |

> **한 메뉴에는 상품 그룹 하나만 걸 수 있습니다.** 이미 걸린 메뉴를 다시 고르면 `이 메뉴에는 이미 다른 상품 그룹이 걸려 있습니다.` 오류가 납니다. 기존 그룹의 메뉴 지정을 먼저 해제하세요.

> **구성 방식을 바꾸면 화면 아래 영역이 통째로 바뀝니다.** `수동 선택`이면 담긴 상품 목록이, `조건 검색`이면 수집 조건 패널이 나타납니다. 반대쪽 설정은 저장되어 있다가 다시 바꾸면 살아납니다.

**메뉴 상단 노출을 지정한 그룹은 예외 — 담긴 상품이 우선, 없으면 조건**

쇼핑특가의 `추천 특가` 처럼 메뉴 맨 위에 뜨는 캐러셀은 구성 방식을 고르지 않습니다. 대신 다음 순서로 채워집니다.

| 순서 | 조건 | 노출되는 것 |
|---|---|---|
| ① | **담긴 상품이 1개 이상** | 담은 상품이 담은 순서 그대로 |
| ② | 담긴 상품이 하나도 없음 | **수집 조건**으로 자동 수집한 결과 |
| ③ | 담긴 상품도 없고 수집 조건도 비어 있음 | **아무것도 노출되지 않습니다** |

그래서 화면에 **담긴 상품**과 **수집 조건**이 동시에 보입니다. 특가 상품을 직접 고르고 싶으면 담고, 신경 쓰기 싫으면 비워 두고 조건에만 맡기면 됩니다.

### 8-2. 수동 선택 — 상품 담기

**담긴 상품** 영역에서 상품 조회 창을 열어 담습니다. 조회 창에서는 다음으로 거를 수 있습니다.

| 필터 | 값 |
|---|---|
| **검색어** | 상품명·상품코드 |
| **카테고리** | 전체 또는 특정 카테고리. **이 몰에 상품이 있는 카테고리만** 나옵니다. |
| **브랜드** | 입력칸을 **클릭만 해도** 이 몰에 상품이 있는 브랜드가 상품 수와 함께 목록으로 뜹니다. 브랜드명·초성으로 검색해 좁히고, 오른쪽 ⊗ 로 비웁니다. |
| **재고** | 전체 / 있음 / 없음 |
| **노출** | 전체 / PUBLIC / HIDDEN / MEMBER_ONLY |

> **카테고리·브랜드 목록은 이 몰이 실제로 쓰는 것만 나옵니다.** 전 몰 공용 한 벌(카테고리 2,300여 개 · 브랜드 1,300여 개)을 그대로 그리면 읽을 수 없고, 대부분은 이 몰에 상품이 없어 골라도 0건이기 때문입니다. 찾는 브랜드가 안 보이면 **그 브랜드 상품이 이 몰에 아직 없다는 뜻**입니다. 쇼핑특가·추천 그룹·기획전의 상품 조회 창도 모두 같은 방식입니다.

여러 상품을 체크해 **한 번에** 담을 수 있습니다. 이미 담긴 상품·다른 몰 상품은 자동으로 제외됩니다.

담긴 상품은 **위에서 아래 순서로 노출**되며, 화살표 버튼으로 순서를 바꾸고 **제거** 버튼으로 뺍니다.

> **담긴 상품의 추가·삭제·순서 변경은 즉시 저장됩니다.** 기본 정보와 달리 별도의 저장 버튼이 필요 없습니다.

> **메뉴 상단 노출을 지정한 그룹은 후보 상품이 좁혀집니다.** 특가 메뉴에 건 그룹이면 [쇼핑특가 관리]의 특가 상세에 등록한 상품 중 **아직 종료되지 않은 특가의 상품**만, 베스트 메뉴면 **베스트 랭킹 상위 50위**만, 신상품 메뉴면 **신상품**만 고를 수 있습니다. "특가 중 선택"이 되도록 설계된 동작이며, 같은 내용이 **담긴 상품** 영역 위 안내에도 표시됩니다.

### 8-3. 조건 검색 — 자동 수집 조건

| 입력란 | 설명 |
|---|---|
| **배지** | `BEST` / `NEW` / `RECOMMEND` / `DEADLINE_SALE` / `GREENHUB_SPECIAL` 중 하나, 또는 전체 |
| **카테고리** | 특정 카테고리로 한정 |
| **최소 할인율 (%)** | 이 할인율 이상인 상품만. 비우면 제한 없음 |
| **재고가 있는 상품만** | 체크하면 품절 상품이 빠집니다 |
| **정렬** | `최신순` / `할인율 높은순` / `가격 낮은순` / `가격 높은순` / `조회수순` / `등록순 (최신순과 동일)` |

> **모든 조건은 AND 로 결합됩니다.** 조건을 여러 개 걸면 그 모두를 만족하는 상품만 모입니다. 조건을 좁게 걸면 결과가 0건이 되기 쉽습니다.

> **판매중·노출중 상품만 대상입니다.** 판매중지·비공개 상품은 조건에 맞아도 들어오지 않습니다.

기본 정보와 수집 조건을 정한 뒤 **저장** 버튼을 누릅니다.

### 8-4. 미리보기

화면 아래 **미리보기** 영역이 **스토어프론트와 같은 경로로 뽑은 실제 결과**를 최대 12개까지 보여 줍니다(비로그인 기준). 조건 검색 그룹이 실제로 무엇을 골라 오는지 여기서 확인하세요.

> **"조건에 맞는 상품이 없습니다"가 뜨면 이 그룹을 쓰는 섹션도 빈 상태로 노출됩니다.** 조건을 넓히거나 상품을 더 등록하세요.

> **메뉴 상단 노출 그룹은 제목 옆에 출처가 표시됩니다.** `담긴 상품` 이면 직접 담은 상품이, `수집 조건 (담긴 상품 없음)` 이면 자동 수집 결과가 나가고 있다는 뜻입니다.

### 8-5. 삭제·비활성화 제한

목록 화면에서는 각 그룹이 **어느 섹션에 쓰이고 있는지** 함께 표시됩니다.

> **섹션이 쓰고 있는 그룹은 삭제할 수 없습니다.** `섹션 N개가 이 그룹을 사용 중이라 삭제할 수 없습니다.` 오류가 뜹니다. 먼저 그 섹션을 지우거나 데이터 소스를 다른 그룹으로 바꾸세요.

> **비활성화(`사용` 끄기)도 같은 제한을 받습니다.** 활성 섹션이 쓰고 있으면 끌 수 없습니다. 끄면 그 섹션이 조용히 빈 상태가 되기 때문입니다.

## 9. 자주 막히는 상황

**Q. 섹션을 고쳤는데 고객 화면이 그대로입니다.**
**발행**을 누르지 않았습니다. 상단에 `미발행 변경사항` 배지가 떠 있는지 확인하세요.

**Q. 섹션에 "프론트 미노출" 배지가 붙었습니다.**
배지 아래에 이유가 적혀 있습니다. 비활성·기간·기기 문제는 오른쪽 설정에서 바로 고치고, 데이터가 없는 경우에는 옆에 붙은 버튼으로 해당 관리 화면에 가서 채우세요.

**Q. 상품 캐러셀이 비어 있습니다.**
① 상품 그룹을 연결했는가, ② 그 그룹의 `사용`이 켜져 있는가, ③ 그룹 미리보기에 상품이 나오는가 를 확인하세요.

**Q. 조건 검색 그룹에 상품이 하나도 안 모입니다.**
조건이 너무 좁습니다. 배지·카테고리·최소 할인율을 하나씩 풀어 보세요. 판매중지·비공개 상품은 애초에 들어오지 않습니다.

**Q. 상품 그룹을 삭제할 수 없습니다.**
어떤 섹션이 그 그룹을 쓰고 있습니다. 오류 메시지에 섹션 번호가 나오니, 그 섹션의 데이터 소스를 바꾸거나 삭제하세요.

**Q. 메뉴에 상품 그룹을 걸려고 하니 오류가 납니다.**
그 메뉴에는 이미 다른 그룹이 걸려 있습니다. 한 메뉴에 하나만 가능합니다.

**Q. 프로모션 배너 섹션이 비어 있습니다.**
섹션의 **배너 그룹 키**와 배너 관리에 등록한 배너의 그룹 키가 정확히 같아야 합니다.

**Q. 히어로 섹션에 아무것도 안 나옵니다.**
배너 관리 &gt; 메인 슬라이더에서 내용을 등록해야 합니다. 페이지 빌더의 테마 히어로 섹션은 **배치만** 정합니다.

**Q. "최근 본 상품" 섹션이 관리자에게 비어 보입니다.**
정상입니다. 방문자마다 다르게 보이는 섹션이라 관리자 화면에서는 비어 보입니다.

**Q. 테마를 바꿨더니 꾸며 둔 섹션이 사라졌습니다.**
`홈 섹션까지 기본 구성으로 교체` 체크박스를 켜고 적용했습니다. 되돌릴 수 없으니, 발행 이력이 있다면 **롤백**을 시도하세요.

**Q. 잘못 발행했습니다.**
상단의 **리비전 선택(롤백)** 에서 이전 리비전을 고르고 **롤백**을 누르세요.

## 10. 알아 두면 좋은 것

- **홈 말고 다른 페이지도 편집·발행할 수 있습니다.** 페이지 선택 목록에 `미발행` 이라 표시된 페이지가 있다면 한 번 발행해 두는 편이 안전합니다.
- **섹션은 복제가 됩니다.** 같은 설정으로 값만 바꾼 캐러셀을 여러 개 만들 때 편합니다.
- **상품 그룹 하나를 여러 섹션이 함께 쓸 수 있습니다.** 대신 그 그룹을 끄거나 지우면 모든 섹션이 함께 영향을 받습니다.
