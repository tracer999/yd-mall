# 회원 관리

쇼핑몰에 **가입한 회원을 찾아보고, 한 명의 활동 내역을 전부 확인하는 화면**입니다. 그 회원이 주문을 몇 번 했는지, 쿠폰을 언제 받아 어디에 썼는지, 포인트가 어떻게 늘고 줄었는지를 한 페이지에서 봅니다. 문제가 있는 계정은 **비활성화**해 로그인을 막을 수 있습니다.

> ⚠️ **`삭제` 버튼은 회원 정보를 실제로 지웁니다.** 휴지통도 복구도 없습니다. 그래서 **탈퇴했거나 활동이 전혀 없는 회원만** 지울 수 있게 막아 두었습니다. 나머지는 삭제가 아니라 **비활성화**로 처리합니다. 4장을 반드시 읽으세요.

> 🏢 **이 목록에는 기업(사업자) 회원이 나오지 않습니다.** 기업회원은 승인 상태·계약기간·전용 할인율과 함께 봐야 하므로 <span class="manual-path">B2B 관리 &gt; 기업회원 승인</span>에서 따로 관리합니다. 목록 위에 기업회원이 몇 명인지와 바로 가는 링크가 표시됩니다. 자세한 내용은 [B2B(사업자) 관리](/manual/admin/b2b) 문서를 보세요.

> **회원은 몰마다 나뉘지 않습니다.** 어느 몰을 편집 중이든 이 목록에는 **가입한 회원 전체**가 나옵니다. 몰별로 다른 것은 회원 자체가 아니라 **멤버십 등급**이며, 그것은 멤버십 관리에서 다룹니다.

## 1. 회원 찾기

목록 위에 **상태 셀렉트**와 **검색창**이 있습니다.

| 조건 | 설명 |
|---|---|
| **상태** | `전체 상태` / `활성` / `탈퇴` 중 선택합니다. |
| **검색어** | **이름 · 이메일 · 전화번호** 중 어디에든 포함되면 찾아 줍니다. 일부만 입력해도 됩니다. |

값을 넣고 **`검색`** 을 누릅니다. 결과는 **가입일 최신순**으로 정렬됩니다.

> **`탈퇴` 필터는 "스스로 탈퇴한 회원"만 봅니다.** 관리자가 비활성화한 회원은 여기에 포함되지 않습니다. 비활성 회원까지 보려면 `전체 상태`로 검색한 뒤 이름 옆 배지로 구분하세요.

## 2. 목록에서 보이는 것

| 열 | 내용 |
|---|---|
| **ID** | 회원 번호. 문의 응대나 확인 요청 때 이 번호로 특정하면 정확합니다. |
| **이름** | 클릭하면 회원 상세로 갑니다. 상태에 따라 **`탈퇴`**(빨강) 또는 **`비활성`**(회색) 배지가 붙습니다. |
| **생년월일** | 없으면 `-`. 생일 쿠폰 발급의 기준이 되는 값입니다. |
| **가입일** | 회원가입한 날짜. |
| **최근 접속** | 마지막으로 로그인한 날짜. 휴면 회원을 가려낼 때 봅니다. |
| **약관 버전** | 이 회원이 동의한 `이용약관`·`개인정보` 버전. 없으면 `-`. |
| **마케팅 동의** | `동의` / `미동의`. **미동의 회원에게 광고성 메시지를 보내면 안 됩니다.** |
| **로그인** | 가입 경로 — `Kakao` / `Google` / `Email`. |
| **관리** | `비활성`(또는 `활성`) 버튼과 `삭제` 버튼. **`삭제`가 회색으로 눌리지 않으면** 지울 수 없는 회원입니다 — 버튼에 마우스를 올리면 이유가 뜹니다(4-2 참고). |

## 3. 회원 상세 보기

이름을 클릭하면 그 회원의 모든 활동이 한 페이지에 나옵니다. **오른쪽 위 `목록으로 돌아가기`** 로 되돌아옵니다.

### 3-1. 상단 요약

프로필 이미지, 이름, 상태 배지(`활성`/`비활성`/`탈퇴`), 가입 경로 배지, 회원 ID, 이메일이 표시됩니다.

**탈퇴한 회원이면** 맨 위에 빨간 상자로 **탈퇴일**과 **탈퇴 사유**가 따로 나옵니다. (사유를 안 적고 탈퇴했으면 `미입력`)

### 3-2. 기본 정보 · 약관 동의 정보

| 구역 | 항목 |
|---|---|
| **기본 정보** | 이름 / 이메일 / 전화번호 / 생년월일 / 주소 / 상세 주소 / 우편번호 |
| **약관 동의 정보** | 이용약관 버전 / 개인정보 처리방침 버전 |

> **여기의 값은 조회만 됩니다.** 관리자가 회원의 이름·연락처·주소를 **고칠 수 없습니다.** 정보 수정은 고객이 직접 마이페이지에서 해야 합니다. 배송지를 잘못 넣은 주문이라면 회원 정보가 아니라 그 **주문 건**을 다뤄야 합니다.

### 3-3. 계정 상태 카드

활성 여부 / 마케팅 동의 / 가입일 / 최근 로그인 / **보유 포인트**가 한눈에 보입니다.

### 3-4. 발급된 쿠폰 및 사용 내역

이 회원이 받은 쿠폰이 **최근 100건**까지 나옵니다.

| 열 | 내용 |
|---|---|
| **쿠폰명** | 쿠폰 이름(없으면 코드). |
| **할인액** | 그 쿠폰의 할인 금액. |
| **지급일** | 발급된 시각. |
| **지급유형** | `자동`(시스템 발급) / `관리자`(직접 지급) / `코드`(고객이 코드 입력). |
| **사용일** | 아직 안 썼으면 `-`. |
| **주문번호** | 어느 주문에 썼는지. 클릭하면 그 주문 상세로 갑니다. |

**"쿠폰을 받았는데 없어졌다"는 문의는 여기서 바로 확인됩니다.** 사용일과 주문번호가 찍혀 있으면 이미 쓴 것입니다.

### 3-5. 포인트 내역

포인트가 오간 기록이 **최근 100건**까지 나옵니다. 유형은 네 가지입니다.

| 유형 | 뜻 |
|---|---|
| **구매적립** | 주문으로 쌓인 포인트. |
| **구매사용** | 주문 결제에 쓴 포인트. |
| **관리자지급** | 관리자가 직접 넣어 준 포인트. |
| **관리자차감** | 관리자가 직접 뺀 포인트. |

늘어난 건 초록(`+`), 줄어든 건 빨강으로 표시됩니다. 주문이 취소되면 여기에 환급·회수 기록이 함께 남습니다.

### 3-6. 주문 내역

이 회원의 주문이 **최근 100건**까지 나옵니다. 맨 위에 **결제완료 기준 주문 금액 합계**가 표시됩니다.

> **이 합계는 `결제완료` 상태 주문만 더한 값입니다.** `배송중`·`배송완료`로 넘어간 주문은 여기에 안 잡힙니다. **이 회원이 지금까지 쓴 총액이 아닙니다.** 실제 누적 실적을 보려면 멤버십의 **회원 등급 현황**에서 인정금액을 확인하세요.

## 4. 비활성화 · 삭제 (주의)

목록의 **관리** 열에 버튼 두 개가 있습니다.

### 4-1. 비활성 / 활성 (권장)

`비활성` 버튼을 누르면 확인 창이 뜹니다 — *"이 회원을 비활성화하시겠습니까?"*

확인하면 그 회원은 **비활성** 상태가 되고 이름 옆에 회색 배지가 붙습니다. **회원 정보·주문·쿠폰·포인트는 전부 그대로 남습니다.**

같은 자리 버튼이 `활성`으로 바뀌므로, **언제든 다시 되돌릴 수 있습니다.** 이것이 안전한 방법입니다.

부정 사용·스팸·분쟁 중인 계정은 삭제하지 말고 **비활성화**하세요. 나중에 확인해야 할 근거가 남습니다.

### 4-2. 삭제 (되돌릴 수 없음)

> ⚠️ **회원 정보를 데이터베이스에서 실제로 지웁니다. 복구할 수 없습니다.**

**아무 회원이나 지울 수 있는 것이 아닙니다.** 실수로 활동 중인 고객을 지우는 일을 막기 위해, 아래 둘 중 하나에 해당할 때만 `삭제` 버튼이 활성화됩니다.

| 지울 수 있는 회원 | 설명 |
|---|---|
| **탈퇴한 회원** | 고객이 스스로 탈퇴한 계정. 이름 옆에 빨간 `탈퇴` 배지가 있습니다. |
| **활동이 전혀 없는 회원** | **비활성** 상태이면서 **주문 0건**, **보유 포인트 0P** 인 계정. 가입만 하고 쓰지 않은 계정이나 테스트 계정이 여기 해당합니다. |

그 외에는 `삭제`가 **회색으로 비활성화**되고, 마우스를 올리면 막힌 이유가 표시됩니다.

| 뜨는 메시지 | 뜻과 대응 |
|---|---|
| *이용 중인 회원입니다. 비활성 처리 후 삭제할 수 있습니다.* | 아직 활성 계정입니다. 꼭 지워야 하면 **먼저 `비활성`을 누른 뒤** 다시 보세요. |
| *주문 이력이 N건 있습니다. 삭제할 수 없습니다.* | 주문 기록이 있는 고객입니다. **비활성화로 처리하세요.** |
| *보유 포인트가 NP 남아 있습니다. 삭제할 수 없습니다.* | 포인트가 남아 있습니다. 정산·소멸 처리 후에 다시 보세요. |
| *기업회원입니다. B2B 관리 &gt; 기업회원 승인에서 처리해 주세요.* | 기업회원은 이 화면에서 다루지 않습니다. |

삭제할 수 있는 회원이면 *"이 회원을 완전히 삭제합니다…"* 확인 창이 뜨고, 확인하면 즉시 삭제됩니다.

삭제 시 유의할 점:

- **되돌릴 방법이 없습니다.** 휴지통도 복구 기능도 없습니다.
- 그 회원의 **장바구니·포인트 내역·쿠폰·리뷰·문의가 함께 삭제**됩니다.
- 그 회원이 남긴 **주문 기록은 남지만, 주문자 정보가 비어 `비회원`으로 표시**됩니다. (다만 주문이 있으면 애초에 삭제가 막힙니다.)

**삭제는 개인정보 파기 요청처럼 지워야 할 법적 근거가 있을 때만 사용하세요.** 그 외 모든 경우에는 비활성화가 정답입니다.

> **테스트용으로 만든 계정을 지울 때도 주의하세요.** 그 계정으로 남긴 테스트 주문의 주문자 정보가 함께 사라져 판매 관리 화면이 `비회원`투성이가 됩니다.

## 5. 자주 막히는 상황

**Q. 회원 정보를 관리자가 고칠 수 없나요?**
없습니다. 이름·연락처·주소 등은 조회만 됩니다. 고객이 직접 마이페이지에서 수정해야 합니다.

**Q. 비활성화한 회원이 `탈퇴` 필터에 안 나옵니다.**
`탈퇴`는 고객이 스스로 탈퇴한 경우만 해당합니다. 관리자가 비활성화한 회원은 `전체 상태`로 검색해서 회색 `비활성` 배지로 찾으세요.

**Q. 회원의 등급을 바꾸고 싶습니다.**
이 화면에는 등급 기능이 없습니다. 멤버십 관리의 **회원 등급 현황**에서 바꿉니다.

**Q. 회원에게 포인트나 쿠폰을 주고 싶습니다.**
포인트는 **포인트 관리**, 쿠폰은 **쿠폰 관리**에서 지급합니다. 지급 결과는 이 회원 상세에서 확인할 수 있습니다.

**Q. "결제완료 기준 합계"가 실제보다 훨씬 적습니다.**
`결제완료` 상태 주문만 더하기 때문입니다. 배송이 진행된 주문은 빠집니다(위 3-6 참고).

**Q. `삭제` 버튼이 회색이라 눌리지 않습니다.**
지울 수 없는 회원입니다. 버튼에 마우스를 올리면 이유가 뜹니다 — 활성 계정이거나, 주문 이력이 있거나, 포인트가 남아 있는 경우입니다. 대부분은 **비활성화**로 처리하는 것이 맞습니다(4-2 참고).

**Q. 사업자(기업) 회원이 목록에 안 보입니다.**
정상입니다. 기업회원은 승인 상태·계약기간과 함께 봐야 하므로 <span class="manual-path">B2B 관리 &gt; 기업회원 승인</span>에서 관리합니다. 목록 위 파란 안내줄의 링크로 바로 갈 수 있습니다.

**Q. 기업회원을 비활성화하거나 삭제하려는데 막힙니다.**
의도된 동작입니다. 이 화면에서 기업회원의 계정을 끄면 기업회원 승인 화면에는 여전히 `승인`으로 보여 두 화면이 어긋납니다. 거래를 중지하려면 **기업회원 승인 화면의 `정지`** 를 쓰세요.

**Q. 마케팅 동의를 관리자가 켜 줄 수 있나요?**
없습니다. 동의는 고객 본인만 할 수 있습니다. **`미동의` 회원에게 광고성 메시지를 보내면 안 됩니다.**

## 6. 아직 지원하지 않는 것

- 관리자의 회원 정보 수정
- 관리자의 회원 직접 등록
- 회원 목록 엑셀 내려받기
- 회원 비밀번호 초기화
- 회원 등급 조정 (멤버십 관리에서 처리)
- 회원 대상 단체 메일·문자 발송
