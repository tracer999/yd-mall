# B2B(사업자) 관리

일반 소비자에게 파는 것과 **같은 몰·같은 상품·같은 재고**로, **사업자 고객에게만 다른 가격과 다른 주문 절차**를 적용하는 기능입니다.

B2B 전용 쇼핑몰을 따로 만들지 않습니다. 고객이 **사업자로 가입하고 승인**을 받으면, 그 고객이 로그인했을 때만 같은 화면에서 **가격 · 최소 주문수량 · 주문 버튼 · 결제 방식**이 바뀝니다. 일반 고객이 보는 화면은 전혀 달라지지 않습니다.

화면 위치: <span class="manual-path">관리자 &gt; B2B 관리</span> (`/admin/b2b/members`)

> **B2B 기능은 모든 몰에서 항상 켜져 있습니다.** 몰마다 켜고 끄는 설정이 없습니다. 다만 **승인된 사업자 회원이 한 명도 없고, `B2B 판매`를 켠 상품이 하나도 없으면 아무 변화도 일어나지 않습니다.** 즉 아무것도 설정하지 않은 몰은 지금까지와 똑같이 동작합니다.

> ⚠️ **주문을 `승인`하면 그 즉시 재고가 빠집니다.** 입금 전이라도 빠집니다. 자세한 이유와 회수 방법은 5장을 보세요.

> **설정은 최대한 단순하게 두었습니다.** 상품마다 정하는 건 *팔 것인가 · 몇 개부터 · 몇 % 할인* 세 가지뿐입니다. 그 밖의 조건은 **견적 단계에서 협의**합니다. 필요해지면 그때 기능을 더합니다.

## 1. 전체 흐름 한눈에 보기

```
① 상품에 B2B 할인율 넣기     (상품 등록/수정 화면 안)
        ↓
② 고객이 사업자로 가입·신청   (고객 화면)
        ↓
③ 기업회원 승인             ← 사업자등록증 확인
        ↓
④ 고객이 전용가로 주문 요청   (결제 아님)
        ↓
⑤ 주문 승인 → 재고 차감 → 입금 안내
        ↓
⑥ 입금 확인 → 출고
```

**①을 안 하면 ④에서 아무 일도 안 생깁니다.** 승인된 사업자라도 B2B 판매를 켜지 않은 상품은 일반 판매가로 삽니다.

**수량별 단가·납기 같은 조건은 미리 정해 두지 않습니다.** 필요하면 ④ 대신 **견적**(6장)에서 담당자가 협의합니다.

---

## 2. 메뉴 구성

| 메뉴 | 무엇을 하나 |
|------|------------|
| **기업회원 승인** | 사업자 신청을 검토하고 승인·반려·정지합니다 |
| **거래처 할인** | 거래처마다 추가로 얹어 줄 할인율을 정합니다 |
| **B2B 주문** | 기업 주문의 승인·입금확인·세금계산서를 처리합니다 |
| **견적 관리** | 견적 요청을 받아 단가를 제안하고 협상합니다 |
| **B2B 설정** | 입금 기한·계좌 안내·세액 표기 등 공통 설정 |

**상품별 B2B 설정은 이 메뉴가 아니라 상품 등록/수정 화면 안에 있습니다**(3장).

---

## 3. 상품에 B2B 할인율 넣기 — 여기서 시작합니다

**별도 화면이 없습니다.** <span class="manual-path">관리자 &gt; 상품 관리</span> 에서 상품을 **등록하거나 수정할 때**, 가격 설정 바로 아래 **`B2B(사업자) 판매`** 섹션에서 함께 정합니다.

### 체크박스를 켜야 항목이 나옵니다

**「이 상품을 사업자 고객에게도 판매」** 를 체크하면 아래 세 항목이 펼쳐집니다.

| 항목 | 설명 |
|---|---|
| **B2B 할인율 (%)** | 판매가 대비 몇 % 깎아 줄지. 입력하면 **적용가가 바로 옆에 표시**됩니다 |
| **최소 주문수량** | 몇 개부터 살 수 있는지. **1이면 1개부터** 가능 |
| **과세 구분** | 과세(세금계산서) / 면세(계산서) / 영세율 |

체크를 끄면 그 상품은 **사업자에게도 일반 판매가**로 팔립니다. 일반 고객 화면은 어느 쪽이든 바뀌지 않습니다.

> **가격은 금액이 아니라 할인율로 정합니다.** 나중에 판매가를 바꾸면 전용가도 **자동으로 따라 움직입니다.** 상품마다 전용가 금액을 다시 계산해 넣을 필요가 없습니다.

> **수량별 단가·납기·결제조건 같은 조건은 여기서 정하지 않습니다.** 거래처가 대량 주문을 요청하면 **견적**(6장)에서 담당자가 그때그때 협의합니다. 미리 다 설정해 두려 하지 마세요.

## 4. 기업회원 승인

### 신청은 고객이 합니다

고객은 두 경로로 신청합니다.

- **사업자로 새로 가입** — 회원가입 화면에서 `사업자 회원가입`
- **이미 가입한 회원이 전환 신청** — 마이페이지 > `사업자 회원 신청`

두 경우 모두 상호 · 사업자등록번호 · 대표자명 · 업태/종목 · 세금계산서 이메일 · 담당자 · **사업자등록증 사본**을 받습니다.

### 승인 화면에서 할 일

목록에서 신청 건을 열면 왼쪽에 **신청 내용**, 오른쪽에 **[등록증 열기]** 버튼이 있습니다.

1. **[등록증 열기]** 를 눌러 사업자등록증을 봅니다.
2. **상호 · 사업자등록번호 · 대표자명이 등록증과 같은지** 대조합니다.
3. 맞으면 **[승인]**, 아니면 사유를 적고 **[반려]**.

> **이 대조가 진위 확인의 전부입니다.** 시스템은 사업자등록번호의 **형식(체크섬)만** 검사합니다. 형식이 맞아도 실재하지 않는 번호일 수 있으므로, 등록증을 반드시 눈으로 확인하세요.

### 상태의 의미

| 상태 | 고객 화면에서 |
|---|---|
| **심사 대기 · 심사 중** | 전용가가 보이지 않습니다. "승인 후 확인 가능" 안내만 |
| **승인** | 전용가와 사업자 주문 절차 사용 |
| **반려** | 사유를 보고 다시 신청할 수 있습니다 |
| **정지** | 기업 거래 차단. 일반 회원으로는 계속 구매 가능 |

승인 · 반려 · 정지 시 **신청 계정 이메일로 안내가 발송**됩니다.

### 거래 조건

승인 화면 아래에서 거래처마다 **추가 할인율 · 계약 기간 · 관리자 메모**를 설정합니다. **계약 기간이 지나면 전용가가 자동으로 멈춥니다**(일반가로 돌아갑니다).

## 5. B2B 주문 — 즉시 결제가 아닙니다

일반 주문은 고객이 카드로 결제하면 끝입니다. **기업 주문은 결제 없이 접수만 되고**, 담당자가 확인한 뒤 입금 안내를 보냅니다.

```
접수         고객이 주문 요청. 재고는 그대로
  ↓
승인         ← 여기서 재고가 빠집니다. 입금 기한 시작
  ↓
입금 확인     담당자가 통장을 보고 확인
  ↓
출고         이후는 일반 주문과 동일 (배송 관리에서)
```

### 왜 승인할 때 재고를 빼나

입금까지 기본 **7일**이 걸립니다. 그동안 재고를 잡아 두지 않으면, 마지막 재고를 두 거래처가 동시에 승인받고 **한쪽은 입금하고도 물건을 못 받는** 일이 생깁니다.

> ⚠️ **그래서 승인한 주문은 반드시 회수하거나 확정해야 합니다.** 미입금 상태로 방치하면 그 수량은 계속 팔 수 없습니다.

### 기한이 지난 주문 회수하기

B2B 주문 목록 맨 위에 **입금 기한이 지난 주문**이 빨간 상자로 뜹니다. 체크한 뒤 **[선택 주문 취소 · 재고 회수]** 를 누르면 주문이 취소되고 재고가 되돌아옵니다.

> **자동으로 취소되지 않습니다.** 담당자가 이 화면에서 직접 회수해야 합니다.

### 처리 버튼

| 버튼 | 언제 |
|---|---|
| **검토 시작** | 접수 건을 살펴보는 중이라고 표시 |
| **승인 · 입금 안내** | 재고 차감 + 입금 기한 설정 + 고객에게 계좌 안내 메일 |
| **입금 확인** | 통장에 입금이 찍힌 뒤. 주문이 확정되고 출고 준비로 넘어갑니다 |
| **반려 · 재고 회수** | 사유를 적으면 고객에게 안내됩니다. 재고는 자동 복원 |

**재고가 부족하면 승인되지 않습니다.** 수량을 조정하거나 반려하세요.

### 세금계산서

주문 상세 오른쪽에서 발행 상태(미발행 / 요청 / 발행 완료 / 취소)와 승인번호를 기록합니다.

> **발행 자체는 시스템이 하지 않습니다.** 홈택스나 쓰시는 계산서 서비스에서 발행한 뒤, 여기에 **상태와 승인번호만 기록**하는 화면입니다.

### 출고·배송·클레임은 기존 메뉴에서

주문번호가 `B2B-` 로 시작할 뿐, 출고·송장 입력·취소·반품은 **[배송 관리]**, **[클레임 관리]** 를 그대로 씁니다. B2B 전용 화면을 따로 쓰지 않습니다.

## 6. 견적 · 협상

대량 주문이라 단가를 협의해야 할 때 씁니다. 고객이 장바구니에서 **[견적 요청]** 을 누르면 시작됩니다.

```
고객: 견적 요청 (희망 단가·수량)
  ↓
관리자: 검토 시작 → 단가·배송비·납기·유효기간 제안
  ↓
고객: 수락  또는  재제안(다시 협의)
  ↓
수락되면 단가 확정 → 주문으로 전환
```

### 관리자가 할 수 있는 것

품목별 **수량 · 제안 단가 · 메모**, 전체 **배송비 · 할인 · 유효기간 · 결제조건 · 납기**를 정해 **[이 조건으로 제안하기]** 를 누릅니다.

- **협상 메시지** — 고객과 주고받는 대화. `내부 메모로 남기기` 를 체크하면 **고객에게 보이지 않습니다**.
- **변경 이력** — 금액·조건이 바뀔 때마다 v1, v2, v3… 로 남습니다. **덮어써서 사라지지 않습니다.**
- **견적서 미리보기 / 발행·보관** — PDF 로 만듭니다. `발행·보관` 을 누르면 그 시점 내용이 파일로 저장되어 나중에 "그때 무엇을 보냈는지" 확인할 수 있습니다.

### 주문으로 전환

수락된 견적에서 **[주문으로 전환]** 을 누르면 B2B 주문이 만들어집니다.

> **전환 시점의 상품 가격을 다시 계산하지 않습니다.** 합의한 확정 단가를 그대로 씁니다. 그 사이 상품 가격이 올랐든 내렸든 상관없습니다.

> 전환으로 만들어진 주문도 **접수 상태**입니다. 재고는 5장처럼 **승인할 때** 빠집니다.

**유효기간이 지나면 자동으로 만료**되어 전환할 수 없습니다.

## 7. 거래처 할인

<span class="manual-path">관리자 &gt; B2B 관리 &gt; 거래처 할인</span>

특정 거래처에 **더 얹어 줄 할인율**입니다. 상품에 정한 B2B 할인율과 **단순히 더해집니다.**

```
최종 할인율 = 상품 할인율 + 거래처 추가 할인율

예) 판매가 10,000원 · 상품 30% · A상사 추가 5%
    → 35% 할인 → 6,500원
```

- 목록에서 거래처마다 숫자를 넣고 **[저장]** 하면 끝입니다.
- 추가 할인이 필요 없는 거래처는 **0** 으로 둡니다(기본값).
- 기업회원 승인 상세 화면의 **거래 조건**에서도 같은 값을 고칠 수 있습니다.

> **상품별로 거래처마다 다른 단가를 정하는 기능은 없습니다.** 그렇게까지 세밀한 조정이 필요하면 **견적**에서 품목별 단가를 직접 제안하세요. 견적에서 합의한 단가가 모든 것에 우선합니다.

## 8. B2B 설정

<span class="manual-path">관리자 &gt; B2B 관리 &gt; B2B 설정</span>

한 번도 저장하지 않아도 **괄호 안 기본값으로 동작**합니다.

| 항목 | 기본값 | 설명 |
|---|---|---|
| **B2B 가격 표기** | 공급가 별도 | 기업 화면에 `공급가 + VAT` 로 나눠 보여줍니다 |
| **쿠폰·포인트 사용 허용** | 사용 안 함 | 전용가는 이미 계약가라 기본은 중복 적용하지 않습니다 |
| **B2B 무료배송 기준액** | 비움 | 비우면 일반 배송정책을 그대로 따릅니다 |
| **입금 기한(일)** | 7 | **길수록 미입금 주문이 재고를 오래 잡습니다** |
| **입금 계좌 안내** | 비움 | 승인 안내 메일과 고객 화면에 그대로 노출됩니다 |
| **견적 기본 유효기간(일)** | 14 | |
| **가입 즉시 자동 승인** | 사용 안 함 | ⚠️ 켜면 **등록증을 확인하지 않고** 전용가가 열립니다. 테스트 전용 |

## 9. 자주 겪는 일

| 증상 | 원인 · 확인할 곳 |
|---|---|
| 승인했는데 고객 화면에 전용가가 안 보인다 | 그 상품의 **B2B 판매 체크가 꺼져 있습니다.** 상품 수정 화면의 `B2B(사업자) 판매` 섹션 |
| 할인율을 넣었는데 값이 그대로다 | 할인율이 **0** 이거나, 고객이 **개인 구매 모드**로 로그인했습니다 |
| 고객이 "장바구니에 담기지 않는다"고 한다 | 최소 주문수량에 걸렸습니다. 화면에 사유가 표시됩니다 |
| 고객이 개인 상품과 기업 상품을 같이 못 담는다 | **의도된 동작입니다.** 가격·주문절차가 달라 한 장바구니에 섞을 수 없습니다 |
| 재고가 있는데 판매되지 않는다 | **승인했지만 미입금인 주문**이 재고를 잡고 있습니다. B2B 주문 화면 상단의 기한초과 목록을 확인하세요 |
| 승인 메일이 안 간다 | 시스템 설정 > SMTP 설정을 확인하세요. `example.com` 같은 주소는 **의도적으로 차단**됩니다 |
| 견적을 주문으로 못 바꾼다 | 유효기간이 지났거나, 이미 전환됐거나, 확정 단가가 없습니다(다시 수락하세요) |
