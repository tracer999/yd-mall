# 상품 관리

쇼핑몰에서 **실제로 파는 물건을 만드는 화면**입니다. 상품명·가격·이미지·상세설명을 여기서 넣고, 색상이나 용량처럼 **고객이 고르는 옵션(SKU)** 도 여기서 붙입니다. **재고는 SKU 로 관리**하므로 등록할 때 한 번 넣고, 이후에는 옵션·SKU 관리 화면에서 고칩니다.

이 화면은 쇼핑몰의 **가장 아래층**입니다. 기획전·베스트·쇼핑특가·세트상품은 모두 "여기 등록된 상품을 골라 쓰는" 화면이라서, 상품이 없으면 그 화면들은 만들 수가 없습니다. 반대로 이 화면에서 상품을 **판매중지**로 바꾸면, 그 상품을 담아 둔 모든 화면에서 함께 팔리지 않게 됩니다.

## 2. 전체 흐름 한눈에 보기

```
① 상품 등록 화면 열기
        ↓
② 기본 정보 입력      (상품명 · 카테고리 · 브랜드 · 판매 상태 · 노출 설정)
        ↓
③ 가격 설정           (판매가 필수)
        ↓
④ 이미지 · 상세 설명 입력
        ↓
⑤ [상품 등록] 저장    ← 여기까지 해야 상품이 "생깁니다"
        ↓
⑥ (색상·용량 등이 있으면) 옵션·SKU 붙이기
        ↓
⑦ 고객 화면에서 확인
```

**⑤번 저장 전에는 [옵션·SKU 관리] 버튼이 나오지 않습니다.** 옵션은 상품을 한 번 만든 뒤에 붙이는 작업입니다.

### 3-1. 검색과 필터

목록 맨 위에서 아래 조건을 조합해 상품을 찾습니다.

| 항목 | 설명 |
|---|---|
| **검색어** | 상품명·브랜드·카테고리 이름에서 찾습니다. |
| **재고 여부** | `전체` / `재고 있음` / `재고 없음`. 옵션상품은 **모든 옵션 재고를 더한 값**으로 판단합니다. |
| **상태** | `판매중` / `판매중지` / `품절` / `출시예정` / `재입고예정` |
| **노출 여부** | `전체공개` / `숨김` / `회원전용` |
| **사업자 상품** | `전체` / `사업자 상품` / `일반 상품`. B2B(사업자) 전용가를 설정한 상품만, 또는 그 외 일반 상품만 골라 봅니다. |
| **상품 출처** | 그 상품이 **어디서 온 것인지**로 거릅니다. `몰상품` / `도매꾹` / `도매매` / `온채널` / `네이버 스토어` |
| **카테고리 (대 / 중 / 소분류)** | 3단계로 좁혀 갑니다. 상위 분류를 고르면 **그 아래 하위 분류 상품까지** 함께 나옵니다. |
| **브랜드** | 브랜드 하나를 고릅니다. `미설정 (브랜드 없음)` 을 고르면 브랜드를 넣지 않은 상품만 나옵니다. |

조건을 넣었으면 **검색**을 누릅니다. 조건이 걸려 있을 때만 **초기화** 버튼이 나타납니다.
한 페이지에 보여 줄 개수는 `10 / 20 / 30 / 50개씩 보기` 중에서 고릅니다.

### 3-2. 목록에 보이는 정보

이미지 · 상품명/카테고리 · 재고 · 정가 · 판매가 · 판매시작일 · 상태 · 노출관리 · 관리 순으로 표시됩니다.

- 상품명 아래 첫 번째 표시가 **출처**입니다. 직접 등록한 상품은 회색 **`몰상품`**, 외부에서 가져온 상품은 **`도매꾹`**·**`도매매`**·**`온채널`**·**`네이버 스토어`** 로 색이 붙어 나옵니다.
- 옵션이 있는 상품은 이름 옆에 **`옵션 N종`**, 없으면 **`단일`** 표시가 붙습니다.
- 옵션상품의 **재고**는 모든 옵션 재고의 합계, **판매가**는 옵션 가격의 범위로 보입니다.
- 판매시작일이 최근이면 **`신상품`** 표시가 함께 붙습니다.
- 판매시작일이 없으면 **`미지정`** 으로 나옵니다.

> **출처는 상품이 만들어질 때 자동으로 정해지며, 나중에 바꿀 수 없습니다.**
> 이 화면에서 **[상품 등록]** 이나 **[일괄 등록·수정]** 으로 만든 상품은 전부 `몰상품` 입니다.
> `도매꾹`·`도매매`·`온채널` 은 **외부몰 연동 → 가져온 상품(중간)** 에서 등록한 상품에 붙습니다.
>
> 외부에서 가져온 상품은 **원래 가격·재고가 공급처 사정으로 바뀔 수 있고**, 카테고리·브랜드가 비어 있는 채로 들어옵니다.
> 그래서 여기서 값을 고치기 전에 출처를 먼저 확인하는 것이 좋습니다.
>
> `네이버 스토어` 는 **네이버에서 가져온** 상품이라는 뜻입니다. 우리 몰 상품을 네이버에 **내보낸** 것과는 다릅니다 — 내보내도 출처는 그대로 유지됩니다.

**상품명을 누르면 상세 보기 화면**이 열립니다. 수정 화면에 들어가지 않고도 그 상품의 설정값·가격·판매 이력을 한 번에 확인할 수 있습니다. (9장 참고)

오른쪽 **관리** 칸에는 **[수정]** · **[옵션·SKU]** · **[삭제]** 버튼이 있습니다.
**[삭제]** 는 **그 줄의 상품 하나만** 지웁니다 — 왼쪽 체크박스를 켜 두었든 아니든 상관없습니다. 다만 아무 상품이나 지워지는 것은 아니고 **판매중지 상태이면서 주문 이력이 없어야** 합니다. (8장 참고)

### 3-3. 여러 상품 한꺼번에 바꾸기

왼쪽 체크박스로 상품을 여러 개 고른 뒤, 목록 위쪽 버튼을 누릅니다.

| 버튼 | 하는 일 |
|---|---|
| **판매중 처리** | 고른 상품을 모두 `판매중` 으로 바꿉니다. |
| **품절 처리** | 고른 상품을 모두 `품절` 로 바꿉니다. |
| **판매중지 처리** | 고른 상품을 모두 `판매중지` 로 바꿉니다. |
| **판매 시작일 지정** | 옆 날짜 칸에 넣은 날짜를 고른 상품에 한꺼번에 넣습니다. |
| **카테고리 매핑처리** | 고른 상품의 **비어 있는 카테고리**를 자동으로 채웁니다. 기존 카테고리에만 연결합니다. 외부에서 가져온 상품 정리에 씁니다. (10장 참고) |
| **브랜드 매핑처리** | 고른 상품의 **비어 있는 브랜드**를 자동으로 채웁니다. 판매사 기준이며 없으면 새로 만듭니다. |
| **B2B(사업자) 판매등록** | 고른 상품을 사업자 전용가로 판매하도록 등록합니다. 누르면 할인율·최소 주문수량을 입력하는 창이 뜹니다. |

> **판매 상태를 바꿔도 옵션(SKU)의 `판매`/`중지` 는 그대로입니다.** 둘은 다른 것을 뜻합니다. 상품의 판매 상태는 **상품 자체를 파느냐**이고, 옵션의 상태는 **그 옵션을 고를 수 있게 하느냐**입니다. 상품을 판매중지하면 고객 화면·장바구니·결제가 모두 막히므로 옵션까지 함께 내릴 필요가 없고, 내려 두면 나중에 판매를 다시 켰을 때 **고를 수 있는 옵션이 하나도 없는 상태**가 됩니다.

> **날짜 칸을 비운 채 [판매 시작일 지정]을 누르면 날짜가 지워집니다.** 지워진 상품은 **신상품에서 빠집니다.** 실수로 전체 선택 후 빈 날짜로 누르면 신상품 코너가 통째로 비게 되니 주의하세요.

**노출관리** 칸의 선택 상자(`노출` / `노출중지` / `로그인전용`)는 **고르는 즉시 저장**됩니다. 따로 저장 버튼이 없습니다.

### 4-1. URL 로 상품 가져오기 (등록 화면에만 있음)

다른 쇼핑몰의 **상품 상세 페이지 주소**를 넣고 **가져오기**를 누르면, 그 페이지에서 읽을 수 있는 상품명·가격·이미지·설명을 아래 폼에 채워 줍니다.

- **저장은 되지 않습니다.** 값을 확인하고 직접 **[상품 등록]** 을 눌러야 상품이 만들어집니다.
- 읽을 수 있는 항목만 채워집니다. 못 읽은 칸은 비어 있으니 직접 채우세요.
- 이 영역은 **새로 등록할 때만** 나옵니다. 수정 화면에는 없습니다.

### 4-2. 이름과 코드

| 입력란 | 필수 | 설명 | 예시 |
|---|---|---|---|
| **상품명** | 필수 | 고객에게 보이는 이름입니다. | `유기농 홍삼정 240g` |
| **상품코드** | 선택 | 내부 관리용 코드입니다. | `GH-001` |
| **상품 URL (슬러그)** | 선택 | 상품 주소에 들어가는 영문 이름입니다. **비우면 상품명으로 자동 생성**되고, 같은 값이 이미 있으면 `-1`, `-2` 가 붙습니다. | `hongsam-240g` |

> **슬러그는 나중에 바꾸지 마세요.** 고객에게 상품 링크를 뿌린 뒤 이 값을 바꾸면 **기존 링크가 전부 깨집니다.** 문자·카카오톡으로 발송한 링크도 마찬가지입니다.

### 4-3. 카테고리와 브랜드

**카테고리** — 두 가지 방법으로 지정합니다.

1. **목록에서 고르기** — `1단계 선택` → `2단계` → `3단계` 순으로 좁혀 갑니다.
2. **직접 입력** — 목록에 없으면 아래 칸에 이름을 적습니다. `대>중>소` 형태로 경로를 적으면 계층까지 만들어 줍니다. (예: `여성신발>부츠/워커>미들부츠`)

> **직접 입력은 "비슷한 게 있으면 연결, 없으면 새로 생성"입니다.** 남발하면 비슷비슷한 카테고리가 계속 늘어납니다. **가능하면 목록에서 고르세요.**
> 카테고리를 아무것도 지정하지 않으면 상품은 **`미분류`** 로 들어갑니다. (목록·검색에서 사라지지 않게 하기 위한 안전장치입니다.)

**브랜드** — 목록에서 고르거나, 없으면 아래 칸에 브랜드명을 직접 적습니다. 카테고리와 같은 규칙으로 비슷한 브랜드에 연결되거나 새로 만들어집니다. 브랜드는 비워 둘 수 있습니다.

### 4-4. 판매 상태 — 값마다 무슨 일이 벌어지나

| 판매 상태 | 고객 화면에서 |
|---|---|
| **판매중** | 정상적으로 구매할 수 있습니다. |
| **품절** | 상품은 보이지만 구매할 수 없습니다. |
| **중지(판매중단)** | 판매를 내립니다. |
| **판매예정** | 아직 팔지 않는 예정 상품으로 표시됩니다. |
| **재입고예정** | 재입고 예정 상품으로 표시됩니다. |

### 4-5. 노출 설정 — 값마다 무슨 일이 벌어지나

| 노출 설정 | 동작 |
|---|---|
| **노출 (기본)** | 누구에게나 보입니다. |
| **노출중지 (프론트 전체 숨김)** | 고객 화면 어디에도 나오지 않습니다. |
| **로그인전용 (로그인 회원만 노출)** | 로그인한 회원에게만 보입니다. |

> **판매 상태와 노출 설정은 서로 다른 스위치입니다.** `판매중` 이어도 `노출중지` 면 고객은 상품을 볼 수 없습니다. "등록했는데 안 보인다"의 가장 흔한 원인입니다.

### 4-6. 판매 시작일 (신상품 판정 기준)

- 이 날짜로부터 **정해진 기간 동안 신상품으로 자동 노출**됩니다. 기간은 입력란 아래 안내 문구에 표시됩니다.
- **새로 등록할 때는 오늘 날짜가 미리 들어가 있습니다.**
- **비우면 신상품에 포함되지 않습니다.**

### 4-7. 재고 수량

**재고는 SKU 로 관리합니다.** 그래서 이 칸은 화면에 따라 다르게 나옵니다.

| 화면 | 재고 칸 |
|---|---|
| **새 상품 등록** | 숫자를 넣는 입력칸. 여기 넣은 값이 그 상품의 **기본 SKU** 재고가 됩니다. |
| **상품 수정** | 입력칸이 없습니다. 현재 재고를 보여주고 **[옵션·SKU 에서 수정]** 버튼이 붙습니다. |

수정 화면에서 재고를 못 고치게 한 이유는 **재고를 넣는 곳이 두 군데면 서로 덮어쓰기 때문**입니다. 등록한 뒤로는 옵션·SKU 관리 화면이 재고를 고치는 유일한 곳입니다. (7장 참고)

> **여기 보이는 재고는 "팔 수 있는 재고"입니다.** 옵션상품이면 모든 옵션 재고의 합계이고, 상태가 `중지` 인 옵션의 재고는 빼고 셉니다.

### 4-8. 유통채널 구분

| 값 | 동작 |
|---|---|
| **미선택 (뱃지 미노출)** | 아무 표시도 붙지 않습니다. 기본값입니다. |
| **온라인판매가능** | 상품에 `온라인판매가능` 표시가 붙습니다. |

### 4-9. 상품구분 (복수 선택 가능)

체크한 만큼 상품에 표시가 붙습니다. 여러 개를 함께 켤 수 있습니다.

| 항목 | 동작 |
|---|---|
| **BEST** | 베스트 표시가 붙습니다. |
| **NEW** | **판매 시작일 기준 신상품 기간이 지나도 신상품에 강제로 남습니다.** |
| **추천** | 추천 표시가 붙습니다. |
| **기간임박할인** | 임박 할인 표시가 붙습니다. 체크하면 아래에 **기간임박할인 만료일** 칸이 나타납니다. |
| **와이디몰특가** | 몰 자체 특가 표시가 붙습니다. |

> **`기간임박할인` 만료일이 지나면 그 상품은 자동으로 품절 처리됩니다.** 날짜를 짧게 넣어 두고 잊어버리면 어느 날 갑자기 안 팔립니다. 만료일은 반드시 의도한 날짜로 넣으세요.

## 7. 4단계 — 옵션 · SKU 붙이기 (색상·용량 등)

색상·용량처럼 **고객이 고르는 항목**이 있는 상품은 옵션을 붙입니다. 옵션 조합마다 가격과 재고를 따로 관리할 수 있습니다. 이 조합 하나하나를 **SKU** 라고 부릅니다.

**상품을 먼저 저장해야 합니다.** 저장하면 수정 화면 아래쪽에 **[옵션·SKU 관리]** 버튼이 생깁니다.

### 7-1. 만드는 순서

```
① [+ 옵션 추가] → 옵션명·옵션값 입력   (예: 색상 / 블랙, 화이트)
        ↓
② [조합 생성] 누르기                   (색상 2개 × 용량 3개 = 6줄 자동 생성)
        ↓
③ 줄마다 판매가 · 재고 · SKU코드 · 상태 입력
        ↓
④ [저장]
```

### 7-2. 옵션 입력

| 입력란 | 설명 | 예시 |
|---|---|---|
| **옵션명** | 고객이 고를 항목의 이름 | `색상` |
| **옵션값** | 고를 수 있는 값들. **쉼표로 구분**해 한 줄에 적습니다. | `블랙, 화이트, 네이비` |

**[카테고리 추천 옵션 불러오기]** 버튼을 누르면 이 상품의 카테고리에 미리 정해 둔 추천 옵션이 채워집니다. 지정된 추천 옵션이 없으면 안내 창이 뜹니다.

### 7-3. SKU 입력

표에는 두 종류의 줄이 나옵니다.

| 줄 | 언제 나오나 | 삭제 |
|---|---|---|
| **기본** (회색 배경, `기본` 뱃지) | 옵션이 없는 상품 | **삭제 불가** |
| **옵션 조합** | **[조합 생성]** 을 누르면 조합마다 한 줄 | **[삭제]** 버튼으로 지울 수 있음 |

**[조합 생성]** 을 누르면 옵션값의 모든 조합이 한 줄씩 만들어집니다. 기본 줄은 이때도 그대로 남습니다.

| 칸 | 설명 |
|---|---|
| **판매가** | 이 줄의 가격. 처음에는 상품 판매가가 들어갑니다. |
| **재고** | 이 줄의 재고. **상품의 실제 재고는 이 값입니다.** |
| **SKU코드** | 줄마다 붙이는 내부 관리 코드 (선택). |
| **상태** | `판매` / `중지` |

- 조합 줄을 `중지` 로 두면 **그 옵션만** 고를 수 없게 됩니다.
- **기본 줄을 `중지` 로 두면 그 상품이 통째로 품절이 됩니다.** 판매 상태는 `판매중` 인데 살 수는 없는 상태가 되니, 상품을 아예 안 팔 거라면 상태를 여기서 내리지 말고 **상품의 판매 상태를 `판매중지`** 로 바꾸세요.

**[저장]** 을 눌러야 반영됩니다. 저장하면 화면 위 표시가 `OPTION` 으로 바뀝니다.

> **기본 줄은 왜 못 지우나요?** 모든 상품은 SKU 를 최소 한 줄 가져야 재고가 잡힙니다. 기본 줄이 없으면 그 상품은 재고 0(품절)이 됩니다. 그래서 지울 수 없게 막아 뒀습니다.

> **기본 줄에서 고친 가격·재고는 상품의 값이 됩니다.** 상품 수정 화면의 재고 표시도 이 값을 따라갑니다.

> **옵션을 붙여 저장하면 기본 줄이 사라집니다.** 옵션상품은 고객이 조합을 골라서 사기 때문에 기본 줄이 필요 없고, 남겨 두면 아무도 살 수 없는 재고가 합계에 섞입니다. 반대로 **옵션을 모두 비우고 저장하면 기본 줄이 다시 생깁니다.**

> **조합 줄을 [삭제]하고 저장하면 그 조합은 팔지 않게 됩니다.** 잠깐만 안 팔 거라면 지우는 대신 **상태를 `중지`** 로 두세요. 재고와 SKU코드가 남아 나중에 그대로 되살릴 수 있습니다.

> **세트·묶음 구성에 쓰이고 있는 SKU 는 지울 수 없습니다.** 저장할 때 `세트/묶음 구성에 사용 중인 SKU가 있어 변경할 수 없습니다.` 메시지가 뜹니다. 세트 쪽 구성을 먼저 풀어야 합니다.

## 8. 상품 삭제

지우는 곳은 두 군데이고, 둘 다 **같은 규칙**으로 동작합니다.

| 위치 | 방법 |
|---|---|
| **상품 목록** | 오른쪽 **관리** 칸의 **[삭제]** — 그 줄의 상품 하나만 지웁니다. **왼쪽 체크박스와는 아무 상관이 없습니다.** 체크를 하나도 하지 않아도, 다른 상품을 체크해 두었어도, 지워지는 것은 버튼을 누른 그 상품뿐입니다. |
| **상품 수정 화면** | 맨 아래 **[상품 삭제]** |

어느 쪽이든 확인 창이 한 번 뜨고, 삭제되면 목록 위에 **초록 띠**로 결과가 나옵니다. 목록에서 지운 경우 **보고 있던 검색·필터·페이지가 그대로 유지**됩니다.

### 8-1. 지울 수 있는 상품의 조건

**아래 세 가지를 모두 만족해야 지워집니다.** 하나라도 걸리면 삭제되지 않고, 목록 위에 **빨간 띠**로 이유가 나옵니다.

| 조건 | 안 지워질 때 나오는 말 | 해야 할 일 |
|---|---|---|
| **판매 상태가 `판매중지`** | `… 은(는) 판매중지 상태가 아니라 삭제할 수 없습니다.` | 먼저 그 상품을 **[판매중지 처리]** 한 뒤 다시 삭제하세요. `품절`·`출시예정`·`재입고예정` 도 마찬가지로 먼저 판매중지로 바꿔야 합니다. |
| **주문 이력이 없음** | `… 은(는) 주문 N건에 포함돼 있어 삭제할 수 없습니다.` | **지울 수 없습니다.** 판매중지 상태로 두세요. |
| **세트·기획상품의 구성이 아님** | `… 은(는) 세트·기획상품의 구성으로 쓰이고 있어 삭제할 수 없습니다.` | 세트·기획상품 관리에서 그 구성에서 먼저 빼세요. |

> **한 번이라도 팔린 상품은 지울 수 없습니다. 이건 막아 둔 것이지 오류가 아닙니다.**
> 지워 버리면 **과거 주문에서 "무엇을 팔았는지"가 사라집니다.** 주문 내역에는 그 줄이 남지만 어떤 상품이었는지 연결이 끊겨, 나중에 매출을 확인하거나 고객 문의를 처리할 때 확인할 방법이 없습니다.
> 그래서 오래된 주문 한 건이라도 있으면 삭제를 막습니다. 더 이상 팔지 않으려는 것이라면 **`판매 상태 = 중지(판매중단)`** 로 두면 됩니다 — 고객 화면에서는 똑같이 사라집니다.

### 8-2. 삭제하면 함께 사라지는 것

> **삭제하면 되돌릴 수 없습니다.** 그 상품의 옵션·SKU, 추가 이미지, **고객이 쓴 리뷰**, 기획전·쇼핑특가·공동구매·아울렛에 넣어 둔 연결, 좋아요, 장바구니에 담겨 있던 것까지 함께 사라집니다. 다른 상품이 쓰지 않는 이미지 파일도 정리됩니다.
>
> **판매를 잠깐 멈추려는 것이라면 삭제하지 마세요.** `판매 상태 = 중지(판매중단)` 또는 `노출 설정 = 노출중지` 를 쓰면 고객 화면에서만 빠지고 자료는 그대로 남습니다.

### 9-1. 화면 구성

```
① 위쪽    이미지 · 상품명 · 판매가 · 판매 상태 · 재고   ← 고객에게 보이는 모습
        ↓
② 가운데  기본 설정 / 가격 정보                        ← 관리자용 설정값 요약
        ↓
③ 아래쪽  상세 정보 · 관련 영상 · AI 추천 (눌러서 펼침)
        ↓
④ 맨 아래  판매 이력
```

오른쪽 위 버튼 네 개로 이동합니다.

| 버튼 | 하는 일 |
|---|---|
| **SEO 보기** | 검색엔진에 이 상품이 어떻게 보이는지 새 창으로 확인합니다. |
| **목록으로** | 상품 목록으로 돌아갑니다. |
| **수정하기** | 이 상품의 수정 화면을 엽니다. **값을 고치려면 여기로** 들어갑니다. |
| **옵션·SKU 관리** | 색상·용량 등 옵션 화면으로 갑니다. (7장 참고) |

### 9-2. 기본 설정 — 여기서 확인되는 값

수정 화면의 **기본 정보**에 넣은 값이 그대로 나옵니다.

| 항목 | 비어 있을 때 표시 |
|---|---|
| **상품 출처** | 항상 값이 있습니다. 직접 등록한 상품은 `몰상품`, 외부에서 가져온 상품은 `도매꾹`·`도매매`·`온채널`·`네이버 스토어` 로 나오고 옆에 "외부에서 가져온 상품입니다." 안내가 붙습니다. |
| **상품코드** | `-` |
| **카테고리** | `미분류` |
| **브랜드** | `-` |
| **판매 상태** | (4-4 표의 값이 그대로) |
| **노출 설정** | `노출 (기본)` |
| **재고 수량** | `0개` — **팔 수 있는 재고**(SKU 합계)입니다. 옵션상품이면 모든 옵션 재고를 더한 값이고, 상태가 `중지` 인 옵션은 빠집니다. |
| **판매 시작일** | `-` — 비어 있으면 **신상품에 포함되지 않습니다.** |
| **유통채널 구분** | `미선택 (뱃지 미노출)` |
| **상품구분** | `-` — 켜 둔 표시가 있으면 `BEST` `NEW` 처럼 나란히 보입니다. |
| **기간임박 만료일** | 상품구분에서 `기간임박할인` 을 켠 상품에만 나타납니다. |
| **네이버 카테고리** | `설정됨` / `미설정` |

> **"고객 화면에 안 보인다" 는 이 표를 먼저 보세요.** `판매 상태` 와 `노출 설정` 두 줄만 확인하면 원인의 대부분이 잡힙니다. 수정 화면까지 들어갈 필요가 없습니다.

> **`상품구분` 에 `기간임박할인` 이 있으면 바로 아래 만료일을 함께 확인하세요.** 그 날짜가 지나면 상품이 자동으로 품절 처리됩니다.

### 9-3. 가격 정보

> **사업자(B2B) 판매도 이 화면에서 함께 정합니다.** 가격 설정 바로 아래 **`B2B(사업자) 판매`** 섹션의 체크박스를 켜면 **B2B 할인율 · 최소 주문수량** 두 항목이 나옵니다. (과세 구분은 상품이 아니라 **거래처**에 정합니다) 켜지 않으면 사업자 고객에게도 일반 판매가로 팔립니다. 자세한 내용은 [B2B(사업자) 관리](/manual/admin/b2b) 3장을 보세요.

> **이 체크를 켜면 고객 상품 목록에도 표시가 생깁니다.** 카드 왼쪽 위에 파란 **`B2B` 뱃지**가 붙고 상단 **`B2B 상품` 필터**로 걸러집니다 — **비로그인 고객에게도 보입니다.** 금액(전용가)은 승인된 사업자에게만 보입니다.


| 항목 | 설명 |
|---|---|
| **판매가** | 실제 결제되는 금액입니다. |
| **정가** | 취소선 표시용. 넣지 않았으면 `-` |
| **할인율** | `SALE` 표시에 쓰이는 숫자입니다. 실제로 깎지 않습니다. |
| **원가 (관리자 전용)** | 매입 단가입니다. 넣지 않았으면 `-` |

> **원가는 관리자에게만 보입니다.** 고객 화면 어디에도 나오지 않습니다. 다만 **상품 상세를 볼 수 있는 운영자라면 누구나 이 값을 보게 되므로**, 원가를 알리고 싶지 않은 담당자가 있다면 운영자 권한 설정을 함께 확인하세요.

### 9-4. 판매 이력

맨 아래에 **그 상품의 판매 이력**이 나옵니다. 주문번호·주문일시·주문자·수량·금액·상태가 최신순으로 표시되고, 주문번호를 누르면 해당 주문 상세로 이동합니다.

결제가 끝난 주문(**결제완료·준비중·배송중·배송완료**)만 집계되며, **취소·환불된 주문은 빠집니다.** 그래서 여기 금액 합계는 정산 금액과 다를 수 있습니다.

## 10. 자주 막히는 상황

**Q. 이 상품이 지금 어떻게 설정돼 있는지 빠르게 보고 싶습니다.**
목록에서 **상품명을 눌러 상세 보기**로 들어가세요. 판매 상태·노출 설정·재고·판매 시작일·상품구분·가격이 한 화면에 정리돼 있습니다. 수정 화면을 열 필요가 없습니다. (9장 참고)

**Q. 상품을 등록했는데 고객 화면에 안 보입니다.**
목록에서 상품명을 눌러 **상세 보기의 `기본 설정`** 을 먼저 확인하면 빠릅니다. 아래를 순서대로 보세요.
1. **노출 설정**이 `노출 (기본)` 인가? (`노출중지` 면 어디에도 안 나옵니다)
2. **판매 상태**가 `중지(판매중단)` 는 아닌가?
3. **편집 중인 몰**이 맞는가? (다른 몰에 등록했을 수 있습니다)
4. **카테고리**가 `미분류` 로 들어가 특정 분류 화면에서만 안 보이는 것은 아닌가?

**Q. 이 상품이 우리가 직접 등록한 건지, 외부에서 가져온 건지 알고 싶습니다.**
목록에서 **상품명 아래 첫 번째 표시**를 보세요. 회색 `몰상품` 이면 이 화면에서 직접 등록한 것이고, `도매꾹`·`도매매`·`온채널`·`네이버 스토어` 처럼 색이 있으면 외부에서 가져온 상품입니다. 상세 보기 화면 맨 위에서도 확인할 수 있습니다.
목록 위 **상품 출처** 필터로 한 종류만 모아 볼 수도 있습니다.

**Q. 출처를 `몰상품` 으로 바꾸고 싶습니다.**
바꿀 수 없습니다. 출처는 **상품이 만들어질 때 자동으로 정해지는 기록**이라 나중에 고치는 칸이 없습니다. 굳이 바꾸려면 그 상품을 지우고 이 화면에서 새로 등록해야 하는데, 그러면 지금까지의 판매 이력·리뷰가 함께 사라지므로 권하지 않습니다.

**Q. 가져온 상품인데 카테고리·브랜드가 비어 있습니다.**
외부에서 가져온 상품은 공급처에 우리 몰 분류가 없어 비어 있는 채로 들어옵니다. 목록에서 **상품 출처**로 그 공급처를 고르고 **카테고리**를 `미설정` 으로 함께 걸면 채워야 할 상품만 남습니다. 전체 선택 후 **[카테고리 매핑처리]** · **[브랜드 매핑처리]** 를 누르세요. (3-3 참고)

**Q. 할인율에 30 을 넣었는데 가격이 그대로입니다.**
정상입니다. 할인율은 `SALE` 표시에 쓰이는 **표시용 숫자**입니다. 실제로 깎으려면 판매가를 낮추거나 쇼핑특가를 만드세요.

**Q. [옵션·SKU 관리] 버튼이 안 보입니다.**
아직 상품을 저장하지 않았습니다. 기본 정보를 먼저 저장하면 수정 화면에 나타납니다.

**Q. 옵션 재고를 바꿨는데 목록의 재고가 안 맞습니다.**
목록의 재고는 **팔 수 있는 옵션 재고의 합계**입니다. 상태가 `중지` 인 옵션은 재고가 있어도 빠지니, 합계가 적게 보이면 그 옵션의 상태부터 확인하세요.

**Q. 상품 수정 화면에 재고 입력칸이 없습니다.**
정상입니다. 재고는 **옵션·SKU 관리** 화면에서만 고칩니다. 수정 화면에는 현재 재고와 **[옵션·SKU 에서 수정]** 버튼만 있습니다. 넣는 곳이 두 군데면 서로 덮어쓰기 때문에 한 곳으로 모았습니다. (새 상품을 **등록**할 때는 재고 입력칸이 있습니다.)

**Q. 카테고리를 직접 입력했더니 비슷한 카테고리가 여러 개 생겼습니다.**
직접 입력은 "비슷한 게 없으면 새로 만드는" 동작입니다. 카테고리 관리에서 정돈하고, 앞으로는 목록에서 고르세요.

**Q. 신상품 코너에 상품이 안 나옵니다.**
`판매 시작일` 이 비어 있거나 기간이 지났습니다. 날짜를 넣거나, 기간이 지나도 계속 노출하려면 **상품구분에서 `NEW` 를 체크**하세요.

**Q. 잘 팔던 상품이 갑자기 품절로 바뀌었습니다.**
**상품구분 `기간임박할인`** 을 켜 두고 **기간임박할인 만료일**이 지났을 수 있습니다. 만료일이 지나면 자동으로 품절 처리됩니다.

**Q. 목록에서 [삭제] 를 눌렀는데 아무 일도 일어나지 않습니다.**
확인 창에서 **취소**를 눌렀는지 먼저 확인하세요. 확인을 눌렀는데도 상품이 그대로라면 목록 위 **빨간 띠**에 이유가 적혀 있습니다 — 판매중이거나, 주문 이력이 있거나, 세트 구성으로 쓰이는 상품입니다. (8-1 참고)

**Q. [삭제] 를 누르면 체크해 둔 상품들이 같이 지워지나요?**
아니요. **버튼을 누른 그 줄의 상품 하나만** 지워집니다. 체크박스는 위쪽의 일괄 처리 버튼(판매중 처리·매핑처리 등)에만 쓰입니다.

**Q. 다 팔린 옛날 상품을 정리하고 싶은데 삭제가 안 됩니다.**
주문 이력이 있는 상품은 지울 수 없습니다. 지우면 과거 주문에서 무엇을 팔았는지 확인할 수 없게 되기 때문입니다. **판매 상태를 `중지(판매중단)`** 로 바꾸면 고객 화면에서는 사라지므로 정리 목적은 같습니다. 목록에서 **상태 = `판매중지`** 로 걸러 두면 관리 화면에서도 섞이지 않습니다.

**Q. 상품 목록에 세트·기획상품이 안 보입니다.**
정상입니다. 세트·묶음·선물세트·선택형세트는 별도 메뉴에서 관리합니다.

**Q. 옵션을 지우려는데 저장이 안 됩니다.**
그 옵션(SKU)이 세트·기획상품의 구성으로 쓰이고 있습니다. 세트 쪽 구성에서 먼저 빼세요.

**Q. 상품 목록에 표시된 가격과 고객이 보는 가격이 다릅니다.**
그 상품에 **쇼핑특가**가 걸려 있을 수 있습니다. 특가가 진행 중이면 고객은 특가 금액으로 결제합니다.

## 11. 상품 일괄 등록·수정 (엑셀)

목록 위 **[일괄 등록·수정]** 버튼으로 들어갑니다. 초기 상품을 한 번에 올리거나, 가격·재고를 주기적으로 일괄 조정할 때 씁니다.

<div class="manual-goto-bar">
  <a class="manual-goto is-sub" href="/admin/products/bulk" target="_blank">일괄 등록·수정 열기</a>
</div>

### 11-1. 순서

1. **[현재 상품 내려받기]** 를 누릅니다. **이 파일이 곧 수정 양식**입니다.
2. 엑셀에서 고칠 값을 바꿉니다. 새 상품은 아래에 줄을 추가하면 됩니다.
3. **[다른 이름으로 저장] → 파일 형식 `CSV UTF-8(쉼표로 분리)`** 로 저장합니다.
4. 파일을 올리고 **[일괄 반영]** 을 누릅니다.

> ⚠️ **`.xlsx` 파일은 그대로 읽지 못합니다.** 반드시 **CSV UTF-8** 로 저장해 올리세요.

### 11-2. 동작 규칙 — 이것만 알면 됩니다

| 규칙 | 뜻 |
|---|---|
| **상품코드가 있으면 수정, 없으면 등록** | 내려받은 파일을 고쳐 올리면 그게 곧 수정입니다. 상품코드를 비우면 언제나 새 상품입니다 |
| **빈 칸은 "그대로 둠"** | 가격만 고치려면 가격 칸만 채우면 됩니다. 나머지 값은 손대지 않습니다 |
| **새 상품은 판매가 필수** | 재고를 비우면 0으로 등록됩니다 |
| **카테고리·브랜드는 이름으로** | 없는 이름이면 그 줄만 실패합니다. [카테고리 관리](/admin/categories)에서 먼저 만드세요 |
| **한 줄이 실패해도 나머지는 반영** | 실패한 줄만 고쳐 다시 올리면 됩니다 |
| **지금 편집 중인 몰에 반영** | 다른 몰에 넣으려면 몰을 바꾼 뒤 올리세요 |

> 값을 **일부러 비우려면** 상품 수정 화면에서 지워야 합니다. 파일의 빈 칸으로는 지워지지 않습니다 —
> 가격만 고치려고 올린 파일이 설명·분류를 통째로 지우는 사고를 막기 위해서입니다.

### 11-3. 파일의 칸

`상품코드 · 상품명(필수) · 카테고리 · 브랜드 · 판매가 · 정가 · 매입가 · 재고 · 판매상태 · 노출 · 과세 · 제조사 · 간단설명`

순서는 상관없고, 필요 없는 칸은 빼도 됩니다.

| 칸 | 넣을 수 있는 값 |
|---|---|
| **판매상태** | 판매중 · 판매중지 · 품절 · 입고예정 · 재입고 |
| **노출** | 공개 · 숨김 · 회원전용 |
| **과세** | 과세 · 면세 · 영세 |

> **이미지 · 상세설명 · 옵션 · SKU 는 이 파일로 다루지 않습니다.** 표로 표현할 수 없는 값이라
> 상품 등록·수정 화면에서 개별로 작업해야 합니다.

### 11-4. 결과 확인

**전체 몇 줄 / 새로 등록 몇 건 / 수정 몇 건 / 실패 몇 건**이 나오고, 실패한 줄은 **몇 번째 줄이 왜 실패했는지** 표로 보여 줍니다.
성공한 줄을 다시 올려도 같은 값으로 덮어쓸 뿐이라 안전합니다.

## 12. 아직 지원하지 않는 것

- 엑셀로 **이미지·상세설명·옵션** 넣기
- 등록한 상품을 다른 몰로 복사·이동
- 상품 삭제 취소(휴지통)
- 엑셀로 상품 **삭제**하기
