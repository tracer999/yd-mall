# 멤버십 관리

회원을 **구매 실적에 따라 등급**(예: 베이직 → 실버 → 골드 → VIP)으로 나누고, 등급마다 **할인 · 추가 적립 · 무료배송 · 쿠폰**을 자동으로 주는 기능입니다. 등급을 만들고, 어떤 실적으로 그 등급이 되는지 기준을 정하고, 평가를 돌려 회원을 배치하는 것까지 이 메뉴에서 합니다.

화면 위치: <span class="manual-path">관리자 &gt; 멤버십 관리</span> (`/admin/membership`)

> **등급·혜택·기준은 모두 몰별로 따로 저장됩니다.** 관리자 상단의 **편집 중인 몰**이 A몰인데 B몰에 만들 생각이었다면, 저장해도 B몰에는 아무것도 생기지 않습니다. 설정을 시작하기 전에 반드시 확인하세요.

> ⚠️ **`지금 평가 실행`은 전 회원의 등급을 실제로 바꿉니다.** 되돌리는 버튼이 없습니다. 반드시 **시뮬레이션**으로 먼저 확인하세요(6장).

## 1. 전체 흐름 한눈에 보기

```
① 등급 만들기            (베이직 / 실버 / 골드 …)
        ↓
② 등급별 혜택 정하기      (할인 % · 적립 % · 무료배송 · 쿠폰)
        ↓
③ 평가 정책 정하기        (실적 기간 · 인정금액 산식 · 승급/강등 방식)
        ↓
④ 등급별 진입/유지 기준 입력
        ↓
⑤ 시뮬레이션으로 확인     (실제로 반영되지 않음)
        ↓
⑥ 평가 실행 → 회원 배치
```

**①②를 먼저 해야 ④가 가능합니다.** 등급이 하나도 없으면 평가 정책 화면의 기준 표가 비어 있어 아무것도 입력할 수 없습니다.

## 3. 1단계 — 등급 만들기

<div class="manual-goto-bar">
  <span class="manual-goto-label">이 단계 화면</span>
  <a class="manual-goto" href="/admin/membership/grades" target="_blank">등급 관리 열기</a>
</div>

목록 오른쪽 위 **`등급 등록`** 을 누르면 입력 화면이 열립니다.

### 3-1. 기본 정보

| 입력란 | 필수 | 설명 | 예시 |
|---|---|---|---|
| **등급명** | 필수 | 고객에게 보이는 이름입니다. | `골드` |
| **등급 코드** | 필수 | 시스템이 쓰는 영문 코드. **등록할 때만 입력할 수 있고 이후 수정 불가**입니다. | `GOLD` |
| **순위 (1이 최상위)** | 선택 | 숫자가 **작을수록 높은 등급**입니다. 기본값 100. | `20` |
| **노출 색상** | 선택 | 등급 배지 색. 기본값은 회색입니다. | `#d4af37` |
| **설명** | 선택 | 관리용 메모. | `연 100만원 이상 구매` |
| **마이페이지 안내 문구** | 선택 | 고객 마이페이지의 등급 영역에 뜨는 한 줄. | `골드 회원 전용 혜택을 누리세요` |

> **등급 코드는 나중에 바꿀 수 없습니다.** 수정 화면에서는 회색으로 잠깁니다. 등록 전에 신중히 정하세요.

> **순위는 반드시 서로 다르게, 간격을 두고 매기세요.** `10 / 20 / 30 / 40` 처럼 두면 나중에 중간 등급을 끼워 넣기 쉽습니다. 순위가 같은 등급이 둘 있으면 승급·강등 판정이 뒤엉킵니다.

### 3-2. 체크박스 3개

| 항목 | 켜면 |
|---|---|
| **기본 가입 등급** | 신규 회원이 처음 받는 등급이 됩니다. **몰당 하나만** 가능하며, 다른 등급을 기본으로 지정하면 기존 기본은 자동으로 해제됩니다. |
| **사용** | 이 등급을 실제로 씁니다. 끄면 평가에서도 제외되고 고객 화면 등급표에도 안 나옵니다. |
| **자동 평가 대상** | 자동 승급·강등의 대상이 됩니다. **끄면 관리자가 수동으로만 지정할 수 있는 등급**이 됩니다(임직원 등급 등). |

> **기본 가입 등급이 없으면 신규 회원이 등급 없이 남습니다.** 등급을 처음 만들 때 가장 낮은 등급 하나는 반드시 `기본 가입 등급`으로 지정하세요.

### 3-3. 등급 혜택

먼저 **사용 여부 스위치 3개**가 있습니다.

| 스위치 | 끄면 |
|---|---|
| **할인 사용** | 아래 할인율이 들어 있어도 **결제에 적용되지 않습니다.** |
| **적립 사용** | 아래 적립률이 적용되지 않습니다. |
| **배송 혜택 사용** | 무료배송 설정이 적용되지 않습니다. |

> **값을 지우지 않고 혜택만 잠시 멈출 때 씁니다.** 예를 들어 이벤트 기간에 등급 할인을 잠깐 중단하고 싶으면 할인율을 `0`으로 지웠다가 되돌리는 대신 **`할인 사용` 체크만 해제**하면 됩니다. 나중에 다시 켜면 원래 값이 그대로 살아납니다.

**할인 관련**

| 입력란 | 설명 |
|---|---|
| **주문 정률 할인 (%)** | 상품 금액에서 이 비율만큼 깎아 줍니다. |
| **할인 최대액 (원, 비우면 무제한)** | 할인 상한. **비워 두면 상한이 없습니다.** |
| **할인 최소 주문금액 (원)** | 이 금액 이상 주문해야 할인이 붙습니다. |

> ⚠️ **`할인 최대액`을 비워 두면 상한 없이 할인됩니다.** 5% 할인 등급에서 500만 원 주문이 들어오면 25만 원이 그대로 깎입니다. **상한을 반드시 넣으세요.**

**적립 관련**

| 입력란 | 설명 |
|---|---|
| **등급 적립률 (%, 비우면 없음)** | 이 등급 회원이 받는 추가 적립률. |
| **적립 방식** | `기본 적립률에 가산` — 사이트 기본 적립률에 **더합니다.**<br>`기본 적립률 대체` — 기본 적립률 **대신** 이 값을 씁니다. |

> **`가산`과 `대체`를 혼동하면 적립금이 몇 배로 나갑니다.** 기본 적립률이 1%이고 등급 적립률이 3%일 때, `가산`이면 총 4%, `대체`면 총 3%입니다. 등급 관리 목록에서 `추가 +3%` / `대체 3%` 로 구분해 표시됩니다.

**배송 관련**

| 입력란 | 설명 |
|---|---|
| **상시 무료배송 (지역할증 제외)** | 켜면 **금액에 상관없이** 기본 배송비가 0원. |
| **무료배송 문턱 override (원, 비우면 몰 기본)** | 이 등급만 다른 무료배송 기준을 씁니다. 몰 기준과 이 값 중 **더 낮은 쪽**이 적용됩니다. |

> **상시 무료배송이어도 제주·도서산간 지역 할증은 청구됩니다.** 무료가 되는 것은 기본 배송비뿐입니다.

### 3-4. 지급 쿠폰 3종

등급에 쿠폰을 연결하면 **정해진 시점에 자동으로 발급**됩니다.

| 구역 | 언제 발급되나 |
|---|---|
| **등급 진입 시 지급 쿠폰 (쿠폰팩)** | 회원이 이 등급으로 **승급하거나 관리자가 수동으로 상향**할 때. 이미 안 쓴 같은 쿠폰을 갖고 있으면 건너뜁니다. |
| **생일 지급 쿠폰** | 이 등급 회원의 **생일 당일**, 연 1회. 일일 배치가 처리합니다. |
| **정기(월) 지급 쿠폰** | 이 등급 회원에게 **매월 1회**. 월 배치가 처리합니다. |

각 구역에서 쿠폰 목록의 체크박스를 **켜면 연결, 끄면 해제**입니다.

> **쿠폰을 먼저 만들어야 연결할 수 있습니다.** "연결할 수 있는 활성 쿠폰이 없습니다"가 뜬다면 쿠폰이 하나도 없거나 전부 비활성인 것입니다.

<div class="manual-goto-bar">
  <span class="manual-goto-label">쿠폰이 없다면</span>
  <a class="manual-goto" href="/admin/coupons" target="_blank">쿠폰 관리에서 먼저 만들기</a>
</div>

입력이 끝나면 **`저장`** 을 누릅니다.

### 3-5. 등급 목록에서 보이는 것

| 열 | 내용 |
|---|---|
| **순위** | 숫자가 작을수록 상위. |
| **등급** | 색상 점 + 등급명 + (등급 코드). |
| **기본** | 기본 가입 등급이면 배지. |
| **회원** | 현재 이 등급인 회원 수. |
| **할인율** | 정률 할인 %. |
| **적립** | `추가 +N%` 또는 `대체 N%`. |
| **배송** | `무료배송` 또는 `N원↑ 무료`. |
| **상태** | `사용` / `중지`. |
| **관리** | `수정` · `삭제`. |

### 3-6. 등급 삭제

`삭제`를 누르면 확인 창이 뜹니다. 하지만 **다음 중 하나라도 해당하면 삭제되지 않습니다.**

| 막히는 이유 | 화면에 뜨는 문구 |
|---|---|
| 이 등급에 속한 회원이 있음 | 소속 회원이 있어 삭제할 수 없습니다. 비활성화하세요. |
| 평가 정책의 기준에 쓰이고 있음 | 평가 기준에 참조되어 삭제할 수 없습니다. |
| 등급 변경 이력에 등장함 | 변경 이력이 있어 삭제할 수 없습니다. 비활성화하세요. |

> **운영을 시작한 뒤에는 사실상 삭제할 수 없다고 보세요.** 쓰지 않을 등급은 삭제 대신 **`사용` 체크를 해제**하면 됩니다. 회원 이력이 안전하게 보존됩니다.

## 4. 2단계 — 평가 정책 정하기

<div class="manual-goto-bar">
  <span class="manual-goto-label">이 단계 화면</span>
  <a class="manual-goto" href="/admin/membership/policy" target="_blank">등급 평가 정책 열기</a>
</div>

"어떤 실적으로 어느 등급이 되는가"를 정하는 화면입니다.

### 4-1. 평가 기준

| 입력란 | 설명 | 기본값 |
|---|---|---|
| **정책명** | 관리용 이름. | `기본 등급 평가 정책` |
| **실적 인정 기간 (개월)** | 최근 몇 개월치 구매를 볼지. | `12` |
| **평가 주기** | 아래 표 참고. | `월 1회` |
| **인정금액 산식** | 아래 표 참고. | `상품금액 − 쿠폰 (권장)` |
| **금액/건수 결합** | 아래 표 참고. | `금액 또는 건수 (OR)` |
| **승급 반영** | 아래 표 참고. | `정기 평가 시` |
| **강등 반영** | 아래 표 참고. | `정기 평가 시` |
| **신규 회원 보호기간 (일)** | 가입 직후 이 기간 동안 보호합니다. | `0` |
| **승급 후 최소 유지 (일)** | 승급한 뒤 이 기간은 유지합니다. | `0` |

**평가 주기**

| 값 | 뜻 |
|---|---|
| **월 1회** | 예약된 배치가 한 달에 한 번 전체를 평가합니다. |
| **일 1회** | 매일 평가합니다. |
| **수동** | 자동 평가를 하지 않고, 관리자가 `지금 평가 실행`을 누를 때만 평가합니다. |

**인정금액 산식** — 실적으로 인정할 금액을 무엇으로 볼지 정합니다.

| 값 | 뜻 |
|---|---|
| **상품금액 − 쿠폰 (권장)** | 상품값에서 쿠폰 할인을 뺀 금액. 가장 무난합니다. |
| **상품 판매가 합계** | 할인 전 상품값 총액. 쿠폰을 많이 쓴 고객이 유리해집니다. |
| **실결제 상품액** | 실제로 결제한 상품 금액. |
| **실결제액(배송비 포함)** | 배송비까지 포함한 결제 총액. |

**금액/건수 결합** — 진입 기준을 어떻게 판정할지 정합니다.

| 값 | 뜻 |
|---|---|
| **금액 또는 건수 (OR)** | 둘 중 **하나만** 넘으면 승급. 승급이 쉬워집니다. |
| **금액 그리고 건수 (AND)** | **둘 다** 넘어야 승급. 까다로워집니다. |
| **금액만** | 건수를 무시하고 금액만 봅니다. |

> **`OR`에서 건수 기준을 `0`으로 두면 모든 회원이 조건을 통과합니다.** 0건 이상은 누구나 만족하기 때문입니다. `OR`을 쓸 거라면 건수 기준을 반드시 의미 있는 값으로 채우거나, `금액만`을 선택하세요. 이것이 "모두가 최상위 등급이 되어 버리는" 가장 흔한 원인입니다.

**승급 반영 / 강등 반영**

| 항목 | 값 | 뜻 |
|---|---|---|
| **승급 반영** | `즉시(결제 시)` | 고객이 결제해 기준을 넘으면 그 자리에서 올라갑니다. |
| | `정기 평가 시` | 다음 정기 평가 때 반영됩니다. |
| **강등 반영** | `정기 평가 시` | 정기 평가에서만 내립니다. (권장) |
| | `즉시` | 조건을 벗어나면 곧바로 내립니다. |
| | `강등 안 함` | **한 번 올라간 등급은 절대 내려가지 않습니다.** |

> **원칙은 "승급은 빠르게, 강등은 정기적으로"입니다.** 강등을 `즉시`로 두면 고객이 주문을 취소한 순간 등급이 내려가 혜택이 사라지고, 그대로 문의가 됩니다. 특별한 이유가 없으면 강등은 `정기 평가 시`로 두세요.

### 4-2. 혜택 적용 설정 — 등급 할인 × 쿠폰 중복

| 값 | 뜻 |
|---|---|
| **중복 적용 (등급 할인 + 쿠폰 함께)** | 등급 할인과 쿠폰이 **둘 다** 적용됩니다. |
| **쿠폰 우선 (쿠폰 사용 시 등급 할인 미적용)** | 쿠폰을 쓴 주문에는 **등급 할인이 붙지 않습니다.** |

> ⚠️ **`중복 적용`은 할인이 겹칩니다.** 골드 5% 할인 + 1만 원 쿠폰이면 두 개가 모두 빠집니다. 여기에 포인트 사용까지 겹치면 결제액이 예상보다 훨씬 낮아집니다. 마진을 확인하고 고르세요.

### 4-3. 등급별 진입/유지 기준

**활성 등급마다 한 줄**씩 표에 나옵니다.

| 열 | 뜻 |
|---|---|
| **진입 금액 / 진입 건수** | 이 등급으로 **올라가기 위한** 최소 실적. |
| **유지 금액 / 유지 건수** | 이 등급을 **지키기 위한** 최소 실적. **비우면 진입 기준과 동일**하게 적용됩니다. |

> **유지 기준은 진입보다 낮게 잡으세요.** 진입 100만 원 / 유지 80만 원처럼 두면, 90만 원 근처의 고객이 매달 승급·강등을 반복하는 일을 막을 수 있습니다. 두 값을 같게 두면 경계선 고객이 등급을 오르내리며 혜택이 붙었다 사라졌다 합니다.

입력이 끝나면 화면 맨 아래 **`정책 저장`** 을 누릅니다.

> **`정책 저장`을 눌러야 반영됩니다.** 표만 채우고 화면을 벗어나면 전부 사라집니다.

## 5. 승급·강등이 판정되는 방식

기준을 채웠다면 시스템이 회원 한 명을 이렇게 판정합니다.

```
① 최근 N개월 실적(금액·건수)을 집계한다
        ↓
② 진입 기준을 만족하는 가장 높은 등급을 찾는다
        ↓
③ 그 등급이 현재 등급보다 높으면       → 승급
   아니고 현재 등급의 유지 기준을 만족하면 → 유지
   둘 다 아니면                          → 유지 기준을 만족하는 가장 높은 등급으로 강등
```

**판정에서 제외되는 회원**

- **등급이 고정된 회원** (회원 등급 현황에서 `고정` 처리한 회원)
- **`자동 평가 대상` 체크가 꺼진 등급** — 그 등급 자체가 판정 대상에서 빠집니다
- **`사용` 체크가 꺼진 등급**

**주문 취소·환불이 생기면** 그만큼 인정 실적이 자동으로 빠집니다. 다만 **그 자리에서 등급이 내려가지는 않고**, 다음 정기 평가에서 반영됩니다.

## 6. 3단계 — 시뮬레이션과 평가 실행

평가 정책 화면 위쪽에 버튼 두 개가 있습니다.

### 6-1. `시뮬레이션` — 먼저 이걸 누르세요

**실제로는 아무것도 바뀌지 않습니다.** "지금 평가하면 어떻게 되는지"만 계산해서 보여 줍니다.

결과는 파란 상자로 나옵니다.

```
대상 N · 승급 N · 강등 N · 유지 N · 실패 N
```

그 아래 표에 **회원ID / 변경(UPGRADE·DOWNGRADE) / 등급 이동(A → B) / 인정금액 / 건수**가 **최대 100건**까지 나옵니다.

> **기준을 바꿨다면 반드시 시뮬레이션을 먼저 돌리세요.** 여기서 "승급 300명"처럼 이상한 숫자가 나오면 기준이 잘못된 것입니다(대개 위 4-1의 `OR` + 건수 0 문제). **정책을 저장한 뒤에 눌러야** 방금 바꾼 기준이 반영됩니다.

시뮬레이션 결과는 **한 번만 표시**되고 화면을 새로고침하면 사라집니다.

### 6-2. `지금 평가 실행` — 되돌릴 수 없음

> ⚠️ **전 회원의 등급이 실제로 바뀝니다.**

확인 창이 뜹니다 — *"지금 전체 회원 등급을 평가·반영할까요? (승급/강등이 실제 적용됩니다)"*

확인하면 편집 중인 몰의 전 회원을 평가하고, 결과가 상단에 나옵니다.

```
평가 완료 — 대상 N, 승급 N, 강등 N, 유지 N
```

승급한 회원에게는 **등급 진입 쿠폰이 자동 발급**되고, 다음 주문부터 새 등급 혜택이 적용됩니다. 모든 변경은 **등급 변경·평가 이력**에 남습니다.

**되돌리는 버튼은 없습니다.** 잘못 실행했다면 기준을 고쳐 다시 평가하거나, 회원 등급 현황에서 한 명씩 손으로 되돌려야 합니다.

## 7. 회원 등급 현황 — 개별 조정

<div class="manual-goto-bar">
  <span class="manual-goto-label">이 장의 화면</span>
  <a class="manual-goto" href="/admin/membership/customers" target="_blank">회원 등급 현황 열기</a>
</div>

**검색 조건** — 회원 검색(이름·이메일·연락처), 등급 셀렉트, `고정만` 체크박스.

목록은 **인정금액이 높은 순**으로 나오며 한 페이지에 30명씩 표시됩니다.

| 열 | 내용 |
|---|---|
| **회원** | 이름과 이메일. |
| **현재 등급** | 색상 점 + 등급명. 없으면 `미지정`. |
| **인정금액 / 건수** | 마지막 평가 때 계산된 실적. |
| **고정** | `고정` / `고정됨 · 해제` 버튼. |
| **등급 변경** | 등급 셀렉트 + 사유 입력 + `변경` 버튼. |

### 7-1. 등급 수동 변경

바꿀 등급을 고르고 **사유**를 적은 뒤 **`변경`** 을 누릅니다. CS 보상, 임직원 지정, 평가 오류 정정 등에 씁니다.

- **상향 변경이면 그 등급의 진입 쿠폰이 발급됩니다.**
- 변경 기록은 사유와 함께 이력에 남습니다.
- **다음 정기 평가에서 다시 자동 판정되어 원래 등급으로 돌아갈 수 있습니다.** 계속 유지하려면 아래 `고정`을 함께 켜세요.

> **사유를 반드시 적으세요.** 비워도 저장되지만, 나중에 "이 회원은 왜 VIP지?"를 확인할 방법이 사라집니다.

### 7-2. 등급 고정

`고정`을 누르면 그 회원은 **자동 평가에서 완전히 제외**됩니다. 승급도 강등도 하지 않습니다. 임직원, 협력사, 분쟁 조정 중인 회원 등에 사용합니다.

버튼이 `고정됨 · 해제`로 바뀌며, 다시 누르면 해제됩니다. 고정된 회원 수는 대시보드에서 확인할 수 있습니다.

## 9. 멤버십 대시보드 읽는 법

<div class="manual-goto-bar">
  <span class="manual-goto-label">이 장의 화면</span>

**상단 카드 4개** — 전체 회원(등급 보유) / 등급 고정 회원 / 활성 등급 수 / 마지막 평가(시각·승급·강등 수).

**등급 구성 및 분포** — 등급별 회원 수와 **구성비(%)**, 혜택 요약, 사용 여부.

> **분포로 기준의 적정성을 판단하세요.** 최하위 등급에 95%가 몰려 있으면 진입 기준이 너무 높은 것이고, 반대로 최상위에 절반이 있으면 기준이 너무 낮은 것입니다.

**등급별 최근 30일 실적 · 혜택 비용** — `결제완료` 주문 기준으로 등급마다 주문 수 / 매출 / 객단가 / **등급 할인 비용** / **적립 예정액**을 보여 줍니다.

> **오른쪽 두 열이 멤버십 운영 비용입니다.** 할인 비용과 적립 예정액을 매출과 비교해 보세요. 매출 대비 비율이 계속 커진다면 등급 혜택을 조정할 시점입니다.

"활성 평가 정책이 없습니다"라는 노란 안내가 떠 있으면 아직 평가 정책을 저장한 적이 없는 것입니다. 평가 정책 화면에서 한 번 저장하세요.

## 10. 고객에게는 어떻게 보이나

- **마이페이지** — 자기 등급, 인정 실적, "다음 등급까지 얼마 남았는지"가 나옵니다. 등급 등록 때 적은 **마이페이지 안내 문구**도 여기에 표시됩니다.
- **멤버십 안내 페이지(`/membership`)** — 전체 등급표가 나옵니다. 등급명, 진입 기준, 적립률, 혜택 요약이 **관리자가 설정한 값 그대로** 표시됩니다.
- **활성 등급이 하나도 없으면** 고객 화면은 기본 예시 등급표로 대체됩니다.

즉 **등급명·혜택·기준을 바꾸면 고객 화면도 즉시 바뀝니다.** 운영 중에 등급명을 바꾸면 고객이 혼란스러워하니 신중히 하세요.

## 11. 자주 막히는 상황

**Q. 평가를 돌렸더니 대부분이 최상위 등급이 됐습니다.**
`금액/건수 결합`이 `OR`인데 **진입 건수가 0**으로 되어 있을 가능성이 큽니다. 0건 이상은 누구나 만족하므로 전원이 통과합니다. 건수를 의미 있는 값으로 채우거나 `금액만`으로 바꾸고, **시뮬레이션으로 확인한 뒤** 다시 평가하세요.

**Q. 반대로 모든 회원이 최하위 등급입니다.**
진입 금액이 실제 구매 규모보다 높습니다. 회원 등급 현황에서 상위 회원들의 인정금액을 확인하고, 그 분포에 맞춰 기준을 낮추세요.

**Q. 등급 평가 정책 화면의 기준 표가 비어 있습니다.**
활성 등급이 하나도 없습니다. 등급 관리에서 먼저 등급을 만들고 `사용`을 켜세요.

**Q. 승급했는데 쿠폰이 안 갔습니다.**
그 등급에 **등급 진입 쿠폰이 연결돼 있는지** 확인하세요. 연결돼 있어도 회원이 **이미 그 쿠폰을 안 쓰고 갖고 있으면 발급이 건너뛰어집니다.** 회원 관리의 회원 상세에서 보유 쿠폰을 확인해 보세요.

**Q. 등급 할인이 결제에 적용되지 않습니다.**
① 등급의 **`할인 사용` 체크**가 켜져 있는지, ② **할인 최소 주문금액**을 넘겼는지, ③ 정책의 중복 설정이 **`쿠폰 우선`**인데 고객이 쿠폰을 썼는지 순서대로 확인하세요.

**Q. 적립금이 예상보다 훨씬 많이 나갑니다.**
적립 방식이 `기본 적립률에 가산`인지 `대체`인지 확인하세요. 기본 적립률은 사이트 설정에서 관리합니다.

**Q. 무료배송 등급인데 배송비가 붙었습니다.**
배송지가 제주·도서산간이면 지역 할증은 그대로 청구됩니다. 정상 동작입니다.

**Q. 수동으로 올린 등급이 며칠 뒤 원래대로 돌아갔습니다.**
정기 평가가 자동 판정한 것입니다. 유지하려면 회원 등급 현황에서 **`고정`** 을 함께 켜세요.

**Q. 등급 삭제가 안 됩니다.**
소속 회원·평가 기준·변경 이력 중 하나에 연결돼 있습니다. 삭제 대신 **`사용` 체크 해제**로 중지하세요.

**Q. 다른 몰에 만든 등급이 안 보입니다.**
등급은 몰별로 따로 저장됩니다. 상단에서 편집 몰을 바꾸고 다시 확인하세요.

**Q. 등급 코드를 잘못 지었습니다.**
수정할 수 없습니다. 새 등급을 만들고 기존 등급은 `사용`을 끄세요. 다만 회원이 이미 소속돼 있다면 수동으로 옮겨야 합니다.

## 12. 아직 지원하지 않는 것

- 등급 코드 수정
- 여러 회원의 등급 일괄 변경
- 등급별 전용 상품·전용 가격
- 평가 실행 결과 되돌리기(롤백)
- 등급 변경 시 고객에게 자동 알림 발송 (강등 사전 안내 이메일은 예외)
