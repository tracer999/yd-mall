# 이메일 템플릿 관리

주문·배송 과정에서 **고객과 거래처에게 자동으로 나가는 안내 메일**의 제목과 내용을 직접 고치는 메뉴입니다. 주문이 접수됐을 때, 상품이 출고됐을 때, 취소·반품이 처리됐을 때 나가는 메일이 모두 여기에 모여 있습니다.

> **아무것도 하지 않아도 메일은 나갑니다.** 모든 템플릿에는 처음부터 기본 문구가 들어 있습니다. 이 화면은 그 문구를 **우리 몰에 맞게 고치고 싶을 때** 들어오는 곳입니다.

## 1. 이 화면에서 할 수 있는 것

| 하고 싶은 일 | 방법 |
|---|---|
| 메일 문구를 우리 몰 말투로 바꾸기 | 템플릿의 [내용 수정] → 제목·내용 고치고 [저장] |
| 특정 안내 메일을 아예 안 보내기 | 목록에서 [발송 중지] |
| 고친 문구를 처음 상태로 되돌리기 | 편집 화면 오른쪽 [되돌리기] |
| 실제로 어떻게 보이는지 확인하기 | 편집 화면 아래 **미리보기**, 또는 [테스트 발송] |
| 취소·반품 접수 알림을 받을 담당자 주소 지정 | 목록 위쪽 **운영자 알림 수신 주소** |

---

## 2. 어떤 메일이 있나

메일은 **받는 사람 기준**으로 묶여 있습니다.

### 공통

| 템플릿 | 언제 쓰이나 |
|---|---|
| 공통 레이아웃 (머리말·꼬리말) | 모든 안내 메일을 감싸는 틀입니다. 맨 위 쇼핑몰 이름, 맨 아래 고객센터 안내가 여기서 나옵니다. 한 번만 고치면 모든 메일에 반영됩니다. |

### 개인회원 주문·배송

| 템플릿 | 언제 나가나 |
|---|---|
| 주문·결제 완료 안내 | 고객이 결제를 마쳤을 때 |
| 배송 시작 안내 | 배송 관리에서 **송장을 등록**했을 때 |
| 배송 완료 안내 | 배송 관리에서 **배송완료**로 바꿨을 때 |
| 취소·반품 접수 안내 | 고객이 마이페이지에서 취소·반품을 신청했을 때 |
| 취소·반품 승인(환불) 안내 | 클레임 관리에서 **승인**했을 때 |
| 취소·반품 반려 안내 | 클레임 관리에서 **반려**했을 때 |

### 기업회원 주문·배송

| 템플릿 | 언제 나가나 |
|---|---|
| 주문 접수 안내 | 기업회원이 주문을 넣었을 때 |
| 주문 승인·입금 안내 | B2B 주문을 승인했을 때 (입금 계좌·기한이 함께 나갑니다) |
| 입금 확인 안내 | 입금을 확인 처리했을 때 |
| 출고 안내 | B2B 주문을 출고 처리했을 때 |
| 배송 완료 안내 | B2B 주문을 배송완료로 바꿨을 때 |
| 주문 반려·취소 안내 | 주문을 반려하거나, 입금 기한이 지나 자동 취소됐을 때 |
| 취소·반품 승인 안내 | B2B 클레임을 승인했을 때 |
| 취소·반품 반려 안내 | B2B 클레임을 반려했을 때 |
| 환불 완료 안내 | 계좌로 환불을 이체하고 **[계좌 환불 완료]** 를 눌렀을 때 |

### 운영자 알림

| 템플릿 | 언제 나가나 |
|---|---|
| 취소·반품 접수 알림 | 고객이 취소·반품을 신청했을 때, **운영자에게** |

## 3. 문구 고치는 순서

```
① 목록에서 고치고 싶은 메일의 [내용 수정]
        ↓
② 제목·내용을 고침 (오른쪽 [넣을 수 있는 정보]로 주문번호 등 삽입)
        ↓
③ 아래 미리보기로 모양 확인
        ↓
④ [저장]
        ↓
⑤ (선택) [테스트 발송]으로 내 메일함에서 직접 확인
```

### 3.1 "넣을 수 있는 정보"란

메일마다 주문번호나 고객 이름처럼 **매번 달라지는 값**이 들어갑니다. 이런 값은 직접 타이핑하는 게 아니라, 편집 화면 오른쪽의 **[넣을 수 있는 정보]** 버튼을 눌러 넣습니다.

예를 들어 `주문번호` 버튼을 누르면 글자 위치에 `{{order_number}}` 가 들어가고, 실제 메일이 나갈 때 `ORD-20260716-51116` 같은 진짜 주문번호로 바뀝니다.

> 버튼을 누르기 **직전에 커서를 둔 자리**에 들어갑니다. 제목 칸을 클릭한 뒤 버튼을 누르면 제목에, 내용 칸을 클릭한 뒤 누르면 내용에 들어갑니다.

자주 쓰는 것들:

| 버튼 이름 | 실제로 들어가는 값 |
|---|---|
| 쇼핑몰 이름 | 몰 이름 (몰 리스트 관리에서 정한 이름) |
| 주문번호 | ORD-20260716-51116 |
| 주문자명 / 거래처명 | 홍길동 / (주)와이디상사 |
| 상품 요약(한 줄) | 홍삼정 스틱 30포 외 2건 |
| 상품 목록 표 | 상품명·수량·금액이 든 표 |
| 결제 금액 | 51,000원 |
| 택배사 / 송장번호 | CJ대한통운 / 123456789012 |
| 주문 상세 링크 | 고객이 눌러서 주문을 볼 수 있는 주소 |

### 3.2 미리보기

편집 화면 맨 아래에 **미리보기**가 있습니다. 여기에는 예시 주문 정보(홍삼정 스틱 등)가 채워진 상태로 보입니다. 쇼핑몰 이름·고객센터 번호처럼 **이미 아는 값은 진짜 값**이 들어갑니다.

내용을 고친 뒤 [미리보기 새로고침]을 누르면 최신 상태로 다시 그립니다.

### 3.3 테스트 발송

편집 화면 오른쪽 **테스트 발송**에 본인 메일 주소를 넣고 [보내기]를 누르면, 지금 화면의 내용이 실제 메일로 옵니다. 제목 앞에 `[테스트]` 가 붙습니다.

> 테스트 발송이 실패하면 대부분 **메일 발송 계정이 설정되지 않은 것**입니다. <span class="manual-path">관리자 &gt; 사이트 설정 &gt; 시스템 설정</span>의 SMTP 항목을 확인하세요.

## 4. 특정 메일을 안 보내고 싶을 때

목록에서 해당 템플릿의 **[발송 중지]** 를 누릅니다. 빨간색으로 `발송 안 함` 표시가 붙고, 그 메일은 더 이상 나가지 않습니다. 다시 보내려면 **[발송 켜기]** 를 누릅니다.

편집 화면 위쪽의 **"이 안내 메일 발송"** 체크박스로도 같은 설정을 바꿀 수 있습니다.

> **공통 레이아웃**을 끄면 머리말·꼬리말 없이 각 메일의 본문만 나갑니다. 메일을 안 보내게 되는 게 아닙니다.

---

## 5. 운영자 알림 수신 주소

고객이 취소·반품을 신청하면 운영자에게도 알림 메일이 갑니다. 그 메일을 **누가 받을지**를 목록 위쪽에서 지정합니다.

- 주소를 넣고 [저장] → 그 주소로 갑니다.
- 비워 두면 <span class="manual-path">사이트 설정</span>의 **대표 이메일**로 갑니다.
- 둘 다 없으면 운영자 알림은 나가지 않습니다(고객에게 가는 안내는 정상 발송).

---

## 6. 몰마다 다른 문구 쓰기

목록 위쪽 **몰 선택**에서 몰을 바꾸면, 그 몰의 문구를 따로 저장할 수 있습니다. 저장하지 않은 몰은 계속 기본 문구를 씁니다.

> 새로 만든 몰은 아무 설정 없이도 기본 문구로 메일이 나갑니다. 몰을 만들 때마다 이 화면에 들어올 필요는 없습니다.

---

## 7. 자주 겪는 상황

| 증상 | 확인할 것 |
|---|---|
| 메일이 하나도 안 온다 | ① 시스템 설정의 SMTP 계정 ② 해당 템플릿이 `발송 안 함` 상태인지 ③ 받는 사람 주소가 `example.com` 같은 예시 주소인지(이런 주소로는 보내지 않습니다) |
| 특정 메일만 안 온다 | 목록에서 그 템플릿이 `발송 안 함`인지 확인 |
| 메일에 `{{order_number}}` 같은 글자가 그대로 보인다 | 버튼으로 넣지 않고 직접 타이핑하면서 철자가 틀렸을 수 있습니다. [되돌리기] 후 다시 넣어 보세요 |
| 고친 내용이 반영되지 않는다 | [저장]을 눌렀는지 확인. 목록에 `직접 수정함` 표시가 붙어야 저장된 것입니다 |
| 운영자 알림만 안 온다 | 목록 위쪽 **운영자 알림 수신 주소**가 비어 있고, 사이트 설정의 대표 이메일도 비어 있는 경우입니다 |
