# 메뉴 관리

고객 화면 **상단 메뉴(GNB)** 와 **헤더 유틸 · 우측 레일**, 그리고 **관리자 사이드바 메뉴**를 다루는 화면들입니다. 메뉴는 하나의 화면이 아니라 **역할이 다른 5개 화면**으로 나뉘어 있습니다. 이 문서는 그 5개를 한 번에 설명합니다.

> **메뉴 설정은 몰마다 따로 저장됩니다.** 작업 전에 화면 상단의 **편집 중인 몰**이 맞는지 확인하세요. 다른 몰을 편집 중이면 저장해도 원하는 몰에는 아무 변화가 없습니다.

## 1. 5개 화면이 각각 무슨 일을 하나

| 화면 | 주소 | 무엇을 하나 |
|---|---|---|
| **메뉴 미리보기** | `/admin/menu-preview` | 지금 설정대로 고객 화면이 어떻게 보이는지 확인하고, **GNB 순서·노출을 드래그로 정합니다.** 여기서 시작하는 것이 가장 빠릅니다. |
| **일반 메뉴 관리** | `/admin/feature-menus` | 베스트·기획전·아울렛처럼 **시스템이 이미 만들어 둔 상단 메뉴**를 켜고 끕니다. |
| **시스템 메뉴 설정** | `/admin/system-menus` | 로그인·장바구니·검색 같은 **헤더 유틸**과 화면 오른쪽에 붙는 **우측 레일**을 다룹니다. |
| **커스텀 메뉴 관리** | `/admin/custom-menus` | "건강식품관" 같은 **운영자가 직접 만드는 메뉴**를 추가합니다. |
| **관리자 메뉴 관리** | `/admin/menus` | **관리자 사이트의 왼쪽 사이드바**입니다. **고객 화면과는 아무 관계가 없습니다.** |

## 2. 메뉴 구조 이해하기 (여기가 제일 헷갈립니다)

메뉴는 **4겹**으로 되어 있습니다. 쉽게 풀면 이렇습니다.

```
① 기능 메뉴 카탈로그   "이 솔루션이 제공하는 메뉴는 이런 것들이 있다"  (모든 몰 공통 · 고정)
        ↓  몰마다 골라 담기
② 몰별 설정            "우리 몰은 이 중 6개만 쓰고, 이름은 이렇게, 순서는 이렇게"
        ↓  모자라면 직접 추가
③ 커스텀 메뉴          "우리 몰만의 '건강식품관' 메뉴를 하나 더 만든다"
        ↓  전체 틀
④ 헤더 설정            "상단 메뉴는 최대 몇 개까지, 어떤 모양(스킨)으로 보일까"
```

### ① 기능 메뉴 카탈로그 — 손댈 수 없는 목록

"쇼핑특가", "베스트/랭킹", "기획전", "브랜드"처럼 **솔루션이 미리 만들어 둔 메뉴 목록**입니다. **이름의 기본값과 주소(URL)는 코드로 고정**되어 있어 운영자가 바꿀 수 없습니다. 화면에서는 **[기본 명칭 (고정)]**, **[표준 URL (고정)]** 칸으로 보입니다.

### ② 몰별 설정 — 실제로 만지는 곳

같은 카탈로그를 두고 **몰마다 다르게** 정합니다. "A몰은 쇼핑특가를 켜고 B몰은 끈다", "A몰은 이름을 '오늘의 특가'로 바꾼다" 같은 것이 여기 저장됩니다. **일반 메뉴 관리 · 시스템 메뉴 설정**이 이 층을 다룹니다.

> **그래서 "일반 메뉴 관리에서 메뉴를 추가할 수 없다"가 정상입니다.** 카탈로그에 없는 메뉴는 만들 수 없습니다. 새 메뉴가 필요하면 **커스텀 메뉴**로 만듭니다.

### ③ 커스텀 메뉴 — 운영자가 만드는 메뉴

기존 기획전·카테고리·브랜드 하나를 골라 **상단 메뉴 자리에 꽂는** 기능입니다. 예를 들어 "전문관" 목록 메뉴를 끄고, 그 자리에 **개별 전문관 '건강식품관'** 을 올리는 식입니다.

> **커스텀 메뉴는 기능 메뉴와 같은 순서 축을 씁니다.** 커스텀 메뉴 순서를 3으로 두면 기능 메뉴 3번 근처에 끼어듭니다. 별도 줄이 아닙니다.

### ④ 헤더 설정 — 전체 틀

상단 메뉴 **최대 노출 개수**, 그중 **커스텀 메뉴가 쓸 수 있는 슬롯 수**, 카테고리 **최대 단계 수**, 헤더 **스킨**을 정합니다.

## 3. 메뉴 미리보기 — 여기서 시작하세요

<div class="manual-goto-bar">
  <span class="manual-goto-label">이 단계 화면</span>
  <a class="manual-goto" href="/admin/menu-preview" target="_blank">메뉴 미리보기 열기</a>
</div>

이 화면은 **고객 화면과 똑같은 계산 방식**으로 메뉴를 뽑아 보여 줍니다. 여기에 보이는 것이 곧 고객이 보는 것입니다.

### 3-1. 조건 토글

화면 위의 버튼으로 조건을 바꿔 가며 확인합니다.

| 토글 | 값 |
|---|---|
| 기기 | **PC** / **모바일** |
| 로그인 상태 | **비로그인** / **로그인** |

> **로그인 필요로 설정한 메뉴는 비로그인 화면에서 사라집니다.** "메뉴가 안 보인다"는 문의가 오면 두 토글을 모두 바꿔 가며 확인하세요.

### 3-2. 헤더 모형

로고 · 헤더 유틸 · 상단 메뉴가 실제 배치대로 그려집니다. 색으로 종류를 구분합니다.

- **초록색** = 카테고리 (드로어형 스킨에서만)
- **보라색 + `커스텀`** = 커스텀 메뉴
- 그 외 = 기능 메뉴

상단에는 `GNB N개 노출 / 상한 8` 처럼 **현재 개수와 상한**이 표시됩니다.

### 3-3. GNB 구성 — 순서와 노출을 정하는 곳

표의 줄을 **끌어서(드래그) 순서를 바꾸고**, 사용 · PC · 모바일 체크를 켜고 끈 뒤 **[저장]**을 누릅니다.

| 칸 | 설명 |
|---|---|
| **순서** | 드래그한 순서대로 1부터 다시 매겨집니다. |
| **항목** | 메뉴 이름 |
| **종류** | 카테고리 / 커스텀 / 기능 |
| **사용** | 끄면 이 메뉴가 상단에서 빠집니다. 카테고리는 `항상` 으로 표시되어 끌 수 없습니다. |
| **PC / 모바일** | 기기별 노출 |

> **여기서 저장하면 카테고리 관리·일반 메뉴 관리 화면의 값도 함께 바뀝니다.** 세 화면이 같은 데이터를 보기 때문입니다. 순서를 맞추는 작업은 **이 화면 한 곳에서** 하는 것이 가장 정확합니다.

> **카테고리의 [사용]을 이 화면에서 끌 수 없는 이유**: 카테고리를 끄면 상단 메뉴뿐 아니라 상품 목록 등 **다른 화면까지 함께 죽습니다.** 그래서 여기서는 기기별 노출만 다루고, 완전히 끄는 것은 카테고리 관리에서 하도록 나눠 두었습니다.

### 3-4. 노출되지 않는 메뉴 — 원인을 여기서 확인

화면 아래쪽 표에 **빠진 메뉴와 그 사유**가 나옵니다. 추측하지 말고 이 표부터 보세요.

| 사유 | 뜻 | 해결 |
|---|---|---|
| **모듈 미구현 (켜도 노출되지 않음)** | 아직 기능이 없는 메뉴입니다. | 기다려야 합니다. 켜도 소용없습니다. |
| **사용 안 함 (운영자가 끔)** | 꺼져 있습니다. | 일반 메뉴 관리 또는 GNB 구성에서 켭니다. |
| **로그인 필요** | 비로그인 조건에서 숨겨집니다. | 정상입니다. 조건 토글을 로그인으로 바꿔 확인하세요. |
| **노출 기간 전 / 노출 기간 종료** | 지정한 노출 기간 밖입니다. | 커스텀 메뉴의 노출 시작·종료를 확인합니다. |
| **콘텐츠 부족 — 채울 내용이 없어 자동으로 숨겨짐** | 보여 줄 내용이 없어 **스스로 숨은** 상태입니다. | 아래 3-5 참고 |

### 3-5. "콘텐츠 부족"으로 자동으로 숨는 메뉴 3가지

메뉴를 켜 두어도 **안에 채울 내용이 없으면 자동으로 감춰집니다.** 빈 화면으로 고객을 보내지 않기 위한 장치이며 **고장이 아닙니다.**

| 메뉴 | 숨는 조건 | 해결 |
|---|---|---|
| **아울렛** | 판매 중인 아울렛 상품이 **설정한 최소 개수에 미달** | 상품을 더 담거나, 아울렛 관리에서 최소 노출 상품 수를 낮춥니다. |
| **공동구매** | 공개 중인 공동구매가 **0건** | 공동구매를 하나 이상 만들고 공개합니다. |
| **쇼핑라이브** | 공개 중인 라이브가 **0건** | 라이브를 하나 이상 만들고 공개합니다. |

같은 화면 아래에서 **지금 상단 메뉴에 걸리는 카테고리 나무**와 **우측 레일 항목**도 함께 확인할 수 있습니다. 우측 레일은 **화면 폭이 1600px 이상일 때만** 나오고, **모바일에서는 나오지 않습니다.**

## 4. 일반 메뉴 관리 — 상단 메뉴 켜고 끄기

<div class="manual-goto-bar">
  <span class="manual-goto-label">이 단계 화면</span>
  <a class="manual-goto" href="/admin/feature-menus" target="_blank">일반 메뉴 관리 열기</a>
</div>

상단 메뉴에 올릴 수 있는 기능 메뉴 목록입니다. **쇼핑특가 · 베스트/랭킹 · 신상품 · 추천 · 이벤트&혜택 · 기획전 · 브랜드 · 전문관 · 아울렛 · 쿠폰 · 멤버십 · 공동구매 · 쇼핑라이브** 그리고 맨 앞의 **카테고리** 버튼이 여기 있습니다.

> 예전에 있던 **'랭킹'** 메뉴는 **없어졌습니다**(2026-07-22). 베스트가 랭킹 기능을 그대로 가지고 있어 같은 화면이 두 개였습니다. 이름도 이미 **베스트/랭킹** 입니다. 기존 `/ranking` 주소로 들어오면 자동으로 베스트로 넘어가므로 링크가 깨지지 않습니다.

### 4-1. 각 칸의 뜻

| 칸 | 바꿀 수 있나 | 설명 |
|---|---|---|
| **사용** | 대부분 가능 | 체크하면 상단 메뉴에 나옵니다. |
| **순서** | 가능 | 0~99. 숫자가 작을수록 앞입니다. |
| **기본 명칭 (고정)** | **불가** | 솔루션이 정한 이름과 내부 코드입니다. |
| **표시 명칭 (변경 가능)** | 가능 | 고객에게 보일 이름. 20자 이내. **비우면 기본 명칭이 쓰입니다.** |
| **표준 URL (고정)** | **불가** | 메뉴를 눌렀을 때 가는 주소입니다. |
| **배지** | 가능 | 없음 / **NEW** / **HOT** / **SALE** 중 하나. 자유 입력은 안 됩니다. |
| **PC / 모바일** | 가능 | 기기별 노출 |
| **로그인 필요** | 가능 | 켜면 비로그인 고객에게 숨겨집니다. |

고친 뒤 아래의 **[저장]**을 누릅니다. **[되돌리기]**를 누르면 저장하지 않고 화면을 다시 불러옵니다.

> **저장하면 고객 화면에 즉시 반영됩니다.** 별도 발행 절차가 없습니다.

### 4-2. 잠겨 있는 메뉴 두 종류

| 표시 | 뜻 |
|---|---|
| **모듈 미구현** | 아직 기능이 없어 **켜도 화면에 안 나옵니다.** 죽은 링크를 막기 위해 체크박스가 잠겨 있습니다. |
| **필수** | 쇼핑몰 핵심 기능이라 **끌 수 없습니다.** 이름과 순서만 바꿀 수 있습니다. |

### 4-3. 상단 메뉴 개수 상한 — 조용히 잘려 나갑니다

상단 메뉴에는 **최대 노출 개수**(기본 8개)가 있습니다. 이를 넘으면 **뒤쪽 메뉴부터 소리 없이 잘립니다.** 관리자 화면에는 여전히 "켜짐"으로 보이기 때문에 원인을 찾기 어렵습니다.

그래서 이 화면은 상황을 미리 알려 줍니다.

| 경고 | 뜻 |
|---|---|
| 빨간 **"GNB 상한을 N개 초과했습니다"** | 뒤쪽 N개가 지금 고객 화면에 안 나오고 있습니다. |
| 노란 **"GNB 슬롯이 가득 찼습니다"** | 지금 하나 더 켜면 맨 뒤가 밀려납니다. |

> **해결은 둘 중 하나입니다.** 안 쓰는 메뉴를 끄거나, **Header 설정에서 최대 노출 개수를 늘리세요.** 커스텀 메뉴도 같은 상한을 나눠 쓰므로 합쳐서 계산됩니다. **맨 앞 카테고리 버튼은 상한 계산에서 빠집니다.**

편집 방법은 일반 메뉴 관리와 **완전히 같습니다.** 다루는 위치만 다릅니다.

| 그룹 | 담긴 메뉴 | 특징 |
|---|---|---|
| **헤더 유틸** | 검색 · 로그인 · 마이쇼핑 · 장바구니 · 고객센터 | **검색·로그인·마이쇼핑·장바구니는 필수라 끌 수 없습니다.** 고객센터만 끌 수 있습니다. |
| **우측 유틸 레일** | 장바구니 · 찜 · 찜한 브랜드 · 최근본상품 · TOP | **화면 폭 1600px 이상에서만** 나옵니다. TOP 은 필수입니다. |

> **여기서는 메뉴를 추가하거나 삭제할 수 없습니다.** 쇼핑몰에 고정된 기능이라 **노출 여부·표시 명칭·순서**만 조정합니다.

> **모바일에서는 우측 유틸 레일이 아예 나오지 않습니다.** 모바일 노출을 켜도 마찬가지입니다.

## 6. 커스텀 메뉴 관리 — 직접 메뉴 만들기

<div class="manual-goto-bar">
  <span class="manual-goto-label">이 단계 화면</span>
  <a class="manual-goto" href="/admin/custom-menus" target="_blank">커스텀 메뉴 목록 열기</a>
  <a class="manual-goto" href="/admin/custom-menus/add" target="_blank">새 커스텀 메뉴 추가</a>
</div>

기능 메뉴는 "기획전 **목록**"처럼 묶음 단위입니다. 반면 **"건강식품관" 하나만 상단에 올리고 싶을 때** 쓰는 것이 커스텀 메뉴입니다.

### 6-1. 입력란

| 입력란 | 필수 | 설명 |
|---|---|---|
| **메뉴 이름** | 필수 | 상단 메뉴에 그대로 표시됩니다. **20자 이내.** |
| **노출 위치** | 필수 | `GNB (상단 메뉴)` / `푸터` / `모바일 퀵 메뉴` |
| **순서** | 선택 | 0~99. **일반 메뉴와 같은 축**입니다. |
| **링크 유형** | 필수 | 아래 6-2 표 참고 |
| **연결 대상 / 링크 주소** | 필수 | 링크 유형에 따라 선택기 또는 입력칸이 나타납니다. |
| **사용 / PC 노출 / 모바일 노출 / 로그인 필요 / 새 창** | — | 체크박스 |
| **배지** | 선택 | 없음 / NEW / HOT / SALE |
| **노출 시작 · 노출 종료** | 선택 | **비우면 상시 노출.** 기간이 지나면 자동으로 숨겨집니다. |

### 6-2. 링크 유형 5가지

| 유형 | 무엇에 연결되나 |
|---|---|
| **기획전 · 전문관** | 발행된 기획전/전문관 하나를 골라 상세로 바로 보냅니다. **전문관(상시)** 과 **기획전**이 나뉘어 표시됩니다. |
| **카테고리** | 카테고리 상품 목록으로 보냅니다. **드로어형 스킨에서는 하위 카테고리가 자동으로 하위 메뉴로 붙습니다.** |
| **브랜드** | 브랜드 상품 목록으로 보냅니다. |
| **내부 페이지 (직접 입력)** | 이 쇼핑몰 안의 경로. **반드시 `/` 로 시작해야** 합니다. |
| **외부 링크** | 다른 사이트 주소. **반드시 `http://` 또는 `https://` 로 시작해야** 합니다. **항상 새 창으로 열립니다.** |

> **발행되지 않았거나 삭제된 대상은 고를 수 없습니다.** 목록에 없다면 기획전 관리에서 **먼저 만들고 발행**하세요.

<div class="manual-goto-bar">
  <span class="manual-goto-label">연결할 대상이 없다면</span>
  <a class="manual-goto is-sub" href="/admin/exhibitions" target="_blank">기획전 · 전문관 관리</a>
  <a class="manual-goto is-sub" href="/admin/categories" target="_blank">카테고리 관리</a>
</div>

### 6-3. 목록 화면에서 할 수 있는 일

- 왼쪽 **토글**을 누르면 사용 여부가 바로 바뀝니다.
- **연결 주소**를 눌러 실제로 어디로 가는지 새 창에서 확인할 수 있습니다.
- 줄마다 **[수정] / [삭제]**

**빨간 `연결 끊김` 배지**가 붙은 줄은 **대상이 삭제·미발행·비활성 상태**라, 켜 두어도 고객 화면에 나오지 않습니다. 죽은 링크를 막기 위한 처리입니다. 대상을 되살리거나 다른 대상으로 바꾸세요.

### 6-4. 슬롯 상한

상단 메뉴에 켤 수 있는 커스텀 메뉴 수에는 **별도 상한**(기본 3개)이 있습니다. 초과하면 저장이 거부되고 다음 메시지가 나옵니다.

`GNB 커스텀 메뉴 슬롯(N개)을 초과했습니다. Header 설정에서 슬롯 수를 늘리거나 다른 커스텀 메뉴를 끄세요.`

> **끄는 것은 언제나 되고, 켤 때만 검사합니다.** 상한이 걸리면 안 쓰는 커스텀 메뉴를 먼저 끄세요.

## 7. 헤더 스킨 2종 — 카테고리가 어디에 놓이는가

**Header 설정**에서 고르며, 이 선택에 따라 **상단 메뉴의 구조 자체가 달라집니다.**

| 스킨 | 카테고리 위치 | GNB 구성 화면에 카테고리가 |
|---|---|---|
| **기본형 (카테고리 분리)** | 상단 맨 왼쪽의 **버튼 하나**에 매달린 별도 패널 | **나오지 않습니다.** 카테고리 순서는 카테고리 관리에서 정합니다. |
| **드로어형 (통합 메뉴)** | 카테고리 1단계가 **상단 메뉴 항목 자체**가 됩니다 | **나옵니다.** 카테고리와 기능 메뉴가 하나의 순서 축에 놓입니다. |

> **드로어형에서는 카테고리가 개수 상한에 포함되지 않습니다.** 카테고리를 자르면 매장이 반토막 나기 때문에, 상한은 **기능·커스텀 메뉴에만** 적용됩니다.

## 8. 관리자 메뉴 관리 — 고객 화면과 무관합니다

<div class="manual-goto-bar">
  <span class="manual-goto-label">이 단계 화면</span>
  <a class="manual-goto" href="/admin/menus" target="_blank">관리자 메뉴 관리 열기</a>
</div>

**관리자 사이트 왼쪽 사이드바**를 설정하는 화면입니다. 고객 화면 메뉴와는 **아무 관계가 없습니다.**

| 칸 | 설명 |
|---|---|
| **이동** | 손잡이를 끌어 순서를 바꿉니다. |
| **메뉴명** | 사이드바에 보이는 이름 |
| **경로 (URL)** | 예: `/admin/products`. **비우면 그룹 줄**이 되어 하위 메뉴만 담습니다. |
| **활성** | 끄면 사이드바에서 사라집니다. |
| **[상세]** | 아이콘과 접근 권한을 펼칩니다. |
| **[삭제]** | 줄을 지웁니다. |

**[상세]** 안에는 두 가지가 있습니다.

- **아이콘** — **[아이콘 선택]** 버튼을 누르면 아이콘 목록이 열립니다. 대시보드·상품·주문·회원 같은 묶음으로 나뉘어 있고, 위 검색칸에 **한글로** 찾을 수 있습니다(예: `주문`, `설정`, `배송`). 고르면 바로 적용되고 왼쪽 미리보기가 함께 바뀝니다. **아이콘 이름을 직접 적을 필요는 없습니다.**
- **접근 가능한 역할** — **최고관리자 / 관리자 / 컨텐츠관리자 / 고객관리자** 중 선택. **비워 두면 전체 허용**입니다.

바꾼 뒤 **[저장하기]**를 눌러야 반영됩니다.

> **자기가 쓰는 메뉴를 숨기거나 권한에서 제외하면 다음부터 그 메뉴가 안 보입니다.** 최고관리자는 모든 메뉴가 보이도록 두는 편이 안전합니다.

> **하위 메뉴가 남아 있는 그룹은 삭제할 수 없습니다.** `하위 메뉴가 있는 그룹은 삭제할 수 없습니다` 메시지가 뜨면 하위를 먼저 다른 그룹으로 옮기거나 지우세요.

> 이 화면은 **최고관리자(super_admin) 또는 관리자(admin)만** 들어올 수 있습니다.

## 9. 자주 막히는 상황

**Q. 메뉴를 켰는데 고객 화면에 안 보입니다.**
**메뉴 미리보기**의 "노출되지 않는 메뉴" 표를 먼저 확인하세요. 사유가 그대로 적혀 있습니다. 가장 흔한 원인은 ① 개수 상한 초과로 잘림, ② 콘텐츠 부족으로 자동 숨김(아울렛·공동구매·쇼핑라이브), ③ 로그인 필요, ④ 모듈 미구현입니다.

**Q. 일반 메뉴 관리에 메뉴를 추가할 수 없습니다.**
정상입니다. 기능 메뉴 목록은 고정입니다. 새 메뉴가 필요하면 **커스텀 메뉴**로 만드세요.

**Q. 메뉴 이름은 바꿨는데 주소가 그대로입니다.**
표준 URL 은 바꿀 수 없습니다. 다른 주소로 보내려면 **커스텀 메뉴**를 만들어 기존 메뉴를 끄세요.

**Q. 체크박스가 회색으로 눌리지 않습니다.**
**모듈 미구현**(아직 기능 없음)이거나 **필수**(끌 수 없음) 메뉴입니다. 옆의 배지를 확인하세요.

**Q. 커스텀 메뉴를 켜려는데 저장이 거부됩니다.**
GNB 커스텀 슬롯이 가득 찼습니다. 다른 커스텀 메뉴를 끄거나 Header 설정에서 슬롯을 늘리세요.

**Q. 커스텀 메뉴에 빨간 `연결 끊김` 이 붙었습니다.**
연결한 기획전·카테고리·브랜드가 **삭제·미발행·비활성** 상태입니다. 대상을 되살리거나 다른 대상으로 바꾸세요.

**Q. 메뉴 미리보기의 GNB 구성에 카테고리가 없습니다.**
헤더 스킨이 **기본형**입니다. 기본형에서는 카테고리가 별도 패널이라 상단 메뉴 순서와 무관하며, 순서는 카테고리 관리에서 정합니다.

**Q. 순서를 바꿨는데 원하는 자리에 안 갑니다.**
기능 메뉴 · 커스텀 메뉴 · (드로어형이면) 카테고리가 **하나의 순서 축**을 공유합니다. 각 화면에서 따로 맞추지 말고 **메뉴 미리보기의 GNB 구성**에서 한 번에 끌어 정렬하세요.

**Q. 우측 레일이 안 보입니다.**
화면 폭이 **1600px 미만**이거나 모바일입니다. 정상입니다.

**Q. 관리자 사이드바에서 메뉴가 사라졌습니다.**
관리자 메뉴 관리에서 **활성**이 꺼졌거나, **접근 가능한 역할**에서 내 역할이 빠졌습니다.

---

## 10. 아직 지원하지 않는 것

- 기능 메뉴의 **주소(URL)와 위치 변경** — 코드로 고정되어 있습니다.
- 기능 메뉴 **추가·삭제** — 카탈로그가 고정입니다.
- **자유 배지 문구** — NEW / HOT / SALE 세 가지만 됩니다.
- 기능 메뉴의 **노출 기간 설정** — 기간 지정은 커스텀 메뉴에만 있습니다.
- **커스텀 메뉴에 하위 메뉴 직접 만들기** — 드로어형에서 카테고리 하위가 자동으로 붙는 것 외에는 없습니다.
