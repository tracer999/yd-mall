# 공지사항 관리

고객에게 알릴 **공지사항**과 **상품안내** 글을 쓰고 고치는 메뉴입니다. (**고객지원 관리 → 공지사항 관리**, 주소 `/admin/notices`)

---

## ⚠️ 먼저 알아둘 것 — 공지는 몰마다 따로입니다

**공지사항은 지금 편집 중인 몰에만 저장되고, 그 몰의 고객에게만 보입니다.** 상품·카테고리·배너·자주 묻는 질문과 똑같이 동작합니다.

그래서 글을 쓰기 전에 **상단에서 편집 중인 몰이 맞는지 반드시 확인**하세요. 몰을 잘못 고른 채 쓰면 **엉뚱한 몰에 공지가 올라갑니다.**

- 목록에도 **지금 편집 중인 몰의 공지만** 나옵니다. 써 둔 공지가 안 보인다면 대부분 **다른 몰을 편집 중인 것**입니다. 몰을 바꿔 보세요.
- **상단 고정 3개 제한도 몰마다 따로** 셉니다. A 몰에서 3개를 고정해도 B 몰에서 또 3개를 고정할 수 있습니다.

> 몰을 옮기는 기능은 없습니다. 다른 몰에도 같은 공지가 필요하면 **그 몰로 바꾼 뒤 다시 등록**하세요.

---

## 목록 화면

상단에 **전체 · 공지사항 · 상품안내** 탭이 있어 구분별로 걸러 볼 수 있습니다.
(화면 제목은 **"게시판 관리"** 로 표시되지만, 사이드바 메뉴 이름은 "공지사항 관리" 입니다. 같은 화면입니다.)

| 항목 | 설명 |
|------|------|
| **구분** | 공지사항 / 상품안내 뱃지 |
| **제목** | 클릭하면 상세 보기. 상단 고정 글은 앞에 **[중요]** 가 붙고 줄 전체가 붉게 표시됩니다. |
| **조회수** | 고객이 그 글을 열어본 횟수 (자동 집계) |
| **등록일** | 글을 처음 저장한 날짜 |
| **관리** | **수정** / **삭제** |

- 오른쪽 위 **글쓰기** 버튼으로 새 글을 씁니다.
- 정렬은 **상단 고정 글이 먼저**, 그다음 **최신순**으로 고정입니다. 순서를 직접 바꿀 수는 없습니다.

---

## 등록·수정 폼의 입력 항목

| 항목 | 설명 |
|------|------|
| **구분** | **공지사항** 또는 **상품안내** 중 선택. 이 값에 따라 **고객 화면에서 보이는 위치가 달라집니다.** (아래 참고) |
| **제목** | 필수. **100자까지**만 저장됩니다. 넘기면 잘립니다. |
| **중요 공지 (상단 고정)** | 체크하면 목록 맨 위에 붙고, 고객 화면에서 **필독** 표시가 붙습니다. |
| **내용** | 서식 편집기(굵게·정렬·목록·표·링크·이미지)로 작성합니다. |

### 상단 고정은 최대 3개

이미 3개가 고정된 상태에서 또 체크하고 저장하면 **"상단 고정은 최대 3개까지만 가능합니다"** 경고가 뜨고 **저장되지 않은 채 이전 화면으로 돌아갑니다.** 새로 고정하려면 기존 글 중 하나의 체크를 먼저 해제하세요.

### 이미지는 본문에 넣을 수 있습니다

편집기의 **이미지 버튼**으로 그림을 올려 본문 중간에 넣을 수 있습니다. 올린 이미지는 서버에 저장되어 고객 화면에 그대로 보입니다.

다만 **한글 문서·PDF 같은 첨부파일 기능은 없습니다.** 안내문을 파일로 내려받게 하는 건 불가능하고, 내용은 본문에 직접 적거나 이미지로 넣어야 합니다.

## 고객 화면에서 어디에 보이나요

공지는 **고객센터**를 중심으로 노출됩니다. 쇼핑몰 **첫 화면(홈)에는 공지가 나오지 않습니다.**

| 고객 화면 | 무엇이 보이나 |
|------|------|
| **고객센터** (`/cs`) | 헤더의 **고객센터** 메뉴로 들어갑니다. 본문 아래쪽에 **구분이 "공지사항"인 글만 최신 5건** 보입니다. **상품안내 글은 여기 안 나옵니다.** |
| **공지사항 목록** (`/boards/notice`) | 고객센터의 **"공지사항 전체보기"** 링크로 이동. 공지사항 글 전체를 **한 쪽에 10건씩** 나눠 보여줍니다. |
| **상품안내 목록** (`/boards/guide`) | 구분이 "상품안내"인 글 목록. **고객이 메뉴로 찾아갈 링크가 없어**, 주소를 직접 입력해야 들어갈 수 있습니다. |

고객이 글을 열면 **조회수가 1 올라갑니다.** 관리자가 상세를 봐도 조회수는 오르지 않습니다.

> **참고:** 이와 별개로 `/notices` 주소의 옛 공지 목록 화면이 남아 있습니다. 이 화면은 **공지사항과 상품안내를 구분 없이 전부 섞어서** 보여주고 쪽 나눔도 없습니다. 지금은 어떤 메뉴에서도 연결되지 않지만, 검색엔진에는 수집될 수 있습니다.

## 삭제

목록의 **삭제** 버튼을 누르면 확인창이 뜨고, 확인하면 지워집니다.

> **삭제하면 되돌릴 수 없습니다.** 휴지통이나 복구 기능이 없어 글이 즉시 완전히 사라집니다. 잠시 안 보이게만 하고 싶어도 **숨김 기능이 없으므로**, 지우지 말고 제목을 바꿔 두거나 내용을 비워 두는 편이 안전합니다.

---

## ⚠️ 주의할 점

- **편집 중인 몰을 먼저 확인하세요.** 공지는 그 몰에만 저장됩니다. 몰을 잘못 고르면 엉뚱한 몰에 올라가고, 목록에도 안 보입니다.
- **저장 = 즉시 공개.** 노출 여부·임시저장이 없어서 **저장하는 순간 고객 화면에 바로 뜹니다.** 작성 중인 글을 미리 저장해 두면 그대로 고객에게 보입니다. 완성한 뒤 한 번에 저장하세요.
- **예약 발행이 없습니다.** 원하는 시각에 맞춰 직접 저장해야 합니다.
- **상단 고정 3개를 넘기면 저장 자체가 취소됩니다.** 경고창을 닫고 돌아오면 작성 내용이 사라질 수 있으니, 긴 글은 미리 복사해 두세요.
- **구분을 잘못 고르면 고객이 못 봅니다.** 중요한 안내를 "상품안내"로 저장하면 **고객센터에 나오지 않고**, 링크도 없는 화면에만 들어가 사실상 아무도 못 봅니다. 일반 공지는 반드시 **공지사항**으로 저장하세요.
- **관리자 목록에는 검색·쪽 나눔이 없습니다.** 글이 쌓이면 한 화면에 전부 나열되어 원하는 글을 찾기 어려워집니다. 브라우저 찾기(Ctrl+F)를 쓰세요.

## 아직 안 되는 것

- **여러 몰에 한 번에 올리기** — 공지는 한 몰에만 속합니다. 몰마다 따로 등록해야 합니다.
- **다른 몰로 옮기기** — 이미 쓴 공지의 소속 몰을 바꿀 수 없습니다.
- **노출 여부(숨김) · 임시저장** — 저장하면 곧바로 공개됩니다.
- **예약 발행 · 게시 기간 설정** — 시작·종료일을 지정할 수 없습니다.
- **삭제 복구(휴지통)** — 지우면 영구 삭제입니다.
- **첨부파일** — 본문 이미지만 가능하고 문서 파일은 올릴 수 없습니다.
- **작성자 · 수정일 표시** — 누가 언제 고쳤는지 기록이 남지 않습니다. 등록일만 보입니다.
- **관리자 목록 검색 · 쪽 나눔 · 정렬 변경** — 지원하지 않습니다.
- **고정 순서 조절** — 상단 고정 글끼리의 순서는 최신순으로 고정입니다.
- **상품안내 메뉴 링크** — 고객이 상품안내 목록으로 갈 수 있는 메뉴가 없습니다.
