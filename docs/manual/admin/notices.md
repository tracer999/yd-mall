# 공지사항 관리

고객에게 알릴 **공지사항**과 **상품안내** 글을 쓰고 고치는 메뉴입니다. 배송 지연 안내, 휴무일 공지, 리뉴얼 안내처럼 **모든 고객에게 똑같이 알려야 하는 소식**을 여기에 올립니다. 관리자 화면에서는 이 메뉴가 **게시판 관리**라는 제목으로 열립니다.

## 1. 가장 먼저 — 공지는 몰마다 완전히 따로입니다

이 앱은 하나의 관리자 화면에서 **여러 개의 몰**을 만들고 관리하는 몰 빌더입니다. 공지사항은 상품·카테고리·배너와 마찬가지로 **몰별로 완전히 분리**되어 있습니다.

| 상황 | 결과 |
|---|---|
| 편집 몰이 **A몰**인 상태에서 글을 씀 | **A몰 고객에게만** 보입니다. B몰 고객에게는 아예 없는 글입니다. |
| 편집 몰을 **B몰**로 바꿔 목록을 봄 | A몰에서 쓴 글은 **목록에 나오지 않습니다.** |
| A몰 글의 주소를 알고 B몰 편집 상태에서 직접 입력 | **열리지 않고 목록으로 되돌아갑니다.** |
| 같은 내용을 두 몰 모두에 올리고 싶음 | **몰을 바꿔 가며 두 번 써야 합니다.** 복사 기능은 없습니다. |

> **글을 쓰기 전에 관리자 화면 상단의 편집 몰부터 확인하세요.** "분명히 등록했는데 목록에 없다"는 문의는 거의 전부 **다른 몰에 쓴 것**입니다. 글은 사라지지 않았고, 편집 몰을 되돌리면 그대로 나타납니다.

> **상단 고정 개수(최대 3개)도 몰마다 따로 셉니다.** A몰에 3개를 고정해 두었어도 B몰에서는 다시 3개까지 고정할 수 있습니다.

## 2. 전체 흐름 한눈에 보기

```
① 편집 중인 몰 확인          (제일 먼저!)
        ↓
② 글쓰기 → 구분 선택          (공지사항 / 상품안내)
        ↓
③ 제목 · 내용 작성            (필요하면 이미지 첨부)
        ↓
④ 중요 공지 여부 결정         (상단 고정 · 몰당 최대 3개)
        ↓
⑤ 저장 → 목록에서 확인
```

**등록하면 곧바로 고객에게 공개됩니다.** 임시저장·예약 발행·노출 켜기/끄기 같은 단계가 없습니다. 아직 확정되지 않은 내용은 **저장하지 마세요.**

### 3-1. 구분 탭

목록 위쪽에 탭 세 개가 있습니다.

| 탭 | 보이는 글 |
|---|---|
| **전체** | 공지사항 + 상품안내 모두 |
| **공지사항** | 구분이 `공지사항` 인 글만 |
| **상품안내** | 구분이 `상품안내` 인 글만 |

### 3-2. 목록 항목

| 칸 | 내용 |
|---|---|
| **구분** | `공지사항`(회색 배지) 또는 `상품안내`(초록 배지) |
| **제목** | 중요 공지는 제목 앞에 빨간 **[중요]** 표시가 붙고 **줄 전체가 연한 빨간색**이 됩니다. 누르면 상세 화면으로 갑니다. |
| **조회수** | 고객이 이 글을 몇 번 열어 봤는지입니다. |
| **등록일** | 글을 저장한 날짜 |
| **관리** | `수정` · `삭제` |

### 3-3. 정렬 규칙

목록은 **중요 공지 먼저, 그다음 최신순**으로 정렬됩니다. 즉 상단 고정한 글이 항상 맨 위에 모여 있고, 그 아래로 나머지가 최근 작성 순서대로 이어집니다.

**중요 공지끼리의 순서를 직접 지정할 수는 없습니다.** 중요 공지 3개 중 특정 글을 맨 위에 두고 싶다면, 그 글을 나중에 등록하거나 다른 글의 중요 표시를 잠시 꺼야 합니다.

## 4. 글 쓰기

<div class="manual-goto-bar">
  <span class="manual-goto-label">이 단계 화면</span>
  <a class="manual-goto" href="/admin/notices/create" target="_blank">새 글 쓰기 화면 열기</a>
</div>

목록 오른쪽 위 **글쓰기** 버튼을 누르면 작성 화면이 열립니다.

### 4-1. 입력란 전체

| 입력란 | 필수 | 설명 | 예시 |
|---|---|---|---|
| **구분** | 필수 | `공지사항` 또는 `상품안내` 중 선택합니다. 기본값은 `공지사항`. 아래 4-2 참고. | `공지사항` |
| **제목** | 필수 | 고객이 목록에서 보는 제목입니다. 비우면 저장되지 않습니다. | `설 연휴 배송 일정 안내` |
| **중요 공지 (상단 고정)** | 선택 | 체크하면 목록 맨 위에 고정되고 **[중요]** 표시가 붙습니다. 아래 4-3 참고. | |
| **내용** | 필수 | 서식 편집기로 작성합니다. 굵게·정렬·목록·링크·표·이미지를 넣을 수 있습니다. | 여러 문단 가능 |

입력을 마치면 **저장** 버튼을 누릅니다. 취소를 누르면 작성한 내용이 **저장되지 않고 목록으로 돌아갑니다.**

### 4-2. 구분 두 가지 — 무엇이 다른가

| 구분 | 어떤 글을 올리나 |
|---|---|
| **공지사항** | 몰 운영 전반에 관한 소식. 휴무·배송 지연·점검·정책 변경·이벤트 안내 등. |
| **상품안내** | 상품 자체에 관한 안내. 원산지 변경, 리뉴얼, 성분 변경, 보관 방법, 주의사항 등. |

**두 구분의 차이는 분류와 배지 색뿐입니다.** 어느 쪽을 골라도 고객에게 공개되는 방식은 같습니다. 다만 나중에 글이 쌓였을 때 **탭으로 걸러 보기 위해** 처음부터 구분을 제대로 정해 두는 편이 좋습니다.

### 4-3. 중요 공지(상단 고정) — 최대 3개

체크하면 그 글이 목록 맨 위에 고정되고 고객 화면에서도 강조되어 표시됩니다.

> **한 몰에 상단 고정은 최대 3개까지입니다.** 이미 3개가 고정된 상태에서 네 번째 글에 체크하고 저장하면 **"상단 고정은 최대 3개까지만 가능합니다."** 라는 알림이 뜨고 이전 화면으로 되돌아갑니다. **이때 작성 중이던 내용은 남아 있으니 당황하지 마세요.** 체크를 풀고 다시 저장하거나, 기존 고정 글 하나의 체크를 먼저 해제한 뒤 다시 시도하세요.

**중요 공지는 아껴 쓰세요.** 3개가 항상 채워져 있으면 고객은 그 자리를 배경처럼 여기게 되어 정작 급한 공지를 놓칩니다. 상황이 끝난 공지는 수정 화면에서 체크를 해제해 일반 글로 내리세요.

**상단 고정에 어울리는 글**

- 연휴·휴무로 배송이 며칠 밀린다는 안내
- 시스템 점검으로 주문이 막히는 시간대 안내
- 진행 중인 대형 프로모션 안내

**어울리지 않는 글**

- 이미 끝난 이벤트 안내
- 몇 달 전 등록한 일반 정책 안내

### 4-4. 내용 편집기 사용법

내용 칸은 서식 편집기입니다. 도구 모음에서 다음을 쓸 수 있습니다.

- **되돌리기 / 다시 실행**
- **글머리 형식(제목·본문)**
- **굵게 · 기울임**
- **정렬(왼쪽·가운데·오른쪽)**
- **번호 목록 · 글머리 기호 목록**
- **링크 · 이미지 · 표**
- **소스 코드 보기**

**이미지 넣기**

도구 모음의 이미지 버튼으로 **PC에 있는 이미지 파일을 바로 올릴 수 있습니다.** 업로드된 이미지는 서버에 저장되고 글 안에 자동으로 삽입됩니다.

- 이미지를 여러 장 넣으면 글이 무거워집니다. 안내 이미지는 **1~2장**이 적당합니다.
- **모바일에서 보는 고객이 많습니다.** 가로로 아주 긴 이미지는 글씨가 읽히지 않을 수 있으니, 중요한 내용은 이미지 대신 글로도 함께 적어 주세요.
- **글을 삭제해도 업로드한 이미지 파일 자체는 서버에 남습니다.**

## 5. 글 수정하기

목록에서 **수정** 을 누르면 작성할 때와 같은 화면이 열리고 기존 내용이 채워져 나옵니다. 고칠 부분만 바꾸고 **저장**을 누르면 됩니다.

- **저장 버튼을 눌러야 반영됩니다.** 내용만 고치고 뒤로 가기를 누르면 변경이 사라집니다.
- **수정해도 등록일은 바뀌지 않습니다.** 그래서 오래된 글을 고쳐도 목록 위로 올라오지 않습니다. 꼭 위로 올려야 한다면 **중요 공지로 체크**하거나 새 글로 다시 쓰세요.
- **조회수는 수정해도 초기화되지 않습니다.**
- 중요 공지 체크를 새로 켜는 경우, 이미 3개가 고정되어 있으면 4-3의 알림이 뜹니다. (자기 자신은 개수에서 빠지므로, 이미 고정된 글을 수정하는 것은 문제없습니다.)

## 6. 글 삭제하기

목록에서 **삭제** 를 누르면 "정말 삭제하시겠습니까?" 확인 창이 뜹니다. 확인하면 **글이 완전히 지워집니다.**

> **삭제는 되돌릴 수 없습니다.** 내용도 조회수도 함께 사라집니다. 휴지통이나 복구 기능은 없습니다.

**지우기 전에 한 번 더 생각하세요.** 지난 공지는 "언제 무엇을 안내했는지"에 대한 기록이기도 합니다. 배송 지연·환불 정책 관련 공지는 나중에 분쟁이 생겼을 때 근거가 됩니다. 화면을 정리하려는 목적이라면 삭제 대신 **중요 공지 체크만 해제**해 아래로 내려 두는 편이 안전합니다.

## 9. 자주 막히는 상황

**Q. 글을 등록했는데 목록에 없습니다.**
거의 대부분 **다른 몰에 등록**한 경우입니다. 관리자 화면 상단의 편집 몰을 글 쓸 때의 몰로 되돌려 보세요. 글은 사라지지 않았습니다.

**Q. 고객 화면에는 안 보이는데 관리자 목록에는 있습니다.**
고객이 보고 있는 몰과 글이 저장된 몰이 다릅니다. 편집 몰과 고객이 접속한 몰이 같은지 확인하세요.

**Q. 중요 공지로 체크하고 저장했더니 알림이 뜨고 되돌아왔습니다.**
그 몰에 이미 상단 고정이 3개 있습니다. 기존 고정 글 중 하나를 수정 화면에서 체크 해제한 뒤 다시 시도하세요.

**Q. 중요 공지 3개의 순서를 바꾸고 싶습니다.**
순서를 직접 지정하는 기능은 없습니다. 중요 공지끼리는 **최근에 등록한 글이 위**로 옵니다.

**Q. 오래된 글을 수정했는데 목록 위로 안 올라옵니다.**
정상입니다. 수정해도 등록일은 그대로입니다. 위로 올리려면 중요 공지로 체크하거나 새 글로 다시 쓰세요.

**Q. 예약 발행이나 임시저장을 하고 싶습니다.**
지원하지 않습니다. **저장하는 즉시 고객에게 공개**됩니다. 미리 준비만 하려면 별도 문서에 적어 두었다가 시점이 되면 옮겨 붙이세요.

**Q. 글을 고객에게 안 보이게 잠시 숨기고 싶습니다.**
노출을 끄는 기능은 없습니다. 삭제하거나, 그대로 두는 두 가지 선택뿐입니다.

**Q. 붙여넣은 표나 서식이 깨집니다.**
워드·한글·웹페이지에서 그대로 복사하면 서식이 어긋날 수 있습니다. 편집기 안에서 표 도구로 직접 만들거나, 글자만 붙여넣은 뒤 서식을 다시 적용하세요.

**Q. 특정 회원 그룹에게만 공지하고 싶습니다.**
지원하지 않습니다. 공지사항은 그 몰의 모든 고객에게 동일하게 보입니다.

**Q. 같은 공지를 여러 몰에 한 번에 올리고 싶습니다.**
현재는 몰을 바꿔 가며 각각 등록해야 합니다. 내용을 미리 만들어 두고 복사·붙여넣기 하는 것이 가장 빠릅니다.

---

## 10. 아직 지원하지 않는 것

- 예약 발행 · 임시저장
- 노출 켜기/끄기 (등록하면 즉시 공개, 내리려면 삭제)
- 중요 공지끼리의 순서 지정
- 여러 몰에 한 번에 등록 · 몰 간 복사
- 특정 회원 그룹에만 공개
- 삭제한 글 복구
- 목록에서 제목·내용 검색
- 공지 등록 시 고객에게 알림 발송
