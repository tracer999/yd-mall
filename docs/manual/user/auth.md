# 로그인 · 회원가입

고객이 **쇼핑몰에 로그인**하거나 **새로 가입**하는 화면입니다. 주소는 `/auth/login`, `/auth/signup` 입니다.

가입 방법은 두 가지입니다. **이메일 + 비밀번호로 직접 가입**하거나, **구글 · 카카오 · 네이버 계정으로 간편 가입**할 수 있습니다. 소셜 버튼은 관리자에 키가 등록된 것만 화면에 나타납니다.

어느 쪽으로 가입하든 **휴대폰 번호와 약관 동의는 반드시 받습니다.** 소셜로 로그인했더라도 휴대폰 번호가 없으면 로그인할 때마다 **추가 정보 입력** 화면(`/auth/signup-finish`)으로 계속 되돌아옵니다.

---

## 이 화면에 무엇이 보이나

- **로그인** — 이메일 / 비밀번호 입력칸, **`사업자 회원으로 로그인` 체크박스**, 소셜 로그인 버튼(구글 · 카카오 · 네이버)
- **회원가입** — 소셜 가입 버튼 + 이메일 가입폼(비밀번호는 **8자 이상, 영문·숫자 포함**) + `사업자 회원으로 가입하기` 링크
- **추가 정보 입력** — 성함 · 이메일 · 휴대폰 번호 · 기본 배송지, 약관 동의
- **약관 재동의**(`/auth/terms-update`) — 약관이 개정되면 로그인 시 자동으로 뜹니다
- **가입 완료** 화면

휴대폰 번호와 이메일은 입력 직후 **중복 여부를 바로 확인**해 줍니다. 이미 가입된 값이면 진행되지 않습니다.

---

## 사업자 회원 가입 · 로그인

일반 회원과 **같은 화면**을 씁니다. 별도의 사업자 전용 로그인 페이지는 없습니다.

**가입** — 회원가입 화면의 `사업자 회원으로 가입하기` 를 누르면 일반 가입 항목 아래에 **사업자 정보 입력란**이 붙습니다(상호 · 사업자등록번호 · 대표자명 · 업태/종목 · 사업장 주소 · 세금계산서 이메일 · 담당자 · **사업자등록증 사본**). 이미 가입한 회원은 마이페이지에서 **사업자 회원 신청**으로 전환할 수 있습니다.

- 사업자등록번호는 **형식(체크섬)을 검사**하므로 아무 숫자나 넣으면 진행되지 않습니다.
- **사업자등록증 첨부는 필수**입니다. 담당자가 이 파일과 입력값을 대조해 승인합니다.
- 가입만으로는 전용가를 볼 수 없습니다. **승인을 받아야** 합니다.
- 사업자 가입에는 **소셜 간편 가입 버튼이 나오지 않습니다**(등록증 첨부가 필요해서입니다).

**로그인** — `사업자 회원으로 로그인` 체크박스로 **어느 자격으로 들어올지**를 밝힙니다. 자격은 **서로 배타적**이어서, 계정과 맞지 않으면 로그인이 거부됩니다.

| 상황 | 결과 |
|---|---|
| 체크 + 승인된 사업자 | **로그인 성공** — 전용가·사업자 주문 절차 |
| 체크 + 심사 중 / 반려 | 로그인은 되고 **신청 상태 화면**으로 이동 (전용가는 승인 후) |
| 체크 + 사업자 회원이 아님 | **로그인 거부** — "사업자 회원이 아닙니다" 안내 |
| 체크 안 함 + 사업자 회원 | **로그인 거부** — "기업회원 계정입니다. `사업자 회원으로 로그인`을 선택해 주세요" |
| 체크 안 함 + 일반 회원 | **로그인 성공** |

> **사업자 회원으로 신청한 계정은 반드시 체크해야 로그인됩니다.** 일반 회원 자격으로는 들어올 수 없고, 반대로 일반 회원이 체크하면 거부됩니다. 로그인한 뒤에 자격을 바꾸는 기능은 없습니다.

자세한 내용은 [사업자(B2B) 구매](/manual/user/b2b)를 보세요.

## 이 화면은 관리자에서 이렇게 만들어집니다

| 고객 화면의 이 부분 | 관리자 어느 기능으로 만드나 | 비고 |
|---|---|---|
| **구글 / 카카오 / 네이버** 버튼 표시 여부 | 사이트 설정의 소셜 로그인 항목 | **클라이언트 ID + 콜백 URL 둘 다** 있어야 버튼이 나옵니다 |
| 콜백 URL (개발/운영 구분) | 사이트 설정의 `개발용 / 운영용 콜백 URL` | 환경에 맞는 쪽이 비어 있으면 버튼이 사라집니다 |
| 약관 동의 화면에 뜨는 **이용약관 본문** | 약관/정책 관리에서 **시행 중(활성)** 인 이용약관 | |
| 약관 동의 화면의 **개인정보 처리방침 본문** | 약관/정책 관리에서 시행 중인 개인정보 처리방침 | |
| **약관 재동의** 화면이 뜨는 것 | 약관/정책 관리에서 **새 버전을 만들고 활성화**했을 때 | 기존 회원 전원이 다음 로그인에 재동의 화면을 봅니다 |
| 가입 직후 자동 지급되는 쿠폰 | 쿠폰 관리에서 만든 **가입 쿠폰** | 없으면 아무것도 지급되지 않습니다 |
| 가입한 회원 목록 · 활성/비활성 | 회원 관리 | |
| **사업자 가입 신청 승인 여부** | B2B 관리 > 기업회원 승인 | 승인 전에는 전용가가 보이지 않습니다 |

> **약관을 새 버전으로 활성화하면 기존 회원 전원이 재동의 화면을 보게 됩니다.** 오탈자 수정 정도라면 새 버전을 만들지 말고 기존 버전을 고치는 편이 낫습니다.

> **소셜 로그인 키를 바꾸면 앱을 재기동하지 않아도 바로 반영됩니다.** 다만 콜백 URL은 각 소셜 서비스(구글·카카오·네이버) 콘솔에도 **똑같이 등록**돼 있어야 로그인이 성공합니다.

## 안 보이거나 비어 있을 때

**"카카오(또는 네이버·구글) 버튼이 안 보입니다"**
사이트 설정에서 그 서비스의 **클라이언트 ID** 또는 **현재 환경에 맞는 콜백 URL** 중 하나가 비어 있습니다. 둘 다 채워야 버튼이 나타납니다.

**"버튼은 있는데 누르면 오류가 납니다"**
콜백 URL이 소셜 서비스 콘솔에 등록된 주소와 다릅니다. 관리자에 넣은 값과 콘솔의 값을 글자 그대로 맞추세요.

**"약관 화면에 '약관 내용이 등록되지 않았습니다'라고 나옵니다"**
약관/정책 관리에 **시행 중(활성)인 약관 버전이 없습니다.** 이용약관과 개인정보 처리방침을 각각 하나씩 만들고 활성화하세요.

**"로그인만 하면 계속 추가 정보 입력 화면이 나옵니다"**
그 회원에게 **휴대폰 번호가 저장되지 않은** 상태입니다. 입력을 끝내야 통과됩니다.

**"가입했는데 쿠폰이 안 들어옵니다"**
쿠폰 관리에 가입 시 지급되는 쿠폰이 설정되어 있지 않습니다.
