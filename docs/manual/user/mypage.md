# 마이페이지

고객이 **자기 주문·쿠폰·포인트·찜한 상품**을 확인하고, **회원 정보를 고치거나 취소·반품을 신청**하는 화면입니다. 주소는 `/mypage` 이며 **로그인해야** 들어갈 수 있습니다.

PC에서는 화면 위 오른쪽의 **마이쇼핑**, 모바일에서는 아래 고정 바의 **마이** 로 들어갑니다.

마이페이지는 고객이 직접 무언가를 만드는 화면이 아니라, **관리자에서 만들어 둔 데이터를 고객별로 비춰 주는 거울**에 가깝습니다. 주문·쿠폰·포인트·등급이 관리자에 없으면 마이페이지도 비어 있습니다.

---

## 이 화면에 무엇이 보이나

첫 화면 위쪽에 **찜 상품 · 최근 본 상품 · 보유 포인트**, 그 아래에 **주문중 · 배송중 · 반품/취소** 건수가 카드로 요약됩니다(아래 [상단 요약 카드](#상단-요약-카드)). 왼쪽 메뉴는 다음과 같습니다.

| 메뉴 | 주소 | 내용 |
|---|---|---|
| 장바구니 | `/cart` | |
| 주문내역 | `/mypage/orders` | 주문 목록 · 주문 상세 · 취소/반품 신청 |
| 취소/반품 내역 | `/mypage/claims` | 진행 상황 확인 · 신청 철회 |
| 리뷰 관리 | `/mypage/reviews` | 쓸 수 있는 리뷰 · 내가 쓴 리뷰 |
| 쿠폰함 | `/mypage/coupons` | 사용가능 / 주문진행중 / 사용완료 / 기간만료 |
| 포인트 | `/mypage/points` | 잔액 + 적립·사용 내역 |
| 1:1 문의 | `/mypage/activities` | 내 문의와 답변 상태 |
| 회원정보 수정 | `/mypage/profile` | |

이 밖에 찜한 상품(`/mypage/likes`), 찜한 브랜드(`/mypage/brand-likes`), 최근 본 상품(`/mypage/recent-views`, 최근 15일), 회원 탈퇴(`/mypage/withdraw`) 화면이 있습니다.

### 상단 요약 카드

메뉴로 들어가지 않고 **첫 화면에서 바로 확인하고 눌러 들어가는** 자리입니다. 두 줄로 되어 있습니다.

**첫째 줄** — 찜 상품 · 최근 본 상품 · 보유 포인트

**둘째 줄(주문 현황)** — 숫자를 누르면 그 조건으로 걸러진 주문 목록이 열립니다.

| 카드 | 세는 것 | 누르면 |
|---|---|---|
| **주문중** | 입금대기 + 결제완료 + 배송준비 | `/mypage/orders?filter=ongoing` |
| **배송중** | 배송중 | `/mypage/orders?filter=shipping` |
| **반품/취소** | 취소·환불된 주문 **+** 취소·반품·교환을 신청한 주문 | `/mypage/orders?filter=claim` |

- **배송완료는 카드로 두지 않습니다.** 끝난 일이라 전체 주문 내역에서 봅니다(목록의 `배송완료` 탭).
- **반품/취소 카드**는 두 가지를 합쳐 셉니다 — 고객이 신청한 건(부분 취소·부분 반품·교환은 주문이 살아 있어 주문 상태만으로는 잡히지 않습니다)과 판매자가 취소한 주문(신청 기록이 없습니다). 둘 다 고객에게는 "내 취소·반품 건"이므로 함께 보여 줍니다.
- 아직 처리되지 않은 신청이 있으면 카드에 **빨간 배지와 `N건 처리 중`** 이 표시됩니다.
- 카드 숫자와 눌러서 열리는 목록의 건수는 **같은 조건으로 세므로 항상 일치**합니다.

주문 목록 위쪽에는 `전체 · 주문중 · 배송중 · 배송완료 · 반품/취소` 탭이 있어 화면 안에서도 바꿔 볼 수 있습니다.
취소·반품 **신청 건의 진행 상황**(승인·반려·회수)은 [취소·반품 내역](/mypage/claims)에서 봅니다.

### 취소 · 반품 · 교환 — 언제 무엇이 가능한가

**셋을 가르는 기준은 딱 하나, "물건이 출고됐는가" 입니다.**

| 주문 상태 | 신청할 수 있는 것 | 왜 |
|---|---|---|
| 결제 전(대기) · 결제완료 · 상품준비중 | **취소** | 아직 안 보냈으니 회수할 물건이 없습니다 |
| 배송중 · 배송완료 | **반품** 또는 **교환** (고객이 선택) | 이미 보냈으니 **되돌려받아야** 합니다 |
| 이미 취소·환불된 주문 | 없음 | — |

- **반품·교환은 배송완료 처리 후 7일 이내**만 신청할 수 있습니다. 7일은 판매자가 배송 관리에서 `배송완료 처리`를 누른 시각부터 셉니다.
- 이미 처리 중인 신청이 있으면 같은 주문에 또 신청할 수 없습니다. 먼저 결과를 기다리거나 **철회**하세요.

#### 세 가지가 어떻게 다른가

| | **취소** | **반품** | **교환** |
|---|---|---|---|
| 언제 | 출고 전 | 출고 후 7일 내 | 출고 후 7일 내 |
| 물건 | 회수 없음 | **회수함** | **회수 후 같은 상품 재발송** |
| 돈 | 환불 | 환불 | **환불 없음** |
| 배송비 | 해당 없음 | 단순 변심이면 **왕복 배송비 차감** | 좌동 |

#### 환불은 언제 이루어지나

| 주문 상태 | 처리 방식 |
|---|---|
| **결제 전 · 결제완료** | **신청 즉시 자동 승인 + 환불** — 담당자 확인을 기다리지 않습니다 |
| **상품 준비중** | 담당자 확인 후 승인되면 환불 |
| **배송중 · 배송완료(반품)** | 담당자가 **물건을 회수해 상태를 확인한 뒤** 승인 → 환불 |

- 환불은 결제수단으로 되돌아가며, 카드사 사정에 따라 **영업일 기준 3~5일**이 걸립니다.
- 사용한 포인트는 포인트로, 쓴 쿠폰은 쿠폰으로 돌아옵니다(전체 취소 시).
- **단순 변심 반품**은 왕복 배송비가 환불액에서 빠집니다. **상품 불량·오배송**은 빠지지 않습니다.

> **자동 승인된 취소는 되돌릴 수 없습니다.** 결제 직후 취소는 승인 대기 없이 즉시 환불로 이어집니다.

#### 신청하는 방법 — 전용 화면에서

주문 상세의 **[주문 취소]** 또는 **[반품 · 교환 신청]** 을 누르면 **신청 화면**(`/mypage/orders/:id/claim`)이 열립니다.
팝업이 아니라 한 화면에서 상품을 보며 고릅니다.

1. **무엇을 신청할지** — 출고 후에만 반품/교환 중에서 고릅니다(출고 전이면 취소로 고정).
2. **사유** — 단순 변심 / 상품 불량 / 오배송 / 기타. `기타`는 직접 적습니다.
3. **대상 상품** — 둘 중 하나로 신청합니다.
   - 상품 줄의 **[이 상품만 신청]** — 그 상품만
   - 아래의 **[주문 전체 신청하기]** — 주문에 담긴 상품 전부

> **수량은 고르지 않습니다.** 상품 단위로 통째로 처리합니다.
> 이미 취소·반품한 상품은 `처리 완료`로 표시되고 버튼이 사라집니다.

#### 일부 상품만 신청했을 때

- **주문은 그대로 살아 있습니다.** 남은 상품은 계속 배송됩니다. 결제 상태만 `부분환불`로 바뀝니다.
- 환불액은 **그 상품이 차지한 금액 비율만큼** 쿠폰·등급 할인과 사용 포인트를 나눠 뺀 금액입니다. 할인받고 산 물건을 정가로 돌려받지 않습니다.
- **배송비는 남은 상품이 있는 동안 환불되지 않습니다.** 나머지 상품이 그 배송으로 가기 때문입니다. 마지막 상품까지 처리되면 그때 배송비도 함께 환불됩니다.
- 나눠서 여러 번 신청해도 **환불액 합계는 결제금액과 정확히 일치**합니다.

### 구매확정과 적립금

**적립금은 결제할 때가 아니라 구매확정할 때 지급됩니다.**

반품하는 고객에게도 일단 포인트를 줬다가 도로 뺏어야 하고, 이미 써 버렸으면 회수할 수도 없기
때문입니다. 구매확정 시 지급하면 그 문제가 없어지고, 부분 반품한 금액을 뺀 **실제 결제액** 기준으로
정확히 계산됩니다.

| 언제 | 무슨 일이 |
|---|---|
| 결제 완료 | 적립 **예정** 금액만 계산됩니다(아직 지급 안 됨) |
| **구매확정** | **적립금 지급** + 반품·교환 신청 종료 |

#### 구매확정하는 방법

주문 상세에서 **[구매확정]** 버튼을 누릅니다. **배송완료된 주문**에만 나타나며,
취소·반품 신청이 처리 중이면 나오지 않습니다.

> ⚠️ **구매확정하면 반품·교환을 신청할 수 없습니다.** 물건을 확인한 뒤에 누르세요.

#### 누르지 않으면 어떻게 되나

배송완료 후 일정 기간이 지나면 **자동으로 확정되고 적립금이 지급됩니다**(기본 7일).
기간은 관리자의 [사이트 설정](/manual/admin/settings)에서 정하며, 반품 가능 기간(7일)보다
짧게 설정해도 7일이 지나야 확정됩니다 — 반품할 시간을 뺏지 않기 위해서입니다.

> 관리자가 이 값을 **0(자동 확정 안 함)** 으로 두면, 고객이 직접 누르지 않는 한
> 적립금이 지급되지 않습니다.

#### 적립금이 0원으로 나올 때

- 사이트 설정의 **구매 적립률**이 0입니다.
- 또는 주문 당시 **회원 등급의 적립률**이 0입니다(주문 시점 등급이 기준입니다).
- 또는 부분 취소·반품으로 **환불된 금액을 빼고 나면 남는 결제액이 없습니다.**
- 배송비에는 적립이 붙지 않습니다.

## 리뷰

### 누가 쓸 수 있나

**산 사람만, 물건을 받은 뒤에** 쓸 수 있습니다. 조건은 셋입니다.

1. 본인이 주문한 상품
2. 주문이 **배송완료**된 상태
3. 그 주문 상품으로 **아직 쓰지 않음**

- **취소·반품한 상품은 쓸 수 없습니다.** 돌려보낸 물건의 후기를 받을 이유가 없고, 적립까지 나가면 "사서 반품하고 리뷰로 포인트만 받는" 길이 열리기 때문입니다.
- 같은 상품을 **두 번 사면 두 번** 쓸 수 있습니다(주문 상품 단위로 하나씩).

### 어디서 쓰나

마이페이지 왼쪽 메뉴의 **리뷰 관리**(`/mypage/reviews`)입니다.

- **작성할 수 있는 리뷰** — 배송완료된 상품이 목록으로 나오고, 각 줄에 **받게 될 적립금**이 함께 표시됩니다.
- **내가 쓴 리뷰** — 지금까지 쓴 리뷰와 받은 적립금. 여기서 삭제할 수 있습니다.

작성 화면에서는 **별점(필수) · 내용(5자 이상) · 사진 1장(선택)** 을 입력합니다.

> 리뷰를 삭제해도 **이미 받은 적립금은 회수되지 않습니다.**

### 리뷰 적립금

**구매 금액에 따라 정해진 금액**을 드립니다. 비율이 아니라 **정액**이며, **회원 등급과는 무관**합니다.

| | |
|---|---|
| 기준 | 그 주문의 **결제금액** |
| 구간 | 관리자가 정합니다 (예: 1만·5만·10만·50만·100만원 이상) |
| 적용 | 결제금액이 걸리는 구간 중 **가장 높은 구간** |
| 사진 리뷰 | **사진 적립액으로 대체**됩니다(글 적립에 더하지 않습니다) |
| 지급 시점 | 리뷰 등록 **즉시** |

예를 들어 구간이 `1만원 이상 500P / 5만원 이상 1,000P` 이고 5만 4천원을 결제했다면 **1,000P**를 받습니다.
사진 적립액이 2,000P로 설정돼 있다면 사진을 올렸을 때 **2,000P**를 받습니다.

**적립금이 0원일 때** — 관리자가 구간을 하나도 만들지 않았거나, 결제금액이 가장 낮은 구간보다 적은 경우입니다.

### 내 리뷰가 상품 상세에 안 보입니다

판매자가 **비공개 처리**했을 수 있습니다. 내 리뷰 목록에는 `비공개 처리됨` 으로 표시됩니다.

## 사업자(B2B) 고객이 볼 때

일반 회원에게는 **`사업자 회원 신청`** 링크 한 줄만 보입니다. 신청하거나 승인된 회원에게는 다음이 추가됩니다.

- **현재 구매 자격** — 승인된 사업자로 로그인했으면 `기업 구매(상호)` 가 표시됩니다. 자격은 로그인한 계정에 고정되므로 **전환 버튼은 없습니다.**
- **사업자 정보** — 신청 상태(심사 대기 · 승인 · 반려 · 정지 · 계약 만료)와 등록한 사업자 내용. 반려된 경우 사유를 보고 다시 신청할 수 있습니다.
- **견적함** — 진행 중인 견적 목록. **승인된 사업자에게만** 보입니다.

자세한 내용은 [사업자(B2B) 구매](/manual/user/b2b)를 보세요.

## 안 보이거나 비어 있을 때

**"쿠폰함이 비어 있습니다"**
쿠폰 관리에서 그 회원에게 쿠폰이 **발급**되지 않았습니다. 쿠폰을 만들기만 하고 발급하지 않으면 고객 쿠폰함에는 아무것도 없습니다.

**"쿠폰이 '주문진행중'으로 묶여 있습니다"**
다른 주문서에서 그 쿠폰을 선택한 채 결제를 끝내지 않은 상태입니다. **30분이 지나면 자동으로 풀립니다.**

**"포인트가 안 쌓입니다"**
사이트 설정의 **구매 적립률**이 0이면 적립이 없습니다. 값이 비어 있을 때만 기본값이 쓰이고, **0을 저장하면 "적립 없음"으로 동작**합니다.

**"등급·멤버십 정보가 안 나옵니다"**
멤버십 관리에서 등급이 설정되지 않았습니다.

**"배송 조회 버튼이 없습니다"**
배송 관리에서 해당 주문에 **송장번호를 등록**하지 않았습니다.

**"휴대폰 번호나 주소를 못 고칩니다 / 탈퇴가 안 됩니다"**
이 항목들은 **카카오 본인 확인**을 거쳐야 합니다. 사이트 설정에 카카오 로그인 키와 콜백 URL이 들어 있어야 동작합니다. 확인 후 **15분이 지나면 다시 인증**해야 합니다.
