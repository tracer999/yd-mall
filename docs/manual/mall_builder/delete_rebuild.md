# 몰 지우고 다시 만들기

몰 빌더에서는 몰을 **만들고 → 지우고 → 다시 만드는** 반복이 잦습니다. 이 문서는 만든 몰을 안전하게 지우는 방법을 설명합니다.

## 삭제 규칙

| 몰 종류 | 삭제 | 이유 |
|---------|------|------|
| **기본몰** | ❌ 불가 | 몰 해석기의 폴백 대상이라 없으면 몰 표시가 깨집니다. 먼저 다른 몰을 기본몰로 지정하세요. |
| 그 외 몰 (데이터 없음) | ✅ 바로 삭제 | 확인 후 즉시 삭제 |
| 그 외 몰 (데이터 있음) | ✅ 강제 삭제 | 딸린 데이터까지 함께 지웁니다(아래 참고) |

> 기본몰이 아니면 **카테고리·상품이 있어도 삭제할 수 있습니다.** 예전에는 데이터가 있으면 삭제가 막혔지만, 몰 빌더 특성상 "만들고 지우기"가 잦아 **강제 삭제**를 지원합니다.

## 데이터가 없는 몰 삭제

몰 관리(`/admin/malls`) 표에서 해당 몰의 **삭제** 버튼을 누르면 확인창이 뜨고, 확인하면 바로 지워집니다. 몰 전용 설정(내비·메뉴·테마)도 함께 정리됩니다.

## 데이터가 있는 몰 강제 삭제

카테고리·상품이 있는 몰의 **삭제** 버튼을 누르면 **강제 삭제 확인 창**이 뜹니다.

1. 창에 **무엇이 지워지는지**가 표시됩니다:
   - 카테고리 N개 · 상품 M개 (딸린 리뷰·장바구니·이미지 포함)
   - 몰 전용 설정: 내비·GNB 메뉴·테마·페이지·배너·딜·이벤트·쿠폰 등
2. 오삭제를 막기 위해 **몰 코드를 그대로 입력**해야 삭제 버튼이 활성화됩니다.
3. **강제 삭제**를 누르면 몰과 딸린 모든 데이터가 **한 번에** 지워집니다.

> **되돌릴 수 없습니다.** 지운 데이터는 복구되지 않습니다.
>
> 단, **주문 이력은 지워지지 않고** 상품 링크만 끊깁니다(주문 기록 보존).

### 무엇이 함께 지워지나

몰에 `mall_id` 로 묶인 데이터에는 데이터베이스 외래키가 없어서, 몰만 지우면 **고아 데이터**가 남습니다. 그래서 강제 삭제는 다음을 **정해진 순서로 함께** 지웁니다.

- 상품과 그에 딸린 것들(리뷰·장바구니·상품 이미지·추천·SEO·랭킹 등)
- 카테고리(깊은 단계부터)
- 딜·이벤트·기획전·공동구매·라이브·쿠폰·FAQ·아웃렛·상품그룹 등 몰 전용 데이터
- 내비게이션·GNB 메뉴·커스텀 메뉴·배송 정책·사이트 설정·테마
- 페이지(홈 등)와 그 섹션

이 전체 삭제는 하나의 트랜잭션으로 처리되어, 중간에 실패하면 아무것도 지워지지 않고 되돌아갑니다.

## 다시 만들기

삭제가 끝나면 몰 목록에서 그 몰이 사라집니다. 새로 시작하려면 [몰 만들기](create_mall) 단계를 처음부터 다시 진행하면 됩니다. 같은 코드를 재사용해도 됩니다.
