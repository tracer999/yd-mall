# 4. 몰 지우고 다시 만들기

몰 빌더에서는 만들고 → 지우고 → 다시 만드는 반복이 잦습니다. 만든 몰은 **데이터가 있어도 지울 수 있습니다.**

## 삭제 규칙

| 몰 종류 | 삭제 | 방법 |
|---------|------|------|
| **기본몰** | 불가 | 다른 몰을 먼저 기본몰로 지정해야 합니다 |
| 그 외 · 데이터 없음 | 가능 | **삭제** 버튼 → 확인창 → 즉시 삭제 |
| 그 외 · 데이터 있음 | 가능 | **삭제** 버튼 → 강제 삭제 창에서 **몰 코드 재입력** |

## 강제 삭제 순서

1. 몰 리스트에서 해당 몰의 **삭제** 버튼을 누릅니다.
2. 창에 지워질 내용(상품 수·카테고리 수, 몰 전용 설정)이 표시됩니다.
3. 오삭제 방지를 위해 **몰 코드를 그대로 입력**해야 **강제 삭제** 버튼이 활성화됩니다.
4. 누르면 몰과 딸린 데이터가 한 트랜잭션으로 함께 지워집니다.

**완료 판단:** 몰 목록에서 그 몰이 사라진다.

> ⚠️ **되돌릴 수 없습니다.** 중간에 실패하면 아무것도 지워지지 않고 되돌아갑니다.

## 무엇이 지워지고 무엇이 남나

| | 대상 |
|---|---|
| **함께 지워짐** | 상품(리뷰·장바구니·이미지 포함), 딜·이벤트·기획전·공동구매·라이브·쿠폰·FAQ·아웃렛·상품그룹, 내비·GNB 메뉴·커스텀 메뉴·배송 정책·사이트 설정·테마, 페이지와 섹션, 배너, 외부몰 연동 상품·이력, **멤버십 등급·혜택·평가 정책과 회원 등급 상태·이력** |
| **보존됨** | **카테고리·브랜드·브랜드 프로필**(전 몰 공용 마스터), **주문 이력**(상품 링크만 끊김), 외부몰 연동 계정 자격증명 |

> 이 몰 전용 분류(테마·아웃렛 타입 카테고리)만 정리되고, 공용 카테고리 트리는 그대로 남습니다.
