# 솔루션 개요 — 이 프로젝트는 "몰 빌더"입니다

## 한 줄 요약

이 프로그램은 **하나의 쇼핑몰을 운영하기 위한 것이 아니라, 쇼핑몰이 필요한 사용자에게 몰을 만들어 포팅해 주는 "몰 빌더(mall builder)" 솔루션**입니다.

## 무엇이 다른가

일반적인 쇼핑몰 프로그램은 "이 몰 하나를 잘 운영한다"가 목표입니다. 이 프로젝트는 다릅니다.

| | 일반 쇼핑몰 | 이 몰 빌더 |
|---|---|---|
| 목적 | 특정 몰 운영 | 몰을 **찍어내서 납품** |
| 몰 개수 | 보통 1개 | **여러 개**(멀티몰) |
| 데이터 | 진짜 운영 데이터 | **예시/샘플 데이터** |
| 몰 삭제 | 거의 안 함 | **자주 함**(만들고 지우고 반복) |
| 사용자 | 몰 운영자 | 몰을 **새로 만들려는 사람** |

즉 이 저장소의 모든 데이터와 몰은 **개발·빌드용**이며, 실제 상거래가 일어나는 운영 환경이 아닙니다.

## 멀티몰 구조

이 빌더는 하나의 앱 안에서 **여러 몰**을 동시에 정의할 수 있습니다.

- 각 몰은 `mall` 테이블의 한 행이고, 고유한 **코드**(예: `health`, `general`)와 이름을 가집니다.
- 스토어프론트는 `?mall=<코드>` 로 어떤 몰을 볼지 정합니다. (예: `/?mall=health`)
- 카테고리·상품·메뉴·테마·홈 화면은 모두 `mall_id` 로 몰에 묶여, **몰마다 완전히 다른 쇼핑몰**이 됩니다.
- **기본몰**(is_default) 하나는 항상 존재하며, 몰을 특정하지 않았을 때의 폴백입니다. 그래서 **기본몰은 삭제할 수 없습니다.**

## 몰 하나를 만드는 큰 그림

```
1) 몰 등록      → 코드·이름·테마 선택(+샘플 데이터) → 초기 골격 자동 생성(테마·헤더·메뉴·홈)
2) 몰 선택      → 편집 대상으로 이 몰을 지정
3) 디자인 다듬기 → 사이트 설정(색상·로고) → 테마 설정(모서리·글꼴) → Header 설정
4) 내용 채우기  → 카테고리 → 상품 → 메뉴 → 페이지 빌더에서 홈 발행
5) 확인         → /?mall=<코드> 로 스토어프론트 미리보기
6) (필요 시) 삭제 → 강제 삭제로 몰과 데이터를 전부 지우고 다시 시작
```

**1번만 해도 볼 수 있는 몰이 하나 나옵니다.** 테마를 고르고 **샘플 데이터 포함**을 켠 채 저장하면 디자인·헤더·메뉴·메인 화면·샘플 상품까지 자동으로 채워지기 때문입니다. 2~4번은 그 몰을 **내 것으로 바꾸는** 과정입니다.
