
쇼핑몰 쿠폰 종류 및 관리 설계

쇼핑몰 쿠폰은 하나의 기준으로만 분류하기보다 다음 3개 축을 분리해서 관리하는 것이 좋습니다.

할인 혜택 종류: 무엇을 할인하는가
적용 단위: 상품·주문·배송 중 어디에 적용하는가
발급 방식: 고객에게 어떻게 지급하는가

Shopify도 정액·정률 할인, 무료배송, Buy X Get Y 등을 주요 할인 유형으로 제공하며, Adobe Commerce는 쿠폰 코드와 별도로 장바구니 조건과 할인 규칙을 관리하는 구조를 사용합니다.

1. 일반적인 쿠폰 종류
1.1 혜택 기준
쿠폰 종류	설명	예시
정액 할인	일정 금액 차감	5,000원 할인
정률 할인	구매금액의 일정 비율 할인	10% 할인
배송비 할인	배송비 일부 또는 전액 할인	무료배송
수량 할인	특정 수량 구매 시 할인	2개 구매 시 10%
Buy X Get Y	특정 상품 구매 시 다른 상품 할인·증정	2개 구매 시 1개 무료
적립금 지급	할인 대신 적립금 지급	구매 후 3,000P 지급
사은품 쿠폰	조건 충족 시 사은품 제공	10만원 이상 구매 시 증정

카페24도 상품 할인, 주문서 할인, 배송비 할인, 적립금 지급 쿠폰을 구분해 제공합니다.

1.2 적용 범위 기준
구분	적용 위치	주요 용도
상품 쿠폰	개별 상품 또는 상품군	특정 상품 판매 촉진
카테고리 쿠폰	특정 카테고리 상품	패션·식품 등 카테고리 행사
브랜드 쿠폰	특정 브랜드 상품	브랜드 프로모션
장바구니 쿠폰	주문 상품 합계	일정 금액 이상 구매 유도
주문서 쿠폰	최종 주문 단위	신규회원·등급·이벤트 혜택
배송비 쿠폰	배송비	무료배송 행사
판매자 쿠폰	특정 판매자 상품	입점형·오픈마켓
전 상품 쿠폰	전체 상품	대규모 프로모션
권장 구분

시스템에서는 최소한 다음 세 가지를 명확하게 분리해야 합니다.

PRODUCT     상품 단위 쿠폰
ORDER       주문 단위 쿠폰
SHIPPING    배송비 쿠폰

상품 쿠폰과 주문 쿠폰을 같은 계산 단계에서 처리하면 할인 중복과 환불 금액 계산이 복잡해지므로 별도 유형으로 관리하는 것이 안전합니다.

1.3 발급 방식 기준
발급 방식	설명	사례
자동 발급	조건 충족 시 자동 지급	회원가입, 생일, 등급 상승
다운로드	고객이 쿠폰존에서 직접 다운로드	월간 쿠폰팩
쿠폰 코드	코드 입력 후 사용	WELCOME10
개별 발급	관리자가 회원을 지정하여 지급	CS 보상 쿠폰
일괄 발급	회원 그룹에 일괄 지급	VIP 회원 대상
이벤트 발급	출석·룰렛·응모 결과로 지급	출석 체크 쿠폰
주문 조건 발급	주문·구매확정 후 지급	재구매 쿠폰
API 발급	외부 CRM·제휴 시스템에서 지급	제휴사 캠페인

쿠폰 코드 방식은 공개 코드, 고객별 고유 코드, 자동 생성 코드 등으로 세분화할 수 있습니다. Adobe Commerce도 고객 그룹이나 최소 구매금액 등의 조건을 충족할 때 쿠폰 코드를 적용하는 구조를 제공합니다.

2. 쿠폰 관리 구조

쿠폰 관리는 다음 네 개 영역으로 분리하는 것이 좋습니다.

쿠폰 정책
  └─ 할인액, 조건, 대상, 기간, 중복 정책

쿠폰 발급
  └─ 누구에게, 언제, 어떤 방식으로 지급할지

회원 보유 쿠폰
  └─ 발급·사용·만료·회수 상태

쿠폰 사용 이력
  └─ 주문 적용, 취소, 환불, 복원 이력

정률 쿠폰에는 반드시 최대 할인금액을 설정하는 것이 좋습니다.

4. 중복 할인 정책

쿠폰 시스템에서 가장 중요한 부분입니다.

권장 계산 순서
1. 상품 기본 할인
2. 상품 쿠폰
3. 장바구니·주문 쿠폰
4. 배송비 쿠폰
5. 적립금·예치금
6. 결제수단 할인
중복 정책 예시
조합	기본 정책
상품 쿠폰 + 상품 쿠폰	동일 상품에는 1개만
상품 쿠폰 + 주문 쿠폰	허용 가능
주문 쿠폰 + 주문 쿠폰	일반적으로 1개만
배송비 쿠폰 + 상품 쿠폰	허용
쿠폰 + 적립금	운영 정책에 따라 허용
자동 할인 + 쿠폰	최적 할인 또는 중복 적용
Buy X Get Y + 상품 쿠폰	기본적으로 중복 제한

실제 플랫폼에서도 할인 유형 간 조합 가능 여부를 별도 정책으로 관리합니다.

추천 구현

쿠폰마다 조합 그룹을 지정합니다.

combination_group
- PRODUCT
- ORDER
- SHIPPING
- PAYMENT

그리고 다음 속성을 둡니다.

stackable                   다른 쿠폰과 중복 가능 여부
same_group_stackable        동일 그룹 중복 가능 여부
max_stack_count             최대 적용 개수
exclusive                   단독 적용 쿠폰 여부
priority                    자동 적용 우선순위
best_discount_only          최적 할인만 적용할지 여부
5. 쿠폰 상태 설계
쿠폰 정책 상태
DRAFT       작성 중
SCHEDULED   발급 예정
ACTIVE      발급·사용 가능
PAUSED      발급 중지
ENDED       기간 종료
CANCELLED   운영 취소
회원 보유 쿠폰 상태
ISSUED      발급됨
AVAILABLE   사용 가능
RESERVED    주문 진행 중 임시 점유
USED        사용 완료
EXPIRED     만료
REVOKED     회수
RESTORED    주문 취소로 복원

결제 진행 중에는 쿠폰을 바로 USED로 변경하지 말고 RESERVED로 점유한 후 결제 성공 시 USED로 확정해야 합니다.

6. 주문 취소·환불 처리
전체 취소
미사용 처리 가능 기간 내 취소
→ 쿠폰 복원

쿠폰 유효기간이 지난 후 취소
→ 복원하지 않음 또는 별도 유효기간으로 재발급
부분 취소

상품 쿠폰은 해당 주문상품 할인액을 기준으로 반환하고, 주문 쿠폰은 각 상품 금액 비율로 할인액을 배분해야 합니다.

상품별 주문 쿠폰 배분액
= 주문 쿠폰 총 할인액
× 상품별 쿠폰 대상 금액
÷ 전체 쿠폰 대상 금액

복원 여부는 운영 정책으로 설정해야 합니다. 카페24도 주문 취소·교환·반품 시 쿠폰 복원 여부를 별도로 설정하도록 안내합니다.

9. MVP 권장 범위

1차 구축에서는 다음 유형만 구현해도 대부분의 일반 쇼핑몰 운영이 가능합니다.

혜택
- 정액 할인
- 정률 할인
- 무료배송

적용 단위
- 상품 쿠폰
- 주문 쿠폰
- 배송비 쿠폰

발급 방식
- 자동 발급
- 다운로드
- 관리자 수동 발급
- 공용 쿠폰 코드

조건
- 최소 구매금액
- 회원 등급
- 특정 상품·카테고리·브랜드
- 발급 및 사용 기간
- 회원별 사용 횟수
- 전체 발급 한도

핵심은 쿠폰을 단순히 종류별 테이블로 나누는 것이 아니라, 혜택 유형·적용 범위·발급 방식·조건 규칙을 조합하는 정책형 구조로 설계하는 것입니다. 이렇게 해야 신규회원 상품 쿠폰, VIP 주문 쿠폰, 브랜드 무료배송 쿠폰처럼 다양한 쿠폰을 코드 수정 없이 생성할 수 있습니다.
