## ⚠️ 대전제가 뒤집혔다 (먼저 읽을 것)

이 문서의 원래 대전제는 **"분류를 먼저 만들지 않는다 · 상품 등록 행위가 카테고리를 만든다"** 였다.

**네이버 표준 카테고리 재구성**(완료 — 글로벌 `categories` 뼈대를 네이버 L1~L3 로 재구성, `origin='naver'` 2,347건)이 이를 **의도적으로 뒤집었다.** 지금 방침은 **네이버 표준 뼈대를 선(先)-시드**하고, 상품 등록은 "생성"보다 **"기존 표준 노드에 매칭"** 이 기본이다.

유사매칭 엔진·무결성 가드·폭주 방지는 전환 뒤에도 그대로 쓰인다. 하지만 **"자동 생성이 본체"라는 서술은 더 이상 현행 정책이 아니다.**

## 잔여 과제 (5~8단계)

| 단계 | 산출물 | DDL |
|---|---|---|
| **5. Quick Add** | 최소 3필드(상품명·가격·이미지) 등록 + 스마트 기본값. Standard/Premium 공통 동작 | 0 |
| **6. AI 채움 · Premium** | `suggestTaxonomy()` + 키 없음/실패 시 Standard 폴백 | 0 |
| **7. CSV 대량등록** | 업로드 파서 + 배치 resolver + 리포트. 컬럼 최소화(상품명·가격·이미지URL·브랜드텍스트·카테고리텍스트) — 브랜드/카테고리는 **텍스트로만 적어도** 자동 매핑. 결과 리포트(매핑 N · 신규생성 M · 실패 K) + 되돌리기 | 0 |
| **8. 정리 도구** | 자동생성 항목 뱃지 + 카테고리 병합 | (선택) `is_auto_generated` 컬럼 |

그 밖의 제안(의견 수준):

- **`assist` 모드(후보 확인 UI)** — 자동 매핑 결과를 저장 전에 운영자가 확인·수정. 오매핑이 걱정되는 초기 몰의 기본값으로 쓸 값어치가 있다.
- **스마트 기본값 & 템플릿** — 자주 쓰는 배송/반품/옵션 세트를 템플릿화해 원클릭 적용. 직전 등록의 브랜드·배송·카테고리를 기본 프리필.
- **등록 마법사** — 3스텝(기본정보 → 이미지 → 확인). 초보 운영자용이고 기존 단일 폼은 "고급 모드"로 유지. UX 리뉴얼 시.

### 판매 가능 상태 검증기 (미구현)

자동으로 만들어진 분류에 상품이 붙어도 **고객 화면에 실제로 노출되지 않으면** 자동화가 완결되지 않는다("자동으로 분류됐는데 안 보인다"). 등록 완료 상품이 만족해야 할 최소 조건:

- `category_id` 가 자동 매핑/생성됨(최악의 경우 "미분류") → 목록·검색 노출
- `price` > 0, `status` = 판매중, **판매가능재고 > 0** (정의 → [`develop_guide/admin/products.md`](../develop_guide/admin/products.md) §18)
- 대표이미지 1장 이상
- (권장) `brand_category_id` 자동 매핑/생성 → 브랜드 허브 노출. 없으면 provider 텍스트라도 채움

검증기가 저장 시 위를 점검하고 미충족 항목을 **경고로만** 표시한다(저장은 막지 않음 — 마찰 최소화). "판매 가능" / "노출 제한" 배지로 시각화.

## 결정 필요 사항

- **"미분류" 카테고리 정책** — 고객 GNB 에 노출할지, 백오피스 전용으로 숨길지. (권장: 고객에겐 숨기고 검색·전체상품에만 포함)
- **납품 등급 기본값** — 신규 몰의 `product_taxonomy_ai_enabled` 기본을 Standard(0)로 둘지 Premium(1)로 둘지, Premium 의 과금·키 발급을 납품 계약 어느 단계에서 정할지. (권장: 기본 Standard, Premium 은 옵션 상품화 — 키 미입력 시 자동 Standard 동작이라 안전)
- **자동생성 플래그 컬럼** — `is_auto_generated` 추가는 스키마 변경이다. `alter-table` 스킬로 `tables.sql` 동기화 필요.

## 계속 유효한 함정

- **AI 는 필수가 아니라 등급 옵션이다.** 자동 매칭 본체가 AI 를 안 쓰므로 Standard(AI 미사용) / Premium(AI 사용)으로 갈라도 처리층은 한 벌을 공유한다. AI on/off 는 "근거 텍스트를 만드는 입력층"만 바꾸며 `system_settings` 플래그로 **코드 재배포 없이** 전환된다. Premium 인데 키가 없으면 자동으로 Standard 로 폴백돼 장애가 등록을 막지 않는다.
- **자동 생성은 유사매칭 선행 + 임계값 + 폭주 방지가 없으면 유령 분류를 양산한다.** "같은 게 있으면 붙이고, 없을 때만 생성"이 알고리즘의 핵심이다.
- **한글 유사도는 초성·별칭 사전이 빈약한 초기 몰에서 오매핑한다.** `auto` 모드는 신뢰 운영자에게만.
- **브랜드명이 `provider` 텍스트 컬럼에도 중복 저장된다**(`resolveBrandName`). 브랜드 자동매핑을 고칠 때 이 정합을 함께 유지해야 한다.
- 카테고리 생성은 **반드시 `services/tree/depthGuard.js` 가드를 경유**한다(뎁스·순환·동일 type).
