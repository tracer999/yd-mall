결론

현재 설계 방향이라면 관리자 메뉴는 “완전 자유형 빌더 관리자”가 아니라 “쇼핑몰 운영자가 필요한 설정만 안전하게 조정하는 관리자”로 가는 것이 맞습니다.

관리자 메뉴는 크게 아래 8개 영역으로 나누는 것을 추천합니다.

1. 대시보드
2. 쇼핑몰 설정
3. 메뉴/카테고리 관리
4. 페이지/전시 관리
5. 상품 관리
6. 프로모션 관리
7. 주문/회원 관리
8. 운영/시스템 관리

핵심은 메뉴 관리와 페이지 빌더를 분리하는 것입니다.

메뉴는 “상단 GNB에 무엇을 보여줄 것인가”이고, 페이지/전시는 “각 메뉴에 들어갔을 때 어떤 화면을 보여줄 것인가”입니다.
둘을 섞으면 관리자 구조가 빠르게 복잡해집니다.

- SEO URL 구조 유지
- QA 범위 축소

4-2. 상품 그룹 관리
관리자 > 페이지/전시 관리 > 상품 그룹 관리

상품 그룹은 일반 메뉴와 페이지 섹션의 데이터 소스입니다.

예시
- 오늘특가 상품 그룹
- 베스트 상품 그룹
- 여름세일 상품 그룹
- 신상품 자동 그룹
- 리뷰 좋은 상품 그룹
- MD 추천 상품 그룹
상품 그룹 유형
유형	설명
수동 그룹	운영자가 상품 직접 선택
자동 그룹	조건 기반 자동 수집
혼합 그룹	자동 + 수동 고정 상품
랭킹 그룹	판매량/조회수/리뷰 기준
기간 그룹	특정 기간만 노출

최종 판단

현재 구조라면 관리자 메뉴는 “메뉴/카테고리 관리”와 “페이지/전시 관리”를 분리해야 합니다.

정리하면 다음이 핵심입니다.

카테고리 관리
= 상품 탐색 구조 관리

일반 메뉴 관리
= 사전 정의 기능 메뉴 ON/OFF 관리

커스텀 메뉴 관리
= 제한된 자유 메뉴 관리

시스템 메뉴 설정
= 로그인/장바구니/마이페이지 등 고정 기능 관리

페이지/전시 관리
= 메뉴 클릭 후 보여줄 화면 구성 관리

이렇게 설계하면 기본 구조의 안정성은 유지하면서도 쇼핑몰별 운영 자유도는 충분히 확보할 수 있습니다.
