# 샘플 데이터 리소스 — 설계

> 상태: 구현 완료(리소스 테이블 · 시더 전환 · 관리 화면 · 추출 스크립트). 커밋 전.

## 1. 왜 필요한가

납품 시 이 시스템은 **몰이 0개**인 상태로 나간다. 고객이 첫 몰을 만들었을 때 상품이 하나도
없으면 리졸버가 전부 null 을 돌려 **메인 화면이 텅 빈다**. 이지모드라도 화면이 어떻게 구성되고
어떻게 보이는지 알기 어렵다. Shopify 등 다른 빌더의 테마가 예시 상품을 포함하는 것과 같은 이유로,
**몰 생성 시 샘플 데이터로 채워진 상태**로 시작한다.

이 샘플의 **원본**을 코드 상수가 아니라 **리소스 테이블**로 관리한다(납품처마다 코드 수정 없이 교체).

## 2. 핵심 제약

| 제약 | 이유 |
|---|---|
| 샘플 리소스는 **몰과 무관한 전역 데이터**(mall_id 없음) | 납품 시 몰이 0개여도 존재해야 한다. 특정 몰의 데이터를 참조할 수 없다. |
| `mallEraser.MALL_SCOPED_TABLES` 에 **넣지 않는다** | 몰 삭제와 무관한 전역 리소스이기 때문. |
| 이미지는 **`/images/...` 경로만** | `.gitignore:44` 가 `/public/uploads/` 를 제외 → uploads 이미지는 **납품본에 안 실려 깨진다.** |
| `products.slug` 는 **전역 UNIQUE** | 시딩 시 몰별 접두어 `sm{mallId}-{sample_key}` 필수. |
| 시딩은 `provisionMall` **이전** | provisionMall 이 베스트 랭킹을 집계하므로, 샘플 상품이 먼저 있어야 첫 스냅샷에 잡혀 캐러셀이 뜬다. |
| `banners` 에는 mall_id 가 없다 | 몰별 샘플 배너로 쓸 수 없다 → 샘플 슬라이더는 **`hero_slide`**(mall_id 보유)만 사용. |

## 3. 구성 요소

| 파일 | 역할 |
|---|---|
| `scripts/migrate_sample_resources.sql` | 리소스 테이블 3종 + **기본 시드**(빈 설치일 때만, `@fresh` 가드) |
| `services/mall/sampleSeeder.js` | 리소스 테이블 → 새 몰로 복제. 멱등(`sm{mallId}-%` slug 가드) |
| `controllers/admin/sampleDataController.js` + `views/admin/service/samples.ejs` | 관리 화면 |
| `routes/admin/service.js` | `/admin/service/samples` 라우트(super_admin) |
| `scripts/extract_sample_from_mall.js` | 기존 몰에서 샘플 추출 + **이미지 실체화** |
| `public/images/sample/` | 커밋되는 샘플 이미지 자산 |

### 리소스 테이블

| 테이블 | 내용 |
|---|---|
| `sample_category` | 샘플 카테고리 + 브랜드. 실제 `categories` 와 동일하게 `type`(NORMAL/BRAND)으로 분기 |
| `sample_product` | 샘플 상품. `category_key`/`brand_key` 로 위를 참조(FK 아닌 **키 기반** — 지웠다 넣어도 관계 유지). `deal_price` 가 있으면 샘플 특가전에 포함 |
| `sample_hero_slide` | 샘플 슬라이더. `MAIN`=중앙 슬라이더, `FEATURE`=우측 카드. `product_key` 로 상품 연결 |

### 몰 생성 흐름 (`mallController.postAdd`)

```
INSERT mall (트랜잭션) → mallContext.invalidate()
  → [seed_sample 체크 시] sampleSeeder.seedSampleData(mallId)   ← 리소스 테이블에서 복제
  → mallProvisioner.provisionMall(...)                          ← 베스트 집계·발행
```
폼의 **"샘플 데이터 포함" 체크박스(기본 켜짐)** = `views/admin/malls/form.ejs` 의 `seed_sample`.

## 4. 샘플 원본 추출 (`extract_sample_from_mall.js`)

```bash
node scripts/extract_sample_from_mall.js --mall=2 --categories=3 --per-category=2 --dry-run
node scripts/extract_sample_from_mall.js --mall=2 --categories=3 --per-category=2 --replace
```
- 상품 많은 카테고리 N개 → 각 M개 상품 → 그 상품들의 브랜드를 추출
- **동명 카테고리 중복 제거**(종합관에는 '니트' 같은 동명 카테고리가 여러 개 있다)
- 상품명의 선두 `[최초가79,900원]` 류 마케팅 문구 제거(`--keep-names` 로 보존)
- **이미지 실체화**: 외부 CDN URL이든 `/uploads` 든 → `public/images/sample/products/{key}.jpg`
  로 내려받아 sharp 로 800px·q80 리사이즈. 이 단계가 없으면 납품본에서 이미지가 깨진다.

### ⚠️ 현재 샘플 원본에 대한 주의

현재 샘플은 **종합관(mall 2)** 에서 추출했고, 종합관 상품은 `seed_mall2_cjon.js` 로 적재된
**CJ온스타일 수집 데이터**다. 따라서 현재 샘플에는 실제 브랜드(닥스·빈폴·아디다스 골프 등)의
상품명과 이미지가 포함되어 있으며, 이것이 **납품 고객사 몰에 복제**된다.
저작권·상표 리스크를 인지한 상태의 선택이며, 교체가 필요하면:
1. `public/images/sample/products/` 의 이미지를 교체하고
2. `/admin/service/samples` 에서 이름·가격·경로를 수정하거나
3. 다른 몰에서 다시 추출(`--mall=1` 등)

## 5. 관리 화면

**서비스 관리 → 샘플 데이터 관리** (`/admin/service/samples`, super_admin)
- 카테고리/브랜드 · 상품 · 히어로 슬라이드를 한 화면에서 일괄 편집(이름·이미지·가격·특가·순서·사용여부)
- 저장 시 **이미지 경로가 `/images/` 로 시작하는지 검증**(`/uploads/` 는 거부 — 납품본 깨짐 방지)
- 여기서 바꾼 값은 **앞으로 생성될 몰**에만 적용된다(이미 만든 몰은 불변)

메뉴 등록: `scripts/migrate_admin_menu_service_group.js` 의 `SERVICE_LEAVES`.
⚠️ 이 스크립트는 **라우트가 배포된 뒤** 실행해야 한다(dev·prod 공용 DB — 먼저 실행하면 사이드바에 404 링크).

## 6. 확장

`sample_*` 테이블에 행을 추가하면 시더가 자동으로 복제한다. 새 종류(예: 샘플 배너)를 추가하려면
`banners` 에 `mall_id` 가 없는 문제(§2)를 먼저 해결해야 한다.
