## 잔여 과제

1. **경로 D — AI 속성 추출.** 사전 기반 추출(경로 C, 구현됨)의 채움률을 본 뒤 필요하면 붙인다.
2. **경로 A — 고시 값 재활용.** ⚠️ **전제가 틀렸다(2026-07-22 확인).** "고시 입력값을 재활용한다"는 계획은 상품별 고시 값이 어딘가 저장돼 있다는 가정 위에 있었는데, 그런 저장소가 없다. 고시 값은 **몰 단위 기본값**(`naver_publish_profile.notice_defaults_json`, `mall_notice_default`)과 채널 전송 시점 override 로만 다뤄지고(`services/sourcing/channel/naverMapper.js:113-127`), `products` 에도 별도 테이블에도 상품별 고시 입력 컬럼이 없다. 살리려면 **상품별 고시 입력 화면과 저장소를 먼저** 만들어야 한다 — 필터가 아니라 상품 등록 쪽 과제다.
3. **생활/건강 2뎁스별 속성 프로필** — 35개 품목이 이질적이라 1뎁스 단위 필터 셋으로 안 묶인다. 별도 작업.
4. **평점 필터·평점순 정렬** — `reviews` 0건이라 Tier 0 에 정의만 해두고 `is_active=0` 이다. 리뷰가 쌓이면 켠다.

## 계속 유효한 함정

- **파셋 카운트는 자기 자신을 조건에서 빼고 센다.** 안 그러면 브랜드를 하나 고르는 순간 나머지 브랜드가 전부 0 이 된다(`buildPredicates(..., { exclude: ['BRAND'] })`). 속성 카운트도 동일.
- **자동 추출값은 `is_searchable=0`(검수 대기)으로만 들어간다.** 승인해야 고객 필터에 걸리는 2단 구조가 이 기능의 핵심이다. 옵션값이 상품명과 혼재(`캐리어20인치(블랙)`)해 파싱 오류가 섞이기 때문이다.

  ```
  자동 추출 → product_attribute(is_searchable = 0)   ← 필터에 안 걸림
                      ↓ 관리자 검수 화면에서 승인
               product_attribute(is_searchable = 1)   ← 필터에 걸림
  ```

- **`getList` 에 JOIN 을 추가하면 카운트 쿼리가 조용히 깨진다.** `productController.getList` 는 카운트를 `SELECT *` → `SELECT COUNT(*)` 문자열 치환으로 만든다. 신규 필터는 **EXISTS 서브쿼리만** 쓴다. (장기적으로는 `brandService` 처럼 WHERE 배열 + 별도 COUNT 로 리팩터링)
- **브랜드 목록 쿼리는 `products` 를 `p` 로 별칭**하므로 `buildPredicates` 에 `alias` 를 넘겨야 한다.
- **속성값을 스크립트로 채우지 말 것.** 이 인스턴스만 동작하고 납품되는 다른 몰에서는 필터가 텅 빈다. 적재는 전부 관리자 화면 버튼으로만 제공한다(`CLAUDE.md`「사용자 전제」).

## 신규 facet 을 추가할 때

- 쿼리 키 = `facet_code` **소문자**, 값은 `value_code` 콤마 구분. 구간형은 `min-max`(열린 구간 `price=100000-`).
- **예약어(`sort` · `page` · `q` · `view` · `mall`)와 충돌하는 `facet_code` 는 금지.** `facet_definition` 저장 시 검증한다.
- 닫힌 집합 facet 은 `facet_value_definition` 에 없는 값을 거부한다(자유 텍스트로 값이 폭발하는 것을 막는다). 표기 흔들림은 별칭 사전으로 흡수.
- 필터 후보는 상상이 아니라 **네이버 상품정보제공고시 스키마**에서 도출한다. `naver_notice_schema`(36종)가 DB 에 적재돼 있고 카테고리 3뎁스에 `notice_type` 이 매핑돼 있어(1,524개 리프) **카테고리 → 고시타입 → 속성 후보**가 데이터로 이어진다. 필드 전문은 `naver_notice_schema.fields_json` 을 직접 조회한다.
- EXISTS 가 6개 이상 중첩될 만큼 facet 이 늘면 `product_facet_flat` 캐시 테이블로 전환한다.
- `category_facet` 은 글로벌이다. 몰별 on/off 가 필요해지면 `mall_category_visibility` 와 같은 패턴으로 `mall_category_facet` 오버라이드를 추가한다.
