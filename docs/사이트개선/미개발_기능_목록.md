## 2. 화면별 상세

### 주문 · 배송 · 클레임
**주문 관리(sales)**
- 관리자 직접 주문 생성(대리 주문)
- ✅ ~~부분 취소~~ — **구현**: 고객이 상품별 수량 선택 → 할인 비율 안분 환불. 남은 상품이 있으면 주문은 유지되고 결제상태만 `부분환불`
- ✅ ~~주문 목록 엑셀 내려받기~~ — **구현**: 화면 필터 그대로 28개 열(송장번호 포함). BOM 처리로 엑셀에서 바로 열림
- ✅ ~~주문서 · 거래명세서 출력~~ — **구현**: 인쇄 전용 화면. 발행처는 그 몰의 사이트 설정에서 자동. 명세서에 공급가액·부가세 포함
- ✅ ~~주문번호·주문자명 텍스트 검색~~ — **구현**: 주문번호·주문자·받는분·연락처(뒷자리)·이메일 통합 검색
- ✅ ~~매출 집계·정산 리포트~~ — **구현**: 기간·몰·구분별 순매출(매출은 결제일 / 환불은 환불완료일 기준)

**클레임 관리(claims)**
- ✅ ~~교환~~ — **구현**: 결제·재고를 건드리지 않고 회수 → 재발송
- ✅ ~~부분 취소·부분 반품~~ — **구현**: `order_claim_items` 신설, 마지막 클레임이 반올림 잔여 흡수
- 관리자 직접 클레임 생성
- 승인 취소(되돌리기)
- ✅ ~~반품 회수 송장 등록·회수 추적~~ — **구현**: `shipments` 를 `direction=RETURN` 으로 재사용, 회수 완료 시각 기록
- ◑ SMS·알림톡 — 코드 완료, 계약 필요

**배송 관리(shipping)**
- ✅ ~~송장 일괄 등록(엑셀)~~ — **구현**: 미발송 주문이 채워진 양식 제공, 실패한 줄만 사유와 함께 리포트

- ✅ ~~배송 완료 자동 처리~~ — **구현**: 체크박스 일괄 처리 + 발송 N일 후 자동 전환(사이트 설정, `0`=끔)

| 1-7 ✅ | **포인트 유효기간(소멸) 관리** | 적립금이 소멸 없이 쌓이면 회계상 부채가 계속 늘고, 나중에 도입하면 소급 적용이 안 됩니다 | 사이트 설정의 `포인트 유효기간(개월)`(`0`=소멸 없음). 유효기간은 **적립하는 순간** 새겨지므로 **이미 적립된 포인트에는 소급되지 않습니다.** 소멸은 먼저 만료될 것부터(FIFO), **잔액을 넘겨 소멸시키지 않습니다.** 내역에 `유효기간 만료 소멸` 로 기록.<br>`services/point/pointExpiryService.js` |
