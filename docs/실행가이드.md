# 실행 가이드

이 문서는 `dev-mall` 프로젝트를 개발 환경과 상용(Production) 환경에서 실행하는 다양한 방법과 각 명령어의 차이점을 상세히 설명합니다.

`package.json`에 정의된 스크립트를 중심으로, 상황에 맞는 올바른 실행 방법을 안내합니다.

## 1. 개발 환경 실행 (Development)

개발 중에는 **코드 수정 시 서버 자동 재시작**과 **CSS 실시간 빌드**가 핵심입니다.

### 1) 통합 개발 모드 (권장)

**명령어**: `npm run dev:all`

- **실행 내용**: 백엔드(`nodemon`)와 프론트엔드 빌드(`tailwindcss --watch`)를 동시에 실행합니다.
- **특징**:
  - `app.js` 등 서버 코드 수정 시: 서버가 자동으로 재시작됩니다.
  - `views/*.ejs` 등 UI 코드 수정 시: CSS가 자동으로 다시 빌드되어 스타일이 반영됩니다.
- **사용 시점**: UI와 서버 로직을 함께 개발할 때 가장 편리합니다.

### 2) 백엔드 전용 모드

**명령어**: `npm run dev`

- **실행 내용**: `nodemon`만 실행하여 서버 코드 변경만 감지합니다.
- **특징**: CSS 빌더가 돌지 않으므로 시스템 리소스를 적게 사용합니다.
- **사용 시점**: API 개발, DB 작업 등 UI 스타일 변경이 필요 없는 작업을 할 때 유용합니다.

---

## 2. 상용 환경 실행 (Production)

상용 환경에서는 `NODE_ENV=production`으로 설정하여 성능 최적화와 보안을 강화해야 합니다. 실행 방식은 크게 **프로세스 매니저(PM2) 사용**과 **단순 Node 실행**으로 나뉩니다.

### 1) PM2를 이용한 실행 (권장)

**명령어**: `npm run pm2:start`

이 명령어는 내부적으로 `pm2 start ecosystem.config.cjs`를 실행합니다.

- **실행 방식**:
  - `ecosystem.config.cjs` 설정을 기반으로 PM2가 애플리케이션을 관리합니다.
  - 기본적으로 **클러스터 모드**로 동작하여 CPU 코어를 효율적으로 사용(기본 2개 인스턴스)합니다.
- **장점**:
  - **무중단 운영**: 프로세스가 에러로 종료되어도 자동으로 재시작됩니다.
  - **로그 관리**: `logs/` 디렉토리에 실행 로그와 에러 로그가 자동으로 저장되고 관리됩니다.
  - **부하 분산**: 멀티 코어를 활용하여 더 많은 트래픽을 처리할 수 있습니다.
- **주의사항**:
  - **Redis 필수**: 다중 인스턴스 환경에서는 로그인 세션을 공유하기 위해 Redis가 필요합니다. (`.env`에 `REDIS_HOST` 설정)

### 2) 단순 Node 실행 (Docker/테스트용)

**명령어**: `npm start` 또는 `node app.js`

PM2 없이 Node.js 런타임으로 직접 프로세스를 하나만 띄우는 방식입니다.

- **실행 방식**:
  ```bash
  # 환경 변수 명시 필요
  NODE_ENV=production npm start
  # 또는
  NODE_ENV=production node app.js
  ```
- **특징**:
  - 단일 프로세스로 동작합니다.
  - 프로세스 관리 기능(자동 재시작 등)이 _없습니다_.
- **장점**:
  - 구조가 단순하여 Docker 컨테이너 등 이미 외부에서 프로세스를 관리해주는 환경에 적합합니다.
  - PM2 설치가 필요 없습니다.
- **사용 시점**:
  - 로컬에서 상용 환경 설정을 간단히 테스트할 때.
  - Docker 이미지의 `CMD`로 사용할 때 (K8s 등이 재시작을 관리하는 경우).

## 3. 실행 방법 비교표

| 명령어                  | 환경 (ENV)  |   프로세스 관리    | CSS 감시 | 주요 용도              |
| ----------------------- | :---------: | :----------------: | :------: | ---------------------- |
| **`npm run dev:all`**   | development |   Nodemon (자동)   |    O     | **로컬 개발 (메인)**   |
| `npm run dev`           | development |   Nodemon (자동)   |    X     | 백엔드 개발            |
| **`npm run pm2:start`** | production  | **PM2 (클러스터)** |    X     | **상용 서비스 운영**   |
| `npm start`             |   (수동)    |    Node (단일)     |    X     | 컨테이너 배포 / 테스트 |

## 4. 문제 해결

### Q: `npm run pm2:start` 후 로그인이 자꾸 풀립니다.

**A**: PM2 클러스터 모드(인스턴스 2개 이상)에서는 각 프로세스가 메모리를 공유하지 않습니다. 로그인 세션을 공유하려면 **Redis**가 필수입니다.

1. Redis 설치 및 실행
2. `.env` 파일에 `REDIS_HOST=localhost` 등 설정 추가
3. `pm2 restart dev-mall`로 재시작

### Q: `npx node app.js`로 실행해도 되나요?

**A**: 가능합니다. `npx node app.js`는 `node app.js`와 거의 동일하게 동작합니다. 다만, 상용 환경이라면 `NODE_ENV=production`을 꼭 붙여주세요.

```bash
# npx 사용 예시
NODE_ENV=production npx node app.js
```
